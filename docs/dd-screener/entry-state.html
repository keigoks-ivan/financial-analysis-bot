讀法 8 點(動手前必讀)
  1. 與 Quality-Entry 的定位區別:Quality-Entry 是「單一逢回邏輯」的複合評分;本頁是「突破派抓第一棒、回調派抓後續棒」的雙邏輯接力。兩者並存,服務不同進場哲學。
  2. Universe 來源(v1.2 retune):`build_quality_entry.py` 全量 scored(post-veto)中,新進門檻 Quality ≥ 0.75 ∩ Growth ≥ 0.6 ∩ archetype=成長型;昨日已在 universe 者只要 Q ≥ 0.7 / G ≥ 0.55(hysteresis 保留門檻)就維持,避免邊界 ticker 進出抖動。v1.2 變更:原 Q≥0.55/G≥0.50 → 83 太多;Q 實測 min=0.58 不發揮濾用,只 G 才是 dial。新門檻自然 universe ~49,過渡期含 hysteresis ~72。archetype 分類純量化(`growth_pillar ≥ 0.55` = 成長型),不看 market cap;V/MA/MSFT 等「mature 但仍成長」compounder 因 growth_durability 高,會被分到「成長型」(若想額外過濾大型權值股,需另起 mega-cap saturation filter,非本版)。
  3. 兩層結構:第一層 W52/W250 週線 MA 閘(🟢/🟡/🔴/⚪)決定能否進入第二層;第二層才判 BREAKOUT/PULLBACK/RIDING 三狀態。
  4. WEAK ≠ 第二層狀態:WEAK 是第一層否決的標籤(跌破 W52 或 W250),不是第二層的進場狀態。它與三個進場狀態並列顯示是為了完整性。
  5. 突破派抓第一棒、回調派抓後續棒;★ NEW 才是新訊號:`is_new=true` 表示今天剛從別狀態轉入此狀態。BREAKOUT 持續中(`is_new=false`)實質接近 RIDING,不是新訊號。
  6. 狀態判定用 5 年週線 cycle context + 4 週方向過濾:不是單日 snapshot — `dist_250w_high_pct`(真 ATH 距離)+ `weeks_since_250w_high`(ATH 週齡 0-26 才算近期)+ `drift_4w_min_in_8w`(過去 8 週曾真跌過)是 backtest 證實的設計。
  7. 止跌確認需人工;DD 新鮮度紅標 ≠ 閘:PULLBACK_WATCH 升級 PULLBACK_CONFIRMED 由人工看圖判讀(本頁 v1.0 不自動升級,只顯示 WATCH 計數)。DD > 90 天紅標代表 quality 判定可能過期 — Ivan 自行決定是否重跑 DD,本頁不擋訊號。
  8. 【v1.2 新】Filter chips 預設只顯示 BREAKOUT / PULLBACK_WATCH / PULLBACK_CONFIRMED(actionable 訊號)。點 ⚪ RIDING / 🔴 WEAK chip 可展開所有 universe;點「★ 只看新訊號」會再窄到只剩 `is_new=true` 列(BREAKOUT/PULLBACK 才會有 ★)。所有 filter 純 client-side,reload 重置。

方法論(門檻全為圓整、寬鬆數值,backtest 驗證)

第一層 MA 閘:🟢 healthy = above_w52 ∩ above_w250 ∩ slope_w250 > 0 · 🟡 mixed = 雙均線上但 slope 平/負 · 🔴 weak = 跌破任一均線(第一層否決)· 🟢 W250 N/A = < 5y 上市但站上 W52(降級,關 BREAKOUT)
BREAKOUT(僅 🟢 healthy):dist_250w_high_pct ∈ [-3%, 0%] ∩ weeks_since_250w_high ≤ 2 ∩ drift_4w_pct > +5% ∩ ma50_pct ≥ 0 ∩ rs_score ≥ 80
PULLBACK_WATCH(🟢/🟡):dist_250w_high_pct ∈ [-15%, -5%] ∩ weeks_since_250w_high ∈ [2, 26] ∩ drift_4w_min_in_8w < -3%(過去 8w 曾真跌過)∩ ma50_pct ∈ [-3%, +3%]
排序鍵(v1.1):主鍵 = 狀態優先序(PULLBACK_CONFIRMED → PULLBACK_WATCH → BREAKOUT new → BREAKOUT 持續 → RIDING → WEAK);次鍵 = FundScore = (0.40×Q + 0.30×G) / 0.70 = 0.571×Q + 0.429×G 由高到低(缺 Q/G 視為 -∞ 沉底)。RS 自 v1.1 起不再參與排序,僅作顯示欄。
WATCH → CONFIRMED 升級:WATCH 中累積 builds_no_new_low ≥ 3 顯示「✓ 可進場考慮」hint,但本頁 v1.0 不自動升級;由 Ivan 看圖判讀後手動進場。容忍式計數:離開 WATCH 區後 ≤ 7 build 內回來 → 計數延續;超過 → 視為這一波結束、清除狀態。
已知設計邊界:MA 燈號只看 W250 斜率,不看 W52 斜率(避免 W52 抖動干擾大方向閘)— 趨勢早期警示由 🔴(跌破 W52)觸發。Backtest 結果(原 v1.1 5y × 23 ticker sample,Q≥0.55):PULLBACK 命中 ~6%(每檔每 ~17 週,合 ~3 次/年)/ BREAKOUT ~18%(production 加 RS≥80 gate 後實際降至 ~10%)。v1.2 retune 後 universe 擴至 ~49 但同屬高品質區段,命中率應接近;未在新 universe 上重 backtest,訊號量明顯異常時可再調 Q/G 門檻。
