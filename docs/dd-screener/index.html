🔭 形狀掃描 — 正在噴的多倍形狀 × 前瞻確認 × 覆蓋標籤（0 檔，—）

橫切研究狀態的雷達：用「形狀」過濾（不用覆蓋、也不用市值過濾）——掃 RS 領導（510 股日更）∪ 自訂全市場 screen（美股主板 × 市值 ≥ $0.3B × 營收 YoY ≥ +20%，系統性條件掃描、不是當日漲幅榜碰運氣），過爆發形狀閘（市值 ≥ $0.3B 無上限、季營收 YoY ≥ +20%、52 週位置 ≥ 0.6 或 RS ≥ 80、GM ≥ 25%、金融/REIT 剔除）。命中後三層標籤而非剔除：
• 前瞻：trailing 只證明過去，多倍股是「市場還沒 price in 的持續成長」——DD universe 內用 Koyfin FY1→FY3 EPS CAGR（與下方主表同源，附修正動能），universe 外用 yfinance 分析師 FY+1 營收預估。🟢 續強 ≥+15% ➖ 放緩 0~15% 🔴 反轉 <0（trailing 是基期效應/週期頂）⚪ 無預估（零分析師＝真沒被發現，是訊號不是剔除理由）。排序＝min(trailing, 前瞻) 保守取小——trailing +104% 但前瞻 −24% 的週期頂自動沉底。
• 加速：📈 預估下季 YoY 明顯高於已報告 YoY（營運槓桿點火中）／📉 明顯放緩／➖ 平穩。加速 × 🟢 前瞻 × ✅有DD 三者疊加＝本漏斗最強候選訊號。
• 覆蓋 + 市值級距：🔭 盲區零覆蓋 → 立 ID｜🟡 有 ID 無 DD → 形狀轉強＝加速訊號，建 DD｜✅ 有 DD → 懂它且剛突破＝最強訊號，回頭重看 DD（AR Live 只抓回檔、抓不到突破，這裡補上）。大型股照樣翻倍（NVDA $300B→$3T），小/中型看「沒被發現」、大型/mega 看週期/重定價。
• 週數：連續在榜週數（🆕＝本次新進榜）。形狀撐過 3 週比一日遊強得多；歷史同時累積成日後回測這條漏斗命中率的資料。
用途：告訴你注意力放哪——不是買入清單，估值與獲利品質刻意不在此把關（多倍股早期永遠看起來貴），那是管線②（DD 裁決層）的事。最終進場由個股 DD §14 裁決 + 時機決定。

表格欄位說明

🎯 現在的選股方式 — 多倍股發現漏斗

目標是找出營運可能爆發、報酬可能數倍的標的。結構是一個橫切雷達 + 一條縱向管線：雷達（🔭 形狀掃描）告訴你「現在哪些股票在噴、注意力放哪」，管線（候選池 → DD → 掛單）是「一個名字的研究進度」。不靠人翻報告、不靠人記憶。

🔭
🔭 形狀掃描 — 橫切雷達（找候選）（見上方「🔭 形狀掃描」折疊區）
這就是「找多倍股」直接看的表。用形狀過濾（不用覆蓋、不用市值過濾）：系統性掃全市場（美股主板自訂條件 screen，非當日漲幅榜）撈「營收正在爆（+20% 以上）+ 股價轉強 + 毛利不差」的股票——過去多倍股起漲的長相。trailing 之上疊前瞻確認：多倍股的本質是「市場還沒 price in 的持續成長」，所以用分析師預估（universe 內 Koyfin FY1→FY3 EPS CAGR 與主表同源；外用 FY+1 營收預估）標 🟢續強/🔴反轉/⚪無預估，排序取 min(trailing, 前瞻)——trailing 火熱但前瞻反轉的基期效應/週期頂自動沉底。再加加速度（📈 營運槓桿點火中）、覆蓋狀態（🔭盲區/🟡有ID/✅有DD）、市值級距、連續在榜週數四層標籤而非剔除。大型股照樣翻倍（NVDA $300B→$3T、MU 記憶體循環），只是靠週期/重定價非「沒被發現」。用途：這是候選清單——📈×🟢×✅有DD 疊加是最強訊號；挑不懂的（尤其🔭盲區）去建 DD。它找的是「候選」不是「確定多倍股」，要不要買由 DD 判。
🧭
管線① 多倍候選池 — ID 已見、未建 DD（縱向管線起點｜見上方「🧭 多倍候選池」折疊區）
產業報告 §9 標為 🔴 核心受益、但還沒做個股 DD 的名單。排序刻意按市值（小 / 中優先）× 純度 × 5 年需求倍數，不按「被幾份 ID 提到」——後者量的是研究注意力重疊，與未被發現的多倍股反相關。用途：決定下一份 DD 的研究預算花在誰。
📊
管線② 主表 — 已建 DD 的名字（裁決層｜下方各欄位即本表）
有個股深度報告（DD）的名字才進主表。FunnelRank（品質 × 護城河趨勢 × EPS 修正動能）綜合排序 ＋ MLB 五條硬閘（FCF/ROIC/EPS CAGR/PEG/D-E）＋ 每份 DD 的統一裁決（進場 / 觀望 / 迴避）。這是「判得準」的一層——爆發候選路徑會在此判斷「這個價位還有沒有不對稱 upside」。用途：判斷該不該進、扮演什麼角色。
🎯
管線③ AR Live watch — 活賠率掛單（時機層｜見表格上方「🎯 爆發獵場 watch」橫條）
每份 DD 都算過 5 年 Bull / Bear 目標價與機率，系統每日以現價重算不對稱比 AR。當一檔「觀望」股因回檔使 AR ≥ 4 且長跑道 🟢、護城河未惡化，自動亮進 watch——「等回檔」從此是系統掛單，不是記在腦裡的待辦。用途：好公司回到好價格時主動通知。

雷達與主表玩的是兩個不同的遊戲：主表的品質框架（FunnelRank / MLB 五硬閘）選「證明過自己的存活者」——ROIC、FCF、估值合理，怕買錯（低 false positive 優先）；形狀掃描選「正在證明自己的轉折者」——營收爆發 + 前瞻續強 + 股價剛轉強，怕漏掉（低 false negative 優先，漏掉 NVDA 的代價遠大於看錯三檔小票），因此刻意不設估值與獲利品質閘。兩張網撈不同的魚、重疊本來就少：MU/SNDK 這種循環股在雷達是榜首、過不了 MLB 硬閘；COST/V 這種品質模範生永遠不會出現在雷達。它們是接力不是打架——雷達撒寬網產生候選，品質框架在管線②（DD 裁決層）把關殺候選；核心（品質複利）與衛星（爆發候選）的配比是 PM 層的事，不是 screener 層的事。
上方分頁另有六個專項子頁（觸底反轉 / 突破動能 / EPS 加速 / 進場狀態機 / 品質切入 / Alpha 排名），各自從不同角度切主表 universe 的進場時機，是管線②③（主表 + watch）的時機側輔助。
重要：雷達與管線產出的都是「該研究 / 該盯」的排序，不是買入清單。最終進場由個股 DD 的 §14 裁決 ＋ 時機共同決定；名單只負責把注意力放對地方。

🚀 近 5Y 新高 spotlight — 表格上方 chip 列 hero 與 filter 之間的綠色橫條列出當下 站在或極近 5 年高點的 ticker — 表格主流量之上的快速導航 chip。
• 篩選邏輯：is_full_5y === true ∩ dist_5y_high_daily_pct ≥ −2% ∩ days_since_5y_high ≤ 10；資料來自 build_dd_screener.py compute_daily_5y_highs()，每次 build batch 抓 yfinance 5y 日線（不是週線 — 週線會壓平 intraweek 峰值造成 NVDA / AVGO / COST 看似在 ATH 但實際已回檔的問題）。
• 排序：距 ATH 最近的先（dist desc），同距時間最近的先（days_since asc）。cap 25 chip 避免一行爆版。
• 顏色：白底綠邊 = dist ∈ [−2%, 0%)；實心綠 = dist === 0（today 創高）；深綠內凹 = 被選中。
• 多選比較：點 chip 累加 ticker 到下方查 ticker 搜尋欄，自動加 trailing space 觸發 exact match（V → 只剩 Visa，不會噴 V/VIK/VRT）。再點同個 chip = toggle 移除。點 ✕ 一鍵清空 + 取消所有 chip 選取。
• 「暫無 ticker 觸頂」placeholder：當天條件 0 命中（罕見，大盤普跌時可能發生）。
EPS 修正動能 — v1.10 動態 baseline + FY3 加入 FY+1→FY+3 CAGR 欄右方的 ↑+X.X% (vs 5/26) / ↓-X.X% (vs 4/30) 小字，是「本份 Excel forward CAGR」對照「上一份 Excel snapshot」的差值；尾段 (vs M/D) 是**比較基準的日期**（不是資料新鮮度 — 主數字永遠是最新一份 Excel）。Curr / Next / FY+3 EPS 三欄底下也都有同樣 chip — FY1 / FY2 / FY3 各自獨立 vs baseline。hover tooltip 另列 baseline 距今幾天。
• 上修（綠）：分析師對未來 EPS 估算往上調 — 業界已知 alpha factor（Womack 1996 + Bloomberg / FactSet 同名指標），上修股票後續 6-12 個月傾向 outperform
• 下修（紅）：估算往下調，常為基本面領先指標壞訊號
• 持平（|diff| < 0.5%）：不顯示箭頭，避免雜訊
• 「新」chip（灰）：此 ticker 上一份 snapshot 還不在 universe（新加 DD / 新 cover），無對照
• 無顯示：首份 baseline 累積中（snapshot 還沒有 prior）— hover 看 tooltip 確認
CAGR 數字單位細節：CAGR chip 顯示的是「兩個 CAGR % 的絕對差」（兩個百分點之差），但為了視覺與 FY1/FY2/FY3 EPS chip 一致，後綴用 % 而非 pp。tooltip 會標註絕對差語意。
baseline 來源優先順序：(1) 同月內較早的 {YYYY-MM}-DD.json（intra-month 5/20→5/25 這種短期 refresh），(2) 1-2 個月前的月度 snapshot。每月 2 號 09:00 Taipei 自動 snapshot（.github/workflows/monthly-eps-snapshot.yml → scripts/snapshot_eps_estimates.py），存於 docs/dd-screener/eps-estimates-snapshots/YYYY-MM.json。
FunnelRank — 漏斗綜合排序分（v1.3 新增，預設排序鍵） 把「QGM 4/4 → 護城河 → 修正動能 → 時機」四層人工漏斗的前三層（基本面）固化成一個 0–1 排序鍵。時機（第四層）不進排序分，維持 filter —— 與 Entry State Machine 的「基本面排序、時機另看」哲學一致。
FunnelRank = 0.40·QualityGate + 0.30·MoatScore + 0.30·RevisionScore
• QualityGate（0–1）：帶「可原諒條件」的品質閘（4 條件計分：FCF / ROIC / EPS CAGR / PEG；2026-07-03 起 D/E 退出計分、降為 ⚠ advisory）—— 4/4 → 1.00；3/4 唯一 fail 在 PEG（估值，非品質本體）→ 0.85；3/4 fail 在 FCF / ROIC / EPS CAGR（品質本體缺陷）→ 0.50；2/4 → 0.30；≤1/4 → 0.10。任一條件無資料（灰色）→ 該條退出分母、按比例換算後套同一映射，並標 quality_gate_partial。
• MoatScore（0–1）：moat_score/10 × 趨勢乘數（↑ ×1.10 / → ×1.00 / ↓ ×0.80），cap 1.0。無 moat 資料 → 0.50。
• RevisionScore（0–1）：FY1/FY2/FY3 各自上修（diff ≥ +0.5%）= +1 / 持平 = 0 / 下修（≤ −0.5%）= −1；raw = 0.2·FY1 + 0.3·FY2 + 0.5·FY3（FY3 權重最大 = 成長曲線後段），RevisionScore = (raw+1)/2。FY3 上修幅度 > FY1 上修幅度（後段變陡）再 +0.05。「新」chip / 無 baseline → 0.50 中性，標 revision_no_baseline。
• 兩條 hard veto（不進公式，直接處置）：① FY1/FY2/FY3 三欄全部下修 → FunnelRank 強制歸 0、該列沉底 + ⛔（下修是領先指標、品質分是落後指標，打架時信前者）；② 護城河趨勢 ↓ 且非四條件全過（pass_count < 4）→ FunnelRank cap 0.50。
權重與映射值寫在 scripts/build_dd_screener.py 檔頭常數（FUNNEL_*），server 端 build 時算好寫進 latest.json（funnel_rank + 三個子分 + 各 flag）。表頭 Funnel 欄 hover 看子分拆解。
  • Ticker：股票代碼（正規格式）。
    來源：dd-meta JSON → scripts/dd_screener_dd_loader.py
    格式：美股 NVDA / 台股 2330.TW / 日股 6857.T / 歐股 RMS（帶交易所後綴的 EU 用內部 suffix map 轉換，DD 名稱保留無後綴形式）
    ⛔ 標記：FY1/FY2/FY3 三欄全下修的 veto 列，ticker 旁顯示 ⛔、整列降透明度並沉底（FunnelRank = 0）。
  • Funnel（漏斗綜合分）：基本面三層合成的 0–1 排序分，表格預設排序鍵（降序）。詳見上方「FunnelRank」callout。
    公式：0.40·QualityGate + 0.30·MoatScore + 0.30·RevisionScore（時機第四層不進分）。
    來源：全部用既有欄位合成（pass_count / 4 條計分明細 + moat_score/trend + FY1/FY2/FY3 修正動能；D/E advisory 不計分），scripts/build_dd_screener.py build 時算好寫進 latest.json。
    互動：3 位小數；hover 看 QualityGate / Moat / Revision 三個子分拆解 + veto/cap 說明；點表頭可改回任一欄位排序（原排序選項全保留）。
  • 裁決（DD §14 統一裁決）：每份 v13/v14 合併報告決策層收斂出的單一裁決 — 進場 觀望 迴避。點擊跳該 DD 的 #decision 錨點。
    來源：v13/v14 dd-meta 的 dca_verdict（欄位名沿用舊稱，資料是合併報告 Part II 決策層，非已退役的獨立 DCA 檔）；hover 看裁決日期與倉位角色（dca_role）。
    顯示 —：v12 舊 schema 報告無決策層 metadata，該 ticker 重跑 v14 DD 後自動補上；不 fallback 讀 legacy DCA（2026-07 起 DCA 不再作為裁決來源）。
    注意：裁決是寫報告當下的判斷，凍結於 DD 日期——股價漂移後宜搭配 5Y IRR（live 回推）與 AR Live watch 判讀，過舊的裁決以重跑 DD 為準。
  • Moat（護城河等級 + 分數 + 趨勢）：護城河評等，格式 等級 分數 趨勢，例如 S 9.8 ↑。
    來源：dd-meta JSON（moat_grade / moat_score）；趨勢箭頭 v13/v14 以 dd-meta 的 moat_trend 為權威，v12 legacy 過渡期間 fallback 舊 DCA Phase A1 箭頭（collect_dca_moat_trend_map()），兩者皆無時 fallback →（保守假設：穩定）
    等級：S（≥ 9.5）/ A（≥ 8）/ B（≥ 6）/ C（≥ 4）/ X（護城河存疑）
    趨勢：↑ 加深 / → 穩定 / ↓ 縮窄
  • FCF（FCF Margin）：TTM 自由現金流 / 營收，百分比。
    來源：優先 docs/qgm/latest.json 或 docs/qgm-tw/latest.json（hard_filter_details.fcf_margin.value）；不在 QGM 中的 ticker 走 scripts/dd_screener_quality.py → fundamentals_us.calc_fcf_margin()（yfinance）

    來源（live）：從 DD 決策層 frozen IRR 用 (price_at_dd / price_now) 回推：
    live_ev5y = ((1 + ev5y/100) × (price_at_dd / price_now)^0.2 − 1) × 100
    直覺：股價跌 → 起點變便宜 → 同樣 5 年 terminal → IRR 變高（合理）。標 * 提醒是推算非實算。
    Caveat（重要）：假設 DD 寫入時的 terminal value（5 年後目標價）未變。如果 thesis broke（基本面惡化、target 砍半），此 live IRR 會誤導你以為「跌很多 = 更便宜的機會」。建議搭配最新 DD 決策層（§14）判讀，不要單看此欄就加倉。
