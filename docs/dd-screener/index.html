表格欄位說明

🚀 近 5Y 新高 spotlight — 表格上方 chip 列 hero 與 filter 之間的綠色橫條列出當下 站在或極近 5 年高點的 ticker — 表格主流量之上的快速導航 chip。
• 篩選邏輯：is_full_5y === true ∩ dist_5y_high_daily_pct ≥ −2% ∩ days_since_5y_high ≤ 10；資料來自 build_dd_screener.py compute_daily_5y_highs()，每次 build batch 抓 yfinance 5y 日線（不是週線 — 週線會壓平 intraweek 峰值造成 NVDA / AVGO / COST 看似在 ATH 但實際已回檔的問題）。
• 排序：距 ATH 最近的先（dist desc），同距時間最近的先（days_since asc）。cap 25 chip 避免一行爆版。
• 顏色：白底綠邊 = dist ∈ [−2%, 0%)；實心綠 = dist === 0（today 創高）；深綠內凹 = 被選中。
• 多選比較：點 chip 累加 ticker 到下方查 ticker 搜尋欄，自動加 trailing space 觸發 exact match（V → 只剩 Visa，不會噴 V/VIK/VRT）。再點同個 chip = toggle 移除。點 ✕ 一鍵清空 + 取消所有 chip 選取。
• 「暫無 ticker 觸頂」placeholder：當天條件 0 命中（罕見，大盤普跌時可能發生）。
EPS 修正動能 — v1.10 動態 baseline + FY3 加入 FY+1→FY+3 CAGR 欄右方的 ↑+X.X% (vs 5/26) / ↓-X.X% (vs 4/30) 小字，是「本份 Excel forward CAGR」對照「上一份 Excel snapshot」的差值；尾段 (vs M/D) 是**比較基準的日期**（不是資料新鮮度 — 主數字永遠是最新一份 Excel）。Curr / Next / FY+3 EPS 三欄底下也都有同樣 chip — FY1 / FY2 / FY3 各自獨立 vs baseline。hover tooltip 另列 baseline 距今幾天。
• 上修（綠）：分析師對未來 EPS 估算往上調 — 業界已知 alpha factor（Womack 1996 + Bloomberg / FactSet 同名指標），上修股票後續 6-12 個月傾向 outperform
• 下修（紅）：估算往下調，常為基本面領先指標壞訊號
• 持平（|diff| < 0.5%）：不顯示箭頭，避免雜訊
• 「新」chip（灰）：此 ticker 上一份 snapshot 還不在 universe（新加 DD / 新 cover），無對照
• 無顯示：首份 baseline 累積中（snapshot 還沒有 prior）— hover 看 tooltip 確認
CAGR 數字單位細節：CAGR chip 顯示的是「兩個 CAGR % 的絕對差」（兩個百分點之差），但為了視覺與 FY1/FY2/FY3 EPS chip 一致，後綴用 % 而非 pp。tooltip 會標註絕對差語意。
baseline 來源優先順序：(1) 同月內較早的 {YYYY-MM}-DD.json（intra-month 5/20→5/25 這種短期 refresh），(2) 1-2 個月前的月度 snapshot。每月 2 號 09:00 Taipei 自動 snapshot（.github/workflows/monthly-eps-snapshot.yml → scripts/snapshot_eps_estimates.py），存於 docs/dd-screener/eps-estimates-snapshots/YYYY-MM.json。
FunnelRank — 漏斗綜合排序分（v1.3 新增，預設排序鍵） 把「QGM 5/5 → 護城河 → 修正動能 → 時機」四層人工漏斗的前三層（基本面）固化成一個 0–1 排序鍵。時機（第四層）不進排序分，維持 filter —— 與 Entry State Machine 的「基本面排序、時機另看」哲學一致。
FunnelRank = 0.40·QualityGate + 0.30·MoatScore + 0.30·RevisionScore
• QualityGate（0–1）：帶「可原諒條件」的品質閘 —— 5/5 → 1.00；4/5 唯一 fail 在 D/E 或 PEG（資本結構 / 估值，非品質本體）→ 0.85；4/5 fail 在 FCF / ROIC / EPS CAGR（品質本體缺陷）→ 0.50；3/5 → 0.30；≤2/5 → 0.10。任一條件無資料（灰色）→ 該條退出分母、按比例換算後套同一映射，並標 quality_gate_partial。
• MoatScore（0–1）：moat_score/10 × 趨勢乘數（↑ ×1.10 / → ×1.00 / ↓ ×0.80），cap 1.0。無 moat 資料 → 0.50。
• RevisionScore（0–1）：FY1/FY2/FY3 各自上修（diff ≥ +0.5%）= +1 / 持平 = 0 / 下修（≤ −0.5%）= −1；raw = 0.2·FY1 + 0.3·FY2 + 0.5·FY3（FY3 權重最大 = 成長曲線後段），RevisionScore = (raw+1)/2。FY3 上修幅度 > FY1 上修幅度（後段變陡）再 +0.05。「新」chip / 無 baseline → 0.50 中性，標 revision_no_baseline。
• 兩條 hard veto（不進公式，直接處置）：① FY1/FY2/FY3 三欄全部下修 → FunnelRank 強制歸 0、該列沉底 + ⛔（下修是領先指標、品質分是落後指標，打架時信前者）；② 護城河趨勢 ↓ 且 pass_count ≤ 4 → FunnelRank cap 0.50。
權重與映射值寫在 scripts/build_dd_screener.py 檔頭常數（FUNNEL_*），server 端 build 時算好寫進 latest.json（funnel_rank + 三個子分 + 各 flag）。表頭 Funnel 欄 hover 看子分拆解。
  • Ticker：股票代碼（正規格式）。
    來源：dd-meta JSON → scripts/dd_screener_dd_loader.py
    格式：美股 NVDA / 台股 2330.TW / 日股 6857.T / 歐股 RMS（帶交易所後綴的 EU 用內部 suffix map 轉換，DD 名稱保留無後綴形式）
    ⛔ 標記：FY1/FY2/FY3 三欄全下修的 veto 列，ticker 旁顯示 ⛔、整列降透明度並沉底（FunnelRank = 0）。
  • Funnel（漏斗綜合分）：基本面三層合成的 0–1 排序分，表格預設排序鍵（降序）。詳見上方「FunnelRank」callout。
    公式：0.40·QualityGate + 0.30·MoatScore + 0.30·RevisionScore（時機第四層不進分）。
    來源：全部用既有欄位合成（pass_count / 5 條明細 + moat_score/trend + FY1/FY2/FY3 修正動能），scripts/build_dd_screener.py build 時算好寫進 latest.json。
    互動：3 位小數；hover 看 QualityGate / Moat / Revision 三個子分拆解 + veto/cap 說明；點表頭可改回任一欄位排序（原排序選項全保留）。
  • Moat（護城河等級 + 分數 + 趨勢）：護城河評等，格式 等級 分數 趨勢，例如 S 9.8 ↑。
    來源：dd-meta JSON（moat_grade / moat_score）；趨勢箭頭以 DCA Phase A1 為 authoritative source（scripts/update_dd_index.py collect_dca_moat_trend_map()），無 DCA 時 fallback →（保守假設：穩定）
    等級：S（≥ 9.5）/ A（≥ 8）/ B（≥ 6）/ C（≥ 4）/ X（護城河存疑）
    趨勢：↑ 加深 / → 穩定 / ↓ 縮窄
  • FCF（FCF Margin）：TTM 自由現金流 / 營收，百分比。
    來源：優先 docs/qgm/latest.json 或 docs/qgm-tw/latest.json（hard_filter_details.fcf_margin.value）；不在 QGM 中的 ticker 走 scripts/dd_screener_quality.py → fundamentals_us.calc_fcf_margin()（yfinance）
    QGM MLB 門檻：≥ 10%（綠色通過 / 紅色不過 / 灰色無資料）
  • ROIC：NOPAT / Invested Capital，百分比。
    來源：同 FCF — QGM hard_filter_details.roic.value 優先；fallback fundamentals_us.calc_roic()（yfinance）
    QGM MLB 門檻：≥ 15%
  • FY+1→FY+3 CAGR：Excel buy-side forward 2Y CAGR（%）— v1.9 起改用 Excel snapshot。
    來源：Excel cagr_fy1_fy3_pct（FY+1 → FY+3 純前瞻 2 年期複合成長率，目前 202/204 ticker 覆蓋，動態隨 Koyfin watchlist 增刪）；少數 yfinance-only ticker（6146.T / 6857.T 等無 Excel 資料）fallback yfinance「+1y / +2y」row；SU / ABB 因 Koyfin export 資料異常於 scripts/load_eps_estimates_xlsx.py _SKIP_TICKERS 名單中略過。
    窗口語意：跟舊版「yearAgo → FY+1」(含 1 個過去實際 EPS) 不同；現在是 2 個前瞻 estimate (FY+1 vs FY+3) — buy-side 對未來 2 年的純前瞻 expectation。
    修正動能：右方小字 ↑+X.X% (vs 5/26)（綠 = 上修）/ ↓-X.X% (vs 4/30)（紅 = 下修）= 本份 Excel CAGR vs 上一份 Excel snapshot 差值（絕對 pp 差，標 % 為視覺一致）；尾段 (vs M/D) 是**比較基準的日期**（hover 看距今幾天）；|diff| < 0.5 視為持平；「新」chip = 此 ticker 上份 baseline 未收錄。詳見上方「EPS 修正動能」callout。
    QGM MLB 門檻：≥ 15%（顏色基於 Excel forward CAGR；server 端 pass_count 同一欄位 — v1.9 已與 UI 顯示對齊）
  • Curr FY / Next FY / FY+3 EPS：當前 / 下一 / 兩年後會計年度 consensus EPS（純資訊欄無 pass/fail）— v1.8 起 Excel buy-side 為主，yfinance fallback；v1.10 加 FY+3 revision chip + 動態 baseline 間距標籤。
    來源優先順序：(1) Koyfin Excel FY1E / FY2E / FY3E consensus mean — 主路徑，202/204 ticker；(2) yfinance .earnings_estimate 0y / +1y / +2y row — fallback（JP 股、SU/ABB、Excel 缺值）。eps_source 欄記載每個 ticker 走哪一路。
    對應曆年：多數 US Dec-FY 公司：Curr FY ≈ 本曆年、Next FY ≈ 下個曆年；非 Dec-FY 例外（NVDA Jan：Curr = FY27 / Next = FY28；JP 公司 Mar：差約半年）。
    每欄底下修正動能 chip：↑+X.X% (vs 5/26) / ↓-X.X% (vs 4/30) = 此 FY EPS vs 上一份 Excel snapshot 的相對變動 %；(vs M/D) 是**比較基準的日期**（即上一份 Excel snapshot 日；hover tooltip 列距今天數）。「新」chip = 此 ticker 在 prior snapshot 還不在 universe（新加入）。詳見上方「EPS 修正動能」callout。
    非美股 native currency 顯示：.TW / .T / .HK / .KS / .SS / .SZ 後綴的 ticker 自動 back-compute FX（從 yfinance financialCurrency vs Excel USD），表格直接顯示 TWD / JPY / HKD / KRW / CNY 數字 + 後綴 chip；hover 看 USD 原值與 FX rate。
    用途：驗證 forward CAGR 公式（((FY+3 / FY+1)^0.5 − 1) × 100，Excel 直接給）；觀察上修/下修方向強弱對比（FY1 與 FY3 上修幅度差很大 = 成長曲線位移）。
  • PEG：即時前瞻 PEG = live FwdPE / Excel FY+1→FY+3 CAGR — v1.10 起分母改用 Excel forward CAGR，與主欄「FY+1→FY+3 CAGR」一致。之前用 yfinance YearAgo→FY+1 anchored CAGR，對 SBC 重的 SaaS（GAAP TTM 被 SBC 壓低）造成 fake-low PEG（CDNS 0.77 → 修正後 2.32；DDOG 4.42 → 3.79）。當 Excel CAGR 缺值（FY1 負如 NBIS / CRWV / NVTS、或 yfinance-only ticker 如 6857.T）會 fallback 回 yfinance CAGR。
    來源（live）：純 derive — 分子分母都用 live，所以價格動 + 分析師修正都會反映在這欄。
    規則：frozen DD PEG 進 tooltip（hover 看 DD 對比）；live 任一邊失敗時 fallback 顯示 frozen DD 並標 DD。
    QGM MLB 門檻：≤ 2.0x（顏色基於 live 值）
  • 2Y P/E：即時 FY+2 Forward P/E — v1.7.3 簡化為只顯示最新數字。
    來源（live）：每日 scripts/build_dd_screener.py，主路徑用 yfinance FY EPS 估算（live_fpe_method = 'eps'）；EPS 抓不到時 fallback 用 fpe_fy2 × price_now/price_at_dd 漂移推算（標 *，live_fpe_method = 'drift'，假設 EPS 未 revise — 半年以上會高估「變貴」程度）。
    規則：純資訊欄，不參與 pass_count；無配色 / 無 DD 對照 — 想知 DD 寫入時的對比請看 /research/ 表格的 data-pe2y 欄位。
  • D/E（Debt/Equity）：Total Debt / Stockholders Equity。
    來源：QGM hard_filter_details.debt_to_equity.value 優先；QGM 有 D/E 缺口（如 ANET 零負債）時補抓 yfinance balance sheet；Negative equity（如 FICO / DELL / SBUX）回傳 null（負值無意義），顯示「—」
    QGM MLB 門檻：≤ 0.7x（上限）
  • 通過（Pass Count）：以上 5 條 QGM-MLB 門檻（FCF / ROIC / EPS / PEG / D/E）通過的數量。
    計算：client-side 即時計算，隨 Custom preset 的 slider 數值同步更新
    顏色：5/5 深綠 / 4/5 黃棕 / 3/5 淡黃 / < 3 紅
    Tab 分組：完全符合（5/5）/ 差一條（4/5）/ 差兩條（3/5）/ 全部
  • 5Y IRR：即時 5 年期年化 IRR（%）— v1.7.3 簡化為只顯示最新數字（heuristic 推算）。
    來源（live）：從 DCA §4 frozen IRR 用 (price_at_dd / price_now) 回推：
    live_ev5y = ((1 + ev5y/100) × (price_at_dd / price_now)^0.2 − 1) × 100
    直覺：股價跌 → 起點變便宜 → 同樣 5 年 terminal → IRR 變高（合理）。標 * 提醒是推算非實算。
    Caveat（重要）：假設 DCA 寫入時的 terminal value（5 年後目標價）未變。如果 thesis broke（基本面惡化、target 砍半），此 live IRR 會誤導你以為「跌很多 = 更便宜的機會」。建議搭配最新 DCA 報告判讀，不要單看此欄就加倉。
    顏色：≥ 12%/yr 深綠 / 8–12% 中綠 / 0–8% 琥珀 / < 0% 紅（依數值上色，不是對比 DD）
    frozen IRR 原始來源：scripts/update_dd_index.py collect_dca_ev_map() 解析最新 DCA HTML §4 EV
  • MA（長期均線健康度）：股價相對 W52（1 年）與 W250（5 年）週均線的位置與斜率。
    來源：scripts/dd_screener_ma.py → yfinance 抓 5Y 週線資料（1wk interval）
    計算：w52 = 52 週 SMA / w250 = 250 週 SMA / slope_w250_pct = W250 現值 vs 4 週前 W250 的漲幅% / drift_4w_pct = 股價 4 週漲跌%
    燈號：🟢 healthy（W52 ↑ ∩ W250 ↑ ∩ slope > 0）/ 🟡 mixed（W52 ↑ ∩ W250 ↑，slope 平或負）/ 🔴 weak（跌破任一 MA）/ ⚪ n/a（歷史不足，如 ARM / NU / VIK / SNDK 上市 < 5 年，改依 W52 判斷並標示 W250 N/A）
  • 距 ATH（距 52 週高點）：當前股價離 52 週調整後收盤最高點的百分比距離。0% = 站在 52 週高；負值 = 低於高點。
    來源：美股 — docs/screener/latest.json dist_52w_high_pct（每日 06:00 TPE 更新，與 Flow ATH Hunter 同源）；非美股及不在 screener universe 的 DD ticker — build_dd_screener.py compute_yfinance_timing_fallback() 批量抓 300d 日線計算（timing_source: "yfinance_fallback"）；TW / JP / EU 若 fallback 也無法取得則顯示「—」
    顏色：🟢 sweet [-7%, 0%] 起漲點甜區 / 🟡 warm [-15%, -7%] base 築底（回調帶 preset 的 distATH 區間）/ ⚪ cool < -15% 深度回調 / 🟠 hot > +2% extended 已突破延伸
  • 時機 preset（Any / 起漲點 / 回調帶）：在基本面 FunnelRank 排序之上，疊加日線時機篩選（與排序分相互獨立）。
    起漲點：距ATH ∈ [-7%, 0%] ∩ 距50DMA ∈ [0%, +5%] ∩ RS ≥ 80 —— 剛突破、未延伸。
    回調帶（v1.3 新增）：距ATH ∈ [-15%, -7%] ∩ MA 燈號 🟢（W52↑ ∩ W250↑ ∩ slope>0，或 <5y 上市以 W52↑ 判斷）∩ 距50DMA ∈ [-3%, +5%] —— 創過新高、正在回測、長期趨勢未壞。語意與 Entry State Machine 的 PULLBACK_WATCH 同構（日線版；週線版見 Entry State Machine）。
    非美股：TW/JP/EU 無 timing 資料者在起漲點 / 回調帶模式自動排除。
  • vs 50DMA（距 50 日均線）：當前股價相對 50 日 SMA 的百分比偏離。正值 = 股價在均線上方。
    來源：同「距 ATH」— screener ma50_pct 優先；fallback yfinance 批量抓 300d 日線，取末 50 根收盤均值計算
    顏色：🟢 sweet [0%, +5%] 剛收復均線、未過熱 / 🟡 warm [+5%, +15%] 延伸但可控 / 🟠 hot > +15% 過熱 / ⚪ cool < 0% 跌破 50 日均線（趨勢受壓）
  • RS（Relative Strength Score）：相對強度評分 0–100，相對 S&P500 全體成份股的百分位排名（多時間框架加權）。
    來源：美股在 screener universe — docs/screener/latest.json rs_score（EMA 平滑的 1w/4w/13w 加權複合分數，與 Flow ATH Hunter 同源）；不在 screener universe 的 DD ticker — fallback 批量抓 300d 日線，計算原始 1w/4w/13w 報酬率，對全 screener universe（~511 股）做百分位排名後加權（公式：r1w×0.2 + r4w×0.3 + r13w×0.5 + trend_bonus ±5）；TW / JP 本地上市股對同一美股 universe 排名（跨幣種，方法上有 FX 雜訊，以一致性優先）
    顏色：深綠 ≥ 80 強勢 / 中綠 60–79 OK / 灰 40–59 中性 / 淺灰 < 40 弱勢
    起漲點門檻：≥ 80
  • Links（DD / DCA）：快速連結。
