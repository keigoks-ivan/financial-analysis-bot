產品與市場
Product-Market Fit（PMF）
Marc Andreessen · 2007 · 《The Only Thing That Matters》Marc Andreessen部落格文章 / 後續由Andy Rachleff系統化
起源故事
Marc Andreessen 在2007年寫了一篇部落格文章，試圖解釋為什麼有些新創公司在幾乎所有方面都做得很糟糕，卻仍然成功；而有些公司在幾乎所有方面都做得很好，卻最終失敗。他的結論是：唯一真正重要的事情是產品市場契合（Product-Market Fit）——在一個好市場裡，有一個能滿足那個市場需求的產品。在達到PMF之前，什麼都不重要；在達到PMF之後，大多數問題都會變得相對容易解決。他對PMF的描述帶有強烈的感覺成分：「你能感覺到它。用戶把你的產品分享給他人。需求超出你供應的速度。媒體開始報導。」
核心邏輯
PMF的核心是一個簡單但殘酷的真相：大多數新創公司死於沒有找到PMF，而不是死於執行問題。框架的操作定義有幾個版本：Sean Ellis測試（如果你的產品消失了，有多少用戶會非常失望？超過40%意味著你有PMF）；NPS（淨推薦值）持續高於50；留存率曲線趨於平穩（不是一直下降）；有機成長超過付費成長。Andreessen強調，在達到PMF之前，你應該做任何必要的事來找到它——包括改變產品、改變市場、改變用戶群，甚至改變整個商業模式。在達到PMF之前投入資源在行銷和銷售是一種浪費。
▸市場的品質比任何其他因素都更重要——一個糟糕的市場會打敗好的產品和好的團隊
▸PMF不是一個二元狀態，而是一個程度——你可以有「一點PMF」或「強烈的PMF」
▸在達到PMF之前，所有的擴張行動（雇人、行銷、銷售）都是在浪費錢
▸PMF是可以被感覺到的，但感覺往往落後於數據
實戰應用
✓ 該做：每個月追蹤一個核心PMF指標（留存率或Sean Ellis測試）。如果留存率曲線持續下降沒有趨於平穩，你還沒有PMF，應該停止所有增長實驗，專注於產品本身。
✕ 避免：不要在找到PMF之前大量雇人。早期雇人的最大成本不是薪資，而是複雜度——每增加一個人，溝通成本呈指數增長，讓你更難快速迭代找到PMF。
框架的內在矛盾
PMF框架最大的問題是它是一個事後的概念——你只有在已經有了一定規模的用戶之後，才能測量留存率和推薦率。但在達到那個規模之前，你怎麼知道自己是在正確的路上？更深的矛盾是：PMF可能是短暫的。Myspace曾經有強烈的PMF，然後Facebook出現了。PMF描述的是一個時間點的狀態，而不是一個永久的成就。另一個矛盾是：在某些B2B市場，「用戶」和「付費方」是不同的人，你可能有很強的用戶PMF但完全沒有商業PMF——用戶愛你的產品，但付費方不願意為此付錢。
仍然有效 vs 正在演化
✓ 仍然有效
PMF作為北極星指標的概念在任何時代都有效。在AI工具降低了建產品成本的時代，「更快找到PMF」變得更加重要，因為更快速的迭代是現在最大的競爭優勢。
⟳ 正在演化
AI時代的PMF有一個新的複雜性：AI工具讓產品的初始能力門檻大幅提高，用戶的期待也相應提高。2026年，「夠好」的PMF門檻比2015年高得多——用戶現在有太多選擇，你需要更強的PMF信號才能確認自己找到了它。
2026 年版本建議
把Sean Ellis測試的門檻從40%提高到60%。在AI工具氾濫的時代，40%的門檻已經不足以區分真正的PMF和暫時的新鮮感。
如何應用這個框架
如果你還沒有足夠的用戶做Sean Ellis測試，用代理指標：你的用戶有沒有自發地推薦給別人？如果你停止主動推廣，使用量是上升、持平還是下降？答案告訴你距離PMF有多遠。
