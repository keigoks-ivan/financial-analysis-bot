Default Alive vs Default Dead
Paul Graham · 2015 · Paul Graham YC Essay《Default Alive or Default Dead?》(2015)
起源故事
Paul Graham 在2015年寫了一篇短文，起因是他在YC看到太多創辦人對自己公司的財務狀況完全沒有直覺。他們知道銀行裡有多少錢，但他們不知道——也沒有認真想過——按照現在的成長速度和燒錢速度，公司是走向活著還是死亡。更讓他擔心的是：大多數創辦人把這個問題的答案留給「下一輪融資」來解決。Graham認為這是一種極度危險的思維，因為它讓你在不知不覺中把公司的生死控制權交給了外部投資者，而不是掌握在自己手裡。
核心邏輯
這個框架的核心問題只有一個：按照現在的收入成長速度和支出速度，在錢燒完之前，公司能不能在不需要新融資的情況下達到損益平衡？如果可以，你是Default Alive；如果不可以，你是Default Dead。三個關鍵計算：跑道（銀行餘額 ÷ 每月淨燒錢量，只算已到帳的錢）；到達損益平衡需要多久（基於誠實的收入成長預估）；兩者的差距。Default Dead的公司在融資談判中毫無籌碼——投資者知道你急，會給出很差的條款。Graham最重要的警告：很多創辦人在Default Dead的狀態下，仍然像Default Alive一樣行動——繼續雇人、燒錢、追成長。
▸把Default Alive/Dead的計算設為每月例行工作——不能超過15分鐘
▸融資談判需要至少6個月的跑道作為籌碼，跑道少於6個月時開始融資往往條件很差
▸「下一輪融資會解決問題」是最危險的思維模式
▸Default Dead不是死刑——它是一個需要立即決策的警報
實戰應用
框架的內在矛盾
Default Alive框架推到極端，會讓你永遠選擇保守。但很多市場是贏者全拿的，在這些市場裡，太保守的創業者反而輸給了敢於燒錢佔市場的競爭者。Uber和Airbnb在早期都是Default Dead的——如果他們嚴格遵守Graham的框架，可能在「安全」的狀態下被市場淘汰。更深的矛盾是：如何判斷你所在的市場是「需要快速燒錢佔位」還是「需要資本效率」？這個判斷本身沒有公式。
