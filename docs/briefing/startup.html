The Feedback Loop Framework
多位創業者 / 代表：Julie Zhuo（前Facebook VP Design） · 2019 · Julie Zhuo《The Making of a Manager》(2019) / Ray Dalio《原則》的Feedback文化 / 矽谷工程文化的集體積累
起源故事
Julie Zhuo 在Facebook從實習生成長為VP of Design的過程中，發現反饋文化是高績效組織和平庸組織之間最大的差異之一。她觀察到：大多數組織的反饋是單向的、延遲的、和模糊的——管理者在年度評估時給反饋，用委婉的語言表達，讓接受者無法從反饋中學到有用的東西。Ray Dalio在Bridgewater建立的「激進透明」（Radical Transparency）文化走向另一個極端，讓所有的反饋都是即時的、直接的、和公開的。對早期創業公司，建立一個有效的反饋迴路——包括用戶反饋、產品反饋、和團隊間的反饋——往往是快速學習和改善的關鍵。
核心邏輯
有效的反饋迴路有四個特點：及時（越快越好，不要等到事後）；具體（「你的設計很好」是讚美不是反饋，「這個按鈕讓我看不到主要內容」是反饋）；可行動（接受者能做什麼改變）；和清楚的意圖（是為了幫助改善，不是批評或攻擊）。對早期創業公司，最重要的反饋迴路是用戶反饋——不只是定性的（用戶說他們喜歡什麼），還包括定量的（用戶實際在做什麼）。兩者的結合——用戶說的和用戶做的——往往比任何單一來源都更有洞察力。
▸反饋越及時越有效——延遲的反饋失去了大部分的學習價值
▸用戶說的和用戶做的往往不同——兩者都需要收集
▸給反饋是技術，接受反饋是更難的技術
▸建立系統性的反饋迴路，讓學習自動化而不是依賴個別的互動
實戰應用
框架的內在矛盾
反饋迴路框架的問題在於：太多的反饋可能讓你陷入分析癱瘓——每個用戶都有不同的意見，你無法同時滿足所有人。更根本的問題是：用戶的反饋往往是在他們現有的認知框架內的，他們無法告訴你一個他們從未使用過的功能是否有用。過度依賴用戶反饋可能讓你建出一匹更快的馬，而不是一輛汽車。
