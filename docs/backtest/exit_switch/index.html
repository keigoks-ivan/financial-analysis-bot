切換從不打贏純 W52;「最好的切換版」只是最少切換的那個 — 而且切換方向是反的

擁有者假設(2026-06-24):用上漲斜率判別出場法 — 陡/抖的上漲用 ATR 自適應的 Supertrend(10,3)鎖緊獲利， 平緩的趨勢用 W52 讓它跑。在 SMH/QQQ 用四個判別(斜率陡峭度 / 上漲波動度 / Kaufman 效率比 / 陡且抖)、 Schmitt 雙門檻遲滯、停損 ratchet 只升不降、兩種進場(趨勢 / E3)、完整門檻掃描測試。 結論：沒有任何一個切換版打贏純 W52。三個現象把它釘死 —— ①「最好的切換版」(vol / combo)其實是 hot% ≈ 0、幾乎從不切到 ST 的那個，等於 W52 的偽裝； ②門檻掃描單調、沒有內部甜蜜點：越常切 ST(hot% 越高),Calmar 越差； ③切換方向是反的 —— 斜率判別只在上漲腿把停損收緊到 ST(製造 whipsaw 拖累報酬), 但真正崩盤時斜率已轉負 = 判為平緩 = 用回 W52,所以「在不需要保護時收緊、需要保護時放鬆」， MDD 完全沒換到改善。根因：在這兩檔、這段期間，W52 本身就是比 ST(10,3)更好的出場(報酬與回撤兩軸都贏), 不存在「切到 ST 會賺」的 regime。唯一站得住的訊號是跨時代(ST 在 2016 後贏 W52)而非跨斜率 —— 那是另一個(且有前視風險)的設計。驗證錨：純 ST(10,3) 與已發布 supertrend 頁數字一字不差，接線無誤。

設計

切換機制(擁有者選定)

  • 兩出場法:W52(週收 < 52週MA,很鬆)vs ST(10,3)(週線 Supertrend 停損線，ATR 自適應、較緊)
  • 每週動態切換:陡/抖 → 用 ST 線、平緩 → 用 W52 線
  • Schmitt 遲滯:雙門檻(上門檻轉 hot、下門檻轉 mild、中間維持),避免在門檻附近來回跳
  • 停損 ratchet 只升不降:regime 切回 W52 時停損不放鬆，已鎖獲利不吐回
  • 無前視:當週收盤決策、次日生效(沿用已驗證 runner / 7bps / 閒置 SHY-BIL)

四個判別 × 兩個進場(都測)

  • 斜率陡峭度:30週MA 年化斜率 > 門檻 → ST(最貼近「斜率」)
  • 上漲波動度:週報酬年化實現波動 > 門檻 → ST(最貼近「抖」)
  • 效率比 ER:Kaufman 效率比 < 門檻(走勢曲折)→ ST
  • 陡且抖:斜率與波動同時超標 → ST(二維)
  • 進場:趨勢(週收>W52,乾淨隔離出場法)與 E3(W40/W52/TSMOM 多數決)各跑一次
  • 防過擬合:門檻不挑單一最佳，跑完整掃描 + H1/H2 拆段 + 回報 hot%(在場使用 ST 線的比例)

藍底 = 純 W52(全勝)。hot% = 在場期間使用 ST 線的週數比例；純法為 0、B&H 為 —。

門檻敏感度掃描 — 沒有內部甜蜜點

純 ST 在 2016 後的低利率融漲時代 Calmar 確實高於純 W52(SMH 1.35 vs 1.14、QQQ 0.80 vs 0.69), 但在 2006-2015(GFC 時代)大輸，全期被 H1 拖垮。問題是：斜率 / 波動 / ER 這些盤中指標在兩個時代都會觸發， 抓不到「時代」這條界線。要利用 ST 這個優勢，判別變數得是總經 / 利率 regime 或 crash 偵測 —— 完全不同、且有前視風險的設計， 不是本頁的斜率假設。記錄在案，不在此採用。
