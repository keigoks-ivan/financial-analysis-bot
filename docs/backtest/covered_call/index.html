三個 overlay 變體全部淨負貢獻；covered call 是效用交換，不是報酬來源

研究問題：在本站已驗證的 STX50 引擎（SMH/QQQ 進攻趨勢的 QQQ 腿・E3 三訊號分數倉＋週線 Supertrend 半倉閘門）上， 疊加賣 ATM 月 covered call 會如何？用狀態切片法把被 cover 部位的報酬換成 ^BXN（Nasdaq-100 BuyWrite）， 測 baseline vs 全 cover（A）／半倉 cover（B）／猶豫帶 cover（C）三變體。結論：三變體全部淨負貢獻，沒有一個過尺。 毛保費現金流很漂亮（16.7 年累計約 $11.17M），淨貢獻卻是大負（全 cover -$5.56M）——收到的權利金遠遠補不回被 cap 掉的趨勢漲幅。 趨勢系統設計就是要吃右尾，而 covered call 正好把上檔削掉，是結構性錯配。 這頁不是報酬型策略推薦，是把「賣方視角的流水」與「真實經濟貢獻」分開看的一份反例。

逐年毛保費（全 cover）vs 淨貢獻（全 cover／猶豫帶），單位：千美金

現金擔保 put — covered call 的現金版

機制上，賣現金擔保 put 與賣 covered call 由 put-call parity 等價：一個是持股＋賣 call，一個是持現金＋賣 put， 兩者的損益曲線在同一履約價下相同。CBOE 的 ^PUT 指數（S&P 500 PutWrite，賣 ATM put＋一個月期 T-bill 抵押）給了現金版的長期基準。

「用閒置現金賣 put 收租」的兩難：^PUT 比純現金多賺約 5.7pp，但下跌月與 SPY 相關 0.85—— 崩盤時被 assign 在底部，本該當保險的現金桶反而在最需要現金的時候被套牢，失去保險功能； 若拿來對照「直接把現金投入指數」，又輸給 buy-hold。它坐在「比現金好、比 buy-hold 差」的中間地帶。
台股警告（TXO）：台股沒有可測的 put-write 基準指數，實務是賣 TXO 台指選擇權，保證金制度＋跳空風險比美股兇險一個量級。 2018/02/06 選擇權事件示範了最壞情況：深度價外的 call 與 put 同時暴漲、賣方被強制平倉，一夜之間多個帳戶穿倉。 在台股複製「賣方收租」的敘事前，先把這個尾部風險放進場景。

為什麼還是有人推薦？

  1. 波動風險溢酬（VRP）真實存在 — 選擇權的隱含波動長期高於已實現波動（IV > RV），賣方確實收得到這個溢酬。問題是它是否補得回被 cap 掉的右尾。
  2. 經典研究取樣的是黃金年代 — Whaley（2002）、Ibbotson 等常被引用的正面結果，樣本落在 1988-2004；之後策略擁擠（大量 buy-write ETF）＋低波長牛，讓 edge 顯著衰減。
  3. 盤整年與慢速空頭確實正貢獻 — 本回測裡 2011／2016（盤整）與 2022（慢速空頭）cover 都是正的；只是這些年份是少數，牛市年的機會成本把它們全吃回去。
  4. 「收租」敘事好賣 — 每月一張看得見的權利金現金流，行為上比「等右尾」好拿。但高分配率的 covered call ETF，其 total return（含分配再投入）長期落後其標的，分配率高不等於報酬高。

那 covered call 適合誰？五類人

共同點：covered call 不是報酬來源，是在「不能賣／不想賣／不指望漲」的約束下，用右尾換現金流與路徑平滑的效用交換。

① 稅務鎖倉的集中持股者

部位因稅務或流動性無法賣出，賣 call 收到的保費是純增量現金流——因為右尾本來就拿不走，cap 它沒有機會成本。

② 本來就設好目標賣價的人

賣 call 等於掛限價單先收錢：到價被 assign 正是原本就想賣的價位。前提是「賣價」這個決定必須先於「賣 call」存在，而不是為了收租硬設一個賣價。

③ 提領期投資人

提領期的序列風險使「較低波動的報酬路徑」有真實效用；平滑的淨值曲線也讓人在行為上拿得住，不會在低點恐慌賣出。用一部分右尾換路徑穩定，對這類人是划算的。

④ 真心預期長期盤整的人

若真的預期標的長期橫盤，右尾本來就薄，賣掉它的機會成本低。但這要求 regime 預測正確——賭錯方向（實際是牛市）代價很大，本回測的牛市年就是反例。

⑤ 授權受限的機構

不能減持（授權或指令限制）但可以寫選擇權，或投資人明確要求月配息。此時 covered call 是在約束下擠出現金流的工具，不是為了跑贏標的。

已知限制（誠實原則）

  1. ^BXN 起點 2009-09，測不到 2008 — 2008 GFC 正是 covered call 理論上最能緩衝的持續性大空頭；本回測涵蓋的最大空頭是 2022（該窗 cover 確實正貢獻），但 GFC 級事件未驗證，這對全 cover 的評價偏保守。
  2. ^BXN = NDX buy-write，非 QQQ — NDX vs QQQ 總報酬基差（QQQ 有管理費與追蹤誤差、指數沒有）＋指數無交易費用，屬已知近似。
  3. 毛保費是 BS 近似、非真實成交 — 忽略 skew、實際 bid-ask、被 assign 的摩擦。實際成交的價差與被指派成本會讓淨貢獻更負，不會更好。overlay 部分未另計交易成本（^BXN 指數已含 roll），這對 cover 變體偏樂觀。
  4. 指數無摩擦成本，對 cover 變體偏樂觀 — 上一點的延伸：所有近似的方向都讓 cover 看起來比實際成交更好，故「三變體皆負」的結論在加上實務成本後只會更強，不會翻正。
  5. SMH 攻擊腿沒測 — SMH 無對應的 BuyWrite 基準，無法用同一狀態切片法；但半導體是本波 AI 行情漲最兇的一段，covered call 的機會成本方向上只會比 QQQ 腿更嚴重。

狀態切片法與引擎定義

引擎（零參數，結構複製自 repo）：QQQ 每日持倉 wt = 0.5·E3 + 0.5·min(E3, ST)， 其中 E3 =（W40 + W52 + TSMOM）/ 3（三訊號分數倉，值域 {0, 1/3, 2/3, 1}），ST = 週線 Supertrend(10, 3) 方向（0/1，只當半倉出場閘門，只減碼不加碼）。 引擎對齊驗收：用同一套程式重建 SMH/QQQ 50/50 STX50，得 CAGR 13.84% / MDD -21.87% / Calmar 0.63，與 results/e4_ensemble.json 逐位一致。

四條報酬流（成本 7bps／側，僅對 baseline 換手排程，四者一致）： Baseline = w·rQQQ＋（1−w）·rcash； A 全 cover 把 rQQQ 換成 rBXN； B 半倉 cover 用 0.5·rBXN＋0.5·rQQQ； C 猶豫帶 cover 僅在非滿倉日（w < 1）換成 ^BXN，滿倉日照 QQQ。
