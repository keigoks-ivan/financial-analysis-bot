⚠ 候選系統 — 未採用

紅旗 4/4 通過，但「不大賠」依賴收盤 MOC 成交 — 矩陣裡所有次日開盤變體都破限

設計目標是雙軌趨勢系統的互補：在盤整期接拉回、用 200DMA 濾網 + 時間停損切掉左尾，事前明文「不能小賺大賠」的 四條紅旗驗收(最大單筆 ≥ -8%、虧獲比 ≤ 3、PF ≥ 1.2、壓力事件暴險 ≤ 30%)。20 年真實資料：基準版兩標的全數通過， 2008 GFC 暴險 0%。但 32 組合矩陣揭露單點依賴：全部 16 個「次日開盤成交」變體最大單筆虧損 -9.6% ~ -11%,破紅旗 — 自動化實作必須用 MOC 單在收盤成交，這不是可妥協的細節。組合分析的結論分歧：對(已否決的)雙軌，0.34 低相關讓 MDD 單調改善；但對採用中的 E3 / LT SMH/QQQ,加 MR sleeve 相對「同權重改持現金」是 MDD/Sharpe 換 CAGR 的真實取捨 (2026-06-30 MR 加計閒置現金收益後，CAGR 已轉為全面領先現金組，Calmar 伯仲之間；現金組仍勝在 MDD 較淺)—— 不再是單純的「被現金壓制」，是否納入屬另一輪決策，本頁不代為判斷。

規則(基準版)

進出場

  • 濾網:close > 200DMA 才允許持倉；跌破即強制出場
  • 進場:空手 AND RSI(2) < 10 → 當日收盤全額買進(MOC)
  • 出場(先觸發者):RSI(2) > 70 / 持滿 10 交易日 / 跌破 200DMA
  • 無價格停損 — 刻意設計：MR 的價格停損常停在最大不利點；左尾由濾網 + 時間停損 + 短持有期控制
  • 成本：單邊 7bps(來回 14bps),與全站一致
