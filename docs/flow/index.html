雙軌投資流程

70% 指數軌（機械訊號）＋ 30% 個股軌（判斷選股）＋ 8~10% 現金。兩軌各自獨立、不交叉污染，共用一層「市場情境」——情境層只餵個股軌的質性判斷，絕不干預指數軌的機械訊號。本頁把全站工具串成一條可執行的路徑，每個工具名皆可直達。
投資流程 實單主系統 選股主控台 市場監測 量化回測

總架構：兩軌 ＋ 現金 ＋ 共用情境層

資金先切三塊，兩條投資軌用完全不同的決策模式，互不干涉。

指數軌

70%
純機械。唯一訊號源＝W52 × 自適應波動率 150% 實單主系統（美 QQQ+SMH／台 0050+2330）。不看盤、不預測、不手動干預，只執行 email 訊號。

個股軌

30%
純判斷。想法 → 漏斗 → 深度研究 → 三錨裁決 → 席位制進場 → 監控 → 歸帳（下方七步）。質性、逐案、有容量上限。

現金

8~10%
另計的乾火藥。用於個股軌機會與再平衡緩衝，不併入兩軌績效計算。
共用市場情境層（只讀、不驅動指數軌）：市場監測、大類資產 regime、產業輪動、擁擠交易監測——這層是給個股軌的質性背景（該加減哪個主題、哪裡太擠）；指數軌完全無視這層，只認自己的機械訊號，避免「看法污染系統」。

指數軌 · 五步機械

整條軌只有一個訊號源、一組執行規則、一條禁止手動干預的鐵律。無信 ＝ 不看盤。
  1. 唯一訊號源 ＝ W52 × 自適應波動率 150% 實單主系統
    cap 1.5、email 驅動。每交易日台股／美股收盤後自動檢查，只有可行動變化才寄信（任一腿閘門翻轉，或目標與現持差 ≥ 10pp）。沒收到信＝不動作。
  2. 執行
    ≤ 100% 部位用現貨（QQQ/SMH／0050/2330）；槓桿段（100~150%）用期貨（美股 MNQ、台股 2330 小型個股期＋小台指）。口數與名目換算用 期貨部位計算機。
  3. 禁止手動干預條款
    不因新聞／盤感／回撤而提前加減碼；不覆寫閘門；不「這次不一樣」。系統的價值來自紀律地執行機械訊號，任何手動 override 都是在破壞回測所依據的規則。
  4. 回顧點（非前置關卡）
    滿 60 個追蹤交易日或首次 ≥ 10% 組合回撤時回顧一次：主頁 cap 1.5 vs cap 1.0 影子 vs 舊實倉（STX50/E3）三線並讀，確認水下體驗與回測分布一致；背離才按 charter 調整或降回 cap 1.0。
  5. 商品 sleeve（同紀律）
    指數核心之外的 商品 Sleeve（GLD/USO 訊號 → 微型期貨），同樣是機械訊號、同樣不手動干預；作用是與股票趨勢家族低相關的危機分散腿。

個股軌 · 七步漏斗判斷

想法很多、席位很少。用漏斗收斂、用容量控制排隊、用三個獨立錨點裁決，買的那天就定義好賣的條件。
  1. 想法四路
    四條進料管道並行——
    品質路：DD Screener 品質進場、QGM 美股／QGM 台股、Momentum-5、RS+VCP Screener
    主題路：產業輪動 ＋ ID Tier Matrix
    事件路：偵測警報網 ＋ 催化劑日曆
    財報路：財報分析
  2. 漏斗收斂 ＋ 席位制容量控制
    所有想法進 選股主控台 收斂排序；席位有限——新想法必須贏過現有席位才擠得進來，強制排序而非無限加倉。
  3. 深度研究
    留下的候選做深度盡調：個股 DD 家族、產業 深度 ID、供應鏈地圖。
  4. 裁決三錨（三錨不同意 ＝ 不進場）
    蒙格獨立裁決munger-mind 外部裁判，讀語料＋DD 獨立對質
    ID 冷審產業 ID 冷讀 checklist，thesis 是否仍成立
    帳本實績裁決實績，過往同類判斷的實際命中率
    三個錨點互相獨立、任一否決即不進場——避免單一視角過度自信。
  5. 席位制進場
    買的那天就定義賣的條件：進場即寫下否證指標（thesis 被推翻的量化門檻）並存進 DD，之後由監控層盯這些門檻。
  6. 自動監控
    持倉週掃（position-thesis-monitor 每週掃全持倉的否證指標與 thesis 老化）、kill-watch、偵測警報網、催化劑鏈——被動抓到人眼會漏掉的沉默衰退。
  7. 出場歸帳
    出場後歸帳進 裁決實績，餵回 flywheel——每次判斷的對錯都留痕，讓下一輪裁決三錨的「帳本實績」更準。

節奏表

大部分日子什麼都不做。固定節奏把「該看什麼、什麼時候看」寫死，避免盤感驅動。
頻率時間盒做什麼
每日< 5 分只處理 email。四種信：①指數軌可行動變化（閘門翻轉／≥10pp）②偵測警報③催化劑觸發 ④持倉否證指標破線。無信＝關掉；想多看一眼盤面，開首頁市場現況四磚被動掃 30 秒就好，不必逐頁巡、不是盯盤。
每週六早~ 30 分財報分析掃下週 EPS ＋ 總經行事曆，標記要盯的事件。
週日晚／週一早~ 45 分週會：regime／rotation／crowding 過一遍 → 選股主控台席位排序 → 持倉週掃報告 → 定本週動作。
每月（2 號後）~ 20 分EPS 快照更新，對照持倉 thesis 的財務假設是否還成立。
每季事件觸發指數軌回顧點（若滿 60 日或 ≥10% 回撤）、ID staleness 重審、出場歸帳、裁決三錨對帳。
節奏的重點是把「不作為」變成預設：指數軌無信不動、個股軌無席位不加，週會才是唯一主動決策的窗口。
