🔴 大錯 ①：OpenAI "Titan" $100B+ 是 gross infrastructure value 估算，不是 AVGO contract revenue
事實：OpenAI / Broadcom 官方 2025-Q4 聯合公告僅敘述「10 GW custom AI accelerators by end of 2029」— 沒有 dollar 數字。$100B 是 Citi 低端估算（gross infra value），FT 用 $35B/GW 行業 benchmark 推算則 $350-500B。AVGO 實際拿到的是 silicon + design service portion，不是整個 rack 的 gross 數字。原 §2 catalyst（line 310）/ §4 TAM 註（line 430）/ §6.1 player table（line 499）/ §10.5（line 715）/ §12 NC#1（line 786）全部以「$100B+ 合約」anchor 推 40%+ ASIC penetration，這個 anchor 估值偏高。應改為「OpenAI 10 GW infra → AVGO silicon revenue 約 $15-25B over 2026-2029」。

🔴 大錯 ②：AVGO 毛利 50%→<40% 的 mechanism 邏輯倒置（user 點 2 + critic 獨立確認）

灰色地帶（為何我們花時間定義）：

💡 Insight §1
真正的「設計服務鏈」只有 AVGO + MRVL + Alchip + GUC + MediaTek + Socionext 六家公司（加 AVGO/MRVL 的一小部分業務）。市場常把 EDA（SNPS/CDNS）混入，但 EDA 賣工具，設計服務賣「成品交付 + 產能承諾」，兩者毛利結構、客戶黏著度差 3-4 倍。
投資意義：若要 pure-play ASIC 設計服務，首選 Alchip（3661），因其近 100% 營收來自 hyperscaler AI ASIC；AVGO 有 VMware + 網通稀釋，MRVL 有存儲 + 網通稀釋。
💡 Insight §1（二）
MediaTek 從手機 SoC 跨入 ASIC 是 2015-2020 年 AVGO 從網通跨入 AI 的重演。當年市場不信 AVGO 能打敗 NVDA，結果 AVGO 現在拿 70% ASIC 市佔。MediaTek 若在 2028 前拿下 3-5 個 hyperscaler flagship（目前 Google 2 個、傳洽 OpenAI 1 個），可成為第三極。

★ TSMC 官方 2026 symposium：ASIC chiplet 拼接路徑（2026-2029 完整節點對）

本 ID v1.0 對 ASIC 設計服務的節點討論偏向「3nm / 2nm 升級」這種粗略描述。TSMC 在 2026 北美技術論壇直接揭示了未來四年「一個 CoWoS 內」的 chiplet node pair 路徑，這對設計服務商（AVGO / MRVL / Alchip / GUC / MediaTek）的每個世代客戶 NRE 投資、wafer cost 攤提、與 design service margin 結構都有具體影響。

1）2026-2027 「跨代拼接」是設計服務的甜蜜點

2026 SoIC N3P+N4 與 2027 SoIC N2P+N3P 都是跨代拼接（compute die 用最先進節點 + I/O / SerDes / memory controller die 用上一代）。這個設計哲學對台系設計服務鏈是個非常具體的利好：

  • Alchip / GUC / MediaTek 過去的核心競爭力是 N4 / N3 backend 設計。2026-2027 跨代拼接需要他們把 I/O die（N4 / N3P）做完整 backend；compute die 才走 leading-edge（N3P / N2P，多由 AVGO 高 margin 段或 hyperscaler in-house 處理）。
  • NRE 攤提結構： compute die 的 NRE（leading-edge）約 $150-250M / project，I/O die 約 $40-80M。Alchip / GUC 接的多是 I/O / supporting die，NRE 規模較小但 take rate 高。在「跨代拼接」階段（2026-2027），每一個 hyperscaler ASIC 等於是兩套 backend service — 對 design service 廠是「一個專案賺兩次」。
  • 9.5-reticle (2027) 的物理空間擴大允許更多 die-on-CoWoS：5.5-reticle 上是 4 顆 SoC（2026），9.5-reticle 上是 8 顆 SoC（2027），翻倍。這直接放大了「supporting die backend」的工作量。

2）2028 起「同代雙拼」把小型 ASIC startup 擠出，rev 反向集中

2028 SoIC N2P+N2P 與 2029 SoIC A14+A14 都是同代雙拼 — 兩顆 die 都用當代最 leading-edge 節點。對設計服務鏈的衝擊是：

  • NRE 規模拉到 $400-600M / project：兩顆都是 leading-edge die，NRE 翻倍以上。能撐這個規模的客戶只剩 NVDA / AVGO 大客戶 + 大 hyperscaler（Google / AWS / Meta / MSFT）。
  • 對 Groq / Cerebras / SambaNova 等 ASIC startup 是 kill blow：原本還能用「mixed N3P+N5 chiplet」省 NRE 的策略，2028 起變成必須兩顆都 leading-edge，single product NRE 從 $200M 級跳到 $500M+ 級，ASIC startup 撐不住雙 leading-edge 的 NRE 與設計時程。
  • 對 AVGO / Alchip / MediaTek 是長期利好：客戶數量會減少（startup 退場），但每個客戶的單 project rev 大幅放大。AVGO 的 Hock Tan 講過 FY27 $100B AI rev 含 $73B "line of sight"，這個 line of sight 在 2028 起的 「同代雙拼」階段會結構性放大。

3）2029 A14+A14 SoIC + >14-reticle CoWoS：把 Feynman / 後 Rubin 鎖到 TSMC

2029 的 chiplet 規格 A14+A14 SoIC + >14-reticle CoWoS + HBM5E 24 顆，這對應 NVDA Feynman（2029 預期）與 AVGO 客製 ASIC 的下一代。對 ASIC 設計服務的長期 anchor：2029 起 ASIC 的設計平台會被鎖在「A14 / A13 IP（backward compatible）+ 4.5μm SoIC + HBM5/5E + N3P base die」這四個維度上。換言之，過去市場討論 ASIC 設計服務都聚焦在「節點選擇」（用 3nm 還是 2nm），2029 之後這個討論變成「全 stack 平台選擇」 — 而這個 stack 只有 TSMC 能 deliver（Samsung Foundry 沒有對等的 SoIC + CoWoS + HBM base die 三合一能力）。這把 NVDA / AVGO / Alchip / MediaTek 等所有 ASIC 設計服務鏈都鎖到 TSMC 路徑上，沒有 second-source 替代。

💡 Insight §2.5（symposium 對 ASIC 設計服務 thesis 的三個強化）
1. 2026-2027 跨代拼接是台系（Alchip / GUC / MediaTek）甜蜜點：每個 hyperscaler ASIC = 兩套 backend service（compute die + I/O die），台系 take rate 翻倍
2. 2028 起同代雙拼把 startup 擠出：NRE 從 $200M 級跳到 $500M+ 級，Groq / Cerebras / SambaNova 撐不住；rev 結構性集中到 AVGO / Alchip / MediaTek 三家
3. 2029 全 stack 平台鎖在 TSMC（A14+A14 SoIC + 14-reticle CoWoS + HBM5E）：Samsung Foundry 沒有對等的三合一能力，ASIC 設計服務鏈 second-source 風險結構性消失

反方論證 2：hyperscaler 共同標準化（OpenAI Triton + MLPerf）消除 ASIC 效率優勢
