市場把「AI 安全」當成資安股的齊漲利多——AI 來了、攻擊變多、所以資安股都受惠。這個框架對方向沒錯，但問錯了問題。真正該問的是價值捕獲（value capture）：當 CrowdStrike、Palo Alto、SentinelOne 把 Anthropic 的 Claude、OpenAI 的 GPT 嵌進自己的 agentic SOC，誰最後賺到最多錢？資安平台廠賣產品，但每一次 AI 推論的成本，部分流向 LLM 廠（Anthropic/OpenAI）與底層算力（NVDA）——增量毛利可能被分走一塊（此為結構推論，幅度尚待各廠 AI 模組毛利揭露驗證）。

同時，一個全新的資安類別正在「現在」被定義：保護 AI 本身（prompt injection 防護、模型安全、MCP gateway、AI runtime firewall）。100% 的企業把 agentic AI 列入 roadmap，但多數承認「agent 出錯時叫不停」——這個治理缺口就是新 TAM。Palo Alto 用 Prisma AIRS 3.0 最積極搶定義、併購 Protect AI；Check Point 以 $300M 併 Lakera；Cisco（AI Defense）、CrowdStrike、Microsoft、雲端原生新創都在賽跑——贏家未定。

所以本報告不站「因為 AI 所以買資安」，而是拆解「在 AI×資安棧裡，誰捕獲價值」。結論：資安平台廠仍是贏家（握分發 + 資料飛輪），但「agentic SOC 的部分經濟漏給 LLM 廠與 NVDA」是市場最低估的一面；而「保護 AI 本身」是 greenfield、誰先定義誰拿超額利潤。

PART I · DECISION SUMMARY

決策摘要層

三句話看完 AI 安全經濟學的現狀、走向與行動，後接 PM 級監測點。這一層是報告的全部重點——下面各節只是把這一層的每個判斷補上證據。

現在 / NOW

一個全新資安層在成形——保護 AI 本身（prompt injection / model security / MCP gateway / AI runtime）；100% 企業把 agentic AI 列 roadmap 但 63% 無法在 agent 出錯時終止、60% 無法強制限制其行為範圍。平台廠搶定義（PANW Prisma AIRS 3.0、CRWD ASPM）、startup 被併（Check Point 收 Lakera $300M、PANW 收 Protect AI）。

未來 / NEXT

3–5 年 AI runtime security 成 greenfield 新 TAM；最大經濟問題 = 價值捕獲——資安廠付錢給 LLM 廠（Anthropic/OpenAI）+ NVDA（算力）做 agentic SOC，部分增量毛利漏出；最大裂縫 = 誰定義 AI 安全類別未定（PANW vs CRWD vs MSFT vs startup）。

行動 / ACTION

押有平台分發 + 資料飛輪的 PANW（Prisma AIRS）/ CRWD / NET（Firewall for AI）；理解 agentic SOC 部分經濟漏給 LLM 廠 + NVDA、增量毛利被分走；AI runtime security 是 greenfield、盯誰先定義類別。

The Key Call · 本報告最重要的一個判斷

市場把「AI 安全」當資安股的齊漲利多，真正的非共識是價值捕獲——當 CRWD/PANW 把 Anthropic/OpenAI 模型嵌進產品做 agentic SOC，每次推論的部分經濟（token/推論成本）流向 LLM 廠與 NVDA（算力），壓縮資安平台的增量毛利（此漏出幅度為結構推論，尚待各廠 AI 模組毛利揭露驗證）。同時一個全新類別（保護 AI 本身 = AI runtime security）正在「現在」被定義，贏家未定。所以這不是「因為 AI 所以買資安」，而是「在 AI×資安棧裡，誰捕獲價值」。資安平台廠仍贏（握分發 + 資料），但「增量毛利被模型商分走」是被低估的一面，而 MSFT/GOOGL（自有模型、不需付外部 LLM 廠）在這條經濟學上結構占優。

三道裂縫具體是：①增量毛利漏出——agentic SOC 越用 frontier model，推論成本越流向 Anthropic/OpenAI + NVDA，資安廠的「賣 AI 結果」毛利低於傳統 SaaS（除非自有模型）；②類別定義權未定——「保護 AI 本身」是 greenfield，PANW Prisma AIRS 3.0 最積極，但 CRWD/MSFT/雲原生新創都在賽跑，誰先成 default 誰拿超額利潤；③GPU 盲區——傳統 EDR 看不見 GPU / AI 推論工作負載（「GPU Blind Spot」），開出一個必須新建的可見性 TAM。盯這三條，比盯「AI 是不是資安利多」更能看清錢的流向。

PART II · HISTORICAL ANALOGS

兩次「賣鏟子 vs 賣服務」，框住 bear 的天花板

歷史不必重演，但兩個對照各給一個教訓：雲端時代「賣算力的（AWS）vs 賣 SaaS 的」給出「平台 vs 應用誰賺更多」，防毒引擎授權給出「核心技術被誰掌握、利潤被誰分走」。

雲端時代：賣鏟子的也賺、但賣服務的賺更多。AWS 賣算力（鏟子）賺豐厚毛利，但建在上面的 SaaS 公司（Salesforce 等）市值與利潤池更大——因為它們離客戶最近、握分發與資料。對本次的含義：LLM 廠（Anthropic/OpenAI）與 NVDA 是「賣鏟子的」（提供模型與算力、分走一塊推論經濟），但資安平台廠（CRWD/PANW）是「賣服務的」（握客戶、分發、資料飛輪），仍是更大的利潤池——這是 bull 的底。量化差異：與雲端不同，AI 推論成本占資安產品成本的比重，目前還在快速變動（模型降價 vs 用量上升），漏出的幅度未定。

防毒引擎授權：核心技術被誰掌握、利潤被誰分走。早年很多防毒產品其實授權同一個底層引擎，貼牌商賺品牌/通路、引擎商賺技術授權費——利潤在棧裡分層。對本次的 bear 含義：若 frontier model 成為 agentic SOC 不可或缺的「引擎」，而模型只有 Anthropic/OpenAI/自有少數幾家，資安廠（外購模型者）就像當年的貼牌商，要把一塊利潤付給「引擎商」。判別訊號：看資安廠 AI 模組毛利是否明顯低於傳統 SaaS、以及自有模型者（MSFT/GOOGL）的相對優勢是否擴大。

PART III · SUPPLY SIDE

供給側：價值鏈分工，利潤池往分發與算力兩端

反直覺但可操作的結論：AI×資安棧的利潤往「兩端」集中——握客戶分發的資安平台（一端）+ 收推論租金的 NVDA/模型商（另一端），中間的純 AI 安全工具最容易被併。

供給端可拆成五層：底層算力（NVDA，收推論租金）、LLM 模型（Anthropic/OpenAI，分推論經濟、但被多模型壓價）、資安平台（CRWD/PANW/S/NET，握分發+資料飛輪、最大利潤池）、自有模型平台（MSFT/GOOGL，垂直整合不付外部 LLM 廠）、AI 安全新創（Lakera/Protect AI 等，做 prompt injection/guardrail——但正被併進平台）。利潤往「兩端」集中：分發端（資安平台）+ 算力/模型端（NVDA/LLM 廠）；中間的純 AI 安全工具是被整併的對象（Check Point 收 Lakera $300M、PANW 收 Protect AI）。

Inference · 供給
純 AI 安全工具被併，利潤往分發與算力兩端集中
前提：AI 安全工具（guardrail/firewall）單點功能、易被平台內建 + frontier model 只有少數幾家（議價力強）+ 資安平台握客戶分發 → 利潤往「平台分發端 + 算力/模型端」集中，中間被擠

純做 prompt injection 防護或 LLM firewall 的新創，功能容易被平台內建或併購（Check Point/Lakera $300M、PANW/Protect AI 已示範），很難獨立成大廠；而 frontier model 只有少數幾家、議價力強，分走一塊推論經濟。最大利潤池仍在「握客戶分發 + 資料飛輪」的資安平台，但其增量毛利被模型/算力端分走。

可證偽條件：若某 AI 安全新創用「AI-native 平台」獨立長成大廠（不被併、自建分發），或開源安全模型讓資安廠不需付外部 LLM 廠（漏出消失），則「兩端集中」論的某一面失效。

PART IV · DEMAND SIDE

需求側：保護 AI 本身的新 TAM 與治理缺口

需求最剛性的是「企業導入 AI agent 卻叫不停」的治理缺口——100% 把 agentic AI 列 roadmap、但 63% 承認無法在 agent 出錯時終止（60% 無法強制限制其行為範圍）；這個缺口就是必須新建的 AI 安全 TAM。

第一引擎：治理缺口（現需）。Kiteworks 對 225 位安全/IT/風險主管的調查：100% 把 agentic AI 列入 roadmap，但 63% 承認「agent 出錯時無法終止」、60% 無法強制限制 agent 行為範圍——這是 2026 的治理-containment 缺口。另有 86% 企業稱需升級 tech stack、90% 視安全與可觀測為必要（此兩項來自 MintMCP/TrueFoundry 等較低層調查，供方向參考）。需求很硬：企業不敢在沒有「prompt injection 防護 + MCP gateway + AI runtime firewall + 預算/速率上限」的情況下放 agent 進生產。第二引擎：GPU 盲區（未需）。傳統 EDR 看不見 GPU / AI 推論工作負載（研究稱「GPU Blind Spot」），開出一個必須新建的「AI 工作負載可見性」TAM——這是 0 基期的 greenfield。

Inference · 需求
需求是 greenfield、剛性——但成長品質看「誰能把缺口變成可賣的類別」
前提：agentic AI 全面上 roadmap（需求）+ 叫不停的治理缺口（剛需）+ GPU 盲區（0 基期）→ AI 安全是 greenfield 新 TAM，但類別定義權未定

軟體類以新類別營收 / attach rate 取代 book-to-bill。需求是 greenfield（0 基期、剛性），但「誰把治理缺口變成可賣的標準類別」未定——PANW Prisma AIRS 3.0 最積極，但 execution risk 是「能否在 Microsoft/CrowdStrike/新創定義類別前成為 default」。

可證偽條件：① AI 安全新類別營收連兩年停在「概念/試點」、無法商品化（greenfield 是空頭支票）；② 類別被某一家（如 Microsoft 內建）快速壟斷，其他玩家吃不到。

多空為何都對

多頭測的是AI 安全需求（治理缺口、greenfield 新 TAM），這幾乎沒爭議；空頭測的是價值捕獲（增量毛利漏給 LLM 廠/NVDA、類別定義權未定）。兩者可同時為真：AI 安全需求是真的（多頭對），但這份需求的利潤被誰、用什麼比例分走未定（空頭對）。本報告把重心放在「價值捕獲」，而非「AI 安全會不會成長」。

PART V · VERDICT & SCENARIOS

裁決與三情境（24–36M）

「AI 安全是 greenfield 新 TAM」判 INTACT（治理缺口），「資安平台是最大利潤池」判 INTACT，但「增量毛利不被分走 + 類別定義權」判 AT_RISK——故結構看多 AI 安全、但要選「自有模型 or 最先定義類別」者。如硬要對應傳統框架：對「可信 AI 部署」的需求遠大於「能提供 runtime 防護 + 治理」的合格供給，等價於結構性短缺型，稀缺來自類別未成熟而非產能。

Base Case主觀機率 ~50%
平台廠定義 AI 安全類別、吃 greenfield，毛利略漏

PANW（Prisma AIRS）/ CRWD 用平台分發把 AI runtime security 收進產品、吃 greenfield 新 TAM；agentic SOC 的部分經濟漏給 LLM 廠/NVDA，AI 模組毛利略低於傳統 SaaS 但可接受；純 AI 安全工具被併。

觸發確認訊號：PANW Prisma AIRS attach 上升、AI 安全新類別營收放量、AI 模組毛利只略低於 SaaS、M&A 持續整併工具。
Bull Case主觀機率 ~20%
自有模型者結構占優 + AI 安全成大類別

AI 安全成獨立大類別、放量快；自有模型的 MSFT/GOOGL（不付外部 LLM 廠）毛利結構占優、加速吃份額；NVDA 因推論放量持續收租；先定義類別者拿超額利潤。

PART VI · TAXONOMY

個股特徵與 conviction tier

本框架只輸出「特徵」與「分類」，買哪一檔由個股 DD 系統與 Pure MA 訊號決定。以下 tier 為產業推論的特徵歸類，提及公司僅為「該特徵的代表案例」，不附帶買賣建議。

CORE 🔴

PANW

最積極定義 AI 安全類別——Prisma AIRS 3.0（AI runtime firewall / model security / posture management）+ 併 Protect AI。特徵是「賭成 AI runtime security default」，本主題類別定義權的代表案例；execution risk 是能否在 MSFT/CRWD/新創前成 default。conviction：高（賭定義權）
CORE 🔴

CRWD / NET

CRWD：Falcon AI 防護 + ASPM（Gartner Customers' Choice）、資料飛輪嵌 frontier model。NET：Firewall for AI + AI gateway，是 AI 流量的天然入口（每個 LLM 呼叫都過邊緣）。特徵是「平台分發 + AI 流量入口」，吃 greenfield 的代表。conviction：中高
CORE 🔴

MSFT / GOOGL（自有模型）

經濟學上結構占優——自有 frontier model（OpenAI/Gemini），不需付外部 LLM 廠、agentic SOC 毛利不漏出。特徵是「垂直整合者」，價值捕獲最完整的代表（量體稀釋是 caveat）。conviction：高（毛利結構優）
EDGE 🟡

NVDA / S（漏出兩面）

NVDA：算力收租方——agentic SOC 推論成本最終流向 GPU，是「價值漏出」的受益者（從資安角度是 edge，但結構受益）。S：Purple AI 多模型（Claude+GPT+自研），是「外購模型、毛利漏出」最明顯的純玩家（雙面）。特徵是漏出鏈的兩端。conviction：中 · 結構曝險

估值面總結：本主題的核心張力在「AI 安全是真 greenfield（需求確定）vs 價值捕獲未定（毛利漏出 + 類別定義權）」。PANW 押「定義 AI 安全類別」、CRWD/NET 押「平台分發 + 入口」、MSFT/GOOGL 在毛利經濟學上結構占優（自有模型）、NVDA 是漏出受益方。現處 Phase I–II（類別定義中），轉空條件 = AI 模組毛利明顯低於 SaaS（漏出大）或 類別被 Microsoft 內建壟斷；de-rating window 在「市場發現 AI 安全的增量毛利被模型商分走」那一刻。純 AI 安全工具（Lakera/Protect AI）已被併、不可獨立投資。買賣由個股 DD + Pure MA 系統決定。

The One Line

「AI 安全」不是資安股齊漲利多，而是價值捕獲問題——資安平台廠仍贏（握分發+資料），但增量毛利被 LLM 廠與 NVDA 分走、自有模型者結構占優，這是市場最低估的一面。

這份報告刻意不做的三件事

  1. 不把「AI 安全」當無差別利多——明確拆解價值捕獲（誰賺到錢、增量毛利漏給誰）。
  2. 不告訴你哪一檔該買——CORE/EDGE 是「特徵的代表案例」，買賣由你的個股 DD 與 Pure MA 系統決定。
  3. 不假裝知道誰會定義 AI 安全類別——PANW vs CRWD vs MSFT vs 新創的賽跑未定，已誠實標為類別定義權之爭。
