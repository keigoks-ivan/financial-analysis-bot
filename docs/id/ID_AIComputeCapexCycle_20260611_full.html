KEY CALL — 本報告最重要的一個判斷

§1 產業白話定義 + 歷史脈絡

這份報告講的不是某一顆晶片，而是「全球科技巨頭為了跑 AI，正在花多少錢蓋運算工廠、這些錢由誰最後買單、會不會蓋過頭」。買方是 Microsoft、Google、Amazon、Meta 這些雲端巨頭（hyperscaler，指自建超大型資料中心的雲服務商）；他們把錢付給晶片設計商、代工廠與記憶體廠；最終要靠 AI 應用的營收把這筆投資賺回來——而那一步，正是爭議所在。

白話開場：所謂「AI 算力資本週期」（capex supercycle，capex = 資本支出，指買地、蓋廠、買伺服器這類長期投資），就是一條從「花錢蓋運算產能」到「靠 AI 服務變現」的資金鏈——這條鏈在 2026 年規模已達史上首個兆元年（~$1.04T）。鏈的最上游是出錢的 hyperscaler 與新興的「主權 AI」（sovereign AI，指國家為資料在地化自建的 AI 基建）；中間是把錢變成算力的晶片設計（NVIDIA）、代工（TSMC）、記憶體（HBM，高頻寬記憶體，AI 晶片的專用高速記憶體）；最末端是 OpenAI、Anthropic 與雲端 AI 業務這些真正向終端用戶收錢的人。整個週期的命門就一句話：上游花的錢，最末端賺得回來嗎？

邊界界定：本報告把焦點放在「資本週期的永續性與過剩風險」這個母題（涵蓋 2022 年至今的這一輪 build-out），因此 in-scope 的是 capex 規模、供給瓶頸（CoWoS / HBM / 電力）、利潤池分布、終端變現缺口與資本週期訊號（折舊、發債）；out-of-scope 的是單一晶片世代的技術細節（那是子題 ID_AIAcceleratorDemand 的事）與單一資料中心的工程設計。灰色地帶在「neocloud（算力轉售商，如 CoreWeave）」——它既是需求方（向 NVDA 買卡）又是供給方（向 OpenAI 租算力），本報告把它當「資本週期的高槓桿放大器」看，而非獨立環節，因為它的脆弱性正來自跨越供需兩端的舉債。

第一次轉折：2022-11 ChatGPT 發布，兩個月內 MAU 衝破 1 億；自此四大 hyperscaler 的 AI 相關 capex 從 2022 約 $162B 一路翻到 2025 約 $448B（近 3 倍），Epoch AI 直接以「自 GPT-4 以來 hyperscaler capex 翻 4 倍」為標題。轉折的機制是三力共振：① frontier model 訓練成本隨 compute 規模指數上升；② 已部署產品的 inference（推論，指模型實際回答問題的運算）需求超出現有容量；③ 各家自研 silicon（如 Google TPU、Amazon Trainium）以建護城河、降低對 NVDA 的依賴。

第二次轉折：2025-01-21 Stargate Project 宣布——4 年投 $500B 建 AI 基建（供 OpenAI 用），$100B 立即部署，由 SoftBank / OpenAI / Oracle / MGX 領投。這標誌資本週期從「hyperscaler 內生現金流投資」升級到「跨主體、帶 SPV 槓桿的兆元級承諾」——是本輪與 2018-19 那波純內生 capex 最關鍵的結構差異（見下文歷史類比）。

編年史：2022-11 之後的 capex 軍備競賽（逐年推進）。把這條曲線一年一年攤開，會看到一場典型的「囚徒困境式」軍備競賽——沒有人敢先停，因為先停的人會把 frontier 拱手讓人：

  • 2022（ChatGPT 引爆前夕）：四大 hyperscaler capex 合計約 $162B，仍是「雲端常態投資 + 早期 AI 試水」的量級，AI 還沒成為 capex 的主軸。
  • 2023（軍備競賽元年）：ChatGPT 把每家逼上牌桌，但節奏並不齊步——Amazon 2023 capex 反而 −17% YoY 至 $52.7B（最大基期者先收手做 post-pandemic digestion），而 Microsoft 因綁定 OpenAI 逆勢 +45%（+$12.8B）、Google 同步加碼。這個「digestion 與加碼並存」的分歧，預示了 2027 可能重演的非齊步 digestion。
  • 2024（全員加速）：digestion 結束，四大集體爆發——Microsoft $75.6B（+83%）、Amazon $83.9B（+57%，AWS 口徑反彈 +114%）、Google $52.5B（+63%）、Meta $39.2B（+40%）；四大合計達 record $410B，AWS 從 2023 的首次年度衰退一年翻身。
  • 2025（兆元前夜 + SPV 入場）：四大升至約 $448B，含 Oracle 的五大「nearing half a trillion」；Stargate 以 $500B / 10GW 把跨主體 SPV 槓桿正式帶進這場競賽，資金來源從純現金流首度滲入外部結構。
  • 2026（史上首個兆元年）：到 2026-Q1 法說，四大 2026 capex guidance 合計已上看 $725B（YoY +77%）：Amazon ~$200B、Microsoft $190B（其中 $25B 來自記憶體/零組件漲價）、Alphabet up to $190B、Meta $125-145B；含 Oracle 的五大則 widely forecast >$600B（+36%）。值得注意的是，2026 的增量裡已不純是「買更多卡」——記憶體與零組件漲價（HBM 排擠 DRAM，見 §3）被 Microsoft 與 Meta 明確點名為 capex 上修的一部分，這是供給端通膨開始反咬出錢方的早期訊號。

這條 2022→2026 從 $162B 到 $725B（近 4.5 倍）的曲線，本身就是 thesis 的雙刃——它證明需求軍備競賽真實且未停（多頭錨），但也意味著任何一家若率先喊停，就會像 2023 的 Amazon 一樣觸發非齊步 digestion，而這次的基期是兆元級（空頭錨）。

歷史類比一：2000 電信光纖泡沫（最強類比，完整時序）

當年的曲線怎麼長出來的。1996-Q1 通訊設備投資約 $62B/年（constant 1996 USD），到 2000-Q4 衝破 $135B/年，CAGR 約 18%，通膨調整後峰值約 $213B（2000）；單 1999 一年結構性電信投資就 +75%，是投機狂熱的峰。boom 期（1995-2000）累計投入估達 $2 trillion，鋪了 80-90M 哩光纖。驅動這條曲線的是一個後來被證明錯誤的單一敘事——「網路流量每 100 天翻倍」。

「每 100 天翻倍」這個 myth 從哪來、錯在哪。它的出處是 1997 年 WorldCom 旗下 UUNET 一位員工（Stluka）在 Excel 裡做的 best-case 情境模型，而那個成長率只在 1995-1996 短短兩年真實成立過。因為 UUNET 當時是全球最大 backbone 之一，這個數字被賦予了可信度——分析師引用、媒體轉載、1998 年政府文件甚至重述了 WorldCom 1997 的說法，最後「固化成事實」，連總統候選人都在引用。AT&T 等業者據此瘋狂鋪纜（約 2,200 哩/小時）。但 Odlyzko（UMN）後來的研究顯示，實際流量約「每年翻一倍」——遠低於那個病毒式統計。換句話說，泡沫不是建在「需求不存在」上，而是建在「需求成長率被高估一個數量級」上。本次的平行警訊：「token 用量每 X 個月翻倍」「inference 需求無上限」這類敘事，必須用同樣的懷疑去檢——這正是 §7 / §8 把「AI app / 晶片營收比」與「capex 增速」列為核心證偽 metric 的歷史理由。

融資結構：債務 + vendor financing 雙重墊高。這場建設幾乎全靠槓桿。Global Crossing / WorldCom / Qwest / Level 3 借了數百億美元建跨洲跨洋光纖——WorldCom 最終虛增獲利 $3.8B、對應約 $30B 債務（2002-07 成為當時史上最大破產）；Global Crossing 2002-01 破產，留下 $12.4B 債務與數千哩 dark fiber。更關鍵的是 vendor financing：Cisco / Lucent / Nortel 直接替「買不起設備的客戶」融資採購——設備商等於自己掏錢墊高自己的需求。這個結構與本次 NVDA 投資 OpenAI / neocloud、晶片商成為客戶的金主高度同構（見 §3 循環融資圖譜）——當賣方替買方融資，帳面需求就含了一塊「自我循環」的虛胖。

破產順序與設備商崩落（capex digestion 最先見血的環節）。崩落是有順序的：360networks（2001）→ Global Crossing（2002-01）→ WorldCom（2002-07），多家 CLEC / 長途商接連倒。而最劇烈的傷亡在設備商：Lucent FY2001 營收 −26% 至 $21.3B、單財年虧 $16B、裁員 2/3、市值蒸發約 $250B（≈ 美國 GDP 2%）；Nortel 市值從 2000-09 的 C$398B 崩到 2002-08 的 <C$5B（股價 C$124→C$0.47，−95%+），其 optical 營收 2000 還暴衝 +133% 至 $9.2B，隔年就崩。關鍵時序：設備商營收峰值（2000）滯後 capex 峰值僅 0-2 季，但崩落同樣劇烈——這正對應本次 NVDA / AVGO 的角色：它們是 capex digestion 一旦發生時最先見血的環節，而非最後。

§2 技術成熟度 + S 曲線

為什麼算力需求是「現在」爆發，不是三年前、也不是三年後？因為兩件事剛剛同時發生：模型從「會聊天」進化到「會自己做事」（agentic），讓單一任務的 token 消耗暴增 5-30 倍；而推論成本同時年降 5-10 倍，讓「用 AI 做事」第一次在經濟上划算。需求曲線與成本曲線的剪刀差，就是這輪資本週期的引擎。

技術成熟的因果鏈有兩段，而且兩段的機制必須講透，才能理解為什麼需求是「現在」洪流而非三年前的脈衝。

第一段：pre-training 撞牆，前沿改道到 test-time compute。pre-training（預訓練，用海量資料把模型「教大」）長年遵循 scaling law——算力與資料堆得越多、模型越強。但 2025 起這條曲線出現 diminishing returns（收益遞減），根本原因是資料 bottleneck：高品質訓練語料是有限的，把全人類文本喂完之後，再加算力的邊際回報快速下滑。前沿因此轉向 test-time / inference compute（推論時運算，o1/o3、Gemini Flash 這類「邊想邊算」的模型）——不是在訓練時把模型教得更大，而是在回答時讓模型多想幾步、反覆取樣、自我檢查。這個解耦的經濟學意義是關鍵：pre-training 是一次性的資本開銷（訓完就固定），而 test-time compute 是隨每次查詢線性增加的營運負載——需求曲線的形狀因此從「脈衝」（訓練完就降）變成「洪流」（用得越多、算得越多，永不歸零）。這一改道，正是把算力需求從「會飽和」變成「結構性持續成長」的機制底層。

第二段：agentic 是把洪流再放大一個數量級的乘數。NVIDIA 在 GTC 2026（San Jose）給的框架最直白：Jensen 稱過去 2 年 computing demand 增加「100 萬倍」，並把它拆成兩個乘數——單一 work 的 compute 需求增約 10,000x（reasoning 模型每個任務要做的「思考」量暴增）× usage 增約 100x（用的人變多），相乘得百萬倍。機制鏈條是：reasoning 模型（會反思、規劃，一個問題內部展開成幾十步推理）→ Claude Code 這類「第一個 agentic model」（自己讀檔、編碼、編譯、測試、再迭代，一個任務可能跑上百次模型呼叫）→ 多步 workflow 自主執行。關鍵在於 agentic 把「一問一答」變成「一個目標、上百次 token 往返」——這就是為什麼 agentic 每 task 耗 token 是 chatbot 的 5-30x（§4）。Jensen 宣告「inference inflection has arrived」的實質，是 AI 第一次能做 productive work（產生可計價的工作成果），需求結構從訓練轉向推理與即時行動。量化佐證：NVDA 揭露 AI inference 的 token 生成「一年內增 10 倍」——這個 10x 不是訓練在長，是「用 AI 做事」在長。

怎麼讀：第一，瓶頸有明顯的「下移」軌跡——從晶片（CoWoS/HBM）逐步移向 racks、power、cooling（OCP 會議 CEO 共識），這意味著利潤池與 entry 機會也會跟著下移（§3 / §9 的非顯而易見受益者邏輯）。第二，CoWoS 與 HBM 是「雙鎖」：兩者任一鬆動才能緩解 GPU 短缺，而兩者都被前四大客戶包下 ~90%，這是 §5 判 shortage 的供給端硬證據。

為什麼這幾個卡點「卡得住」（機制，不只是列名）。每個 kingmaker 之所以是 kingmaker，是因為它同時滿足「需求暴增 + 供給彈性極低」兩個條件，而供給彈性低的原因各不相同：

  • CoWoS 卡在「製程獨家 + 擴產長前置」：先進封裝把 GPU die 與 HBM 堆疊在矽中介層（interposer）上，這道製程目前近乎只有 TSMC 能量產到 AI 晶片要的良率與規模。卡點不是「想擴擴不了」，而是 interposer / substrate / tooling 整條鏈擴產要 18-24 個月，且 NVIDIA 已把過半 2026-27 產能「booked several years ahead」——新進需求只能排隊到 mid-2027 之後，這就是它「卡得住」的合約 + 製程雙重原因。
  • HBM 卡在「製程稀缺 + 排擠效應」：HBM 把多層 DRAM die 垂直堆疊並用 TSV 打通，每 GB 耗用的 wafer 產能約是普通 DDR5 的 3 倍。這代表廠商每多做一片 HBM，就要犧牲 3 片普通 DRAM 的產能——當 AI 把 HBM 需求拉爆（NVDA 一家吃 ~54%），記憶體廠被迫在「賺 HBM 的高毛利」與「供給傳統 DRAM」之間取捨，結果是兩邊都漲價（§3/§5）。卡點的本質是產能的物理排擠，不是單純缺機台。
  • 電力卡在「外部基建長前置，不在晶片廠控制範圍」：當晶片瓶頸被錢部分解開後，瓶頸下移到電網——高壓變壓器交期已從 pre-2020 的 24-30 個月拉到 5 年、interconnection 4-10 年。這個卡點最致命，因為它不在任何 AI 公司能加速的範圍內：你可以花錢買更多卡，但變壓器和輸電線的 5 年交期是物理與監管的硬牆——這也是為什麼 $662B 的 datacenter lease 仍「尚未開工」（§3/§5）。
技術成熟度告訴我們兩件事可操作的事。其一，需求拐點（agentic + inference inflection）是真的、且剛發生不久，所以「現在是週期早中段、不是末段」——這支撐核心倉位續抱。其二，真正的技術 kingmaker 是 CoWoS 與 HBM 而非 GPU 本身，所以掌握這兩個瓶頸的 TSM（CoWoS 守門人）與 HBM 三寡占（MU 等）享有比 merchant GPU 更深的護城河；而瓶頸下移到電力散熱，則讓 VRT 這類二次受益者進入投資視野（§9）。

§3 供給側：現在與未來 + 利潤池

誰在供給算力、瓶頸卡在哪、賺到的錢被誰拿走？三句話：供給端是 NVDA（設計）+ TSMC（代工）+ HBM 三寡占（記憶體）的高利潤鐵三角；瓶頸是 CoWoS 與 HBM 的雙鎖；利潤池被設計層與代工層牢牢吃住，伺服器組裝廠（ODM）只是「量大錢薄」的搬運工。

三時間欄（2024 / 2026 現在 / 2027 預估）讓你看出加速器市場的 share 是在加速集中還是被侵蝕，而不只是一張 snapshot。

🔴 四家核心玩家的個體敘事（供給策略 / 客戶結構 / 在本 thesis 的位置）

NVIDIA（設計龍頭，最肥利潤池的守門人）。供給策略是「整機架賣 + 生態鎖定」：不只賣 GPU，而是把 compute + networking 整包出（Q1 FY2027 DC networking $14.8B、YoY +199%），用 CUDA 軟體生態把客戶綁死，並以「最佳 perf-per-TCO」對抗自研 ASIC 的攻擊。客戶結構正在廣化——DC 營收 $75.2B 裡 Hyperscale（~$38B）與 ACIE（cloud builder + enterprise + sovereign，~$37B）已近乎對半，代表需求外溢出 Big-4，降低單一客戶砍單的傳導風險（這正是 §8 bear 路徑被稀釋的原因）。在本 thesis 的位置：它是「financial 過剩風險」的最大承重者——倍數不貴（Fwd P/E ~22.5-24x）但 FY27 consensus ~$391B 的 E 全壓在 capex 100% 兌現上，一旦 capex 增速反轉，它是 capex digestion 最先見血的環節（對應 2000 年 Lucent/Nortel 的角色）。

TSMC（代工 + CoWoS 雙守門人，本 thesis 最安全的押注）。供給策略是「稀缺製程 + 罕見擴產」：2026-Q1 GM 66.2%、HPC 平台占晶圓營收 61%（一年前才 51%），先進製程（7nm 以下）占 74%，並罕見地宣布加速擴產。客戶結構是「全鏈通吃」——無論 NVDA GPU 還是 AVGO/Google 的自研 ASIC 贏，都得來台積電流片、配 CoWoS，所以它對「誰贏」不敏感。在本 thesis 的位置：它是利潤池集中論（§7 NC#3）最硬的支點，也是議價權紅利最持久的一家——CoWoS 配置滿到 mid-2027 給了它約兩年的瓶頸租金。風險最低，因為它賺的是「賣鏟子給所有淘金者」的錢。

AVGO（客製 ASIC 設計，第二大且最快膨脹的利潤池）。供給策略是「替 hyperscaler 設計專用晶片」：Google TPU、Meta MTIA 等背後的 ASIC 設計與量產，AVGO 是核心代設計商。Q2 FY2026 AI 半導體 $10.8B（YoY +143%），Q3 guide 拉到 $16.0B（YoY +200%），公司整體 non-GAAP op margin ~67%。客戶結構高度集中在少數 hyperscaler 的客製專案——這是雙刃：綁定深、但也意味單一客戶 roadmap 變動衝擊大。在本 thesis 的位置：它是「ASIC 會不會侵蝕 NVDA、稀釋上游利潤」這條核心張力（§7 NC#3）的化身。最 load-bearing 的 bull claim 是 Hock Tan 於 2026-06-03 Q2 法說重申的 FY2027 AI 營收 >$100B（近 FY26 $56B 的兩倍）；2026-06 selloff（Q3 guide $16B < 市場期望 $17.2B 致 −12.6%）已把估值從歷史高位壓回業界中位（Fwd P/E ~31x），下檔部分定價、風險報酬轉佳——故本報告把它由 ASIC 敘事升 🔴 核心。

AMD（merchant GPU 第二供應源，四檔 🔴 中 conviction 最低）。供給策略是「第二供應源 + 開放生態（ROCm）對抗 CUDA 鎖定」：Q1 2026 Data Center segment $5.8B（YoY +57%）、GM 升至 53%，MI350/MI325 帶動，下一棒押 MI400/MI450 世代。客戶結構最大亮點是 Meta 計畫部署多達 6GW AMD Instinct（首個 1GW 用客製 MI450）——這是 merchant GPU 陣營對 NVDA 壟斷最實質的一次突破，也綁進了 AMD 給 OpenAI 的認股權證安排（見下方循環融資）。在本 thesis 的位置：它列 🔴 是因為 DC $5.8B 占總營收 $10.3B 約 56%（過 40% 門檻），但 conviction 在四檔 🔴 中最低——它是挑戰者而非瓶頸守門人，護城河（ROCm 生態）遠不及 CUDA，且若 capex digestion 來臨，第二供應源通常比龍頭更早被砍單。

循環融資圖譜：誰投誰、誰先付錢、誰承擔終端信用風險

這條鏈最被低估的脆弱點，是「同一批公司出現在交易的多側」——晶片商投 AI lab、AI lab 向雲端商採購、雲端商回頭向晶片商買卡，形成資金循環。2026 有分析估這類循環交易總額 >$800B。多頭（Janus Henderson）稱之為「virtuous circle」（需求自我強化），空頭（Bloomberg）則稱「同一筆錢墊高彼此營收」——這個分歧與 2000 年 vendor financing 的爭論一模一樣。把主幹一條條拆開：

  • NVDA → OpenAI：原 $100B「縮水」為 $30B equity（重大更新）。2025-09-22 原案措辭是「intends to invest up to $100B」，staged 結構——每部署 1GW 算力釋放一筆資本，首 1GW 綁 2026H2 Vera Rubin 平台。但到 2026 年，此案已縮水為 $30B equity stake，屬 OpenAI 2026-02-27 宣布的 $110B funding round 一部分；Jensen Huang 2026-03 親自確認原 $100B「probably not in the cards」。縮水原因：OpenAI 分散硬體供應（AMD 6GW、Cerebras/Groq 探推論）以降低對 NVDA 依賴，使原 staged 結構對雙方吸引力下降。敘事意義：這是「circular lock-in 戰場」最鮮活的案例——投資金額隨議價力消長而動態調整，不是固定承諾，所以任何把 $100B 當 live 數字引用的分析都已過時。
  • OpenAI → Oracle $300B（誰先付錢是關鍵）。OpenAI 承諾 2027-2031 每年付 Oracle $60B（共 $300B）雲端基建。關鍵條款：OpenAI 在機器實際 in-use 前不付錢、付款隨服務交付逐步進帳——也就是 Oracle 先扛資本。Oracle 為填這個 timing gap，計畫 2026 透過債 + 股募 $45-50B。風險前置在 builder 端（Oracle），這與「用自有 cash flow 建設」的 hyperscaler 是完全不同的脆弱結構。
  • OpenAI → Microsoft $250B：OpenAI 同時承諾向其大股東 Microsoft 採購 $250B 雲端服務——又一條「投資人同時是供應商」的循環。
  • AMD → OpenAI warrants 換 6GW 採購：AMD 給 OpenAI 認股權證，可買最多 1.6 億股 AMD（約 10% 公司）@ $0.01/股，performance-based 分批 vest（首 tranche 綁初始 1GW 部署，後續綁 AMD 股價目標 + OpenAI 里程碑）。這是「以股權換採購承諾」的另一種循環墊高。

誰承擔終端信用風險？把鏈條收斂成一句話：在 Oracle / Stargate 案中，基建供應商（Oracle）先舉債扛資本、OpenAI 後付且綁 in-use，所以終端信用風險前置在 builder 端；而 OpenAI 本身的經濟學仍是 spend >> revenue（年化營收 ~$24-33B vs 兆元級採購承諾）。這意味著整個循環的最終承重點，是「OpenAI 等 AI lab 能否把採購承諾轉成真實營收」——一旦這環的變現假設破裂（§4 的 7-9x 缺口），連環營收會同時縮水（§7 反方 2）。這正是與 2000 年 vendor financing 同構的「自我循環虛胖」風險，也是本 thesis 把 circular financing 列為 kill scenario 旁證的原因。

怎麼讀：第一，利潤往「設計（NVDA/AVGO）+ 代工（TSMC）+ HBM」三段集中，這三段同時享有 ~66-75% 毛利且都在 ↑，意味著議價權牢牢在上游——這支撐核心倉位押在 NVDA/TSM/AVGO/MU。第二，ODM 是唯一 ↓ 的環節（GM 僅 ~5-6% 還被 memory 漲價 + consignment 寄售模式進一步擠壓），證明「蓋 AI 工廠」這件事，組裝者賺的是辛苦錢——電力散熱（VRT，↑ 但缺量化）與 neocloud（CRWV，利潤未證）兩格的留白，正是本鏈下一個值得追蹤的不確定區。

利潤池為什麼這樣遷（機制，不只是箭頭方向）。三段上漲、一段下跌，背後各有不同的經濟學引擎：

  • HBM 從虧轉盈的機制：記憶體本是惡名昭彰的週期性虧損業務（產能一上量就殺價）。HBM 改變了這件事——它把 DRAM 從「標準品」變成「綁定 AI 加速器的客製高速記憶體」，ASP 從 HBM3 ~$200/stack、HBM3E ~$300、跳到 HBM4 ~$500/stack。同時前述「每 GB 耗 3 倍 wafer」的排擠效應，讓三寡占（SK hynix / Micron / Samsung）握有絕對定價權：他們可以選擇少做傳統 DRAM、多做 HBM，結果是 HBM 量價齊揚 + 傳統 DRAM 因供給被排擠也漲價（2Q26 傳統 DRAM 合約價 QoQ +58-63%）。由虧轉盈不是週期回升，是產品組合 + 排擠定價權的結構性升級——這也是為什麼 MU 雖列 🟡 卻有被低估的營業槓桿（§9）。
  • ODM 量增利薄的結構原因：伺服器 ODM（鴻海 / 廣達 / 緯穎）GM 僅 ~5-6% 且還在被擠壓，原因有二。其一，consignment（寄售）模式——客戶（hyperscaler）自己採購 GPU 與記憶體等高價料件、寄給 ODM 組裝，ODM 只賺組裝代工費，最肥的料件利潤完全不過它的손익表。其二，memory 漲價反咬——當 HBM/DRAM 漲價，ODM 採購成本上升但無法等比轉嫁，毛利進一步被壓（廣達 Q1 2026 即便營收 / EPS 創高，GM 仍因快速轉向高價 GPU server 而「collapse」）。所以「蓋 AI 工廠」這件事，組裝者注定是「量大錢薄的搬運工」，這是結構而非景氣問題。
  • H100 租價「反漲」的供給合約機制：表面矛盾——H100 on-demand 短租崩 -64%（>$7→~$2.95/GPU-hr，spot 破 $1.03），但一年期租約價反而 2025-10→2026-03 漲 +40%。機制是：on-demand 短租市場湧入 >300 家小型 neocloud 打價格戰（成本曲線最右端的邊際供給者，spot 跌破 $1 時最先停產）；而長約市場反映的是 frontier lab 鎖定產能的真實緊張——願意簽一年約的客戶是要穩定 capacity 跑訓練/推論的，這塊需求 price-inelastic、且受 CoWoS/HBM 雙鎖限制供給。短租打價格戰 vs 長約反漲，正是判 shortage 的核心證據：真需求（長約）緊、投機供給（短租）爛。

成本曲線（本主題版＝算力成本曲線，週期性產業故強制保留）：與大宗商品「誰是邊際生產者」邏輯對應，算力的成本曲線最直接的證據是 GPU 租賃價。H100 從 early-2024 的 >$7/GPU-hr 跌到 2026 cohort median ~$2.95/GPU-hr（峰值至今 -64%），spot 甚至破 $1.03；2025 有 >300 家新供應商湧入掀價格戰。誰先死？答案是「上一代 merchant GPU + 高槓桿小型 neocloud」——他們在成本曲線最右端（成本最高），spot 跌破 $1 時最先停產。誰是低成本供給者？是有自研 ASIC 的 hyperscaler——Google TPU 對 H100 約「4× better value」、功耗低 60-65%，走自有 fab 配額垂直整合攤平 TCO；連 Meta 都傳與 Google 洽談 mid-2026 起多十億美元 TPU 部署。但要注意關鍵反證：H100 的一年期租約價反而 2025-10→2026-03 漲了 +40%（見 §5），顯示 on-demand 短租在打價格戰、但長約鎖定的長期需求仍緊——這個分歧正是判 shortage 的核心。

未來供給（capex pipeline + 進入門檻 + 彈性）：產能正在快速擴張但 lead time 仍長。CoWoS 自 late-2023 起約 10× 擴張，2026 底達 120-130K wpm、2027 底 170K wpm，但 NVIDIA 已包下過半 2026-27 產能、先進封裝線「booked several years ahead」，CoWoS 全配置至少到 mid-2027。Stargate 已超越原 10GW 里程碑（近 90 天 +3GW），五個新站 + Abilene 達 ~7GW 規劃、未來 3 年 >$400B。供給彈性的真瓶頸已下移到電力：高壓變壓器 lead time 2-4 年、interconnection 4-10 年，且 $662B 的 datacenter lease「尚未開工」——對照 2026 約 16GW 的宣告 pipeline，實際動工僅約 5GW，且 30-50% 專案因電力與電氣設備（高壓變壓器、switchgear、電池儲能）的多年交期而面臨延後或取消（Sightline Climate 2026-05 統計）——這意味著即使錢到位，產能上線仍有結構性時滯，反而降低了短期過剩風險。地緣面：H200 對中 2026 放行（附 25% surcharge、每客戶上限 75,000 顆）但截至報導仍無實際交付（法律 limbo），H20 則因出口管制致 NVDA 一次性 $4.5B 庫存 charge。

新進入者的信用結構：neocloud 是「以債換 backlog」的高槓桿放大器。新供給的最大來源之一是 neocloud（算力轉售商），但它們的擴張幾乎全靠槓桿，且結構脆弱性藏在資產負債表內外。以 CoreWeave 為例：Q1 2026 底總債務 >$17.3B（一年前僅 $4.9B）、總負債 $50.8B vs 股東權益 $4.8B（>10x 槓桿），另有 $34B 租賃在表外；利息費用 YoY 翻 3 倍至 $311M。它的融資是雙 facility 利率階梯——DDTL 4.0（$8.5B，首個 investment-grade GPU 抵押融資，Moody's A3）浮動利率 SOFR+2.25% / 固定 ~5.9%，由 Meta 合約（≥$19B）背書；但較新的 DDTL 5.0（$3.1B）pricing 已跳到 SOFR+4.50%，明顯比 4.0 貴——風險定價在上升。最致命的是「抵押品縮水悖論」：貸款以 GPU 為抵押，但 GPU 市場租金已跌 50-70%，抵押品價值在還款表開始引用折舊硬體時同步縮水。同業也在發債承壓（Nebius 2026-03 發 $4.34B 債當月股價跌 >20%）。McKinsey 點出結構性弱點：BMaaS 毛利扣人力 / 電力 / 折舊後僅 14-16%，利用率掉破 80% 報酬即趨平——這個 80% 利用率門檻，就是本次的「點亮率」等價物，一旦掉破即從正報酬翻負，是 neocloud bear case 的具體觸發線（呼應 2000 年光纖的 5% 點亮率）。所以本報告把 CRWV 定位為「過剩風險的高槓桿溫度計」而非核心 long。

因果閉合（回應 §1 提出的結構變數）：§1 點出三個結構變數——CoWoS/HBM 雙鎖、自研 silicon 護城河、token 成長機制。其中 CoWoS/HBM 雙鎖在未來 3-5 年是否仍 binding？答案是「2026-27 仍 binding、2028 後鬆動」：CoWoS 2027 底擴到 170K wpm、HBM4 三家增產，到 2028 供給彈性才追上，故瓶頸的議價權紅利還有約兩年。而自研 silicon 護城河正在從「降低對 NVDA 依賴」變成「實際侵蝕 NVDA 份額」（ASIC 44.6% CAGR），這個結構變數在未來 3-5 年只會更 binding，不會消退——這也是為何 §7 把「利潤池會否被自研侵蝕」列為核心分歧。

供給側給出一個反直覺但可操作的結論：瓶頸（CoWoS/HBM/電力）是這輪資本週期的朋友，不是敵人。因為產能上線有 2-4 年的結構性時滯（封裝滿到 mid-2027、變壓器 2-5 年、interconnection 4-10 年），即使資金狂湧，短期內也很難真的「蓋過頭」——這把過剩風險從 2026 往後推到 2027-28。對 PM 的含義：掌握瓶頸的 TSM（CoWoS 守門人）與 HBM 三寡占享有最持久的議價權紅利（約兩年），是比 merchant GPU 更安全的押注；而 ODM 的 ↓ 利潤池與 neocloud 的未證利潤，則是這條鏈裡該避開的「量大錢薄 / 高槓桿」陷阱。

§4 需求側：現在與未來 + 三角驗證

誰在買算力、為何買、哪塊需求最不可替代？現在的買家高度集中在四大 hyperscaler（2026 占 ~$725B / $1.04T 總 capex），但需求正在外溢到主權 AI 與企業端；最不可替代的是 frontier lab 的訓練需求（連兩代舊卡都租不到），最脆弱的是企業推論需求（MIT 稱 95% pilot 零 ROI）。

現需結構：2026 是史上首個算力 capex 兆元年（~$1.04T），拆分為 Big-4 hyperscaler $725B + Oracle $50B + Apple $13B + Neoclouds $60B + 中國 $80B + 主權 $60B + 其他 $48B。需求廣化的最強訊號來自 NVDA：Q1 FY2027 DC 營收 $75.2B 首次出現 Hyperscale（~$38B）/ ACIE（cloud builder + enterprise + sovereign，~$37B）近乎對半——代表 AI 需求已外溢出 Big-4。pricing power 的分層很關鍵：frontier lab 訓練需求最 price-inelastic（Jensen 指連兩代舊卡 spot 租金仍漲、租不到，capacity-constrained）；企業推論需求較 elastic（對 token 單價 / ROI 敏感），是過剩風險的主要承壓點。

需求集中度正在從「四大獨大」走向「四層買家」。把 $1.04T 攤開看買家結構，會發現一個對 thesis 至關重要的演化——需求不再只靠 Big-4。第一層仍是四大 hyperscaler（$725B，~70%），用自有現金流投資；第二層是 Oracle + neocloud（$110B），帶槓桿、靠 backlog 撐；第三層是主權 AI（$60B），由監管 / 國防的資料在地化要求驅動（Microsoft 已鋪 Austria/Denmark/Greece/Italy/Poland sovereign zone、AWS European Sovereign Cloud，大型企業估占主權市場 ~68%）；第四層是中國（$80B），走自有供應鏈、與西方需求脫鉤。這個「四層化」對 thesis 是雙面刃：好的一面是單一買家砍單的傳導被稀釋（§5 bear 路徑因此延後）；壞的一面是後三層（neocloud / 主權 / 中國）的需求品質遠不如 hyperscaler 自有現金流——neocloud 靠債、主權靠財政預算、中國靠政策，任一資金來源轉緊都可能比 hyperscaler 更早收手。所以「需求廣化」這個多頭訊號，必須拆開看每一層的資金品質，而不是只看總額在增長。

用三情境把 2030 TAM 框起來，每個數字都能回溯到「假設改了什麼」——但先看口徑，三家機構的數字根本不是同一指標，不可直接並列。

需求三角驗證（top-down vs bottom-up 對帳，硬規則）：把三層數字疊起來對帳——下游 capex（誰出錢）vs 上游晶片營收（錢變成什麼）vs 最終 AI app 營收（賺回多少）：

  • 上游 AI 晶片營收（混合口徑）：NVDA DC（FY27 consensus ~$350B+）+ AVGO AI（$56B）+ AMD DC + TSMC AI 製造 → 量級 ~$450-500B（含 NVDA system 與 TSMC wafer 的 double-count 風險，僅供量級）。
  • 最終 AI app 營收（即時 run-rate，gross-basis）：OpenAI ~$24-33B + Anthropic ~$30-47B（口徑爭議：Anthropic $47B 為 gross ARR（Series H, 2026-05；含 AWS/Google/MSFT cloud reseller pass-through），其中 $30B 為 2026-04 較早里程碑；OpenAI 主張 net 約 $22B）≈ 已披露純 AI app ~$55-70B （gross 口徑，非 net 認列）；加上未拆分的 hyperscaler 自家 AI 業務，樂觀補到 <$150B。注：7-9x 變現缺口計算採此 gross 數，若改 net 口徑缺口更大。

對帳結論：capex（~$725B）比晶片營收（~$450-500B）約 1.5-2x，這個缺口合理（含電力 / 土建 / 網路，非晶片）。但「晶片營收 vs 已驗證 AI 終端營收」缺口高達 ~7-9x（$450-500B vs $55-70B）——這正是 Sequoia「$600B question」的核心，也是本報告 thesis 的脆弱點。採信哪邊？短期採信下游 capex（T1 法說承諾最硬、且 RPO 佐證，見 §5），因為終端 ARR 雖落後但成長極快（Anthropic 5 個月 5x）；但 7-9x 的絕對缺口意味著「變現必須在 2027-28 大幅收斂」這個 [A:] 假設若不成立，bear 情境就成立。口徑警告：capex 為曆年 guidance、晶片為財年混合、終端為即時 run-rate，三者非同期，故缺口倍數是量級判斷而非精算。

7-9x 缺口「為什麼可以暫時存在」——三個結構性緩衝。一個 7-9x 的變現缺口若放在傳統製造業早就是泡沫鐵證，但 AI 算力的特殊結構讓它可以暫時存在，而理解這三個緩衝正是判斷它「何時從緩衝變裂縫」的關鍵。其一，企業 AI 預算的「實驗性質」：2026 的企業 AI 支出本質是 R&D / 戰略卡位，不是要求當期 ROI 的營運支出——CFO 們把它當「不投就出局」的選擇權買，所以容忍零回報的窗口比一般 capex 長（這也是為何 MIT「95% 零 ROI」沒有立刻引發砍單）。其二，hyperscaler 的自家補貼：四大用雲端、廣告、軟體的成熟現金流補貼 AI 業務的虧損——Meta 的廣告（$196B、+22%）、Microsoft 的 Office/Azure、Google 的搜尋，都在替 AI 投資「輸血」，使缺口可以靠母體獲利撐住而非立刻反映在破產上。其三，預付結構把時點往後推：ORCL RPO $638B、MSFT RPO $625B 這些是「已簽約、分年認列」的預付承諾，意味著終端買家（OpenAI 等）的付款義務已鎖定、只是還沒到認列時點——帳面上「營收還沒到」不等於「需求不存在」。什麼條件下緩衝變裂縫？當這三者任一鬆動：① 企業把 AI 從「戰略選擇權」重新歸類為「要當期 ROI 的營運支出」（觸發點 = 一輪宏觀衰退逼 CFO 砍實驗性預算）；② hyperscaler 的母體現金流轉弱、無力續補貼（觸發點 = 廣告 / 雲端成長失速）；③ 預付方（OpenAI / Anthropic）的下一輪融資斷檔、無法履行採購承諾（觸發點 = §3 circular financing 一環斷裂）。這三個觸發點，就是 §8 證偽表把「AI app / 晶片營收比」與「capex 增速」列為核心 kill metric 的深層理由——它們是緩衝三柱崩塌的可觀測代理。

需求可持續性辯論：把多空最強版本完整攤開。這是本報告最該寫透的一場辯論，因為 thesis 的「mixed」評級正源於它——量能側多頭與變現側空頭各有最強證據，且都沒被對方駁倒。

多頭最強版本（Jensen / capacity-constrained 派）：需求不是投機，是真實到「供給根本跟不上」。最硬的證據是 Jensen Huang 公開指出——連兩代舊的 GPU（H100/H200）spot 租金都還在漲、根本租不到，這在投機泡沫裡不可能發生（泡沫的特徵是新產能湧入壓垮舊資產殘值，而非舊資產持續升值）。配上量能數據：Google token 量 3.2 quadrillion/月（YoY +7x）、Gemini API 月請求 85B（YoY +142%）、agentic 每 task 耗 token 為 chatbot 的 5-30x——這些不是預測，是已發生的實測用量。多頭的邏輯閉環是：真實用量爆炸 → 算力被吃滿 → 連舊卡都緊缺 → 故 capex 是在追趕需求而非製造需求，所以 $725B 不是泡沫是「人類史上最大基建」的合理 capex。

空頭最強版本（Cahn / MIT / Fed 派）：用量爆炸是真的，但用量 ≠ 營收 ≠ 利潤。Sequoia 的 David Cahn 用一個簡單算式撕開缺口——把 Nvidia DC 營收 ×2（推總 infra 投入）再 ×2（覆蓋 50% 毛利所需的終端營收），得出每年需要的終端 AI 營收規模，而實際 end-user AI 營收只有「數百億/年」級，gap 在 2026 持續擴大而非收斂。更致命的是 MIT Project NANDA 的實證：基於 $30-40B 企業支出、52 高管訪談 + 153 leader 問卷 + 300 公開部署，95% 企業 GenAI pilot 對 P&L 零可量測影響——失敗點在 pilot→production 的整合，不是模型能力。空頭的邏輯閉環是：token 用量再大，若 95% 的企業部署轉不成 P&L 改善，那這些算力就是「被使用但不被付費（或付費遠低於成本）」，缺口不會靠用量增長自動收斂。Fed 把 AI 列為 2026 頂級系統性風險（僅次地緣）正是這套邏輯的體制背書。

為什麼兩邊都對、且這正是 thesis 脆弱的根源：多頭測的是「供給側的緊俏」（租價、lead time、token 量），空頭測的是「需求側的變現」（終端營收 / 成本比、企業 ROI）——兩者測的根本不是同一件事，所以可以同時為真：算力確實被吃滿（多頭對），但吃滿不代表付得起（空頭對）。本報告的裁決因此不站隊「泡沫 yes/no」，而是接受「短期供給緊 + 中期變現懸」的並存，把判斷重心放在「緩衝三柱（企業實驗預算 / hyperscaler 補貼 / 預付結構）能撐多久」——這就是把一場吵不完的哲學辯論，轉成 §8 可監測的 metric 戰場。

Token 經濟學：量價淨效果，變現缺口的微觀引擎

上面那場辯論其實可以收斂成一條算式，而 token 經濟學就是那條算式。token（模型處理文字的最小計費單位）是這條兆元 capex 鏈最末端的「計價單位」——hyperscaler 蓋的每一瓦資料中心、買的每一顆 GPU，最終都要靠賣 token（或 token 包裝成的訂閱與 agent 服務）回收。所以 capex 回收的公式很短：回收 = 每 token 毛利 × token 量 ÷ 資產壽命。問題在於，這三個變數現在同時在劇烈變動，而且不同向。

量與價的事實，先分開看。量的一側是真實的爆炸：Google 在 I/O 2026 公布全平台月處理 token 量達 3.2 quadrillion（3,200 兆），較一年前的 480 兆年增 7 倍；Microsoft Q2 FY26 法說披露 Foundry 上有 250+ 客戶年處理量將破 1 兆 token、季消費 >$1M 的客戶數年增近 80%——用量端沒有任何放緩跡象。但價的一側崩得更快：Epoch AI 對六個 benchmark 的系統性測量顯示，達到同等能力的 token 單價年降中位數 ~50x，2024 年後加速到 ~200x（區間 9x-900x，依任務而異）；具體錨點是 GPT-4 級能力的價格從 2022 底的 ~$20/M tokens 跌到 ~$0.40/M（~50x），開源與蒸餾模型把「夠用的智能」打到接近免費。這個跌速沒有任何硬體 commodity 可以類比——DRAM 最慘的年份跌 6 成，token 是每年跌九成以上。

量價相乘的淨效果，才是判準。把兩側放進同一條算式：推導：若停在同一能力點，量 ~7x/年 × 價 ~÷50x/年 → 同能力營收線每年縮水到原本的 ~0.1-0.15x——也就是說，光靠用量成長，追不上單價崩跌，「token 量爆炸」本身並不是多頭證據。那為什麼實際 AI 營收還在漲（OpenAI Q1 2026 單季 ~$5.7B、Anthropic 年化 ~$45B 且 Q2 可望再翻倍）？因為有兩個抵消機制在硬撐：其一是能力階梯上移——營收不停留在被打到地板價的舊能力，而是不斷遷移到新 frontier（reasoning、agentic）重新訂高價，等於在一條不斷塌陷的價格曲線上持續往上游搬家；其二是 agentic 乘數——一個 agent task 觸發 10-20 次模型呼叫、RAG 把 context 膨脹 3-5x，第三方估算企業 AI 帳單兩年漲 ~320%，即便同期 token 單價跌了兩個數量級。所以 Jevons 悖論（降價刺激更大用量）在這裡不是定律，是條件式：第三方對 Goldman「2030 用量 24x」預測的敏感度拆解顯示，24 倍用量只有在單價跌幅 ≤~90% 時才救得了營收線；若競價把單價壓掉 95-97%（成熟 commodity 市場的常態結局），同一份用量預測算出來的是營收壓縮。一句話：變現缺口能否收斂，不取決於 token 量漲多快，而取決於「能力階梯 + agentic 乘數」能否持續跑贏單價崩跌。

對接回 capex 主線：token 毛利要先翻過折舊那座山。回到回收公式的分母——第三方估算五大 hyperscaler 的 AI 資產年折舊已達 ~$400B 量級，約當其 2025 年合計利潤：意思是 token 業務（含其包裝形態）的毛利要先蓋過這座每年自動長高的折舊山，capex 才開始真正「賺錢」。這就把 token 經濟學跟 §5 的折舊反身性接成同一條鏈：每 token 經濟學每惡化一分（單價跌快於量增），回收期就拉長一分，折舊年限「誠實化」的壓力就大一分。而 token 價值鏈的利潤分布，與 §3 晶片利潤池驚人地同構——serving 層（neocloud 租 GPU 算 token）的 H100 租價已從峰值跌 64-75%、毛利被擠向零；只有 frontier 能力層（最新模型）保有定價權——token 的錢和晶片的錢一樣，都往「不可替代的那一層」集中，這是 §7 分歧 3 在需求側的鏡像。對 PM 的監測指標因此多一條：盯「token 量增速 vs 同能力單價跌速」的比值——量增速（現 ~7x/年）若降到 2x 以下、或單價跌速再加快，缺口收斂的數學就不成立；Google 每年 I/O 與各季法說的 token 量披露，是這條指標最便宜的觀測窗。

因果閉合（回應 §1 的需求驅動）：§1 提的需求驅動是「生成式 AI + agentic 拐點」。未來 5-10 年走向？需求量能側強多頭：Google token 量 3.2 quadrillion/月（YoY +7x）、agentic 每 task 耗 token 為 chatbot 的 5-30x、Anthropic 5 個月 5x。但拐點的持續性取決於 Gartner 點出的關鍵——「迄今 AI 支出主要由科技公司 / hyperscaler 驅動，企業端尚未真正發力，2026 預期為 inflection year」。也就是說，需求驅動的下一棒（企業端買單）尚未驗證，這是把 thesis 從「結構性」打成「mixed」的原因：量能（token）真實，但變現（企業 ROI）未證。

需求側留給 PM 三個可操作判斷。其一，TAM 的口徑亂象（$1.7T vs $7T）提醒我們別用機構大數字當 thesis 錨——真正該盯的是 7-9x 的變現缺口能否收斂，而不是 TAM 多大。其二，需求的「不可替代性」是分層的：押 frontier 訓練曝險（NVDA 高階 / TSM N2 / HBM4）比押企業推論曝險（吃 token 單價的環節）更安全，因為前者 price-inelastic、後者是 MIT「95% 零 ROI」的承壓點。其三，token 經濟學給了缺口收斂一條可算的判準——「量增速 vs 同能力單價跌速」的比值：量 ~7x/年對上價 ~÷50x/年，缺口靠自然成長收不攏，必須靠能力階梯與 agentic 乘數；當 Google token 量增速降到 2x 以下、或新能力定價開始失守，就是 NC#2 從 AT_RISK 轉 BROKEN 的領先讀數。這直接影響 §9 的 ticker 深度排序。

§5 供需裁決 + 短中長期推估 + 投資時鐘

⚠️ 本章含 judgment：🟡 bullet 代表推論，附事實鏈與信心度 + ⚠ 證偽條件
綜合 §3（供給）與 §4（需求），結論方向是：未來 12 個月供給短缺（lead time 36-52 週、CoWoS 滿到 mid-2027、租價反漲）；未來 3 年轉向平衡偏緊，因為產能 2027-28 才追上而變現缺口同時擴大。這不是過剩週期的頂，而是「build-out 末段、早期裂縫已現」的 Phase II。

資本週期證據（裁決的量化依據，至少引 2 項）：用「高報酬引資本→過剩→均值回歸」框架檢查三個指標：

  • ① 折舊年限變更（最硬訊號）：hyperscaler 集體把 server 折舊由 3-4y 延至 5-6y（Alphabet 2023 server 4y→6y、Amazon 5y→6y、Meta 漸延至 5.5y），合計每年省 ~$18B 折舊、機械性 flatter earnings。但 Amazon Q1 2025 反向把部分 server 6y→5y，理由「AI/ML 技術迭代加速」——這是過剩風險的早期反向訊號（T1 動作）：當折舊開始縮短，代表資產實際壽命被重估、回收期被質疑。但須限定 scope：此次縮短僅適用部分 server 資產（AI/ML 伺服器子集），非 Amazon 全艦隊折舊政策翻轉，財務影響約 $0.7B/年（季度 D&A +~$217M）——故這是「一家、一子集」的早期訊號，本 thesis 監測的是「是否有第二家跟進」（§7 NC#2 / §8 證偽表），而非把它讀成 fleet-wide 政策反轉。矛盾點在於 NVDA 旗艦 GPU 實際 ~3y 換代（每代效能 2-3x），hyperscaler 卻仍多記 5-6y 折舊。
  • ② capital intensity / ROIC（方向性）：capital intensity 達營收 45-57%（歷史「不可想像」水準）[A: 以 Introl 2026 估算為錨，各家口徑略異]；capital base 增速 >> 營收/EPS 增速 → ROIC 受壓（mixed，META 因增收 + 減本 ROIC 反升）。缺各家精確逐年 ROIC 數列，故此項為方向性判斷。
  • ③ FCF 拐點 + 外部發債（最清晰拐點）：2025-09 起 hyperscaler 集體轉外部融資，capex 開始超過 operating cash flow——META 2025-10 發債 $30B、ORCL 2025-09 發債 $18B、2025 全年 AI-related 新債 ~$121B（為 2020-24 年均 $28B 的 4 倍+），MS 估 2026 AI 債近翻倍至 ~$570B、至 2026-05-31 已發 $236B（約去年同期 4 倍），MS/JPM 預估 tech 未來數年需再發 ~$1.5T。這是資本週期進入「外部融資驅動」late-cycle 特徵的定義性訊號。

折舊年限的反饋鏈：會計 → EPS → capex 決策（為什麼這是裁決級訊號）。折舊年限不是枯燥的會計細節，它是一條會自我強化的反饋鏈，理解它才知道為何 Amazon「6y→5y」是早警而非雜訊。鏈條這樣走：第一步（會計）——折舊是 GPU 成本攤提到損益表的速度。把年限從 5y 延到 6y，等於把同一筆 GPU 成本攤更多年、每年認列的折舊更少；hyperscaler 集體把 server 由 3-4y 延至 5-6y，合計每年「省」~$18B 折舊。第二步（EPS）——折舊少 = 帳面成本少 = reported earnings 機械性變高（Alphabet 2023 那次延長就讓 earnings +$3B）。這不是業務變好，是會計假設變寬鬆撐出來的 EPS。第三步（capex 決策）——更高的 reported EPS 撐住更高的股價與更低的資金成本，反過來讓 hyperscaler 更敢加碼 capex；同時，把 GPU 當「6 年資產」記帳，意味著管理層對外宣稱「這些卡能用 6 年」，於是更願意大規模採購。現在反向了：Amazon Q1 2025 把部分 server 6y→5y，理由「AI/ML 技術迭代加速」——這是反饋鏈第一次往回轉。它同時做三件殺傷力疊加的事：壓低未來 EPS（成本攤更快）、公開承認資產實際壽命比帳列短（坐實「NVDA 旗艦 ~3y 換代、卻記 5-6y 折舊」的矛盾）、暗示回收期被高估。矛盾的量級很具體：旗艦 GPU 每代效能 2-3x、實際約 3y 就被換掉，但帳上仍多記 5-6y——這中間 2-3 年的差，就是 Michael Burry 指控的「2026-2028 間低估折舊 $176B、虛增獲利」的來源。所以本 thesis 監測的不是 Amazon 這一家（它只是部分 server 子集、~$0.7B/年的小動作），而是「是否有第二家跟進」——因為第二家跟進就代表這條反饋鏈從「一家的謹慎」變成「行業對資產壽命的集體重估」，那才是裂縫擴大的訊號。

FCF 轉外部發債為什麼是週期後段訊號——對照 2000 年的 vendor financing。capex 的資金來源從「內生現金流」轉向「外部發債」，是資本週期理論裡最典型的 late-cycle 標誌，原因在於它揭示了「報酬已經追不上投入速度」。機制是：早期 capex 由經營現金流自然支應，代表投資仍在「賺得回來」的舒適區；一旦 capex 超過 operating cash flow、必須舉債補差（2025-09 起的拐點），就意味著產業為了維持軍備競賽的節奏，開始預支未來——這正是「高報酬引資本 → 資本湧入超過內生供給 → 轉外部融資墊高 → 最終報酬均值回歸」這條均值回歸曲線的後半段。但本輪與 2000 有一個關鍵差異，必須誠實標出：2000 電信的外部融資是垃圾債 + vendor financing（高槓桿、且設備商替買不起的客戶融資，帳面需求含虛胖），而本輪的 hyperscaler 發債後 leverage 多在 0.4-0.7x（vs IG 均 ~3x），主體信用仍穩——市場對此的補償要求也已升高（AI 相關新債 new-issue concession 平均 ~12bp，vs 大盤 ~2.5bp，代表債市已開始為 AI 長天期科技債要求溢價）。所以這條訊號的正確讀法不是「立刻崩」，而是「安全邊際正在被預支」：主體（四大）還很穩，但邊緣（neocloud，見 §3 CRWV 的 SOFR+450bp 與抵押品縮水悖論）已在用更貴的錢、更危險的結構續命——當主體的 0.4-0.7x 槓桿開始往上爬、或邊緣的違約開始發生，就是這條訊號從「late-cycle 預警」轉為「digestion 觸發」的時刻。

§3 + §4 → 供需裁決

依本章 §3 與 §4 的推導，未來 12 個月供給短缺、未來 3 年轉向平衡偏緊。短期的證據是硬的：GPU lead time 36-52 週、Blackwell sold out 至 mid-2026（backlog 3.6M units）、CoWoS full allocation 至 mid-2027，加上 H100 一年租約價反漲 +40%——現貨價格與訂單能見度兩端都找不到 glut。中期的關鍵則是一個因果倒置：教科書資本週期是「高報酬引資本 → 產能過剩 → 報酬崩」，但本輪供給端的實體瓶頸（變壓器 5 年、電網 4-10 年、封裝配滿）把「產能過剩」這條路封死了——資本湧入的壓力無法在實體面宣洩，於是改道財務面：表現為 7-9x 變現缺口、折舊年限被迫誠實（Amazon 6y→5y）、capex 超過經營現金流後轉外部發債。所以裁決的完整版是：實體面短缺 → 平衡偏緊；財務面的「過剩」已在累積——2027-28 變現若未收斂，需求端急煞會比產能過剩先到。這就是為什麼本報告說本輪的 kill 是 financial 過剩而非 physical 過剩：不是修辭，是瓶頸結構決定的宣洩路徑。

怎麼讀：第一，12M 的 trigger 是租價（領先指標），3Y 是 capex 增速（傳導指標），5Y+ 是變現比（終局指標）——三層 trigger 對應三層時滯，盯對指標才不會用滯後的營收做決策。第二，bear 都不是「需求消失」而是「變現追不上」，這呼應裁決——本輪的 kill 是 financial 不是 physical。

投資時鐘 phase 判定：當前處 Phase II（build-out / capex 爆發中後段），更精確說是「late-build-out 早期裂縫」。各 phase 贏家切換邏輯：Phase I-II（現在，capex 狂奔）= 設計 + 代工 + HBM 上游通吃（NVDA/TSM/AVGO/MU）；Phase III（digestion，若 capex 增速反轉）= 上游營收先受傷、低成本自研 ASIC 與軟體變現方相對抗跌；Phase IV（復甦）= 變現驗證後的二次成長。換 phase 的必要 ∩ 充分條件：從 II 進 III 的必要條件 = capex YoY 增速首次跌破 +20%；充分條件 = 必要條件 且 RPO 增速同步反轉（光是 capex 降速但 RPO 仍強，代表只是節奏調整非 digestion）。兩條件齊備才算真正換 phase。

庫存 / 訂單週期指標（phase 判定的證據）：AI 算力沒有傳統「channel inventory 週數」，故用三個對應的領先指標——① GPU lead time 36-52 週（仍長，shortage）；② Blackwell backlog 3.6M units、sold out to mid-2026（訂單能見度極高）；③ RPO（剩餘履約義務，雲端需求能見度的最佳 proxy）爆發：ORCL RPO $638B（YoY +363%）、MSFT commercial RPO $625B（YoY +110%）、Google Cloud RPO >$460B（季增近一倍）。RPO 是訂閱型業務的 backlog，這三家齊爆代表雲端需求能見度未轉弱。唯一的庫存堆積反證是 NVDA H20 的 $4.5B charge——但那是地緣 specific（出口管制致 H20 需求轉弱），非全面 glut。口徑警告：RPO 含大量 AI 預付合約，集中度與兌現風險需與 §4 終端缺口交叉看，不能只看大數字。

Bear 世界長什麼樣：用 2022-23 的傳導實證映射本輪

裁決說「未來 3 年轉平衡偏緊」，但 PM 必須先看清「如果 bear 成真，那個世界長什麼樣、按什麼順序展開」——而我們不必純靠想像，因為 2022-23 剛剛真實演過一次 digestion，可以拿來逐步映射。

2022-23 的傳導實證（誰先被砍、時滯多長、股價領先幾個月）。那一輪的順序是：最成熟、基期最大的買家先收手——Amazon 2023 capex −17% YoY 至 $52.7B（AWS capex −~10% 至 $24.8B）率先砍，而 Microsoft / Google 因綁定 ChatGPT 反而同年加碼（MSFT +45% / +$12.8B）。這告訴我們 digestion 不是齊步，而是從邊際最弱的需求開始。傳導到上游時：即便有 Amazon 砍單，NVDA DC 營收 Q3 FY2023 仍 +31% YoY——單一買家砍單在多元客戶結構下被稀釋，傳導的時滯與幅度都比直覺小。而股價的領先性最關鍵：§1 已量化半導體股價（SOX）平均領先基本面約 3 季，2018-19 那次更精確——capex 增速反轉後約 2-3 個 quarter，NVDA DC 營收就從 +58% 翻成 -10%。

映射到本輪：bear 若觸發，會這樣一步步展開。把上面的實證套到 2026 的結構上，bear 世界的劇本是：觸發——四大某一家（最可能是基期最大、變現壓力最先浮現的一家，類比當年的 Amazon）在 2027 capex guidance 上首次明顯降速（YoY 跌破 +20%），或第二家跟進折舊縮短坐實資產壽命疑慮。誰先砍單——不是龍頭 NVDA 先掉，而是邊際需求先斷：高槓桿 neocloud（CRWV 等，利用率掉破 80% 即翻負）、merchant GPU 第二供應源（AMD，digestion 時第二源比龍頭更早被砍）、以及資金品質最差的那層買家（neocloud / 部分主權）。營收傳導順序——邊際 neocloud 砍單 → AMD/merchant GPU 訂單先軟 → 約 2-3 季後傳到 NVDA DC 營收增速放緩 → 再傳到 TSMC（CoWoS 訂單）與 HBM。股價時序——按 §1 領先性，SOX / 上游晶片股會在營收見頂前約 2-3 季先轉弱，所以「等 NVDA 財報確認增速放緩才賣」必然晚一個身位。這就是為何本報告把租價、capex guidance、折舊政策列為「比財報早 2-3 季」的領先訊號。

三重 buffer 為何延後但不取消傳導。本輪相較 2000 / 2018-19 多了三道緩衝，它們會把上述劇本的時鐘往後撥，但不會讓它消失。其一，NVDA backlog 覆蓋：~$500B backlog 延伸至 2026、三年產品能見度（Rubin 2026 / Rubin Ultra 2027 / Feynman 2028）、Q1 FY2027 total supply-related commitments $119.0B——這意味著即使需求轉弱，已簽的 backlog 會撐住營收數季，延後傳導。其二，TSMC 的 prepay / LTA：2026 capex $52-56B、NVDA 已鎖定多數 HBM4（SK hynix + Samsung），供給端的長約與產能預訂讓上游不會在需求一軟就立刻見骨。其三，多元客戶結構：NVDA DC 營收 Hyperscale / ACIE 近對半，單一買家砍單被稀釋（2022-23 已驗證）。但 buffer ≠ 免疫：backlog 會耗盡、LTA 會到期、多元客戶在系統性砍單時會同向——三重 buffer 的作用是把 2-3 季的傳導時滯拉長到可能 3-4 季，給 PM 更多反應時間，但它改變的是「何時」而非「會不會」。所以正確的 bear 準備不是「不持有」，而是「持有但盯著三道 buffer 的耗損速度」——backlog 覆蓋月數縮短、LTA 到期未續、客戶集中度回升，任一就是 buffer 在變薄的訊號。

二階效應：AI capex 外溢到電力、記憶體、債市與總體經濟

這輪資本週期的規模已大到會反咬鄰接領域，而這些二階效應同時是「需求真實」的佐證、也是「政治 / 監管反作用力」的來源——PM 必須兩面都看。

  • 電力：從 capex 的下游變成 capex 的天花板。IEA 估 2026 全球 DC 用電可達 ~1,050 TWh（若 DC 是國家，是全球第 5 大用電國，介於日本與俄羅斯之間）；美國 DC 用電 2024→2030 增 ~240 TWh（+130%），占 2030 前全美需求成長的 ~50%。瓶頸已下移到實體裝備：高壓變壓器交期由 pre-2020 的 24-30 個月拉長到 5 年、gas turbine（GE Vernova / Siemens / Mitsubishi）backlog 各約 5 年、turbine 價估 2027 底達 $600/kW（較 2019 近 3 倍）。這條二階效應是雙面的：一面是限制過度建設（與 2000 光纖「想建就建」不同，5 年交期的長前置裝備反而壓住了 glut 速度，是 §3「瓶頸是朋友」的延伸）；另一面是政治反作用力——美國住宅電價 2025 已漲 11.5%（超通膨）、2030 前估漲最高 40%，Virginia 發電成本恐飆 57%，Dallas Fed 估 DC 單獨推升 headline PCE 通膨 +0.05pp（2026）→ +0.13pp（2030）。電價外溢到一般家庭，就是 §7 kill scenario「社會許可撤回」的種子。
  • 記憶體：HBM 排擠 DRAM，AI capex 漲價反咬出錢方。HBM 每 GB 耗 wafer 約為 DDR5 的 3 倍，AI 加速器吃掉封裝產能 + high-bin DRAM die，結構性排擠傳統 DRAM。2026-Q1 是拐點——HBM 每片 wafer 營收與獲利雙雙被 DDR5 64GB RDIMM 超越，廠商重配產能並用激進 HBM 漲價回應，結果是 HBM 量價齊揚 + 傳統 DRAM 因供給被排擠也漲（2Q26 傳統 DRAM 合約價 QoQ +58-63%、NAND 部分口徑 +75%）。這條外溢直接反映在出錢方的帳上：Microsoft 與 Meta 已把「記憶體 / 零組件漲價」明確點名為 2026 capex 上修的一部分（§1）——也就是說，AI 軍備競賽正在推高自己的投入成本，這是「供給端通膨反咬需求端」的閉環，會讓 capex 數字「虛胖」一塊（增量不全是買更多算力，部分是同樣算力付更多錢）。
  • 總體經濟：AI capex 占 GDP 與成長貢獻雙雙超越 2000 電信峰值。這是「集中度風險」最強的一句——AI capex 約占 GDP 1.2-1.28%（Mag-7，Q2 2025 annualized），已超過 2000 電信 capex 占 GDP 峰值的 ~1.0%；2025 前三季 AI 相關貢獻實質 GDP 成長 0.97pp（= 總成長 39%，vs 2000 電信 28%）；Harvard 的 Furman 甚至估「無 DC 則 2025H1 美國 GDP 成長僅 0.1%」。含義很尖銳：AI capex 對總體經濟的支撐已大於 2000 電信，這意味著一旦 digestion 發生，對總體的拖累也將大於 2000——這把「AI 過剩」從一個產業問題升級成一個總體問題，也是 Fed 把它列為頂級系統性風險的根本理由。對 PM 的含義：本 thesis 的 bear 不只殺半導體股，會透過 GDP 拖累殺到大盤 beta，故 bear 情境下的對沖不能只在 sector 內做。
本章給 PM 最重要的可操作結論：現在 act（續抱核心倉），但盯的是領先指標不是營收。供給端三證據 + RPO 爆發 = 短期 shortage 無疑，所以「等過剩確認才賣」會晚 2-3 季（§1 領先性）。真正的賣出觸發是 §8 會列的三個領先訊號——租價跌破 $2.0、第二家折舊縮短、capex 增速跌破 +20%——任一出現都比財報更早預告 Phase II→III 切換。bear 世界的劇本（邊際 neocloud 先斷 → AMD → NVDA → TSMC/HBM，三重 buffer 延後 2-3 季傳導但不取消）告訴我們持有順序：核心壓最有 buffer 的瓶頸玩家（NVDA backlog / TSM LTA），避開最先被砍的邊際曝險（高槓桿 neocloud / merchant 第二源）。而二階效應（電價政治反作用、AI 占 GDP 超 2000）提醒：bear 情境的對沖要延伸到大盤 beta，不能只在 sector 內。

§6 產業經濟學與估值傳導

這條鏈的錢怎麼賺、估值跟著什麼變數動？簡單說：上游靠「瓶頸 + 高毛利 + 量價齊揚」賺結構性利潤，估值則跟著「成長假設兌現度」走——而非倍數本身。當倍數已不貴（NVDA Fwd P/E 低於歷史均值），re-rating 的鑰匙就交到 E（盈餘）能否追上 consensus 手裡。

看上游各環節的 unit economics 與 ASP 動態，判斷誰最能抗 commoditization（商品化，即被殺價）。

怎麼讀：第一，抗 commoditization 能力由強到弱是「代工（TSMC）> 設計（NVDA）> HBM > GPU 短租」——這直接對應 §9 ticker 深度。第二，唯一在跌的 ASP 是 GPU on-demand 短租（-64%），但這是 neocloud 之間的價格戰、不是上游設計/代工的毛利在崩；上游 GM 全數維持 66-75%，證明議價權仍牢牢在瓶頸端。

為什麼上游能維持 66-75% 毛利而下游被殺到 5-6%——產業經濟學的根因。同一條鏈，毛利從 75% 到 5% 落差 15 倍，不是偶然而是結構決定的。上游（設計 + 代工 + HBM）賺的是「稀缺性租金」：NVDA 靠 CUDA 生態的轉換成本鎖客（軟體護城河，客戶換掉要重寫整套 stack）、TSMC 靠先進製程近壟斷 + CoWoS 配置權（產能稀缺，誰都得排隊）、HBM 三寡占靠排擠定價權（每 GB 耗 3 倍 wafer，可選擇少做傳統 DRAM 抬價）。這三段的共同點是「供給無法快速複製」——蓋一座先進 fab 要 2-3 年 + 數百億、CUDA 生態十年累積、HBM 良率是高門檻——所以即使需求爆炸，供給也跟不上，租金（毛利）就守得住。下游 ODM 反之賺「組裝代工費」：consignment 模式下，最肥的料件（GPU / HBM）由客戶自購寄售、利潤不過 ODM 損益表，ODM 只賺裝配費，且裝配能力可複製（多家鴻海 / 廣達 / 緯穎競爭），故毛利被壓到 5-6%。對估值傳導的含義：上游的稀缺性租金讓它的 E 在 shortage 期極肥，但也讓它在 digestion 期跌得最重（租金消失最快）；下游的薄毛利反而讓它的 E 波動小、更像「量的 beta」。這解釋了為何本報告把核心倉壓在上游稀缺性玩家而非下游量能玩家——要的是 shortage 租金的 upside，代價是承擔它在 bear 情境的高 beta。

估值傳導：為什麼 Fwd P/E 低於歷史均值，卻仍叫「未 priced」？這是本章最反直覺、也最容易被誤讀的一點。表面上 NVDA Fwd P/E ~22.5-24x 低於半導體 median 34（-33.8%）與自身 3/5/10 年均值，看起來「便宜、安全」；但「未 priced」指的不是「倍數沒漲」，而是「市場還沒把 shortage 紅利定價進倍數，也沒把變現缺口風險定價進 E」——這是兩個方向相反的「未定價」同時存在。要拆解這個悖論，得先理解 P/E 是 P/E = Price /（EPS × 倍數）這個恆等式裡，本輪的全部動作都發生在 E（盈餘）而非倍數。

機制是這樣的：當一家公司的 E 以極高速度成長（NVDA FY26 ~$316B → FY27 consensus ~$391B），分母 EPS 衝得比股價還快，Fwd P/E（用未來 12 個月 EPS 當分母）就會機械性地被壓低——這不是市場給了保守倍數，而是「高速成長的 E 把倍數自己稀釋掉了」。所以低 Fwd P/E 在這裡不是估值安全的證據，而是「市場已經把高成長 plug 進 E」的證據。真正決定股價的，從「倍數會不會擴張」轉移成「那個高 E 假設能不能兌現」——而 E 的可信度，正取決於本報告反覆強調的變現缺口（§4 的 7-9x）能否收斂。換句話說，倍數低 = 風險沒消失，只是換了地方藏：從「估值倍數泡沫」搬到「盈餘成長假設」裡。這與 2000 電信「E 還沒影子、倍數先瘋狂」恰好相反——2000 是倍數定價了不存在的未來，本輪是 E 定價了高度依賴 capex 兌現的未來。

de-rating 的觸發鏈：倍數安全不代表股價安全。既然風險藏在 E 裡，de-rating（殺估值）的觸發就不會來自「倍數太貴被壓縮」，而會走這條鏈：capex 增速反轉（§8 領先訊號）→ 分析師下修 FY27/FY28 營收 consensus → E 假設裂縫被定價 → 即使倍數不變，股價隨 E 下修等比例下殺。敏感度錨點很直接：若 FY27 營收兌現度從 100% 降到 75%（bear），E 下修約 25%，在倍數不變下股價約 -25%；若市場同時因「成長故事破裂」而把倍數從 22-24x 壓回 18x（double de-rating），跌幅會疊加到 -40% 以上——這正是 §1 統計「SOX 歷史 drawdown -30% 至 -82%」的來源，週期股的下行從來不是溫和的。反方向也成立：若變現缺口收斂、consensus 上修，因為倍數基期已低（low base），re-rating 的彈性反而比高倍數股更大。所以這是一個不對稱的「倍數安全、E 危險」結構——上行靠 E 兌現 + 低基期 re-rating，下行靠 E 落空 + 可能的 double de-rating。

出錢方（hyperscaler）的估值傳導不同步。值得對照的是：付這筆 capex 的四大，估值傳導路徑與上游 NVDA 相反。四大目前 Fwd P/E 多低於自身 5-10Y band（MSFT ~21x vs 5Y avg ~32、META ~17x、AMZN ~30x vs 5Y ~57、GOOGL ~29x ≈ 10Y 均），代表市場對「出錢方能把 capex 變現」這件事並未給溢價——樂觀的「變現會收斂」假設沒被 price 進買方倍數，而是 price 進上游晶片商的 E。傳導含義：當 bear 觸發、市場重新定價變現缺口，殺傷力會集中在上游晶片股（E 假設崩）而非買方（本就沒給溢價）——這直接影響 §9 的曝險排序與 bear 情境下的相對抗跌判斷。

估值傳導給 PM 一個關鍵校準：別用「PE 不貴」當安全感。NVDA Fwd P/E 低於歷史均值看似便宜，但那是因為 E 已被 plug 進高速成長——真正的風險不在倍數會不會壓縮，而在那個高 E（FY27 ~$391B）的成長假設會不會落空。這把監測重心從「估值分位」轉到「§8 證偽表的營收 / capex 增速 metric」。對 entry 的含義：倍數低 = 不必等大幅 de-rating 才進場，但要在 E 假設出現裂縫（capex 增速反轉）時果斷降曝險。

§7 Non-Consensus + Priced-in 檢驗 + Kill Scenario

⚠️ 本章含 judgment：每條分歧附 [F:] cornerstone + [X:] 三情境 + priced-in

Steel-man 反方（最有力的 3 條）：先給空頭最強版本，再逐點回應

下面三條不是稻草人——每條先用空頭自己最強的數據 + 邏輯鏈把論點立到最硬，再給本報告的逐點回應與證偽窗口。想不出能站住的反方 = 分析深度不足。

反方 1：2018-19 / 2022-23 digestion 重演（最強，傳導實證派）

空頭最強版本：本輪會像歷史上每一次半導體 capex 超級週期一樣，以 digestion 收尾，而且本次基期是兆元級，回檔幅度會更大。邏輯鏈：① capital intensity 已達營收 45-57%（Introl 估，歷史「不可想像」水準）——資本密度到這個極值，均值回歸是時間問題；② 2025-09 起集體轉外部發債（$121B → 2026 估 $570B），代表內生現金流已撐不住 capex；③ Michael Burry 的具體控訴——Meta/AMZN/MSFT/Google/Oracle 把 GPU 折舊拉到 5-6 年（真實經濟壽命 2-3 年），2026-2028 間低估折舊 $176B、虛增獲利，他的部位是對 SOXX 的槓桿空單（2027-01 到期 put，直接押 digestion 在 2027 顯性化）；④ 傳導實證——2018-19 capex 降速後 2-3 季，NVDA DC 營收從 +58% 翻成 -10%（§1）。Fed 已把 AI 列 2026 頂級系統性風險、Man Group 稱 AI 金融架構「unsustainable」為這套邏輯背書。逐點回應：傳導實證是對的，但 2022-23 同時證明了「digestion 不齊步 + 多元客戶稀釋砍單」——Amazon 砍 17% 那年 NVDA DC 仍 +31%；且本輪有三重 buffer（NVDA $500B backlog / TSMC LTA / Hyperscale-ACIE 對半客戶結構，§5）把傳導時滯從 2-3 季拉長到可能 3-4 季。Burry 的 $176B 折舊控訴是 thesis 最該認真對待的一條，但它的證偽是可觀測的（看是否有第二家跟進縮短折舊），而非已成定局。證偽窗口：2027 capex guidance 公布（2026-Q4 起陸續）+ 第二家折舊縮短揭露。

反方 2：circular financing 崩塌（資金循環派）

空頭最強版本：AI 需求有一塊是「同一筆錢在晶片商 → AI lab → 雲端商之間循環墊高彼此營收」的虛胖，一旦一環斷裂，連環營收同時縮水。Bloomberg 統計這類循環交易總額 >$800B，主幹包括：NVDA→OpenAI（原宣布 up to $100B，2026 已縮水為 $30B equity stake——Jensen Huang 2026-03 親自確認原 $100B「probably not in the cards」，屬 OpenAI 2026-02-27 宣布的 $110B funding round 一部分）、OpenAI→Oracle $300B（5 年）、OpenAI→Microsoft $250B、AMD→OpenAI warrants 換 6GW 採購、Oracle→NVDA 晶片採購（金額回流 NVDA，而 NVDA 又是 OpenAI 投資人）。更硬的數據：Moody's 點出五大 hyperscaler 累積 $662B 未起算的 DC 租賃承諾完全在表外，等於這五家最新調整後債務的 113%；neocloud 端 CRWV 總負債 $50.8B vs 股東權益 $4.8B（>10x 槓桿）、市場隱含 ~40% 違約風險、抵押品（GPU）租金已跌 50-70% 同步縮水（§3）。最脆弱的單點是 OpenAI 經濟學——spend >> revenue（年化 ~$24-33B vs 兆元級採購承諾），整個循環的最終承重點是它能否把承諾轉成真實營收。逐點回應：循環規模確實巨大（>$800B 統計、$662B 表外承諾），論證不倒；但要區分「主體」與「邊緣」——NVDA→OpenAI 從 $100B 縮到 $30B 這件事本身，恰恰證明這些承諾是動態可調、隨議價力消長的，而非 2000 年 vendor financing 那種一簽就鎖死的剛性負債；且 hyperscaler 主體 leverage 仍 0.4-0.7x（vs IG 均 3x），真正的崩塌風險集中在邊緣 neocloud（CRWV 等）而非主體。證偽窗口：任一大型 SPV / neocloud 出現債務違約或合約取消（如 Stargate 傳「stall」、CRWV refinancing 失敗）。

反方 3：企業端 ROI 證偽（MIT 路徑，需求下一棒接不上派）

空頭最強版本：token 用量爆炸是真的，但這些用量轉不成終端營收，因為買單的下一棒（企業端）的 ROI 根本沒出現。最硬的實證是 MIT Project NANDA——基於 $30-40B 企業支出、52 高管訪談 + 153 leader 問卷 + 300 公開部署，95% 企業 GenAI pilot 對 P&L 零可量測影響，稱之為「GenAI Divide」（高採用、低轉化），失敗點在 pilot→production 的整合。配上 Gartner 的觀察——迄今 AI 支出主要由科技公司 / hyperscaler 驅動，企業端尚未真正發力，2026 只是「預期的 inflection year」（尚未驗證）。邏輯鏈：若 2026 這個企業 inflection 落空，需求的第二層（企業買單）接不上第一層（hyperscaler 自建），那麼 §4 的 7-9x 缺口就不會收斂、緩衝三柱的「企業實驗預算」這一柱會最先崩。歷史鏡鑑是 2000 年「流量每 100 天翻倍」——當年用量也在長，只是成長率被高估一個數量級，本次「inference 需求無上限」可能犯同樣的錯（§1）。逐點回應：MIT 數據是 2025-07 的快照，測的是當下的 pilot→production 失敗率，不必然線性外推——agentic 工具（Claude Code 6 個月衝 $1B ARR、Anthropic 5 個月 5x）顯示變現在特定 vertical（coding）已突破；但「特定 vertical 突破」≠「企業普遍 ROI」，這條反方的核心質疑（普遍企業 ROI 未證）至今未被駁倒，故信心給「中」而非「低」。證偽窗口：2026H2 企業 AI 採用率 / ROI 數據、企業端 AI app 營收占比是否上升。

市場最低估的風險：不是「需求消失」（多空都在吵這個），而是折舊政策的反身性。一旦第二、三家 hyperscaler 跟進 Amazon 縮短折舊，它不只是一個訊號——它本身就是傳導機制的第一節：折舊縮短 → reported EPS 機械性下壓（讓本來就高的 E 假設更難達標）→ 股價與資金成本受壓 → capex 決策轉保守 → 上游訂單降速。換言之，這個會計動作會自己啟動它所預警的 digestion。它比任何分析師的 bear note 都更有殺傷力，因為它是 hyperscaler 對自己資產壽命的誠實重估（坐實「旗艦 GPU ~3y 換代、帳列 5-6y」的矛盾），而且不可逆——沒有公司會在縮短折舊後一年又改回去。市場目前幾乎沒在 price 這條路徑，這也是本報告把「第二家跟進」列為三大領先訊號之一的原因。

三條分歧裡，分歧 1（供給短缺）最可操作——它對（租價硬證據）且未 priced（倍數偏低），是清晰的 long 理由。分歧 3（利潤池集中）次之，對且部分已 priced，風險報酬轉佳，支持逢 selloff 建立 AVGO。分歧 2（變現缺口）雖對但樂觀已 price 進 E，故它不是 long 理由而是風險控制線——當折舊縮短擴散或變現比惡化，這條分歧由 INTACT 轉 BROKEN，就是降曝險的訊號。

§8 Catalyst Timeline + 證偽表

⚠️ 本章含 judgment：每條 metric ≥1 [X:] 三情境閾值
未來 18 個月該盯哪些節點、哪些數字一破就要砍曝險？領先順序很重要——租價（最早）→ 折舊政策 → capex guidance（最關鍵）→ 終端變現。盯對順序，比看財報早 2-3 季。

節點為什麼按這個順序排：catalyst 的價值不在「會發生什麼」，而在「它領先營收多久」。本表刻意按領先性排序——租價（供給真相，最難造假、月度可得，領先最久）→ 折舊政策（會計誠實度，季度揭露）→ capex guidance（需求真相，§1 證明 2-3 季傳導到上游營收）→ 上游毛利 / 變現比（滯後確認）。盯前段做決策、用後段驗證，就能比財報季早 2-3 季預判 Phase II→III 切換。以下逐節點附「為什麼這個節點重要」的機制：

2026-Q3 AVGO Q3 FY2026 財報實際 AI 營收 vs 自家 $16B guidance。為什麼重要：AVGO 是 §7 NC#3「ASIC 利潤池」的化身，它 meet/miss 自家已發 guidance 是整個 ASIC ramp 敘事的第一個 reality check——ASIC 是唯一成長率超越 merchant GPU（44.6% vs 16.1%）的環節，它若 miss，市場會質疑整條「自研侵蝕 NVDA 但利潤池仍集中上游」的論證。口徑校正：$16B 不是獨立門檻，而是 AVGO 已於 2026-06-03 法說正式發出的 Q3 AI 半導體 guidance（YoY +200%），故監測點是實際結果 meet / miss 這個已發 guidance，而非「是否 ≥$16B」（後者是 tautology）。檢視 Q3 actual AI 營收是否達到或超過 $16B 官方 guide，且全年仍維持 ≥$56B。若達成（actual ≥$16B 且全年 reiterate ≥$56B） → 確認 ASIC 利潤池論（§7 NC#3），加碼 AVGO。若落空（actual <$16B，即 miss 已發 guidance，或全年下修） → 觸發 §7 反方 1，重檢 AVGO 與整體 ASIC 曝險。🔵 [X: base 很可能 meet / bear 可能 miss 已發 guidance]

2027-Q1 AVGO FY2027 全年 AI 營收指引能否維持 ≥$100B（CEO Hock Tan 於 2026-06-03 Q2 法說重申的 >$100B 正式 guidance，近 FY26 $56B 的兩倍——這是當前最大、且先前未被本表監控的 bull claim）。檢視 隨後各季法說是否續守 >$100B FY27 指引、抑或下修。若達成（FY27 指引維持 >$100B） → ASIC 利潤池論的中期 upside 坐實，AVGO 🔴 核心邏輯強化。若落空（FY27 指引下修至 <$100B 或撤回） → ASIC ramp 不如承諾，§7 NC#3 由 INTACT 轉 AT_RISK，降 AVGO 曝險。🔵 [X: base 很可能維持 / bear 可能下修]

2026-Q3 H100 一年租約價月度追蹤。為什麼重要：這是全表最早、最難造假的供給真相——租價是供需即時撮合的市場價，不像財報有認列時滯、不像 guidance 是管理層 spin。一年期長約反映 frontier lab 鎖定產能的真實緊張（price-inelastic 需求），它若守住就直接否證「即將過剩」，它若崩跌就是 glut 比任何財報都早的第一聲警報。檢視 是否守 ≥$2.0/GPU-hr。隨新世代 GPU 逐步上量，供給研究普遍預期 2026 年中 H100 長約價有機會回落至 sub-$2 區間，故這條最早、最難造假的領先指標正逼近觸發帶，須月度緊盯。若達成（續守 ≥$2.0） → shortage 延續，§7 NC#1 INTACT。若落空（跌破 $1.7 重返 2025-10 低點） → glut 早訊，降全鏈曝險。🔵 [X: base 很可能續守 / bear 不太可能 2026 內崩跌]

2026-Q4 四大 2027 capex guidance 首次披露。為什麼重要：這是全表最關鍵的需求真相節點——§1 用 2018-19 實證了 capex 增速反轉後約 2-3 季就把上游營收從高雙位數翻負，所以 2027 guidance 是「比 NVDA 財報早 2-3 季」的需求領先訊號。尤其要看是否「非齊步」（某一家先降速，類比 2022-23 的 Amazon），因為 digestion 從來不是集體同時，而是從邊際最弱的買家開始。檢視 capex YoY 增速。若達成（仍 >+20%） → build-out 延續，維持核心倉。若落空（跌破 +20%） → 觸發 2018-19 式 digestion 路徑（§1），果斷降曝險。🔵 [X: base 很可能 >+20% / bear 可能單一家先降速]

2026-Q4 hyperscaler 折舊政策（10-K / 10-Q 揭露）。為什麼重要：這是「市場最低估的風險」的觀測點（§7）——折舊縮短是 hyperscaler 對自己資產壽命的誠實重估，比任何分析師 bear note 都更有殺傷力，因為它同時壓低 reported EPS（讓高 E 假設更難達標）+ 坐實 GPU 回收期疑慮。Amazon 已是第一家，第二家跟進就代表這從「一家的謹慎」變成「行業集體重估」（§5 反饋鏈）。檢視 是否有第二家跟進 Amazon 折舊縮短。若達成（無第二家擴散） → §7 NC#2 風險可控。若落空（≥1 家跟進 6y→5y） → 市場最低估的風險成真，重評全鏈 E 假設。🔵 [X: base 可能無擴散 / bear 可能 1-2 家跟進]

2027-Q1 TSMC / NVDA 上游毛利率。檢視 GM 是否守 ≥65% / ≥70%。若達成（GM 守 ≥65%/≥70%） → 利潤池集中論 INTACT。若落空（連兩季跌破） → 上游議價權結構性崩壞，§7 NC#3 BROKEN。🔵 [X: base 很可能守住 / bear 不太可能短期崩]

2027-H1 CoWoS / HBM 供給彈性轉折。檢視 lead time 是否從 36-52 週縮短至 <24 週。若達成（lead time 仍 >36 週） → shortage 紅利延續。若落空（大幅縮短至 <24 週） → 供給追上，議價權紅利進入倒數。🔵 [X: base 很可能 2027H1 仍緊 / bull 可能延到 2028]

非顯而易見受益者（角色機制，不只是名單）：

  • VRT（電力散熱）——瓶頸下移的二次受益，角色是「賭所有 AI 工廠都得通電散熱」。它的受益機制與晶片商根本不同：NVDA/AMD 的曝險綁在「賭對哪顆 GPU 贏」（GPU vs ASIC 之爭直接影響其份額），而 VRT 的曝險是「賭有沒有 AI 工廠在蓋」——不論最後是 NVDA GPU、Google TPU 還是 AVGO ASIC 勝出，每一座資料中心都得配電、散熱、做液冷。§5 二階效應已證瓶頸正從晶片（CoWoS/HBM）下移到電網與散熱（變壓器交期 5 年、gas turbine backlog 5 年、transceiver 吃 15-20% 功耗、OCP 共識瓶頸移向 power/racks），這代表「賣鏟子的人」正在從晶片商換成電力散熱商。市場仍多把 VRT 當「傳統電力設備股」定價，低估了它作為 AI build-out 必經環節的 structural 曝險——這是典型的「需求確定、但市場用錯類比定價」的低估機會。風險面：它不是瓶頸守門人（電力設備有多家供應商，護城河不如 TSMC CoWoS），且若 AI 工廠蓋速放緩，它一樣受 digestion 影響，只是時滯更後（電力是後段裝機）。
  • CRWV（neocloud）——不是 long，而是「過剩風險的高槓桿溫度計」。它在本表內但角色獨特：它既是需求方（向 NVDA 買卡）又是供給方（向 OpenAI 租算力），跨越供需兩端的舉債讓它成為整條鏈槓桿是否過熱的領先放大器。機制是——CRWV 的 backlog $99.4B 靠重債堆出（總負債 $50.8B vs 股東權益 $4.8B、>10x 槓桿、$34B 表外租賃、利息費用 YoY 翻 3 倍、DDTL 5.0 利率已跳到 SOFR+450bp），它的抵押品（GPU）租金已跌 50-70% 同步縮水，McKinsey 點出 BMaaS 利用率掉破 80% 報酬即翻負。所以 CRWV 不該被當成「便宜的 AI 純度 play」買進——它該被當成儀表看：當 CRWV 的 refinancing 利率續升、或利用率 / backlog 兌現出現裂縫，那就是整條鏈「以債換 backlog」的邊際結構開始崩的最早訊號，比 hyperscaler 主體（leverage 0.4-0.7x）出問題早得多。
  • MU / HBM 三寡占的隱性營業槓桿——MU 雖列 🟡，但 HBM4 ~$500/stack vs HBM3 ~$200 的量價齊揚，使記憶體環節從週期性虧損翻成高獲利。這個「由虧轉盈」的營業槓桿幅度被市場低估（記憶體股的歷史估值錨定在週期性虧損的記憶，而非結構性升級的現實），且它是無論 GPU 還是 ASIC 贏都吃得到的位置（ASIC 也得配 HBM），曝險的「不挑邊」特性與 TSMC 同構。
最高純度 play 是 NVDA + TSM（直接掌握設計與瓶頸製程，price-inelastic 訓練曝險最高）；最被低估其曝險的是 VRT（瓶頸下移的二次受益，市場仍當它是傳統電力股而非 AI play）。配置邏輯：核心壓 NVDA/TSM/AVGO（利潤池通吃），衛星布 MU/VRT（瓶頸下移 + HBM 槓桿），而 CRWV 雖在表內但定位是「監測過剩風險的指標」而非 long——它的 backlog 規模與債務結構，是判斷整條鏈槓桿是否過熱的領先溫度計。
