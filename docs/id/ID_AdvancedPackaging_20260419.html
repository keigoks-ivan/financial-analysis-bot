★ TSMC 官方 2026 symposium：CoWoS reticle scaling + SoW-X 重寫上限

本節整合 TSMC 在 2026 北美技術論壇對外揭示的 CoWoS interposer 路線圖與全新的 SoW-X（System-on-Wafer-X）類別。這是兩件事一起改寫了原本「CoWoS 在 2027-2028 撞牆 → 必須跳到 CoPoS / 玻璃」的市場敘事。

1）2026 5.5-reticle 的 yield 數字才是真正的 thesis 強化

roadmap 上最容易被忽略的數字其實不是 14-reticle 或 SoW-X，而是 2026 量產的 5.5-reticle CoWoS yield 已過 98%。這個數字的意義有兩層：第一層是直接的成本意義 — 過去 CoWoS-L 的 yield 假設業界普遍用 60-70% 估，TSMC 公布 98% 等於把 NVIDIA / AVGO / AMD 全鏈的 wafer cost 直接砍 30%+，這部分省下來的成本會在 2026-2027 GP margin 上反映；第二層是路線圖意義 — 5.5-reticle 在 2024 才剛量產，僅僅兩年就把 yield 從 ramp 期推到接近成熟，這證明 TSMC 對「reticle 倍數每兩年提高一階」這種節奏可以維持，等於把後續 9.5x → 14x → >14x 的時程可信度大幅提升。本 ID 原本對 CoPoS 2028-2029 的描述是「TSMC 原訂時程」，現在更精確的講法應該是：CoPoS 不再是 2028-2029 唯一解，CoWoS interposer 在 substrate 上能繼續疊到 14x、>14x，CoPoS 只是其中一條路線，而非必走。

2）9.5-reticle (2027) 與 14-reticle (2028)：把 ASIC chiplet 拼接邊界往外推

9.5-reticle 在 2027 量產意味 12xHBM4E 可以塞在同一個 CoWoS 內，這直接對應 NVIDIA Rubin Ultra（2027 Q2 sampling）與 AVGO 下一代 TPU 的封裝規格 — 過去因為 5.5-reticle 限制，AVGO 的 TPU 必須用「2 個 chiplet 各帶 6 顆 HBM」的拼接方式做出 12xHBM 容量，9.5-reticle 之後可以變成「1 個大 chiplet + 12 顆 HBM」，省掉一層 die-to-die 互連，能效再上修 15-20%。14-reticle (2028) 對應 20xHBM5，這是給「需要 1 TB+ memory」的 AI 訓練 chip 的解法 — Llama 5 / GPT-6 等下一代訓練模型對 HBM 容量的需求已經把單一 GPU 的 memory 從 192GB（HBM3E）推到 1TB+（HBM5），這個 capacity scaling 沒辦法靠 die-to-die 拼接解，只能靠 reticle scaling 解。

3）SoW-X：在 substrate 上做 wafer-scale，把 CoPoS 的「面板級」優勢吸走

SoW-X 是這次 symposium 的 surprise — TSMC 揭示 System-on-Wafer 平台延伸到 >40-reticle (64xHBM) 規模、2029 量產，目的是給 logic + HBM 做 wafer-scale 整合。這是過去市場以為「只有 CoPoS / 玻璃面板」才能做到的尺寸，現在 TSMC 證明在 substrate 上也能做到。對本 ID 既有 thesis 的修正方向是清楚的：

  • 原 thesis（2026-04 base case）：CoWoS-L 在 2027-2028 撞 reticle 牆 → CoPoS 2028-2029 是必走路線 → 玻璃基板 2028+ 是 enabler。
  • 新事實（2026 symposium）：CoWoS interposer 可以疊到 >14-reticle (2029)；wafer-scale 用 SoW-X 解決 64xHBM 規模需求；CoPoS 仍存在（2028-2029 mass production）但定位變成「並行選項」而非「必走」。
  • thesis 修正：本 ID §1 thesis「CoPoS 時程暴露有機基板物理極限」這句話本身仍對（5.5-reticle 之上 substrate warpage 確實是 2027-2028 critical bottleneck），但「CoPoS 必走」需要降級為「CoPoS 是 substrate-on-substrate 的一條路徑，而 SoW-X 是另一條」。

4）對玻璃基板 / 面板級 thesis 的影響：負面 datapoint，但仍未證偽

SoW-X 在 substrate 上做到 >40-reticle 對玻璃基板 / 面板級 thesis 是個負面 datapoint — 過去多頭論述常用「CoWoS interposer 物理上做不到 12-reticle 以上」當作 LPKF / Corning / Ibiden（玻璃路線）的 must-buy 理由，現在 TSMC 證明 substrate 路線可以走更遠。但這不是完全的 kill — 玻璃 TGV 在 CTE 匹配（玻璃 3-5ppm vs 矽 3ppm）的天然優勢仍在，只是必走時程從 2028 推到 2030+，且玩家結構從「TSMC 必須採用」變成「Intel / Samsung / OSAT 用來追趕的差異化武器」。本 ID §9.5 Kill Scenario 增列此項，並把 LPKF / Corning 的緊迫性下修。

💡 Insight §2.6（symposium roadmap 對既有 thesis 的三個修正）
1. 5.5-reticle 2026 yield >98% 是「CoWoS GPM 結構性上修」訊號：NVDA / AVGO / AMD 的 wafer cost 直接砍 30%+，2026-2027 GP margin 上修空間沒被市場 model 進去 — TSMC AP 業務毛利可能突破 60%（vs 既有 50-55% 區間）
2. CoWoS interposer 的「物理天花板」往後延了至少兩個世代：2027 9.5x → 2028 14x → 2029 >14x；NVDA Rubin Ultra（2027）+ Feynman（2029）兩代封裝邊界都在 substrate 上完成，不用先跳到玻璃 — 玻璃路線（LPKF / Corning）緊迫性下修，必走時程從 2028 推到 2030+
3. SoW-X 把 wafer-scale assembly 帶進 substrate 路線：64xHBM 給「下一代訓練模型 1TB+ memory 需求」一個 substrate-only 解法，過去市場以為這需要 panel-level 才能做 — 對 CoPoS 是一個「並行而非必走」的重新定位
