Apple Intelligence 平台威脅
真實，但慢，且 model 是租來的

把母題報告濃縮成一張投資決策卡：Apple 的 OS 級 AI 助手會不會繞過你手機裡的 app？會——但比想像慢，而且它的大腦是向 Google 租的。保留核心判斷、conviction tier、賣出訊號，砍掉考證厚度。

關於「Apple Intelligence 會不會顛覆 app 經濟」，市場有兩種對立的偷懶看法。一種說「Apple 終於做 AI、平台霸主回歸、利多」；另一種說「Apple AI 一團糟、延期再延期、不足為慮」。兩種都漏看了 2026 年最關鍵的事實：Apple 在 WWDC 2026 重啟的 Siri，大腦是向 Google 租的——它建在一個客製化的約 1.2 兆參數 Gemini 模型上，傳每年付 Google 約 $10 億（2026 年 1 月簽約）。

這件事一刀切開了 Apple 的兩個身分：OS 分發層的贏家（它握 iPhone、握 App Intents 這個「Siri 唯一能呼叫第三方 app 的通道」、握 Apple Search Ads），但同時是model 層的輸家（自研失敗、改租 Gemini，還要把產品發現/購買的部分收入分給 Google）。所以「Apple Intelligence 平台威脅」的真相是：威脅真實但慢——Siri 變成凌駕 app 的「總入口」、單一用途的 app（約會、評論、訂位、叫車、外送）會被「幫我訂位 / 幫我找對象」繞過、變得 invisible；但 Apple 的延期史、歐盟 DMA 卡關、裝置限制（iPhone 15 Pro+）把落地拖到 2027 年後。

所以本報告不押「Apple 贏 AI」或「Apple 輸 AI」，而是拆解誰捕獲價值、誰被 Sherlock。結論：價值往 AAPL（OS 分發 + Search Ads）+ GOOGL（model 經濟 + 收入分成）兩端；輸家是 Sherlock-able 的單一用途 app（MTCH/BMBL/YELP）；OTA（BKNG/EXPE）部分受威脅但靠多平台分散；真正的對手其實是跨 OS 的 OpenAI ChatGPT super-app。

PART I · DECISION SUMMARY

決策摘要層

三句話看完 Apple 平台威脅的現狀、走向與行動，後接 PM 級監測點。這一層是報告的全部重點——下面各節只是把這一層的每個判斷補上證據。

現在 / NOW

WWDC 2026 重啟 Siri——建在客製 ~1.2 兆參數 Google Gemini（~$1B/年、2026-01 簽）；SiriKit 棄用、App Intents 2.0 成 Siri 呼叫第三方 app 的唯一路徑；on-device 限 iPhone 15 Pro+；EU 因 DMA 延後（iOS 27 不出）。Apple 自研 model 落後、改租 Gemini 並讓 Google 分潤。

未來 / NEXT

3–5 年 Siri-as-front-door + App Intents 成 OS 級 agent 入口，單一用途 app（Sherlock-able）被「book a table / find a date / order food」繞過、變 invisible；但落地受 Apple 延期史、EU DMA、裝置限拖到 2027+；Apple 2027 才望以自研 ~1 兆 model 取代 Gemini。價值往 AAPL（分發 + Search Ads）+ GOOGL（model + rev share）。

行動 / ACTION

押分發層 AAPL（OS + Search Ads $5B→$14–20B 估）+ model 層贏家 GOOGL（Gemini-Siri rev share）；避開 Sherlock-able 單一用途 app（MTCH/BMBL/YELP）；OTA（BKNG/EXPE）部分受威脅但多平台分散；真正對手是 OpenAI ChatGPT super-app。

The Key Call · 本報告最重要的一個判斷

市場吵「Apple 贏 AI 還是輸 AI」，真正的非共識是——Apple 是「OS 分發層贏家、model 層輸家」的雙面，且它的 agentic 威脅真實但慢。真實：Siri 變成凌駕 app 的總入口、App Intents 成新 gatekeeper，單一用途 app（約會/評論/訂位/叫車/外送）會被 OS agent 繞過。但慢且雙面：Apple 自研 model 失敗、改租 Gemini（$1B/年 + 給 Google 分潤），model 層的錢給了 GOOGL；加上延期史、EU DMA、裝置限，威脅落地拖到 2027+。所以價值往兩端：AAPL 得 OS 分發 + Search Ads、GOOGL 得 model 經濟；輸家是 Sherlock-able 單一用途 app；真正對手是跨 OS 的 OpenAI ChatGPT。買「威脅落地」要押慢、買「誰受益」要分清分發 vs model。

三道裂縫具體是：①App Intents 成 gatekeeper——Siri 只能透過 App Intents 呼叫 app，呼叫不到的 app 在 assistant-first 工作流裡變 invisible，這是新的依賴/守門層；②Apple 租 Gemini = model 價值給 Google——Apple 是分發贏家但 model 輸家，產品發現/購買的收入還要分 Google 一塊，這是市場最少談的價值轉移；③威脅落地慢——Apple 兩年延期史（Siri 成「笑話」）、EU DMA 卡關、on-device 限 iPhone 15 Pro+，把 Sherlock 衝擊推到 2027+。盯這三條，比盯「Apple AI 行不行」更能看清錢的流向。

PART II · HISTORICAL ANALOGS

Sherlocking 與預設搜尋租金，框住多空

歷史不必重演，但兩個對照各給一個教訓：Sherlocking 給出「被 OS 內建吞的 app 會死」的 bear，預設搜尋租金給出「分發 vs model 的議價誰收租」。

類比一：Sherlocking——被 OS 內建功能吞的 app 會死。Apple 把第三方熱門功能做進 OS、讓原 app 失去存在理由，是有名字的歷史模式（源自 2000 年代 Sherlock 吞 Watson）：手電筒 app、剪貼簿、螢幕延伸（Luna Display 被 Sidecar 吞）等無數單點 app 被 Sherlock 後歸零。量化錨：被 Sherlock 的單一用途 app 多數營收歸零或被迫轉型。對本次 bear 含義：Siri + App Intents 是「OS 級 agent 版的 Sherlock」——「幫我訂位 / 找對象 / 叫車」直接由 Siri 完成，單一用途的 MTCH/BMBL/YELP/DASH 最危。

兩個對照 · Apple AI 威脅像 Sherlock 還是收租
對照Sherlocking（OS 吞 app）預設搜尋租金（分發收租）Apple AI 像哪個？
機制OS 內建吞單點 app分發層向 service 收租Siri/App Intents 吞 + Search Ads 收租
輸家單一用途 app—（付租者）MTCH/BMBL/YELP（Sherlock-able）
贏家Apple（OS）Apple（收 Google ~$20B/年）AAPL 分發；但 model 反過來付 Google
速度內建即生效長約穩定慢（延期/DMA/裝置限）
結論單點被吞分發收租分發贏、但 model 議價反轉付 Google

怎麼讀：Apple AI 同時是「Sherlock」（吞單一用途 app）與「收租反轉」（過去 Google 付 Apple ~$20B/年當預設搜尋，現在 Apple 付 Google $1B/年租 Gemini）。bull 的底是 AAPL 分發 + Search Ads，bear 是 Sherlock-able app 被繞過；新洞見是「model 議價反轉」——分發霸主竟要向 model 商付費。

類比二：預設搜尋租金——分發 vs model 誰向誰收租。過去十年，Google 每年付 Apple 約 $200 億，買 Safari 預設搜尋位置——這是「分發層（Apple）向 service 層（Google）收租」的經典。量化差異：AI 時代出現議價反轉的一半——Apple 自研 model 失敗，反過來付 Google 約 $10 億/年租 Gemini 當 Siri 大腦，且讓 Google 分產品發現/購買收入。教訓：分發仍值錢（Apple 收 Search Ads、握 App Intents），但 model 層的稀缺讓 Apple 這個分發霸主也得向 model 商低頭——這是「分發贏家、model 輸家」雙面的根源。判別訊號：Apple 2027 能否以自研 model 取代 Gemini、把 model 價值收回。

PART III · SUPPLY SIDE

供給側：Apple AI 棧，利潤往分發與 model 兩端

反直覺但可操作的結論：Apple AI 的利潤一分為二——分發層（AAPL：OS + App Intents + Search Ads）與 model 層（GOOGL：Gemini + 收入分成）各拿一塊，中間被 Sherlock 的單一用途 app 失血。

供給端可拆三塊。分發層（AAPL）：iPhone 裝機量 + Siri 入口 + App Intents（Siri 呼叫 app 的唯一通道）+ Apple Search Ads——這是 Apple 真正的護城河，AI 強化它（assistant 變新入口）。model 層（GOOGL）：Apple 自研失敗、租 Gemini（~1.2 兆參數客製模型、$1B/年），Google 還分產品發現/購買收入——model 經濟給了 Google。被 Sherlock 的 app 層：單一用途、可被 OS agent 一句話完成的（MTCH/BMBL/YELP/DASH）失血；有網路效應/多平台/host-side 的（UBER/ABNB/BKNG）相對防禦。利潤往「分發 + model」兩端，中間單點 app 被擠。

Inference · 供給
分發給 Apple、model 給 Google，Sherlock-able app 失血
前提：Apple 握 OS 分發 + App Intents gatekeeper + Search Ads（分發值錢）+ 自研 model 失敗租 Gemini（model 價值外流 Google）+ 單一用途 app 可被 OS agent 一句話完成 → 利潤往分發（AAPL）+ model（GOOGL）兩端，單點 app 失血

Apple 的分發護城河被 AI 強化（Siri 變新入口、App Intents 當守門），它靠 Search Ads 變現；但 model 層它租 Gemini、還分潤給 Google，所以 model 經濟給了 GOOGL。被夾在中間的是 Sherlock-able 單一用途 app——它們的功能「幫我訂位/找對象/叫車」正好是 OS agent 能一句話完成的。$1B/年 Gemini 案是「分發霸主竟向 model 商付費」的硬證據。

可證偽條件：若 Apple 2027 成功以自研 ~1 兆 model 取代 Gemini（model 價值收回）、或 App Intents 反而讓 app 更易被觸及（變賦能而非守門），則「model 給 Google / app 失血」論的某一面失效。

PART IV · DEMAND SIDE

需求側：assistant-first 與 Sherlock-able 缺口

需求最剛性的是「用一句話讓助手把事做完」——當用戶習慣對 Siri 說「幫我訂位」，app 的角色從「被打開」變成「被呼叫」；最大的變數是 Apple 把這件事拖了多久才落地。

第一引擎：assistant-first 行為（未需轉現需）。用戶要的是「一句話做完事」——「幫我訂張兩人桌」「找附近評價高的拉麵」「叫車去機場」。當 OS 級 agent 能直接完成，app 從「主動打開」變成「被 Siri 呼叫」，App Intents 成為「能不能被叫到」的生死線。這是真實的行為位移，OpenAI ChatGPT 的 super-app（Booking/Spotify 入駐）已在跨 OS 證明需求。第二引擎：落地速度（供給瓶頸）。需求方向明確，但 Apple 的供給一再卡關：個人化 Siri 2024 預告後兩度跳票、成「笑話」；EU 因 DMA 在 iOS 27 不出 Siri AI；on-device 限 iPhone 15 Pro+。所以 Sherlock 衝擊是「會來、但 2027+」。

需求三角驗證 · 行為位移 vs 落地瓶頸
視角數值 / 事實口徑
行為位移（需求）assistant-first：Siri/ChatGPT「一句話做完事」產品趨勢（量級）

怎麼讀：行為位移（assistant-first）方向明確、需求是真的；但 Apple 供給一再卡關（延期/DMA/裝置），把 Sherlock 衝擊推到 2027+。採信「Apple 實際出貨進度 + Sherlock-able app 財報」作威脅落地的最硬證據，而非 keynote demo。

多空為何都對

多頭（對 app loser 而言是空頭）測的是assistant-first 行為（OS agent 繞過 app），方向沒爭議；防禦方測的是落地速度（Apple 延期、DMA、裝置限、網路效應）。兩者可同時為真：威脅是真的（會來），但落地慢（2027+）、且非所有 app 都 Sherlock-able（有網路效應者防禦）。本報告把重心放在「誰被 Sherlock、誰防禦、價值給誰」，而非「Apple AI 行不行」。

PART VI · TAXONOMY

個股特徵與 conviction tier

本框架只輸出「特徵」與「分類」，買哪一檔由個股 DD 系統與 Pure MA 訊號決定。以下 tier 為產業推論的特徵歸類，提及公司僅為「該特徵的代表案例」，不附帶買賣建議。

CORE 🔴

AAPL（分發層贏家）

OS 分發護城河被 AI 強化——Siri 2.0（租 Gemini）+ App Intents gatekeeper + Apple Search Ads（$5B 2025 → $14–20B 2030 估）。特徵是「分發層贏家、但 model 層落後租 Gemini」，雙面：分發加分、model 失分。conviction：高（分發護城河）· 盯能否 2027 收回 model
CORE 🔴

GOOGL（model 層贏家）

贏下 Gemini-Siri 案——~$1B/年 + 產品發現/購買收入分成，把 Gemini 塞進全球數億 iPhone。特徵是「分發 vs model 議價反轉的受益方」，本主題最被低估的贏家（市場只談 AAPL/Sherlock，少談 GOOGL 收 model 租）。conviction：中高（model 議價）
EDGE 🟡

BKNG / EXPE / ABNB / UBER / DASH（防禦/受威脅）

受 Siri/agent 威脅但有防禦——BKNG（多平台+多 agent 入駐分散）、ABNB（host-side 非 fully sherlockable）、UBER（雙邊網路效應）、DASH（配送網路）、EXPE（中段壓力較大）。特徵是「Sherlock 威脅 vs 網路效應/多平台的拔河」。conviction：中 · 各看防禦力
LOSER 🟢

MTCH / BMBL / YELP（Sherlock-able）

單一用途、最易被 OS agent 繞過——MTCH/BMBL（dating，付費用戶流失）、YELP（評論+訂位，Apple Maps/Siri Sherlock target）。特徵是「功能可被一句話完成、無強網路效應」，結構性 loser（除非轉成不可取代的資料/社群）。conviction：結構性偏空

估值面總結：本主題的核心張力在「assistant-first 威脅真實 vs 落地慢 + 價值兩端分」。分發層 AAPL 與 model 層 GOOGL 是贏家（後者最被低估）；防禦型（UBER/BKNG/ABNB）靠網路效應/多平台拖延威脅；Sherlock-able 單點（MTCH/BMBL/YELP）是結構性輸家。現處 Phase I（威脅成形未落地），轉空（對 AAPL）條件 = Siri 再延期 + ChatGPT 跨 OS 搶 assistant；對單點 app 的轉空已在進行（付費用戶流失）。OpenAI 為私有、不可直接投資、但是真對手。買賣由個股 DD + Pure MA 系統決定。

The One Line

Apple 的 AI 威脅真實但慢，且 model 是向 Google 租的——分發贏家 AAPL 與 model 贏家 GOOGL 兩端收錢，Sherlock-able 單點 app（MTCH/YELP）失血。

這份報告刻意不做的三件事

  1. 不說「Apple 贏 AI」或「Apple 輸 AI」——明確拆成「分發層贏、model 層輸」雙面。
  2. 不把威脅當立即生效——強調延期史/EU DMA/裝置限把 Sherlock 落地推到 2027+。
  3. 不告訴你哪一檔該買——CORE/EDGE/LOSER 是「特徵的代表案例」，買賣由你的個股 DD 與 Pure MA 系統決定。
