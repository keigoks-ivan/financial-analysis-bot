軟體股 × AI infra 的八個非共識（2026 版）

從 6 份雲端 / SaaS 產業 DD（M1 資安 SaaS 平台戰 + M2 AI × 資安 + M5 身份 + M10 資料安全 + M11 LLM × 資安 + Agentic AI 平台）合讀後提煉，加入一個來自 12 年買方經驗的 critique：軟體股並非「AI 時代結構性逆風」那麼簡單，也不是「全面 long」那麼樂觀。這 8 條 thesis 是本站目前對「軟體股 vs AI infra 資金重分配」最完整的決策框架，同時也明確分出 Tier 1 敘事 watchlist 與 Tier 2/3 進場紀律——避免把「故事對」當成「現在買」。

L1 · 產業結構（護城河侵蝕 + 預算重分配 + 威脅分 tier）

三條關於軟體業基本結構的 thesis
3 條
1
CRWD 的 kernel-level 鎖定護城河正被 Microsoft Windows Resiliency Initiative 主動削掉
M1 §3, §9 →

這是 CRWD 估值最大的結構性變化，而且還沒被市場完全消化。2024 年 7 月 19 日 Falcon 全球當機事件之後，微軟 2024 年 9 月公開宣布 Windows Resiliency Initiative，要把 EDR 供應商（包括 CrowdStrike）移出 Windows kernel space——這不是競爭威脅，是微軟自己把 CRWD 最深的那一層護城河拿刀削掉。過去 CRWD 的 pitch deck 裡「內核級別存取 = 不可取代」這句話，2027 年之後會自動失效。

共識CRWD 的 kernel-level access 是它最深護城河，F500 客戶因為技術深度綁定而無法遷移；7/19 outage 後 98% retention 是「不可取代」的證據。
本 IDKernel 鎖定這層護城河的「半衰期」被微軟 2024-09 的 Windows Resiliency Initiative 縮短到 2-3 年。CRWD 仍會是平台贏家，但必須用 data lake（LogScale）+ MDR 服務 + Charlotte AI + 非 Windows 覆蓋重建「非 kernel 依賴」的 moat。**「毫髮無傷」是錯的，正確框架是「moat 換軌」——舊 moat 結構性衰退，新 moat 能否接得上是 2026-2027 最重要的檢核**。
證據(1) MSFT Windows Endpoint Security Platform roadmap 2024-09 公告；(2) Gartner 2025 CIO Survey 顯示 34% F500 評估「dual EDR」架構，此議題在 2024 之前不存在；(3) 7/19 事件後多家 F500（Delta Airlines 提告 CRWD 求償 $500M+）推動 board-level「分散 EDR vendor risk」；(4) CRWD 自身 2025 加速投資 data lake + Charlotte，已顯示內部認知 kernel 不是長期錨
投資行動CRWD 從「platform anchor long」降為「moat 換軌觀察」。進場前先看 2026-2027 Q 財報：(a) 非 Windows endpoint ARR 是否加速；(b) LogScale / Next-Gen SIEM ARR 佔比是否過 25%；(c) Charlotte AI attach rate 是否追上 S Purple 的 50%+。三項任兩項達標才算新 moat 接軌成功。
event-triggered L1 產業結構
2
IT 預算不是 zero-sum，SaaS 與 AI infra 是「兩速跑道」而非零和
Agentic AI §4 / M11 →

這條反駁當前最流行的空頭敘事之一：「NVDA capex 爆發的錢從 SaaS 預算來，軟體股結構性逆風」。這個會計等式 first-order 正確，second-order 卻忽略了兩個關鍵事實——IT budget 總 pie 本身在擴張，而且 AI agent 需要 data substrate，regulated data SaaS 反而是 AI 的補充品而非替代品。

共識NVDA + hyperscaler capex 爆炸的錢，是從 SaaS 訂閱預算裡搬出來的；軟體股面臨結構性逆風，整個類別估值中樞應該下移。
本 IDGartner 2026 全球 IT spending +12.5% YoY 到 $5.7T，AI infra 吃 +5-6pp 增量，但 regulated data SaaS（WDAY / INTU / NOW / DOCU）仍能吃 +5-7pp 增量——不是零和，是兩速跑道。被 AI 真正吃掉的是 per-seat communication SaaS（HUBS mid-market、ZM、TWLO voice、APPF 部分），不是全類軟體。更關鍵的是，AI agent 部署後企業反而要加購 WDAY Integration Platform、NOW Creator、CRM Data Cloud——這些是 agent 行動的 canonical schema，INTU 拿 OpenAI $100M 合約就是這個邏輯的直接驗證。
證據(1) Gartner 2026 IT spending forecast +12.5% YoY；(2) INTU 2025-11 與 OpenAI $100M 合約——OpenAI 必須付錢接 Intuit 稅務資料層才能做稅務 agent；(3) WDAY Illuminate 12 agents + Agent System of Record = 從被吞食對象升級為 agent 治理層；(4) NOW Now Assist ACV $600M + 244 筆 >$1M deals 證明 IT workflow 層反而因 agentic 需求擴大而加速
投資行動短期：不要 blanket short 軟體類股。分層：Long regulated data SaaS（WDAY / INTU / NOW / DOCU），Avoid communication SaaS（HUBS mid-market、ZM、TWLO、APPF）。中期：建立 Top 20 企業 IT capex tracker 追蹤 SaaS vs AI infra 的 seat count 與合約結構變化——這是判斷整個軟體類別估值中樞是否真的下移的唯一量化訊號。
structural L1 產業結構
3
Capital Cycle 在 SaaS 跑 3-5× 慢於商品類，AI-native 競爭者威脅必須分 tier 套用
Agentic AI §1 / §9 →

Marathon Asset Management 的「看供給不看需求」資本週期框架在大宗商品（硬體、能源、航運）很準，但套在 SaaS 要打折——因為 enterprise switching cost、regulated data、以及企業銷售分銷這三個摩擦，讓「資本過剩 → 供給氾濫」的理論在 SaaS 跑得慢 3-5 倍。「AI-native 3 人團隊能用 10% 成本挑戰既有軟體廠」這句話不是對所有 SaaS 對稱，必須分 tier 看。

共識VC 錢退 + AI 讓開發成本接近零 → 所有 SaaS 公司都會面對 AI-native 對手用 10% 成本攻擊 → 軟體類全面結構性逆風。
本 IDAI-native 威脅在 SMB / mid-market 是真的且急：Glean 6 個月做出企業 search、Decagon 9 個月衝到 $35M ARR。但在 top-of-pyramid regulated enterprise，瓶頸不是 coding cost，是三個結構摩擦：(1) 合規認證時間——FedRAMP High 平均 24 個月、SOC 2 Type II 18 個月，新創過不了這一關就賣不進 F500 / Fed / 金融；(2) 企業銷售分銷——CISO 關係是 10 年累積，CRWD 有 300+ 專跑 F500 的 enterprise AE，新創要建軍團要燒 5 年；(3) data gravity + 歷史 label——CRWD Falcon Complete 10 年真人分析師 triage label、WDAY ERISA-compliant payroll schema，這些是 path-dependent 資產，LLM 能力再強也複製不了。
證據(1) Glean、Decagon、Cursor 在 SMB / mid-market 快速穿透（Decagon $35M ARR / 9 個月）；(2) 同期 CRWD 2023-2024 漲 2.5×、PANW 漲 2×——「資本週期會殺 SaaS」的論者 short 這些股票是大虧；(3) FedRAMP High 從 zero 到發證統計平均 24 個月；(4) CRWD Fortune 500 滲透率 78%、PANW 85%，新創在頂端市佔 < 2%
投資行動分 tier 套用資本週期框架：Fast-burn（2-3 年被吃） = HUBS mid-market、ZM standalone、TWLO voice、APPF（可 short 或避開）；Slow-burn（5-7 年 slow erosion） = CRWD、PANW、MSFT、CRM（仍可 long，但要追蹤 moat 換軌）；Protected（3-5 年結構安全） = WDAY、INTU、NOW、DOCU、PLTR（regulated schema + 合規護城河；可 core long）。

L2 · 估值與 TAM（市場估算滯後 + LLM 原廠 premium 錯定價）

兩條關於 agentic AI 估值錨點的 thesis
2 條
4
Horizontal 前 5 家合計 agentic ARR > $15B 已是嚴格 TAM 的 2 倍——市場 TAM 估算明顯滯後
Agentic AI §4 →

TAM 研究機構的估算正在被真實商業化速度追過去。MarketsandMarkets 給 Agentic AI Platform 嚴格口徑 2025 $7.84B → 2030 $52.6B，但 2026 年現在 CRM Agentforce + MSFT AI + NOW Now Assist + PLTR US commercial + WDAY Illuminate 五家合計已超 $15B——直接是嚴格 TAM 的 2 倍。這代表公開市場的 TAM-based valuation 可能嚴重低估現實，上修空間還沒被定價。

共識Agentic AI Platform 2030 TAM 約 $52B（MarketsandMarkets），按 CRM / MSFT / PLTR / NOW 等 Horizontal 玩家的 valuation 應該 price in 這個 TAM 上限。
本 ID研究機構 TAM 估算明顯滯後於商業化速度。CRM Agentforce $800M ARR +169%、MSFT AI run rate $13B → $25B FY26 target、NOW Now Assist ACV > $600M、PLTR US commercial +137% YoY、WDAY AI ARR $400M+——五家 Horizontal 2026 年合計 > $15B，已是嚴格 TAM 的 2 倍。廣義 TAM（Bain / McKinsey）才是比較合理的錨：2025 $12-15B → 2030 $80-100B，Horizontal 前 5 在 2030 合計吃到 $50-70B 是 reasonable central case 而不是 bull case。
證據(1) CRM FY26 Q4 Agentforce ARR $800M +169%，包含 29,000 deals 與 2.4B agentic work units 可驗證；(2) PLTR FY26 guide US commercial +115% YoY 隱含從 $1.465B 擴至 $3.14B；(3) MSFT 自己揭露 AI run rate $13B → 目標 FY26 年末 $25B；(4) Gartner 2026 預測 40% 企業應用將含 task-specific agent（vs 2025 < 5%）
投資行動不要用 MarketsandMarkets 的 $52B 當 valuation 天花板。正確的 cross-check 是看 Top 5 Horizontal 2028 能否合計到 $35-50B agentic ARR（目前 run rate 隱含 2028 會到）。這個數字達成就是 Horizontal 類 EV/Sales 上修的 catalyst。
event-triggered L2 估值 TAM
5
Anthropic / OpenAI 從企業 agentic 只拿不到 15% token cost，「AI × 企業」premium 應該打折
M11 LLM × 資安 →

市場把 Anthropic $30B run rate、OpenAI $25B run rate 高估值的最大理由是「他們吃掉 Agentic AI 所有企業錢」。但 M11 的 research 顯示這個假設錯——資安 vertical 在 Anthropic 營收佔比 < 3%，企業 agentic application 類比例類似。真正的 agentic ARR 流向 Horizontal + Vertical SaaS，LLM 原廠只分到 token fee 這一塊，佔整體 agentic 收入大約 10-15%。

共識Anthropic 和 OpenAI 是 Agentic AI 最大受益者，透過 token 費與企業 API 收割整個生態。他們的 run rate 快速擴張證明這個 thesis。
本 IDLLM 原廠的 $55B 合併 run rate 裡，企業 agentic application vertical 佔比低（資安類 < 3%、整體企業 agentic 估 10-15%）。剩下 80%+ 是 consumer（ChatGPT、Claude Pro）、developer（Claude Code、Copilot）、廣義 API 實驗性工作負載。真正拿到企業 agentic 長期 recurring revenue 的是 Horizontal 平台（CRM / MSFT / PLTR / NOW / GOOGL）與 Vertical SaaS（WDAY / INTU / DOCU）。LLM 原廠的「AI × 企業」premium 在當前估值應該打折，因為 pure company-level AI revenue 低於市場想像。
證據(1) Anthropic 資安 API 年度支出估 < $1.5B（M11 cross-ID 計算）vs Anthropic $30B run rate = < 5%；(2) CRM Agentforce $800M + NOW Now Assist $600M + WDAY $400M 這些企業 agentic ARR 流向 SaaS 平台而非 LLM 廠；(3) MSFT 已公開走向自建 Phi-4 / MAI-1 減少 OpenAI 依賴；(4) Amazon $25B 投資 + Google 14% 持股 Anthropic 讓 Anthropic 結構上不會往 vertical SaaS 競爭
投資行動不要把 Anthropic / OpenAI 估值 premium 當成「AI 企業紅利」的全部。Public market 正確 exposure 是 Horizontal 平台 + 吃 regulated data vertical。若押注 LLM 原廠，要用 consumer + developer 兩個 segment 的規模撐 valuation，不要靠 enterprise agentic API。
structural L2 估值 TAM

L3 · 利潤分配（Framework 商品化 vs Infra 結構贏家）

agentic 生態系最尖銳的利潤分化
1 條
6
Framework 層已結構性商品化，Agent Infra（Cloudflare、AWS Bedrock AgentCore）才是真贏家
M11 §5 / Agentic AI §6 →

這是私募估值與公開市場邏輯最大的斷層。LangChain 以 $1.25B 估值融到 Series B，但 LangSmith ARR 只 $12-16M，等於 75-100x ARR multiple——本質是 venture capital 為「未來」買單。反觀公開市場的 Cloudflare，單一合約 $42.5M ARR + Workers AI 請求 YoY +4,000%，是真實 per-invocation 收入的 infrastructure 贏家。這兩者的 pricing mismatch 是未來 18-24 個月 agentic 生態最大的 re-rating 機會。

共識LangChain $1.25B Series B 估值反映 agent framework 類別崛起，未來 agent engineering platform 是大市場。Framework 層是 agentic 生態的核心價值所在。
本 IDFramework 商業化路徑窄——Anthropic、OpenAI、Google、MSFT 四家 SDK 全是 MIT 免費開源，獨立 Framework 廠（LangChain、CrewAI）想收錢但客戶可以免費用原廠 SDK。LangChain $12-16M ARR / $1.25B 估值 = 75-100x 是 venture 定價未來，public market 不會這樣 price。真贏家是 per-invocation / egress 收費的 Infra 層——Cloudflare Workers + Code Mode MCP portal（$42.5M ARR 單合約、Workers AI req YoY +4,000%）、AWS Bedrock AgentCore（2025-10 GA）、MCP protocol（10,000+ servers、97M 月 SDK 下載）——這些才是 agentic 生態系真實現金流所在。
證據(1) LangChain Series B $125M @ $1.25B（2025-10 IVP lead）對應 LangSmith ARR $12-16M（TechCrunch / Fortune 報導），未否認；(2) CrewAI 150 企業客戶僅 ARR $3.2M（ACV < $25K 代表多數是 pilot）；(3) AutoGen 被 MSFT 直接收編（無獨立 exit 價值）；(4) Anthropic 2025-12 把 MCP 捐給 Linux Foundation = 放棄 Framework lock-in，鎖定在 SDK + Claude Code 的 vertical 路線
投資行動Public market 最佳 exposure：Cloudflare（NET）作為 agent infra 最純 play，但注意 EV/S 15x+ 是要 watchlist 不是現在 buy。Avoid LangChain / CrewAI 類 private framework pure-play。LangChain 下輪若 down round，確認 Framework 泡沫破裂，整類 VC-backed agent framework 都會被 de-rate。

L4 · 決策紀律（Tier 1/2/3 分層 + 10 年視野）

把敘事研究與交易決策分開的兩條 meta-thesis
2 條
7
Tier 1 / 2 / 3 紀律必須分層——Watchlist 不等於進場清單，「結構性 Beta」是危險敘事
紀律流程 ↓

這是本 flagship 的 meta-thesis，也是 12 年買方經驗的核心。產業 DD 寫得再漂亮，如果在趨勢未確認時進場，都是違反紀律的接刀。「結構性 Beta」這四個字本身就是危險敘事——它用敘事強度掩蓋 timing 問題。正確的做法是把研究層與交易層明確分三 tier：

共識故事對 → 可以買。敘事邏輯強的標的應該直接進場建立「結構性 Beta」倉位。
本 ID故事對 ≠ 現在買。必須通過三層獨立過濾：Tier 1 敘事研究層（Watchlist Generator）產出候選清單，不下買賣判斷；Tier 2 基本面過濾層（EPS 成長 / ROIC / FCF margin / PEG < 2 / ATH proximity）剔除「故事對但價格錯」者；Tier 3 趨勢確認層（Pure MA 週線站穩 + 月線翻正 + ATH 20% 內）決定時點。三層都過才建倉，缺一不進場。
證據(1) 2022-2025 期間多個 agentic / AI 敘事 thesis 正確但入場時點錯誤的案例（例 2022 Q4 進 CRWD 後半年跌 30%，2024 Q1 才回原價）；(2) 當前 NET、DDOG、ADBE、CRWD 多數週線 MA 還在壓價下方，按 Tier 3 應該等趨勢確認；(3) Tier 2 PEG < 2 過濾當前會剔除 NET（EV/S 15x+）、DDOG（高倍數）、大部分 high-multiple SaaS
投資行動以本 flagship 列出的 watchlist（見下方 §Watchlist）為 Tier 1，自行跑 Tier 2 基本面與 Tier 3 MA + ATH。對於目前 Tier 3 不達標的標的（多數高估值軟體股），耐心等待趨勢確認，不要被敘事推著進場。
structural L4 決策紀律
8
Munger 10 年視野測試是護城河真偽的最終檢驗——模糊就是風險信號
Cross-ID 所有 §11 →

Charlie Munger 常問的問題是「給我看 10 年後這家公司長什麼樣」。如果答案模糊，那就是警訊——不是因為預測能力差，是因為護城河輪廓本身在侵蝕。這個測試應用到本 flagship 涵蓋的 70 檔 ticker，可以區分真正的 core long 與 false-confidence long。

共識Moat 可以靜態量化：market share、gross retention、NRR、switching cost。這些指標好的就是護城河穩固。
本 IDMoat 要通過「10 年後這家公司長什麼樣」的 narrative clarity 測試。靜態指標說明「現在」，但護城河能不能撐 10 年要看輪廓是否清晰。應用於本 flagship 涵蓋的核心持倉候選：10 年圖像清晰者——WDAY（payroll ledger + ERISA 合規不可替代）、INTU（IRS e-file + SMB 財稅）、NOW（IT CMDB workflow 深）、PLTR（ontology + FedRAMP + 國防）、CRM（Data Cloud + 29K Agentforce deals）；10 年圖像模糊者——CRWD（kernel erosion + MSFT 競爭雙重威脅）、ADBE（Figma + Canva + 成長放緩）、ZS（pure-play SASE 被 platform 擠壓）、TWLO（CPaaS 被 Voice AI-first 吃）。模糊不代表立刻賣，代表 position sizing 要小、退場訊號要清楚。
證據(1) 清晰陣營共同點：regulated data + 合規認證 + 深度 workflow schema；(2) 模糊陣營共同點：主要 moat 是「單一技術層」（kernel / 創意工具 / SASE pure-play / voice CPaaS），被結構變化侵蝕後無明顯替代 moat；(3) 歷史驗證：類似模糊信號早期出現在 2015-2020 的 Symantec、2018-2022 的 EMC、2020-2024 的 Cisco legacy firewall，這些公司最後都 de-rate 40-60%
投資行動把 10 年視野測試加進 Watchlist 評分。清晰陣營可進 core（≤ 5 檔）；模糊陣營只能進 satellite（≤ 3 檔）或 avoid。CRWD 與 ADBE 這類「moat 模糊但目前數字還好」者，設停損訊號（例 CRWD gross retention < 95% 即減碼、ADBE revenue growth < 8% 即減碼），不要被 static good metrics 蒙住。

🧩 Cross-ID Synthesis：三層投資判斷整合

第一層：軟體股不是全面逆風，是分化加劇

空頭敘事「AI infra capex → SaaS 結構性逆風」對了 first-order 方向但錯了 second-order 細節。真實的 2026-2028 圖像是軟體類內部劇烈分化——Regulated data SaaS（WDAY / INTU / NOW / DOCU）因為 AI agent 需要 data substrate 反而受益；Horizontal 平台（CRM / MSFT / PLTR）吃到 $15B+ agentic ARR 是市場 TAM 估算沒跟上的結構 tailwind；但 Communication / 輕 CRM SaaS（HUBS / ZM / TWLO）正在被 Agentforce + Copilot + AI-native 新創三路擠壓。不要 blanket short 軟體類，也不要 blanket long。

第二層：LLM 原廠與 Infra 是兩個不同的投資標的

市場把 Anthropic $30B run rate 視為「AI 企業紅利全包」是錯的——企業 agentic vertical 佔比 < 15%，剩下是 consumer / developer / 實驗性 workload。真正吃到企業 agentic recurring revenue 的是 Horizontal 平台與 regulated Vertical SaaS。反過來 Cloudflare / AWS Bedrock AgentCore 這類 per-invocation 收費的 agent infra，才是 MCP 中立化後的結構贏家——獨立 Framework（LangChain、CrewAI）因為 LLM 原廠 SDK 免費會持續商品化。

第三層：紀律整合——Tier 1 研究不等於 Tier 2/3 進場

本 flagship 最重要的 meta-thesis 是把敘事研究與交易決策明確分離。Tier 1（敘事 Watchlist 生成）輸出候選清單；Tier 2（五條件基本面過濾，PEG < 2）剔除「故事對但價格錯」者——當前會擋掉 NET、多數高倍數 SaaS；Tier 3（Pure MA + ATH proximity）決定時點——當前多數軟體股週線 MA 還在壓價下方，要等趨勢翻回。三層都過才建倉，缺一不進場。「結構性 Beta 倉位最佳時機」這種敘事強度掩蓋 timing 的語言必須警惕——過去十年不虧損的核心紀律不是「故事對就進場」，是「趨勢確認後才進場」。
