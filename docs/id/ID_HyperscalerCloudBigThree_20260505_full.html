§1 產業白話定義 + 歷史脈絡

白話版：AWS、Azure、GCP 是「租算力給全世界」的三大房東。2026 年它們都在瘋狂蓋資料中心（合計 capex 數千億美元），但成長速度差很多、回報還沒被證明、而且開始自己造晶片不全買 NVDA。所以別把它們當「一起漲」的籃子。

白話開場：先說三個關鍵詞。hyperscaler是超大規模雲端房東（AWS/Azure/GCP），把運算/儲存/AI 算力租給企業。capex是它們蓋資料中心、買晶片的資本支出——2026 年幾乎全砸在 AI。自研 ASIC是它們自己設計的 AI 晶片（GCP 的 TPU、AWS 的 Trainium、Azure 的 Maia），用來取代部分 NVDA GPU、降自己的成本。本報告問的是：這三大房東該值什麼估值、天量 capex 何時回本、自己造晶片改寫了誰。

邊界界定：本報告（ai_cross_domain 子題，母題 AICrossDomainImpact）聚焦「三巨頭的相對價值 + capex ROI + 自研晶片」。納入三巨頭 + 第四勢力（ORCL/META/中國雲）+ 晶片供應（NVDA/AVGO/MRVL/AMD）+ neocloud。AI 訓練/推論晶片的供需細節見「Token 經濟學」與算力相關 ID；電力見電力相關 ID。

歷史分三段。第一段：2006–2015 雲端誕生——AWS 開創 IaaS、規模先發；Azure/GCP 後追。第二段：2016–2023 三強鼎立 + 分化——Azure 靠企業綁定 + 高毛利追近、GCP 後段；雲端取代自建機房、結構性成長。第三段：2024–2026 AI capex 軍備 + 成長重新分化——AI 需求爆發、Big4 capex 從 ~$246B（'25）跳到 $380B+（'26）；GCP 用 TPU + Gemini 加速到 +63%、Azure +40% + 受電力限的滿手需求、AWS +19% 落後；自研 ASIC 占比衝到 25–30%。capex 是真需求驅動（算力受限）、但 ROI 未證。

歷史類比一：鐵路/光纖超建——capex 軍備的 ROI 風險。19 世紀鐵路、1990s 末光纖都曾因軍備式超建，短期需求真、長期過剩、折舊吞報酬（光纖泡沫後大量暗光纖閒置）。量化錨：dot-com 光纖超建後，鋪設光纖多年僅點亮個位數百分比、電信巨額減記。對 bear 含義：$380B+ AI capex、2/3 是壽命短 GPU，若雲端營收加速跟不上折舊，會重演「超建→折舊吞報酬」。關鍵差異：hyperscaler 蓋的是可出租算力、自身又是最大用戶，不是純投機鋪管。歷史類比二：雲端 SaaS 化——需求真但贏家分化。雲端取代自建機房、需求結構性真實，但贏家分化（AWS 規模、Azure 毛利、GCP 後追）。量化差異：AI 時代延續且更極端——GCP +63%、Azure +40%、AWS +19%。教訓：把三巨頭當一個籃子是錯的，要按成長/毛利/規模分清估值。

本主題有 capex/折舊週期，但用「成長分化 + ROI 兌現」當時序錨。

關於 cycle 統計表的省略：雲端營收為結構性成長、capex 有週期但 ROI 是核心變數，故不立傳統 ASP cycle 表；以「雲端三段」（2006–15 誕生 → 2016–23 三強分化 → 2024–26 AI capex 軍備）的編年呈現時序。算力晶片週期見「Token 經濟學」/算力 ID。

這段歷史的可操作含義：三巨頭 AI capex 同時有「光纖超建」的 ROI 風險與「雲端 SaaS」的真需求。bull 的底是真需求 + 可出租算力 + 自用，bear 是 $380B+ 折舊跑贏營收加速、毛利壓縮。且三家分化——別當一個籃子，按成長（GCP）/毛利（Azure）/規模（AWS）分清估值。

§2 技術成熟度 + S 曲線

為什麼 capex 軍備是現在？因為 AI 訓練/推論需求跨過「算力是瓶頸」門檻、且自研 ASIC 技術成熟到能取代部分 GPU——但電力與先進封裝把產能上線拖慢，形成「需求受限於供給」的奇景。

技術拐點有三。AI 算力需求跨過瓶頸：訓練 frontier model + 推論放量（見「Token 經濟學」Jevons）讓算力成為瓶頸，三家都 compute-constrained。自研 ASIC 成熟：GCP TPU（Trillium v6 → Ironwood v7）、AWS Trainium（Project Rainier ~500K for Anthropic）、Azure Maia 跨過「自研晶片經濟性優於外購 GPU」門檻，占比衝到 25–30%。供給瓶頸：電力（Azure $80B backlog 卡在電力）+ 先進封裝（CoWoS）讓產能上線慢於需求。三者交會，capex 軍備、但 ROI 取決於產能能否轉成租金。

§3 供給側：現在與未來 + 利潤池

三巨頭的供給能力分化（成長/毛利/規模各有所長），且自研晶片把一塊原屬 NVDA 的利潤池移到 AVGO/MRVL 與 hyperscaler 自身。

供給端是「三巨頭 + 第四勢力 + 晶片供應」。三巨頭各有所長：GCP（+63%、TPU 領先、Apple/Anthropic TPU deal）、Azure（+40%、op margin 最厚、$80B 受電力限的滿手需求、OpenAI/Anthropic 進駐、Maia）、AWS（30% 份額規模最大、+19% 落後、Trainium/Project Rainier ~500K）。第四勢力：ORCL（OCI、OpenAI Stargate、增速快毛利薄）、META（capex $125–145B 體量等同）、中國 BABA/騰訊。晶片供應：NVDA（主供）、AVGO（TPU base die + 客製 ASIC ~70%（hyperscaler 整體、業內估，非單一 TPU））、MRVL（Trainium base die）、AMD（dual-source）。利潤往「成長/毛利贏家 + 自研 ASIC 套利者」集中。

利潤池遷移

利潤正從「單純買 NVDA GPU」往「自研 ASIC + ASIC 設計外包」遷移：自研 ASIC（25–30% 且升）把原本全給 NVDA 的加速器採購，分一塊給 AVGO（TPU base die ~70%、客製加速器 +140% YoY）、MRVL（Trainium base die ~55–60%），並降 hyperscaler 自身推論成本（強化雲端毛利）。被搶的是 NVDA 的部分份額，搶到的是 ASIC 套利者 + hyperscaler 自身。同時三巨頭的雲端利潤也分化——GCP 成長、Azure 毛利、AWS 規模。（依 QC-18，雲端利潤池逐段精確百分比難拆，以份額/增速 + 定性呈現。）

Inference · 供給
三巨頭分化、自研晶片把利潤從 NVDA 移到 ASIC 套利者
前提：三巨頭成長/毛利/規模分化 + 自研 ASIC 25–30% 且升 + ASIC 設計外包 AVGO/MRVL → 估值該分化、一塊 GPU 利潤池移到 ASIC 套利者 + hyperscaler 自身

三巨頭不是同質籃子——GCP 賣成長 + TPU、Azure 賣毛利 + 受限需求、AWS 賣規模；估值該按此分化。同時自研 ASIC 把加速器採購分一塊給 AVGO/MRVL、降 hyperscaler 推論成本。NVDA 仍主供但 2027–28 面臨結構性份額侵蝕。

可證偽條件：若 NVDA 下一代（Rubin）效能/TCO 大幅領先使自研 ASIC 經濟性消失、或某一家 hyperscaler 成長/毛利同時碾壓另兩家（分化收斂成單一贏家），則「分化 + ASIC 套利」論的某一面失效。

未來供給與因果閉合：成為雲端 AI 贏家的門檻是「自研 ASIC + 電力產能 + AI 生態 + 規模」。§1 提出「成長/毛利/規模分化」是估值判準——未來 3–5 年仍 binding：AI 差異化（自研矽 + model）拉開三家、capex ROI 決定誰的毛利撐得住；唯一鬆動是 NVDA 效能碾壓使自研失去經濟性——這是 §7 的反方。

§4 需求側：現在與未來 + 三角驗證

第一引擎：AI 算力受限需求（現需）。需求不是問題——三巨頭都 compute-constrained：Azure 約 $80B 訂單因電力（非需求）無法交付、GCP RPO >$70B（多年期 AI 承諾）、AWS AI run-rate >$15B 三位數增。客戶要算力、hyperscaler 蓋不夠快，瓶頸在電力與先進封裝。第二引擎：ROI 兌現（核心賭注）。Big4 capex $380B+、約 2/3 餵壽命短 GPU——折舊打進成本線、壓縮雲端毛利。bull：算力受限 = 定價權，租金抵銷折舊；bear：折舊跑贏營收加速、毛利壓縮。payback 只有「雲端營收續加速」才成立——本主題最該盯的證偽點。

需求三角驗證

受限需求（現需）：Azure $80B backlog（電力限）、GCP RPO >$70B、AWS AI run-rate >$15B（Q1'26 法說 T1）。capex（投入）：Big4 2026 $380B+（vs '25 ~$246B）、2/3 餵 GPU（彙整估計，公司級口徑）。ROI 證偽：雲端營收加速 vs 折舊（定價權 vs 毛利壓縮）。三邊一致——需求受限（蓋不出來）證明真需求 + 定價權，但 ROI 取決於雲端營收能否續加速。注意：capex 含公司級（MSFT ~$190B/AMZN ~$200B）與雲端 segment 兩種口徑、不可混用。

BABA🟡 次要受益Alibaba Cloud 中國雲 #1、2026 重啟 AI capex（Qwen）；ADR 曝險
CRWV🟢 邊緣受害CoreWeave——純 GPU neocloud 結構性 loser（自研 ASIC 擠 + Anthropic 撤退、合約集中）
AAPL🟢 邊緣中性用 Google TPU 訓練 Apple Foundation Models——對 NVDA 負面 binary 訊號（非 hyperscaler）

非顯而易見的判讀：AVGO 是最確定的 ASIC 套利者——市場焦點多在三巨頭與 NVDA，但 AVGO 的客製加速器 +140% YoY（>NVDA DC 增速）+ TPU base die ~70% 市佔，是自研 ASIC 趨勢的最直接受益，且其上行未充分 priced。反向，純 GPU neocloud（CRWV）是結構性 loser——被 hyperscaler 自研 ASIC + 大客戶（Anthropic）撤退至自研晶片雙重夾擊。AAPL 用 Google TPU 是 NVDA 份額侵蝕的 binary 訊號。Anthropic（私募）是平衡三巨頭定價權的 swing customer。

最高純度 play 是 GCP（成長 + TPU）/ Azure（毛利 + 受限需求）/ AVGO（ASIC 套利）——買賣由個股 DD 與 Pure MA 系統決定。AWS 規模大但盯成長回升；NVDA 仍主供但盯 2027–28 份額侵蝕；neocloud（CRWV）結構性偏空。
