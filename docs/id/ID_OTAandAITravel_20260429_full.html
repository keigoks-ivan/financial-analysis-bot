§1 產業白話定義 + 歷史脈絡

OTA 賣的不是「一個網站」，而是「把全世界的旅館、機票、體驗 inventory 聚合起來，加上跨幣別支付、防詐、改退與客服，讓你在一個地方訂完一趟旅程」。付錢的是旅客（佣金內含在房價），而 OTA 賺的是 supplier 付的 commission（take rate）。AI 在 2024–2026 從兩端衝擊這門生意——但方向相反、力道不對稱。

白話開場：傳統旅遊分銷有兩個痛點——supplier（旅館/航空）想直接賣、不想付 OTA 15% 佣金；旅客想一站訂完、不想開二十個分頁比價。OTA（online travel agency，線上旅遊代理）就是站在中間，用規模解決兩邊：它聚合了夠多 supplier inventory（所以旅客來這裡能一次比完）、又帶來夠多旅客流量（所以 supplier 願意付佣金上架）。這是典型的雙邊網路效應生意——越多旅客來、越多 supplier 上架，反之亦然。Booking.com（BKNG 旗下）是全球最大的這種平台，Expedia（EXPE）是美國最大，Trip.com（TCOM）是中國最大。

邊界界定：本報告聚焦「擁有完整訂房漏斗 + 賣 inventory（而非賣 brand）」的 OTA 主軸——納入全球 OTA（BKNG/EXPE/TCOM）、Experiences pure-play（TRIP/Viator）、STR 替代品（ABNB，與兄弟 ID_HotelChains 互鎖）、以及 2024 起進場的 AI 分銷層（OpenAI/Perplexity/Google/MS）。不深入純 GDS（Sabre/Amadeus 是 OTA 上游 wholesale）、純 hotel SaaS（PMS）、連鎖旅館本身（那是 supplier 端、見 ID_HotelChains）。灰色地帶在「Google」——它既是 OTA 最大的上游流量來源（旅客大多先 Google 搜尋），又是部分競品（Google Hotels meta-search），還是 OTA 最大的廣告買家對手——這個三重身份本身就是 §7 的 insight 來源。

歷史脈絡分四個轉折。第一次轉折：1995–2005——把離線旅行社搬上網，Expedia（1996）、Priceline（1997）、Travelocity、Booking.com（2005 被 Priceline 投資）多家分散。第二次轉折：2005–2015——大整合，Priceline 收 Booking.com + Agoda + Kayak、Expedia 收 Hotels.com + Orbitz + HomeAway，雙寡占成形、Booking.com 靠歐洲基本盤全球擴張。第三次轉折：2015–2020——mobile-first + Airbnb 進場 + 國際擴張；BKNG 領先 EXPE、ABNB 起飛。第四次轉折：2020–2024——COVID 全產業崩 -70%、reopen、BKNG 啟動 Connected Trip（跨類別整合 hotel + flight + car + experience）。

第五次轉折，也是本報告的核心：2024–2026——AI 分銷革命啟動。2024-05 Google 開始 rollout AI Overview；2025-10 OpenAI 把 Booking.com / Expedia / Tripadvisor 做成 ChatGPT 內建 app；Perplexity + Selfbook + Tripadvisor 推 hotel booking agent；BKNG 把 GenAI 升級為 agentic Trip Planner。而 2026 年的關鍵戲劇性轉折是：2026-03 OpenAI 悄悄撤掉了 ChatGPT 內的旅遊 instant checkout——它發現「旅遊太複雜」，用戶樂意問點子、但不願在 chatbot 裡輸入信用卡。這個撤退是整個 AI×OTA thesis 的分水嶺：它證明下游 agentic 直接訂房的威脅被高估了，OTA 的信任 + 支付 + 改退層繞不過。真正在 throttle OTA 的是上游 Google AI Overview，不是 ChatGPT。

§2 技術成熟度 + S 曲線

為什麼 AI 對 OTA 的衝擊是「上游已發生、下游退潮」，而不是「ChatGPT 立即取代」？因為兩個技術現實——AI Overview 的「答案頁」已經量產（壓搜尋流量是當下事實），而 agentic 直接訂房卡在「信任 + 支付 + 改退 friction」這個 2026 還沒解開的瓶頸。

AI 對旅遊分銷的衝擊要拆成兩條獨立的技術曲線，它們成熟速度完全不同。上游曲線（AI Overview，已成熟）：Google 把 LLM 摘要塞進搜尋結果頁頂端，旅客一個「東京五天怎麼玩」的 query 就能在頁面上得到答案、不必點任何 OTA 連結——這是純資訊呈現問題，技術已量產，所以 travel 類 zero-click 已升到 65–70%。下游曲線（agentic 訂房，未成熟）：要讓 AI agent 真的「幫你訂下去」，需要解開三個遠比「呈現答案」難的問題——(1) 支付信任（旅客願不願在 chatbot 輸入信用卡訂一筆數千美元的行程）；(2) 改退與爭議（航班取消、旅館 overbooking 時誰處理）；(3) 即時庫存同步（價格與空房秒級變動）。這三個在 2026 還沒解，所以 OpenAI 撤退、滲透率 < 1%。

§3 供給側：現在與未來 + 利潤池

供給端的結構是「2 家全球全平台（BKNG/EXPE）+ 1 家中國巨頭（TCOM）+ 一票被擠到邊緣的區域/單品類玩家」。瓶頸不是產能（無實體產能），而是「supplier inventory 廣度 + payment 基建 + 消費者信任」這道準入門檻——這正是 AI agent 必須跟 OTA 合作的根本原因。

怎麼讀：利潤往「上游 Google」與「OTA 雙寡占內的贏家」集中——Google 把 OTA 的免費 SEO 流量轉成付費（廣告池擴大），OTA 內部 BKNG 擴大、EXPE 守、小 OTA 失血；AI 分銷層自己還沒獲利、只收 referral。（依 QC-18，缺逐家利潤池精確百分比，以定性「主導/均勢/次要」呈現。）

成本曲線（省略說明）：OTA 無實體產能，沒有「價格跌到哪個 level 誰先停產」的邊際成本曲線；競爭的終局不由單位成本決定，而由 inventory 廣度與信任成本 決定。這正是 AI agent（沒有 supplier 直連、沒有 payment rail、沒有客服）無法靠價格戰繞過 OTA、只能接 OTA API 的根本原因——它們沒有 OTA 那種「三道準入門檻」的成本結構。

未來供給與進入門檻：新進入者要成為全球 OTA 贏家，門檻是「夠廣的 supplier 直連 inventory + 自有 payment 基建 + 消費者信任」三者，單靠內生成長要很多年——所以擴張的主要路徑是併購 + 規模飛輪。AI agent 想繞過這道門檻只有兩條路：(a) 自建 inventory/payment/客服（成本極高、且要重建信任）；(b) 直連 GDS（Sabre/Amadeus 的 2M+ 旅館庫存）+ 第三方支付（PayPal）。2026-02 Sabre + PayPal + MindTrip 的 agentic pipeline、RateGain 的 MCP booking 引擎都在試 (b)，但 2026 仍卡在「爭議處理可信度」——OTA 的客服 + 改退層是最難複製的一塊。

§3 因果閉合（回應 §1 的結構變數）

§1 提出「inventory + payment + 信任」三道準入門檻是 OTA 護城河來源——未來 3–5 年它是否仍 binding？答案是對下游 binding、對上游不 binding：對下游 agent，三道門檻越來越 binding（OpenAI 撤退正是承認繞不過），OTA 是 agent 不可繞過的 inventory 層；但對上游 Google，門檻完全不 binding——Google 不需要 inventory/payment，它只要在搜尋入口 throttle，就能把 OTA 的免費流量變成付費流量。所以 OTA 的護城河是「不對稱的」：擋得住下游 agent、擋不住上游搜尋 throttle。這是 §7 要檢驗的核心張力。

§4 需求側：現在與未來 + 三角驗證

需求最不可替代的一塊是「旅遊是高頻、高金額、高信任的剛性消費」——疫後 experience > goods 的消費結構移轉讓旅遊支出有韌性；最改寫分銷的引擎是 AI，但 2026 它對需求是「上游 throttle 已發生、下游 agentic 退潮」的非對稱雙引擎。

現需結構：採購主體是全球旅客，旅遊需求結構性成長（experience economy）+ 中國 outbound 仍在回補（TCOM outbound 已回 2019 的 >120%、國際 OTA bookings +~70%）。兩個 AI 引擎方向相反：①上游搜尋 throttle（已發生）——Google AI Overview / AI Mode 把 travel informational query 變成答案頁，zero-click 升到 65–70%、organic session 掉 20–40%、被引用品牌反而多拿 35% 點擊（贏家更集中）；②下游 agentic 退潮（2026 反轉）——OpenAI 撤掉 ChatGPT instant checkout，僅 ~2–8% 旅客願讓 AI 代付。需求的「量」很穩，被改寫的是「OTA 怎麼拿到這些量」。

§4 因果閉合（回應 §1 的需求驅動）

§1 提到 AI 是分銷的核心驅動——未來 5–10 年它會強化而非消退，但方向是「上游壓力持續、下游威脅延後」。上游 Google throttle 是持續的（Google 不會放棄把搜尋變答案頁），所以 OTA 行銷費率上升是結構性的；下游 agentic 退潮是timing的（2027–2028 信任 friction 若被 MCP/Sabre+PayPal 解開會回來）。需求拐點風險不在「量」（旅遊很難消失），而在「OTA 拿量的成本」（上游）與「OTA 是否被降為 commodity API」（下游長期）。

最有定價權的需求是「OTA 作為 agent 的 inventory + payment + 信任層」——agent 買的不只是庫存，是「不必自建支付與客服」。對投資的含義：需求引擎（旅遊剛韌 + OTA 不可繞過）在 2026 數據上成立、很可能持續，所以「OTA 會被取代」不是當下風險點；真正要盯的是上游 throttle 怎麼壓 margin（§6）與下游威脅何時回歸（§7）。

§5 供需裁決 + 短中長期推估 + 投資時鐘

⚠️ 本章含 judgment：🟡 bullet 代表推論，附事實鏈與信心度 + ⚠ 證偽條件。
綜合 §3（供給門檻擋得住下游 agent、擋不住上游 Google）與 §4（需求剛韌、量穩但拿量成本上升），本產業未來 3–5 年的裁決是「OTA 不可繞過 + 雙寡占內部永久分化」——但籠罩在上游 throttle 壓 margin 的結構性陰影下。

資本週期證據（OTA 版替代指標）：OTA 無實體 capex/折舊週期，改用三個「資本是否過度湧入」的替代訊號：① OTA 估值倍數史——2021 OTA 倍數隨成長股泡沫衝高，2022–23 壓縮，2026 BKNG 拆股後 fwd P/E ~16x、EXPE ~10x 已回理性區間，顯示「無差別湧入」階段已過；② AI 資本配置——資金往「有完整 inventory/payment 能力」的 OTA AI 整合集中（BKNG agentic Trip Planner），而非無差別灑向「AI 直接訂房」新創（OpenAI 撤退正是承認該方向不成熟）；③ 買回 vs 擴張——BKNG 季退還股東 record $4.0B、EXPE $5B 新買回，反映「成熟現金流回饋」而非「燒錢搶份額」。三者一致指向「成熟理性」而非「泡沫式擴張」。[A: 假設旅遊需求不因宏觀急凍；若地緣/宏觀急轉，高 beta OTA 承壓]。

§3 + §4 → 供需裁決

依本章 §3（供給門檻不對稱：擋下游、不擋上游）與 §4（需求剛韌、量穩但拿量成本上升）的推導，未來 3–5 年產業結構是 OTA 不可繞過 + 雙寡占內部分化，因為 agent 必須接 OTA inventory（下游 binding），而 Google 把免費流量變付費（上游 throttle）。這不是「過剩/短缺」的商品框架（OTA 無產能），而是「份額集中度 + 流量成本」的結構框架。（若硬要對應傳統商品框架：旅客需求遠大於「具備完整 inventory/payment/信任的合格 OTA 供給」，等價於結構性稀缺型——只是供給稀缺性來自三道準入門檻而非實體產能。）

  • 共識說 X：AI Overview 只傷 inspiration/blog 流量，booking-intent query 仍受保護、對 OTA 營收影響小。🟢 [F: T3 — informational query AI Overview 觸發率高、transactional 較低]。
  • 我們說 Y：上游 throttle 是結構性且已發生——即使 booking-intent query 受保護，inspiration → booking 的漏斗變窄，OTA 必須加大付費行銷補免費流量缺口（margin 壓力），且被引用進 AI Overview 的品牌多拿 35% 點擊（贏家更集中、輸家更失血）。🔵 [X: base 很可能 — OTA 行銷費率溫和升、BKNG 用品牌直訪對沖；bear 可能 — Google 自做 hotel booking funnel、throttle 失控]。
  • Priced-in：市場目前把 Google throttle 當「可控的行銷費微升」——幾乎沒 price 「Google 自己認真做 hotel booking funnel」的尾部風險（Google Hotels meta-search 已存在）。這是最未被 priced、也最該獨立監測的一條：盯 OTA 行銷費 / GBV 比與 Google Hotels 直接 booking 份額。
→ 推論邏輯：上游 throttle 已發生、市場只 price 了「可控微升」一面。⚠ 證偽：OTA 行銷費率持續上升而 GBV 增速不升 + Google Hotels booking 份額竄升 = 分歧 3 的 bear 成立。

Steel-man 反方（最有力的 3 條）

  • 反方 1（Google 上游終局）：Google 仍佔絕大多數 travel research 入口，AI Overview throttle + 自做 hotel booking funnel，可把 OTA 的 TAM 結構性壓縮。當前證據：travel organic -20–40% 已發生。證偽窗口：OTA 行銷費率 vs GBV 增速持續背離兩年。
  • 反方 2（agentic 回歸）：2026 退潮是 timing 而非永久；MCP/Sabre+PayPal/RateGain 基建持續推，2027–2028 信任 + 支付 friction 若被解，agentic 直接訂房回歸、繞過 OTA。當前證據：基建仍在推進。證偽窗口：AI 代付意願破 15% + 主要 agent 重啟 instant checkout。
  • 反方 3（旅遊週期 + 地緣）：旅遊是高 beta 可選消費，宏觀逼降溫 + 地緣（中東）逆風會放大跌幅。當前證據：BKNG Q2'26 guide 軟化至 room nights +2–4%、中東逆風。證偽窗口：room nights 連兩季破 low-single 且非一次性。

市場最低估的風險：不是 ChatGPT（disintermediation 恐慌已退潮）、也不是雙寡占分化（已 part-priced），而是分歧 3 的 Google 上游終局——市場把 Google throttle 當可控微升，最沒準備好「Google 自己認真做 hotel booking funnel + throttle 失控壓 margin」這個情境。這是本報告認為最該獨立監測的尾部風險，也是比 ChatGPT 更危險的真威脅。

三條分歧裡最可操作的是分歧 2（EXPE B2B deep-value）——EXPE ~10x 隱含結構落後近乎全 price，B2B +22% 若驗證結構性有 re-rate 空間，且有明確領先指標（B2B 增速 + margin）。分歧 1（OTA 繞不過）次之、part-unpriced 但已被 Skift 等部分認知；分歧 3（Google 上游終局）是最未 priced 的尾部風險、但時間軸長。
