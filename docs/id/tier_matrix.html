📐 Framework v3.0 — 為何要這個獨立頁？
產業 DD（ID）描述產業 framework，不替個股下投資結論（§11 玩家矩陣的 🔴🟡🟢 是「產業曝險深度」非「投資建議」）。本頁把跨 ID 累積的「個股投資 conviction」獨立出來，做三軸打分 + 四 Tier 分級。v3.0 重定義三軸（1-5；數字越大 = 風險越高）：

• AI 顛覆（合併舊版「自我蠶食」+「競對直擊」兩軸：自家 AI 替代核心業務 + 競對 AI 直擊 use case 的淨曝險。1=AI 是需求 driver / 5=核心被替代路徑明確）
• 週期（v3.0 新軸，接 AIComputeCapexCycle 框架：若 capex digestion / 產業自身週期反轉，這檔在「見血順序」的位置。1=需求結構性無週期 / 5=cycle 反轉時最先且最重跌 — 上游晶片、memory、WFE 高分；買方 hyperscaler 低分）
• Priced（取代舊「Reset」軸，改用 live 數據錨定：5Y PE 分位 + expectation gap。1=自身 band 低分位 / 5=100% 分位或 priced 完美執行）

四 Tier 分級邏輯（Tier 反映結構性 conviction，不是 entry timing — timing 看各檔 DD 訊號燈與 Priced 分數）：
• Tier 1 真 Moat-core：不可替代的 infra / 矽片 / 平台 / 稀缺資產 + AI 顛覆免疫或受益；可作組合核心持倉（Priced 高低不影響 Tier，只影響 entry timing）
• Tier 2 Conviction Long（衛星池）：結構強 + execution 強，但 PE 已 priced 或 thesis 已獲數據驗證但結構強度未達 moat-core；satellite 級配置
• Tier 3 Thesis-pure / 週期觀望：押 thesis 反轉 / 週期 timing / turnaround；嚴守對應 ID 的 §13 證偽條件，不該無腦 Long
• Tier 4 Avoid / 結構受損：增速、競對、估值多軸不利，或結構性受害 / 高槓桿週期溫度計；除非重大 catalyst 反轉，不該 Long
🧭 2026-06 市場座標（v3.0 新增 — 取自 ID_AIComputeCapexCycle 2026-06-11，critic 裁決 THESIS_INTACT）
週期定位：Phase II（build-out 中後段，late-build-out 早期裂縫）— 非過剩週期的頂。12M 供需 = 短缺（GPU lead time 36-52 週、CoWoS full allocation 至 mid-2027、H100 一年租約價半年反漲 +40%）；3Y = 平衡偏緊但變現缺口同步擴大（上游晶片營收 ~$450-500B vs 已驗證 AI app 營收 ~$55-70B ≈ 7-9x）。本輪的 kill 是 financial 過剩（變現追不上 + 2025-09 起集體轉外部發債，2026 AI 新債估 ~$570B），不是 physical 過剩。估值結構與 2000 年相反：「倍數安全、E 危險」 — 四大 hyperscaler 倍數都在歷史 band 中下緣，樂觀全部 price 進上游晶片商的 E（NVDA FY27 consensus 建基 capex 100% 兌現）。bear 觸發時殺傷力集中在上游晶片股（高週期分）而非 hyperscaler。
