§1 產業白話定義 + 歷史脈絡

白話版：當你習慣對 Siri 說「幫我訂張兩人桌」而不是打開 OpenTable，app 的角色就從「被你打開」變成「被 Siri 呼叫」。問題是——Siri 這顆大腦是 Apple 向 Google 租的，所以這場威脅的錢，一部分流進了 Google 口袋。

白話開場：先說三個關鍵詞。Sherlocking是 Apple 的老招——把第三方熱門功能做進 OS，讓原 app 失去存在理由（名字源自 2000 年代 Sherlock 吞掉 Watson）。App Intents是新 Siri 呼叫第三方 app 的唯一通道（SiriKit 的後繼）——你的 app 若沒接 App Intents，Siri 就叫不到它，在「助手優先」的世界裡等於 invisible。Sherlock-able指那些「單一用途、可被一句話完成」的 app（約會、評論、訂位、叫車、外送）——它們最容易被 OS agent 繞過。本報告問的是：這場 OS 級 agent 威脅，誰被吞、誰收錢、多快發生。

邊界界定：本報告聚焦「Apple Intelligence 作為 OS 級 agent 對 app 經濟的威脅 + 價值捕獲（分發 vs model）」。納入 Apple、其 model 供應商（Google）、被威脅的 consumer app（dating/OTA/reviews/ride/food）、competing agent（OpenAI）。不展開 Apple 硬體本業估值（那是個股 DD）。灰色地帶是「Sherlock-able vs 防禦型」——有強網路效應（UBER）、多平台分散（BKNG）、host-side 供給（ABNB）的相對防禦。

歷史分三段。第一段：2024 WWDC 2024 預告個人化 Siri（能跨 app 行動），市場期待 Apple AI 大爆發。第二段：2024–2025 延期再延期——個人化 Siri 兩度跳票、Apple Intelligence 成「笑話」，自研 model 進展不順。第三段：2026 重啟但租大腦——2026-01 Apple 簽下 Google Gemini（客製 ~1.2 兆參數、~$1B/年）；WWDC 2026 重啟 Siri、棄 SiriKit、推 App Intents 2.0；但 EU 因 DMA 在 iOS 27 不出、on-device 限 iPhone 15 Pro+。威脅成形，但「真實但慢、且 model 租來的」。

歷史類比一：Sherlocking——被 OS 吞的單點 app 會死。Apple 內建功能吞第三方 app 是有名字的模式：手電筒 app、剪貼簿、螢幕延伸（Luna Display 被 Sidecar 吞）等無數單點 app 被 Sherlock 後歸零。量化錨：被 Sherlock 的單一用途 app 多數營收歸零或被迫轉型。對 bear 含義：Siri + App Intents 是「OS 級 agent 版 Sherlock」，MTCH/BMBL/YELP/DASH 最危。歷史類比二：預設搜尋租金的反轉。過去十年 Google 付 Apple 約 $200 億/年買 Safari 預設搜尋位（分發層向 service 層收租）；AI 時代出現議價反轉的一半——Apple 自研 model 失敗，反過來付 Google 約 $10 億/年租 Gemini，且讓 Google 分產品發現/購買收入。量化差異：分發仍值錢（Apple 收 Search Ads），但 model 的稀缺讓分發霸主也得向 model 商低頭——這是「分發贏、model 輸」雙面的根源。

本主題無價格 cycle，用「Siri/agent 採用 + Sherlock 落地」當時序錨。

關於 cycle 統計表的省略：平台威脅為結構性事件、無 ASP cycle，故不立 cycle 統計表；以「Apple AI 三段」（2024 預告 → 2024–25 延期 → 2026 重啟租 Gemini）的編年呈現時序。

這段歷史的可操作含義：Apple AI 同時是「Sherlock」（吞單一用途 app）與「收租反轉」（過去收 Google 租、現在付 Google 租）。bull 的底是 AAPL 分發 + Search Ads，bear 是 Sherlock-able app 被繞過；新洞見是「model 議價反轉」——分發霸主竟向 model 商付費，model 價值給了 GOOGL。

§2 技術成熟度 + S 曲線

為什麼威脅是現在、但慢？因為「OS 級 agent 能呼叫 app 做事」的技術剛到位（App Intents 2.0 + Gemini 大腦），但 Apple 的 execution（延期）、監管（EU DMA）、裝置（iPhone 15 Pro+）三道閘把落地拖到 2027+。

技術拐點是「assistant 能可靠地呼叫 app 完成多步」。三個要件 2026 才湊齊：大腦——Siri 建在客製 ~1.2 兆參數 Gemini，推理/工具呼叫夠強（但是租的）；呼叫通道——App Intents 2.0（richer entity、streaming、多輪、View Annotations API）讓 Siri 能呼叫第三方 app 做事，且是唯一通道（SiriKit 棄用）；on-device 隱私——Foundation Models on-device（限 iPhone 15 Pro+）；惟客製 ~1.2 兆 Gemini-powered Siri 的推論走 Apple Private Cloud Compute（PCC）私有雲、非真正裝置端，on-device 指的是 Apple 自有小模型。三者交會，「對 Siri 說一句話就把事做完」技術可行。但三道閘拖慢落地：Apple 兩年延期史、EU DMA 在 iOS 27 不出、on-device 限新機。

§3 供給側：現在與未來 + 利潤池

Apple AI 的利潤一分為二——分發層（AAPL：OS + App Intents + Search Ads）與 model 層（GOOGL：Gemini + 收入分成）各拿一塊，中間被 Sherlock 的單一用途 app 失血。

供給端可拆三塊。分發層（AAPL）：iPhone 裝機量 + Siri 入口 + App Intents（Siri 呼叫 app 的唯一通道）+ Apple Search Ads——這是 Apple 真正的護城河，AI 強化它（assistant 變新入口）。model 層（GOOGL）：Apple 自研失敗、租 Gemini（~1.2 兆參數客製、$1B/年），Google 還分產品發現/購買收入——model 經濟給了 Google。被 Sherlock 的 app 層：單一用途、可被 OS agent 一句話完成的（MTCH/BMBL/YELP/DASH）失血；有網路效應/多平台/host-side 的（UBER/ABNB/BKNG）相對防禦。利潤往「分發 + model」兩端，中間單點 app 被擠。

Inference · 供給
分發給 Apple、model 給 Google，Sherlock-able app 失血
前提：Apple 握 OS 分發 + App Intents gatekeeper + Search Ads（分發值錢）+ 自研 model 失敗租 Gemini（model 價值外流 Google）+ 單一用途 app 可被 OS agent 一句話完成 → 利潤往分發（AAPL）+ model（GOOGL）兩端，單點 app 失血

Apple 的分發護城河被 AI 強化（Siri 變新入口、App Intents 當守門），靠 Search Ads 變現；但 model 層它租 Gemini、分潤給 Google，所以 model 經濟給了 GOOGL。被夾在中間的是 Sherlock-able 單一用途 app——它們的功能正好是 OS agent 能一句話完成的。$1B/年 Gemini 案是「分發霸主竟向 model 商付費」的硬證據。

可證偽條件：若 Apple 2027 成功以自研 ~1 兆 model 取代 Gemini（model 價值收回）、或 App Intents 反而讓 app 更易被觸及（變賦能而非守門），則「model 給 Google / app 失血」論的某一面失效。

§4 需求側：現在與未來 + 三角驗證

需求最剛性的是「用一句話讓助手把事做完」——當用戶習慣對 Siri/ChatGPT 說「幫我訂位」，app 從「被打開」變「被呼叫」；最大的變數是 Apple 把這件事拖了多久才落地。

第一引擎：assistant-first 行為（未需轉現需）。用戶要「一句話做完事」——「訂兩人桌」「找評價高的拉麵」「叫車去機場」。當 OS 級 agent 能直接完成，app 從「主動打開」變「被 Siri 呼叫」，App Intents 成「能不能被叫到」的生死線。OpenAI ChatGPT super-app（Booking/Spotify 入駐）已在跨 OS 證明需求。第二引擎：落地速度（供給瓶頸）。需求方向明確，但 Apple 供給一再卡關：個人化 Siri 兩度跳票、成「笑話」；EU 因 DMA 在 iOS 27 不出 Siri AI；on-device 限 iPhone 15 Pro+。所以 Sherlock 衝擊「會來、但 2027+」。

需求三角驗證

行為位移（需求）：assistant-first「一句話做完事」，方向明確（ChatGPT super-app 已驗證跨 OS 需求）。供給瓶頸：Apple Siri 兩度跳票、EU DMA 不出、限 iPhone 15 Pro+——落地 2027+。真對手：跨 OS 的 OpenAI ChatGPT（Booking/Spotify 入駐）。三邊一致——威脅是真的、但落地慢、且 Apple 非唯一 agent。注意：keynote demo 與實際出貨口徑不同、勿混用，本報告以「實際出貨進度 + Sherlock-able app 財報」為主錨。

Inference · 需求
assistant-first 需求真實但落地慢——關鍵看「誰被 Sherlock、誰防禦」
前提：assistant-first 行為位移（OS agent 繞過 app）+ Apple 供給卡關（延期/DMA/裝置）+ 非所有 app 都 Sherlock-able（網路效應者防禦）→ 威脅真實但慢、且分化（單點輸、防禦型守）

需求方向（一句話做完事）沒爭議；變數在「速度」與「誰被 Sherlock」。Apple 延期/DMA/裝置限把落地拖到 2027+（給 loser 喘息）；且只有單一用途、可一句話完成、無強網路效應者最危（MTCH/BMBL/YELP），有網路效應/多平台/host-side 者（UBER/ABNB/BKNG）相對防禦。

可證偽條件：① Siri 2.0 又跳票/DMA 持續卡（威脅再延）；② 連有網路效應的 app（UBER/BKNG）也被 OS agent 顯著繞過（防禦失效）。

多空為何都對：威脅論測assistant-first 行為（OS agent 繞過 app），方向沒爭議；防禦方測落地速度（延期/DMA/裝置/網路效應）。兩者可同時為真：威脅是真的（會來），但落地慢（2027+）、且非所有 app 都 Sherlock-able。本報告把重心放在「誰被 Sherlock、誰防禦、價值給誰」。

最高純度 play 是分發 AAPL 與 model GOOGL（後者被低估）——買賣由個股 DD 與 Pure MA 系統決定。防禦型（UBER/BKNG/ABNB）靠網路效應拖延威脅；結構性輸家是 Sherlock-able 單點（MTCH/BMBL/YELP）；OpenAI 是真對手但私有。
