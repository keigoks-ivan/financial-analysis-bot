§1 產業白話定義 + 歷史脈絡

白話版：「Copilot」是坐在你旁邊、你問它才動的助手；「Agent」是你交代一個目標、它自己規劃並動手做完多步任務的員工。Agentic AI 平台這門生意，就是「賣這種會自己動手的 AI，以及讓它跑起來的底座」。

白話開場：先分清四個詞，因為它們是四門不同的生意。LLM（大型語言模型）是底層大腦（Claude/GPT/Gemini），負責理解與生成。agent 平台是把大腦組裝成「會自己規劃、呼叫工具、執行多步」的自主流程（MSFT Copilot Studio、PLTR AIP、Google Vertex）。應用/copilot是把 agent 嵌進具體業務情境（Salesforce 客服 agent、ServiceNow IT agent）。基礎設施是讓 agent 安全地連到工具、資料與彼此的協議與網路（MCP、agent 身份、Cloudflare）。把這四層攪在一起談「Agentic AI」，就會買錯層——每一層的贏家邏輯、護城河、估值都不同。

邊界界定：本報告聚焦「四層棧的價值捕獲 + 哪一層握真護城河 + 商業化證明 vs 夭折」。納入做 agent 平台、應用 copilot、agent 基礎設施的上市與重要私有玩家；底層 LLM 廠的估值另見「Token 經濟學」與 AI 模型 ID。灰色地帶是「copilot vs agent」——多數產品仍在從 copilot（被動）往 agent（主動）過渡，本報告以「是否能自主執行多步 + 有可量化 ROI」為真 agent 的判準。

歷史分三段。第一段：2023 生成式 AI 爆發，產品形態是「聊天助手」（ChatGPT、各家 copilot），人問才答。第二段：2024–2025 agentic 化——AI 從「助手」變「會自己動手的員工」，能規劃、呼叫工具、執行多步；2025 被稱為「agentic 元年」，平台廠（Salesforce Agentforce、Microsoft Copilot Studio、Palantir AIP）大舉推出 agent 框架。第三段：2026 商業化證明年——不再只是 demo，而是付費營收：Agentforce ARR $800M、Copilot Studio 40 萬自建 agent、PLTR 營收 +85%。同時也是「洗牌年」：Gartner 點名 agent washing、預測 40% 專案夭折。

歷史類比一：SaaS 取代 on-prem（握資料+workflow 者贏）。上一個平台換代裡，贏家不是技術最炫的，而是離客戶最近、握住資料與工作流的應用層——Salesforce、ServiceNow 由此長成數百億美元巨頭。量化錨：agentic 正重演，Agentforce 上線 6 個月做到 ARR $800M、平均 ROI ~171%（第三方研究），是「應用層握 CRM 專屬資料 = 真護城河」的早期證據。歷史類比二：貼名泡沫（agent washing 的鏡像）。1999–2000 無數公司名字加「.com」估值即翻倍、泡沫後存活率 <50%；2017 年 Long Island Iced Tea 改名「Long Blockchain」股價單日 +289%（後遭下市）。今天「agent washing」是同一齣：數千家把舊聊天機器人貼上 agentic 標籤，Gartner 估僅約 130 家是真的。量化差異：真 agent 有可量化 ROI（Agentforce 171%）、嵌專屬資料；貼名者沒有，會在 40% 夭折潮裡歸零。

本主題無價格 cycle，用「agentic 採用滲透」當時序錨。

關於 cycle 統計表的省略：agentic 為單向結構成長、無 ASP peak-to-trough，故不立 cycle 統計表；以「產品形態三段」（2023 聊天助手 → 2024–25 agentic 元年 → 2026 商業化證明/洗牌）的編年呈現時序。

這段歷史的可操作含義：agentic 同時有「SaaS 上移」的真（應用層握資料的 CRM/NOW/PLTR）與「貼名泡沫」的假（agent washing、~130/數千家才真）。bull 的底很實（CRM $800M ARR），但 bear 不是「agentic 不重要」（重要），而是「純題材/純工具被 40% 夭折潮洗掉、AI 估值溢價收斂」。

§2 技術成熟度 + S 曲線

為什麼 agentic 是現在？因為三個技術同時成熟：model 的 tool-use/reasoning 能力夠強、MCP 等協議讓 agent 安全連到工具與資料、企業的工作流被數位化到 agent 可介接——三者交會，agent 從 demo 變成可賣的產品。

agentic 的技術拐點有三。model 的 tool-use 與長程推理：frontier model 能可靠地呼叫外部工具、維持多步計畫、自我修正，是 agent「能自己動手」的前提。協議標準化（MCP）：Model Context Protocol 讓 agent 用統一方式連到工具、資料源與彼此——Anthropic 發起，OpenAI、Google DeepMind、Microsoft、AWS 全部採用，到 2025-11 週年已達 97M 月 SDK 下載、10,000+ 部署 server；2025-12 Anthropic 把 MCP 捐給 Linux Foundation 的 Agentic AI Foundation（與 OpenAI、Block 共同創立，白金會員含 AWS/Anthropic/Block/Bloomberg/Cloudflare/Google/Microsoft/OpenAI）。企業工作流數位化：CRM/ITSM/HR 等系統已 SaaS 化，agent 才有可介接的資料與動作。三者交會，agentic 從口號變成可賣的產品。

§3 供給側：現在與未來 + 利潤池

agentic 棧的利潤往「兩端」集中——握客戶資料+workflow+分發的應用層（一端）+ 協議/網路的基礎設施層（另一端），中間的 LLM 商品化、純 agent 工具被擠。

供給端是四層：①LLM 模型層（OpenAI/Anthropic/Google——被多模型壓價、商品化中）；②agent 平台層（MSFT Copilot Studio、Google Vertex、AWS Bedrock Agents、PLTR AIP——把 model 組裝成自主流程）；③應用/copilot 層（CRM Agentforce、NOW、WDAY、INTU、ADBE——嵌業務情境、握專屬資料）；④基礎設施層（MCP 協議、NET agent 網路、agent 身份/治理——讓 agent 連工具與彼此）。利潤往「應用層（握專屬資料+workflow+分發，高轉換成本）+ 基礎設施層（協議/網路收費站，標準鎖定）」兩端集中；LLM 層被壓價、純 agent 工具淹沒在 agent washing。

利潤池遷移

整條鏈的利潤正從「中間」（純呼叫 model、純做 demo agent）往「兩端」遷移：底層 LLM 被多模型壓價（利潤薄）、純 agent 工具被 agent washing 淹沒（無護城河），賺溢價的是握客戶專屬資料 + 工作流 + 分發的應用層（CRM/NOW/PLTR——高轉換成本）與握協議 + 網路的基礎設施層（MCP/NET——標準鎖定）。被搶的是純 LLM wrapper 與貼名工具，搶到的是應用層巨頭與協議/網路。（依 QC-18，缺逐段精確百分比，以定性呈現。）

Inference · 供給
LLM 商品化、純工具被淘汰，利潤往應用與基礎設施兩端
前提：LLM 多模型壓價（商品化）+ 純 agent 工具易被內建/agent washing 淹沒 + 應用層握專屬資料與 workflow（高轉換成本）+ 基礎設施握協議/網路（標準鎖定）→ 利潤往「應用端 + 基礎設施端」集中，中間被擠

當「呼叫一個 model」變成可替換商品、「做一個 demo agent」變成人人能貼名，價值就不在這兩處。賺溢價的是握客戶專屬資料+工作流+分發的應用層（CRM/NOW/PLTR）與握協議標準+agent 網路的基礎設施層（MCP/NET）。CRM Agentforce 的 $800M ARR 與 171% ROI 證明應用層能把 agentic 重新定價（此 ARR 為產品有機成長、非併購灌入）。

可證偽條件：若某純 agent 工具型新創用「AI-native 平台」獨立長成大廠（不被併、自建分發），或開源 agent 框架讓應用層的資料/workflow 優勢被抹平，則「兩端集中」論的某一面失效。

未來供給與進入門檻：成為 agentic 贏家的門檻是「專屬資料 + workflow 嵌入 + 分發通路 +（基礎設施層則是）協議/網路鎖定」。純 LLM wrapper 補不上資料、純工具補不上分發——這正是 CRM（握 CRM 資料）、NOW（握 IT/企業流程）、PLTR（握作戰/營運資料）守得住的原因。長期變數是平台廠（MSFT/GOOGL）會不會用「分發 + 自有 model + 內建治理」把應用層的部分價值收走。

因果閉合（回應 §1 結構變數）：§1 提出「握資料+workflow 是 SaaS 換代不變的贏家邏輯」——未來 3–5 年是否仍 binding？很可能：agent 越自主，越需要可信的專屬資料與既有流程，門檻越高；唯一鬆動是開源 agent 框架 + 通用 model 把「接資料」變廉價——這是 §7 的反方。

12–36 個月內，LLM 持續商品化、純工具被洗，利潤往「應用 + 基礎設施」兩端集中。對投資的含義：押握資料/workflow 的應用層（CRM/NOW/PLTR）+ 分發/自有 model（MSFT/GOOGL）+ 協議網路（NET）；避開純 LLM wrapper 與貼名工具。

  2. [T1]Cloudflare（NET）— Workers + MCP host，agent 網路飛輪

§6 產業經濟學與估值傳導

agentic 的錢怎麼賺：應用層軟體高毛利 + 專屬資料/workflow 溢價，但要扣掉 agent 執行的推論 token 成本（見「Token 經濟學」）。估值對「ARR 增速 × ROI 兌現 × 毛利」敏感——PLTR 的極端高倍數押的是「成 agentic 作業系統」。

估值傳導機制：應用層估值對「ARR 增速 × ROI 兌現 × 毛利能否守住」敏感。傳導鏈：商業化兌現（CRM $800M ARR、PLTR +85%）→ 高 ARR + ROI → 估值溢價；反向，若 40% 夭折擴大、ROI 落空或推論成本侵蝕毛利，溢價 de-rate。敏感度錨點：PLTR 增速從 +85% 掉到 +40%，極端高 EV/S 常見壓縮 40–60%（市場重定價「agentic OS」假設）。

估值現在 price 了什麼：PLTR 極高 EV/S price 了「成 agentic 作業系統 + 完美執行」，沒 price「增速放緩/夭折擴大」；CRM/NOW price 了商業化兌現、part-price 持續加速；MSFT/GOOGL price 了分發但 agentic 增量被量體稀釋。會 re-rate 的是 ROI 普遍兌現 + 協議/網路爆發；會 de-rate 的是 40% 夭折擴大 + 任一季 miss。對 PM：盯 ARR 增速與專案存活率，不是盯「AI 是不是趨勢」。

2026–2027 agentic 專案存活率 vs Gartner 40% 取消預測。若存活率高、取消集中於貼名者 → 真 agent 領導者份額提升。若取消率超 40%、波及真 agent → 觸發夭折潮 bear。

滾動監測 MCP 標準化進度 + NET agent 流量/用量。若用量爆發 → 基礎設施層贏家確立。若標準分裂/用量停滯 → 基礎設施論題受挫。

滾動監測 PLTR 估值 vs 增速續航（AI 板塊整體溫度）。增速維持、估值消化 → 高倍數可持續。任一季 miss → 觸發高估值 de-rate。

3–4 個 kill metric + base/bull/bear 閾值 + 時間窗。bear 閾值即四層分化 thesis（在應用層龍頭上）翻空點。閾值以 current actual 校準（CRM ARR $800M、PLTR +85%、Gartner 40% 取消）。
