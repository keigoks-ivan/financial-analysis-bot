§1 產業白話定義 + 歷史脈絡

CUDA 是 NVIDIA 的軟體平台——讓開發者用 GPU 跑運算的程式語言 + 函式庫 + 工具鏈。它的價值不在「快」，在「過去十年所有 AI 軟體都預設它」。ROCm 是 AMD 的開源對手；framework（PyTorch/JAX）則是更上層的抽象，決定程式跑在哪種硬體。

白話開場：一個 AI 模型從程式碼到跑在晶片上，要經過三層——① 框架（framework）：開發者寫 PyTorch / JAX（高階、硬體無關）；② 運算平台：框架呼叫 CUDA（NVIDIA）或 ROCm（AMD）把運算下放到 GPU；③ 互連：多卡訓練時用 NVLink（NVIDIA 卡間高速線）+ NCCL（其上的通訊庫）把數千張卡串起來。NVDA 的「護城河」橫跨這三層，但每層的厚度差很多——這正是本報告的主軸。

邊界界定：本報告聚焦「NVDA 軟體/互連護城河的厚度與裂解」，含 CUDA vs ROCm（運算平台層）、NCCL/NVLink vs UALink（互連層）、framework 抽象（PyTorch/JAX/Triton 治理層）。GPU vs ASIC 的硬體競爭見母題 AI 加速器需求；推論成本經濟見 AI 推論經濟學。灰色地帶是「framework 層」——它不是晶片也不是 OS，但它決定硬體可否替換，計入因為它是護城河終局的決定者。

第一次轉折：2006 CUDA 1.0 發布、2012 AlexNet 用 2 張 NVIDIA GPU 奪 ImageNet 冠軍（錯誤率自 ~26% 降到 ~16%），此後十年幾乎 100% 深度學習研究與框架預設 CUDA，生態鎖定成形；第二次：2023-2025 ROCm 5.x→6.x 逐步追趕，但出箱可用率長期卡在「能跑、需大量手調」、推論引擎相容性差一截，AMD AI GPU 份額個位數；第三次：2026-03 ROCm 7.2 首次達成 vLLM / llama.cpp / Ollama / SGLang 的出箱 parity、PyTorch 2.7 ROCm build ~95% 功能對等——CUDA 在推論層的護城河第一次出現實質裂縫。

歷史類比（兩有用一誤導）。① x86 生態鎖定（有用）：Intel 靠生態鎖定統治數十年，但在「新平台」（手機 ARM）完全沒進場——對應 CUDA 在既有訓練工作流深鎖、在新推論部署較弱。② Android / JVM 抽象層（有用）：抽象層普及後底層硬體被商品化——對應 PyTorch 正把 GPU 變可替換後端。③ 「開源必勝 Linux 取代 Windows」（誤導）：ROCm 開源不保證取代 CUDA，生態慣性 + 高階核 + 物理互連讓「開源 = 必勝」是錯誤類比；要分層看。

歷史告訴我們：生態鎖定在「既有工作流」極深、在「新平台」較弱；抽象層一旦普及就商品化底層硬體。所以投資框架要從「CUDA 會不會被取代」換成「哪一層、被誰、何時」——這正是裁決分三層的歷史理由。

§2 技術成熟度 + S 曲線

為什麼是現在 ROCm 達 parity？因為兩件事到位：① 開源框架（PyTorch/vLLM）成熟到「硬體後端可插拔」；② AMD 把 ROCm 的相容性與文件補到「出箱可用」門檻，跨過了過去三年卡住的最後一哩。

技術因果鏈：過去 ROCm 的問題不是「不能跑」，而是「要手動編譯、踩坑、缺文件」——對企業是隱性成本。2026-03 的 ROCm 7.2 跨過的拐點是「出箱 parity」：vLLM / llama.cpp / Ollama / SGLang 這些主流推論引擎不用改 code、不用自訂 kernel 就能跑在 AMD GPU 上，PyTorch 2.7 的 ROCm build 達 ~95% 功能對等。這意味著「PyTorch + vLLM、無自訂 kernel」的推論工作負載，遷移成本已降到可接受。但若你依賴 TensorRT-LLM 或 FlashAttention 3（NVIDIA 的高階最佳化核），仍得留在 CUDA——這是 parity 的邊界，也是護城河仍存的證據。

§3 供給側：護城河三層，每層玩家不同

「CUDA 護城河」其實是三道厚度不同的牆疊起來；看清哪層由誰守、哪層誰能攻，才知道價值流向哪。

護城河三層的守方、攻方、牆厚與趨勢。

CUDA 護城河三層 · 守 / 攻 / 牆厚
層守方 (NVDA)攻方牆厚 / 趨勢
推論（單節點）CUDA / TensorRT-LLMAMD ROCm 7.2 / ASIC薄 · ↓ 裂縫擴大
高階 kernelFlashAttention 3 / TRT-LLMROCm 追趕中中 · → 仍 CUDA 優
訓練 scale-outNCCL + NVLinkRCCL / UALink（spec 2026Q2、2026H2 初步商用、大規模 2027）厚 · → 仍獨家

怎麼讀：牆厚由薄到厚＝推論 < 高階核 < 訓練 scale-out；NVDA 最該守的不是 CUDA 品牌，而是 NVLink+NCCL 物理互連。最危險的趨勢是抽象治理層——NVDA 根本守不住（那是框架擁有者的地盤），而 framework owner 正是 hyperscaler 自己。（QC-17：份額非機械推估，取技術社群實測 parity 狀態定性。）

🟡Inference · 供給信心：高
  • 前提：推論多為單節點（軟體 API 層）+ 訓練需數千卡（物理互連 NCCL+NVLink）+ framework 可抽象軟體 API、但抽象不了物理互連。
  • 推論：軟體 API 可被框架抽象、被 ROCm 複製（推論層已發生）；但 NVLink 是卡間物理高速互連、NCCL 是其上的通訊最佳化——數千卡訓練是 latency/頻寬硬約束，不是換個 API 就行。
→ 護城河最硬處不在「CUDA 軟體」，在「NVLink 物理互連 + 其上 NCCL」。⚠ 證偽：UALink 2.0 spec 2026 Q2 已出、硬體 2026 H2 初步商用；若 2027 hyperscaler 大規模採用、或 MSCCL++ 轉 production，訓練 scale-out 護城河鬆動——NVDA 最硬那層牆出現裂縫（監測點 2，2026 H2 起即盯）。

成本曲線（軟體護城河版）：軟體護城河沒有實體成本曲線，但有「遷移成本曲線」——遷離 CUDA 的成本由「自訂 kernel 多寡 + 多卡規模 + 框架依賴」決定。推論（PyTorch+vLLM、無自訂 kernel、單節點）遷移成本最低（已可遷），訓練（數千卡、依賴 NCCL/NVLink、可能用 TRT-LLM）遷移成本最高。因果閉合（回應 §1「生態慣性」變數）：CUDA 生態慣性在「既有訓練工作流」未來 3-5 年仍 binding（換框架 + 重寫 kernel + 重建多卡通訊的成本極高），但在「新建推論部署 + 自研矽」已鬆動——這是裂縫只在推論層、且由有自有框架的巨頭推動的根；只有當 framework 多後端推進到訓練、且 UALink 替代 NVLink，這條 binding 才會在訓練層放鬆。

為什麼物理互連比軟體 API 難複製（機制）。軟體 API（CUDA call）本質是「介面契約」——只要 ROCm 實作同樣契約、框架做一層 shim，就能替換，這是 ROCm 推論 parity 能成的原因。但 NVLink 是「卡與卡之間的實體高速通道 + 交換晶片（NVSwitch）」，NCCL 是其上把數千卡梯度同步壓到最低延遲的通訊演算法——要複製它，得同時做出等價的實體互連矽、交換拓撲、與多年調校的通訊庫，還要說服整個生態改用。這不是「實作一個 API」，是「重建一個物理 + 軟體的垂直棧」。UALink 之所以是唯一真威脅，正因為它是聯盟級的開放實體互連標準——衝著最難複製的那層去。這也解釋為何 NVDA 近年把重心從「CUDA 軟體」往「NVLink/NVSwitch 互連」傾斜：它知道最硬的護城河在物理層，不在軟體品牌。

12-36M 內供給格局：推論層價值往 AMD/ASIC + framework owner 流；訓練層價值 NVDA 暫守；長期看，抽象層誰控制、誰就能決定 GPU 是否可商品化。議價權仍最集中在守住最厚訓練層的 NVDA，但增量話語權正往抽象層滲透。

§4 需求側：誰想逃離 CUDA、為什麼逃得動

想逃離 CUDA 鎖定的，正是 NVDA 最大的客戶——hyperscaler；他們既有逃的動機（成本、議價權），又有逃的能力（自研矽 + 自有框架）。

現需：逃離 CUDA 的需求來自 NVDA 自己的大客戶。動機：CUDA 鎖定 = NVDA 的定價權 = hyperscaler 的成本與議價劣勢，他們每年付數百億給 NVDA，每省一分鎖定就多一分議價。能力：他們同時握有自研矽（TPU/Trainium/MTIA/Maia）與自有框架（GOOGL JAX、META PyTorch、AMZN Neuron），可從「晶片 + 編譯棧 + 框架」三端一起繞過 CUDA。這跟以往「小廠想挑戰 NVDA」最大的不同——這次想逃的是有資源、有框架治理權的巨頭。

誰想逃、用什麼工具逃、逃得動哪一層。

逃離 CUDA 的需求三角驗證
玩家逃離工具逃得動的層
GOOGLTPU + JAX/Pallas + ICI 互連訓練 + 推論（自有棧最完整）
AMZNTrainium + Neuron SDK推論為主，訓練追趕

怎麼讀：逃得最徹底的是 GOOGL（TPU + JAX 自有訓練棧 + ICI 自有互連，frontier 訓練都能繞 NVDA）；META 的關鍵武器是「擁有 PyTorch」——它不只自己逃，還握有讓所有人都能逃的抽象方向盤。逃得動的層目前集中推論；訓練 scale-out 除 GOOGL TPU（自有 ICI）外，多數仍回 NVLink。三角對帳：動機（成本/議價）× 能力（自研矽+框架）兩邊都成立的，只有 hyperscaler 巨頭——這就是「逃離 CUDA」這次玩真的、且集中在巨頭的根。

因果閉合：CUDA 的生態慣性在「既有訓練工作流」仍 binding（換框架成本高），但在「新建推論部署 + 自研矽」場景已鬆動——這是裂縫只在推論層、且由有自有框架的巨頭推動的結構根。

多空為何都對：NVDA 多頭測「訓練 scale-out + 既有工作流慣性」（真的還很硬），空頭測「推論層 parity + 抽象層商品化」（也真的在發生）——測的是不同層，可同時為真。把它讀成「CUDA 整體崩 / 整體穩」都會誤判。

二階效應——CUDA 護城河蝕的外溢。當推論層鎖定鬆動，外溢出三條二階：① GPU 毛利——遷移成本降＝客戶議價權升＝NVDA 推論 GPU 的定價權與毛利長期承壓（即使量仍增）；② 開放生態——ROCm / Triton / UALink 等開放標準受惠，利於「非 NVDA」整條供應鏈（AMD、ASIC、開放互連聯盟）；③ hyperscaler 資本效率——能繞過 CUDA 鎖定的買家（GOOGL/AMZN）每單位算力 TCO 下降，這回饋到 AI 推論經濟學的「垂直整合者毛利領先」——軟體脫鉤是垂直整合成本優勢的一塊拼圖。三條二階同向：護城河蝕的價值，從 NVDA 的軟體溢價，流向買方與開放生態。

哪塊需求最不可替代：frontier 訓練（需 NVLink scale-out + 整棧最佳化）。逃離 CUDA 的力道由「自有框架 + 自研矽」的垂直整合度決定，GOOGL 最深、META 握抽象方向盤、AMD 客戶限推論。

§5 供需裁決 + 短中長期推估 + 投資時鐘

⚠️ 本章含 judgment：🟡 bullet 附事實鏈 + 信心度 + ⚠ 證偽條件。
綜合 §3（護城河三層）與 §4（誰逃得動），裁決是「護城河分層」：推論蝕、訓練守、抽象權上移；不是二元崩塌。

資本週期/競爭證據（裁決依據，引三項）：① 遷移成本趨勢——ROCm 7.2 把推論遷移成本降到可接受（出箱 parity）；② 自研矽 production 滲透——GOOGL TPU、AMZN Trainium 已 production scale，繞過 CUDA 的實證在累積；③ framework 多後端化——PyTorch 2.7 ROCm ~95%、JAX 多後端，抽象層正把硬體商品化。三項合起來：護城河在推論層與抽象層正被結構性侵蝕，但在訓練 scale-out 層（物理互連）仍 binding。

§3 + §4 → 護城河裁決

依本章 §3 與 §4 的推導，CUDA 護城河未來 X 年分層裂解：推論層「已裂」（ROCm 出箱 parity）、訓練 scale-out 層「仍守」（NCCL+NVLink 獨家；UALink 硬體 2026 H2 已初步商用，大規模採用 2027 才是實質威脅）、抽象治理層「權力上移」（framework owner 接管硬體可替換性的決定權）。不騎牆：三層各有明確裁決，整體是「分層」而非「整體崩 / 整體穩」。

§6 產業經濟學與估值傳導

本主題不是 TAM 題，是「NVDA 軟體鎖定溢價值多少」題；估值跟著「護城河分層被市場認知到什麼程度」動。

護城河各層對 NVDA 估值溢價的貢獻。

NVDA 軟體溢價的分層拆解
層對溢價貢獻趨勢該如何定價
訓練 scale-out（NVLink/NCCL）高守溢價保留
高階核（TRT-LLM 等）中緩蝕溢價漸折
推論層（CUDA）中已裂溢價該折讓
抽象治理（守不住）移向 framework不該給 NVDA 溢價

怎麼讀：市場給 NVDA 的軟體溢價該「分層定價」——訓練層保留、推論+抽象折讓。把「CUDA 整體不可取代」當前提，是高估了溢價的持久性。

估值傳導：市場目前給 NVDA 的估值隱含「CUDA 鎖定難以撼動」這個整體前提。護城河分層意味這前提錯了一半——推論層與抽象層的溢價該被逐步 price out。最大的錯誤定價是把「CUDA 整體不可取代」當錨；一旦市場開始 price「推論已商品化 + 抽象權移向 framework owner」，NVDA 的軟體溢價會 de-rate（不是崩，是從「不可取代」回到「最強但可競爭」）。敏感度錨點：若市場認知「訓練護城河也被 UALink 威脅」，NVDA 的溢價折讓幅度會遠大於推論層的折讓——因為訓練層是溢價的主要來源。

估值現在 price 了什麼：CUDA 整體鎖定（錯，該分層）。哪個變數動會 de-rate：ROCm 達訓練 parity、或 UALink 大規模商用。framework owner（META/GOOGL）的抽象治理價值尚未被 price，是未被認知的 optionality。

§7 Non-Consensus + Priced-in 檢驗 + Kill Scenario

⚠️ 本章含 judgment：每條分歧附 [F:] cornerstone + [X:] 三情境 + priced-in。
市場主流框「CUDA 不可取代」或「ROCm 開源必勝」；我們的三條分歧在「護城河分層」「最硬處是物理互連非軟體」「framework owner 才是終局贏家」。
🟡分歧 1：護城河分層——「CUDA 不可取代 / ROCm 必勝」都錯信心：高
  • 共識 X：兩派——「CUDA 整體不可撼動」（NVDA 多頭）vs「ROCm 開源終將取代」（AMD 多頭）🟢 [F: T3 兩種敘事並存]
  • 我們 Y：護城河分層——推論已裂（ROCm 7.2 出箱 parity）、訓練 scale-out 仍守（NVLink）🔵 [X: base 很可能]
  • Priced-in：NVDA 估值 price「整體不可取代」（高估）；市場未分層定價——推論層溢價該折但還沒折。分歧「對且未 priced」。
→ ⚠ 證偽：若 ROCm 連訓練 + 高階核都達 parity（整體崩），或推論層 ROCm parity 倒退（整體穩），分層論失效。
🟡分歧 2：最硬處是 NVLink 物理互連，不是 CUDA 軟體信心：中高
  • 共識 X：市場把 NVDA 護城河等同「CUDA 軟體生態」🟢 [F: T3]
  • 我們 Y：軟體可被抽象/複製（ROCm 已證），真正不可取代的是 NVLink+NCCL 的物理互連 scale-out 🔵 [X: base 很可能]
  • Priced-in：市場把溢價歸給「CUDA」，未區分「可被抽象的軟體」vs「難複製的物理互連」；故 UALink 的威脅被低估（它打的正是最硬那層）。
→ ⚠ 證偽：若 UALink 商用且採用快，最硬那層也鬆——NVDA 溢價大幅重定價。
🟡分歧 3：2027-28 真贏家是 framework owner 不是 GPU 廠信心：中
  • 共識 X：市場聚焦「NVDA vs AMD」誰贏 GPU 🟢 [F: T3]
  • 我們 Y：當 PyTorch/JAX/Triton 把硬體抽象成可替換後端，決定權移到 framework owner（META/GOOGL/OpenAI）🔵 [X: base 很可能]
  • Priced-in：market 完全沒 price「擁有 PyTorch」對 META 的長期戰略價值，這是最未被認知的 optionality。
→ ⚠ 證偽：若 framework 多後端始終無法用於最前沿（frontier 訓練仍綁 CUDA），抽象治理權的價值打折。

Steel-man 反方（最有力的 3 條）

  • 反方 1：整棧最佳化讓 CUDA 護城河再延伸。路徑：frontier 模型越複雜，CUDA + TRT-LLM + NVLink 的整棧協同優勢越大，抽象層在最前沿打折。證據：最前沿訓練仍幾乎全在 NVDA。證偽窗口：frontier 訓練出現顯著非 NVDA 部署。
  • 反方 2：ROCm parity 卡在「夠用但不夠好」。路徑：ROCm 推論 parity 但性能/穩定性略遜，企業為可靠性仍付 CUDA 溢價。證據：TRT-LLM/FlashAttn3 仍 CUDA-only。證偽窗口：ROCm 性能在生產環境持平 CUDA。
  • 反方 3：UALink 採用難起量。路徑：2.0 spec 雖 2026 Q2 已出、硬體 2026 H2 已初步商用，但開放互連聯盟生態協調慢、NVLink 持續領先世代，hyperscaler 大規模採用遲遲不來，scale-out 護城河延續。證據：初步商用 ≠ 大規模採用，frontier 訓練仍綁 NVLink。證偽窗口：UALink hyperscaler 大規模採用率顯著上升。

市場最低估的風險：分歧 2——市場把溢價歸給「CUDA 軟體」，沒區分「可抽象的軟體」與「難複製的物理互連」，因此低估了 UALink（打的正是最硬那層）的潛在衝擊。

三條分歧裡，分歧 1（護城河分層）「對且未 priced」最可操作；分歧 2（物理互連最硬）是估值框架的關鍵校正；分歧 3（framework owner 終局）是 META 最未被 price 的 optionality。
