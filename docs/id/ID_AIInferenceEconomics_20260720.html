Key Points

  • Token 單價自 GPT-3 崩約 98%（$20→~$0.40 / M tokens），但 token 總支出近一年反而翻倍（Silicon Data 指數）——Structural Jevons Paradox（Zhang & Zhang 2026-01 形式化）成立：智能單價暴跌時，下游廠商不是省錢，而是重新設計 agent 架構去吃更多算力——更深推理迴圈、更長 context、多 agent 工作流。
  • 企業 AI 預算裡推論成本占比從 2023 的 20% 衝到約 85%、推論已占算力支出約 2/3——AI 的成本重心已從訓練移到推論。
  • 真正的斷層不是「便宜」，而是毛利分層的分裂：commodity 推論趨零毛利（純 GPU 雲轉售與無差異 LLM API 是輸家）；frontier 推論不在同一條成本曲線上、仍能收高價；custom ASIC 垂直整合與 routing/orchestration 中介層毛利守得住。
  • 盯「推論變不變便宜」的人問錯問題；該問的是「不斷上升的 token 支出最後落在哪一層、那一層有沒有毛利」。
  • Anthropic 衝到 $30B 營收、毛利含 inference 卻只 ~48%（inference 成本比預期高 23%、2026 估虧 $14B）——「量爆發 ≠ 毛利」的活證據。押毛利守得住的層，避開被 Jevons 海嘯沖到零毛利的層。
  • 中國 open-weight 模型（DeepSeek V4／Qwen／Kimi K2.6／GLM-5.2）以 frontier 約 1/7 至 1/34 的 output 單價逼近能力前緣（2026-05 佔 OpenRouter token ~61%）——把 commodity 層地板再砸低一階、層界上移，強化而非推翻毛利分層判斷；閉源能力領先 2026-03 反擴回 3.3%（Stanford），frontier 溢價層變薄但未消失。最新一例是 2026-07-16 發布的 Kimi K3（2.8T open-weight MoE）：稀疏路由把邊際成本壓得更低，但官方牌價反較上一代漲約 3 倍——commodity 層地板持續下修不等於售價只跌不漲，serving 硬體與記憶體門檻是另一條反向約束（詳 Debate 4 與 §3.3，資料時點 2026-06／07）。

Section 1 · Investment Thesis

Investment Thesis

Key Call · 本報告最重要的一個判斷

「推論變便宜」被市場讀成通縮利空，這個直覺錯了。Structural Jevons Paradox 反轉它：單價每跌一階，廠商把省下的算力重新投進更貪婪的 agent 架構（agentic workload 每個 task 吃 5–30x 於 chatbot 的 token），所以總支出隨單價下跌而上升。真正的斷層不是價格，是毛利落在誰手上：同一波 token 海嘯，commodity 推論被沖到零毛利（GPU 雲轉售、無差異 API），而 frontier 推論、垂直整合的 custom ASIC、與 routing 中介層守住毛利。盯「便不便宜」的人問錯問題；該問「上升的支出落在哪一層、那層有沒有毛利」。

毛利三層具體是：① 輸家層——純 GPU 雲轉售（neocloud）與無差異 LLM API，commodity inference 成本趨零、被價格戰打到地板；② 中性收費層——routing/orchestration（在便宜與貴的模型之間做智能路由、抽中介費，token 通縮反而擴大其價值）；③ 贏家層——frontier 推論（最新推理模型不在零成本曲線上）+ 垂直整合 custom ASIC（GOOGL TPU、hyperscaler 自研把 inference 成本內部最佳化）。同樣賣「推論」，這三層的毛利命運完全相反。

供需裁決：未來 12 個月推論需求爆發（Jevons + agentic）；但供需的真議題不是「夠不夠」，而是毛利三層分裂——commodity 推論「過剩 / 零毛利」、frontier + 垂直整合 ASIC「稀缺 / 守毛利」。不騎牆：需求明確爆發、毛利明確分層，整體裁決為「需求爆發但毛利分裂」。

投資時鐘：當前 Phase II（推論放量 + 毛利分化）。轉下階段必要條件＝token 支出見頂或 frontier 毛利明確分歧；充分條件＝兩者同時。領先指標：token 支出指數 + agentic 滲透 + AI lab inference 毛利（推論無傳統庫存，用這三個當 leading indicator）。

Debate 4中國 open-weight 模型會不會摧毀西方推論毛利？信心：中
市場認為
兩極對立：一派把中國 open-weight 讀成西方推論毛利的終結者（同工作負載便宜 60–90%、OpenRouter token 佔 ~61%）；另一派因合規與地緣顧慮認定「西方企業不會用、無影響」🟢 [F: T3 兩派敘事並存，2026-06／07]。
我們認為
都不對。中國 open-weight（DeepSeek V4／Qwen／Kimi K2.6／GLM-5.2）是毛利分層論證的加速器，不是推翻者：它把 commodity 層的價格地板再砸低一階（同級 output 價差 ~7–34×，2026-06 查核），並把層界上移——過去屬 frontier-adjacent 的 coding／agentic 中段工作負載被拉進 commodity 層；但 frontier 溢價層未消失（Stanford 統計閉源對開源能力領先 2026-03 反擴回 3.3%，2024-08 僅 0.5%），垂直整合 ASIC 層與此衝擊正交，routing 層因模型選擇爆炸更受益。邊際新輸家＝以 open-weight 轉售為業的第三方 serving 商（Fireworks GM ~50%、同模型跨供應商價差 6×）——此判斷細分 §3.2 三層表的粗顆粒：「routing ↑」須拆為抽象／路由層（↑）與純 open-weight serving（承壓入 commodity 延伸）。中國自身算力約束（Ascend 產能／HBM、H20 管制）使衝擊主要走「權重開源 → 西方第三方 serving」通道，而非中國雲 hosted API 直接搶單 🔵 [X: base 很可能]。2026-07-16 發布的 Kimi K3 是最新一例：稀疏路由（896 個 expert、每 token 僅啟動 16 個）把邊際成本壓得更低，但 serving 需要的記憶體門檻（同廠上一代 K2.x 級 1T 參數權重近 1TB）不降反升、且 K3 官方牌價較上一代 K2.6 漲約 3 倍——「commodity 層地板持續下修」與「第三方 serving 商被擠壓」兩件事同時成立，互不矛盾（詳 §3.3）。
判別訊號
① OpenRouter 中國 open-weight token 份額（2026-05 ~61%）：續升＝commodity 層界續上移。② frontier 對同級 open-weight 定價倍數（現 ~7–34×）：OpenAI 若實施傳聞中的大幅降價（2026-06 報導）＝frontier 溢價層壓縮確認。③ 閉源-開源能力差（0.5% → 3.3%）：重新收斂至 <1%＝溢價層告急。④ 中國算力約束（Ascend 良率／HBM）：明顯鬆動＝hosted 通道打開、衝擊升級。⑤ 記憶體門檻（DRAM 合約價、SK hynix 短缺展望）：現況＝2026Q1 DRAM 合約價 QoQ 近倍增（TrendForce）、SK hynix 警告短缺恐延續至 2030——持續緊縮＝serving 硬體門檻維持高檔，強化「資本不足的第三方 serving 商最先出局」；明顯鬆動＝serving 門檻降、commodity 層競爭加劇。

市場最低估的風險：Debate 2——市場把推論鏈當同一多頭，低估「commodity 轉售零毛利」這層的下行；這層往往被「AI 成長」敘事掩蓋。

對投資的意義 · Investment Implication四條分歧裡，Debate 2（毛利三層分裂）「對且未 priced」最可操作；Debate 1 部分已 priced；Debate 3 是 frontier lab（私募）的最大未定價風險；Debate 4（中國 open-weight）強化 Debate 2——commodity 層的下行比市場認知更深，且新增第三方 open-weight serving 商為邊際輸家。

Section 3 · Industry Mechanics

產業機制與供需

先看需求引擎（Jevons × agentic 乘數），再看供給三層的成本結構與中國 open-weight 新變數，最後收斂到供需裁決與三情境。每個論點配一張 Exhibit。

3.1 需求引擎：Jevons 悖論與 agentic 乘數

現需：關鍵不是用戶數，而是每 task 的 token 消耗。單價跌 → 開發者採更深推理迴圈、更長 context、多 agent → 每 task token 暴增（agentic 是 chatbot 的 5–30x）。企業 GenAI 支出兩年成長 22x，同期 per-token 價格跌 >90%——量爆發遠超價下跌。這是「單價崩 98% 但總支出翻倍」的需求引擎。

滲透證據（西方端）：OpenRouter 上中國 open-weight 模型 2026-05 佔全平台 token ~61%（DeepSeek 17.6%、Qwen 13.9%）；美系三家（Google／OpenAI／Anthropic）合計份額一年內自 ~70% 跌至 ~30%（2025-06 → 2026-06）。美國企業端：中國模型佔 enterprise token 份額自 2026-02-08 起每週 ≥30%、峰值 46%（此前 12 個月均值僅 11%，CNBC 2026-07-07）。個案：AI agent 新創 Lindy 於 2026-06 把 agent 流量自 Anthropic 移往 DeepSeek V4；Airbnb 與 Cursor（Anysphere）因揭露使用 Qwen／Kimi 遭美國國會調查——合規顧慮真實存在，主流繞道是經 AWS／Azure／Vertex 的 managed 上架或自託管權重、不直連中國雲。生態端：Qwen 在 Hugging Face 累計下載破 10 億、超越 Llama 成最大開源模型系；工作負載結構上 programming 佔 OpenRouter 用量從 2025 初 ~11% 升至 2026 年中 >50%——中國模型恰好在 coding 上最強、最便宜，滲透與工作負載遷移互相放大。

對三層毛利結構的含義（細分，非推翻）：開源權重讓「模型本身」不再是第三方 serving 商的差異化來源——Together／Fireworks／Groq 等以 open-weight 轉售為業者，同一模型跨供應商價差達 6×（Q2 2026 定價矩陣）、Fireworks 毛利 ~50%（低於軟體業 70%+，Sacra；且 margin 是這個類別「沒人揭露的指標」，Forbes 2026-07-18）——它們的經濟本質是本報告 commodity 層的延伸，不屬 routing 層。§3.2 三層表的「routing ↑」因此須拆兩半：抽象／路由層（在爆炸的模型選擇間做智能路由）中介價值隨中國模型湧入而上升；純 open-weight serving 則被同模型比價壓進 commodity 命運。垂直整合 ASIC 層（GOOGL TPU 等自用自算）與此衝擊正交——它的競爭軸是成本結構、不是模型定價。

中國自身的算力枷鎖：DeepSeek 在 2025-12 的 V3.2 技術論文自述最大約束是算力取得（CSIS）。出口管制下 NVIDIA 於 2026-03 停產中國向 H200；國產替代 Ascend 910C 推論效能約 H100 的六成、先進封裝後良率 ~75%、HBM 為產能瓶頸（存料估僅可產約百萬顆，IFP／SemiAnalysis）；ByteDance 計畫 2026 年斥資 $5.6B 採購華為晶片，DeepSeek V4 已為 Ascend 最佳化並在其上跑推論。含義：中國模型的全球衝擊主要走「權重開源 → 西方第三方 serving」通道，而非中國雲 hosted API 直接搶單——算力約束限制其自營服務的擴張半徑，也因此受壓最重的是西方 open-weight serving 層，而非 hyperscaler 或垂直整合層。

Kimi K3：稀疏化把邊際成本壓得更低（資料時點 2026-07）。2026-07-16，月之暗面（Moonshot AI）發布 Kimi K3——2.8T 總參數 open-weight MoE，896 個 expert 中每 token 僅啟動 16 個（約 1.8% 稀疏度），導入 KDA（Kimi Delta Attention）混合線性注意力＋Gated MLA 壓縮 KV cache，並用 IndexShare（每 4 層共用一個輕量稀疏索引器）在 1M-token context 下把單 token FLOPs 再砍約 2.9 倍、解碼速度加快至 6.3 倍（Moonshot 官方 kimi.com/blog/kimi-k3；MarkTechPost，2026-07-16）；完整權重預定 2026-07-27 開源。這與前段單 token 崩 98% 的量化／蒸餾／推測解碼／KV-cache 重用四優化同一因果根——稀疏 MoE 路由與線性注意力是最新一層疊加優化。每一代新 open-weight 模型都把 commodity 層地板再往下砸一階，這強化本節「層界上移」的判斷，是同一機制的最新樣本，不是新機制。

但 serving 門檻由總參數量決定，不隨稀疏度下降。稀疏路由壓的是「跑一個 token 要算多少」，壓不動「把模型擺上線要多少記憶體」——同廠上一代 K2.x（1T 參數）權重達 959GB、閒置需求約 1,347GB vRAM，官方建議原生精度用 8 張 B200 或 16 張 H100 部署（Lambda 部署指南）；以 INT4 量化概算仍約 500GB。B200 用 NVFP4 serving K2.5/K2.6 已可壓到 $0.14/M tokens（SemiAnalysis InferenceX），但 Epoch AI 估算 GPT 系列仍帶約 50% 推論毛利、Kimi 系列則接近成本價 serving（推測性估計）——中西定價的價差有一部分是毛利結構差，不是純效率差。更關鍵的反向訊號：Kimi K3 官方牌價（$3／$15 每 M tokens，cache-miss 檔）較上一代 K2.6（$0.95／$4.00）漲約 3 倍——這是「無限通縮」敘事的第一個逆風數據點：邊際成本持續壓低不代表售價只跌不漲。同期記憶體端全面吃緊：DRAM 合約價 2026 年第一季單季 QoQ 上漲逼近 90-95%（TrendForce，較先前預估大幅上修），SK hynix 直言記憶體短缺恐延續到 2030 年。稀疏度持續下降、但記憶體與 supernode 部署門檻不降反升，這個落差進一步加固前段「邊際新輸家＝第三方 open-weight serving 商」的判斷——資本不足以吃下巨大固定記憶體門檻的玩家，在近成本定價的擠壓下最先出局。

考證與來源 · 死亡螺旋機制 / 單 token 四優化 / 二階效應 / Bear 情境敘事

commodity 層的「零毛利死亡螺旋」機制。為什麼純 GPU 雲轉售（neocloud）毛利被打到地板？三力同時作用：① 無模型差異——跑開源 / 通用模型、客戶無 lock-in、純比價；② spot GPU 定價——GPU 供給漸寬鬆、現貨租價下行，轉售毛利被夾在「固定的長約 GPU 成本」與「下行的現貨售價」之間；③ 利用率死亡螺旋——利用率一旦跌破損益平衡（neocloud 約 80%），固定成本攤不掉、毛利由正轉負。三者疊加，commodity 推論不是「毛利薄」而是「結構性趨零甚至為負」。這與 frontier 層（能力溢價、客戶為最新模型付高價）和垂直整合層（自研 ASIC + 自有模型 + 高利用率）形成鏡像對比——同樣賣推論，一個跌進死亡螺旋、一個收結構性溢價，這正是「毛利分層」不是程度差異而是命運分歧的微觀根。

單 token 為何能崩 98%——四個疊乘的優化。① 量化：權重從 FP16 降到 FP8 / FP4 / INT4，記憶體與算力需求腰斬再腰斬；② 蒸餾：用大模型訓出能力接近、體積小數倍的小模型跑推論；③ 推測解碼（speculative decoding）：小模型先猜、大模型驗證，吞吐量倍增；④ KV-cache 重用 + 連續批次（continuous batching）：把多請求的共用前綴與閒置算力榨乾。四者疊乘，同能力單 token 成本三年崩約 98%。關鍵 insight：這些優化降的是「單位成本」，但每次降價都被 agentic 架構用「跑更多」吃回去——優化越成功，Jevons 引擎轉得越快，這是「技術進步反而推升總支出」的反直覺核心。

二階效應——推論通縮外溢到軟體商業模式與宏觀。token 單價崩跌的衝擊不止 AI 基建，它沿「智能成本」往外重塑三件事：① SaaS 定價模式——當每次 AI 動作成本逼近零，軟體從「按席次（per-seat）」轉向「按用量（usage-based）」收費，定價權往「能把 AI 用量轉成可計費結果」的應用層移動（routing/orchestration 與垂直 SaaS 的機會）；② 知識工作邊際成本——便宜推論把過去「人做才划算」的任務（摘要、翻譯、初稿、客服）邊際成本壓近零，需求量爆發（Jevons 的宏觀版）；③ 能源——agentic 乘數讓 token 量暴增，即使單 token 更省電，總推論電力需求仍上升，把壓力傳導到資料中心電力（見姊妹母題 AI 資料中心）。這三條二階說明：推論通縮的真正受益者，是「能把便宜智能轉成可計費價值」的層，不是賣原始 token 的層——這也呼應 KEY CALL 的毛利分層判斷。

Bear 情境長什麼樣（情境敘事化）。若 agentic 乘數見頂 + 變現失速同時發生，劇本會這樣走：先是某季企業 SaaS 財報揭露「AI 功能採用率高但付費轉換低」，CFO 開始把 AI 從「戰略投入」重新歸類為「要當期 ROI 的營運支出」；接著 agentic 應用的 token 乘數停止上升（開發者不再加深 agent 迴圈，因為更深的推理沒帶來可計費的價值）；token 支出指數先走平、再下滑；frontier lab 的 inference 毛利卡在 ~48% 上不去、融資環境轉冷、估值重定價；commodity 推論層因需求放緩 + 產能過剩進入更慘烈的價格戰死亡螺旋；最後連垂直整合者也因自用需求放緩而利用率下降。這個世界裡唯一相對抗跌的，是「已把 inference 成本內部最佳化、且有自有現金流補貼」的少數垂直整合者（GOOGL）——其餘各層按毛利脆弱度依序見血（commodity 轉售先倒、無差異 API 次之、frontier lab 因高 burn 受傷）。要提前 2-3 季捕捉這個情境，盯的領先訊號是 token 支出指數轉折與 SaaS 財報的「AI 付費轉換率」。

考證與來源 · 錯誤定價機制全文

為什麼「把推論鏈當同一多頭」是危險的錯誤定價。市場給 AI 標的估值時常用「AI 營收成長率」當單一錨，把跑 inference 的公司一律給高倍數——但這忽略了毛利結構的天壤之別。一家純 GPU 雲轉售商即使營收 +100%，若毛利趨零甚至為負，其內在價值可能是負的（燒錢買成長）；而一家垂直整合者營收 +30%、但 inference 毛利結構性領先，內在價值遠高。當市場某天「認清」這個分層（通常由一兩家 commodity 轉售商爆出虧損或下修觸發），會出現劇烈的分化重定價：高營收低毛利者 de-rate、低調高毛利者 re-rate。這正是本報告把判斷重心放在「毛利落在哪層」而非「誰營收成長快」的估值理由，也是 Debate 2「市場把推論鏈當同一多頭」被列為最可操作分歧的根據。

附錄 A · 產業白話定義與歷史脈絡

推論（inference）＝模型訓練好後，每次回答問題、生成內容時跑的運算。訓練是一次性資本支出，推論是隨用量重複發生的營運成本——AI 真正的「電費」。誰付錢？所有用 AI 的企業與消費者，透過 API 或自建。

白話開場：一個 AI 應用的成本分兩塊——訓練（造模型，一次性）與推論（用模型，持續）。早期市場焦點全在訓練（誰的模型大、誰買最多 GPU），但 2026 推論已占算力支出約 2/3、占企業 AI 預算約 85%。推論經濟學問的是：當每個 token（模型處理的最小文字單位）越來越便宜，這門生意的錢從哪賺、毛利在哪。

邊界界定：本報告聚焦「推論的成本 / 毛利結構與價值流向」，含算力（GPU / ASIC 推論）、模型層（frontier vs commodity）、服務層（API / routing）。capex「會不會過剩」見姊妹母題 AI 算力資本週期；GPU vs ASIC 競爭見 AI 加速器需求；軟體生態見 CUDA / ROCm。灰色地帶是 routing/orchestration——它不產生 token 但決定 token 流向哪個模型，計入因為它是通縮下的中介收費層。

第一次轉折：2022-11 ChatGPT 引爆，推論需求從零到爆；第二次：2023-2025 token 單價隨模型效率與競爭崩跌（同能力 −~98%，$20→~$0.40/M）；第三次：2026 推論占 AI 預算衝到 ~85%、agentic workload 普及（每 task 5–30x token），「token 便宜但總帳單上升」成為市場顯學（Fortune 2026-06）。

附錄 B · 歷史類比：頻寬通縮與雲端商品化

歷史類比（兩有用一誤導）。① 頻寬通縮（有用）：每 GB 成本跌數量級但總支出未崩，贏家是差異化服務非純轉售。② 雲端 IaaS 商品化（有用）：基礎運算商品化後價值往編排層上移——對應 routing 層。③ 「便宜 = 利空」（誤導）：把單價下跌直接當需求 / 營收利空，忽略 Jevons 的量放大，方向就錯。

怎麼讀：三個先例都是「單位成本跌、總支出不跌反升、贏家在差異化 / 編排層」。本次差異是通縮更快、需求彈性更高（agentic 主動重設計），所以規律更強烈——但「純轉售被打成 commodity」一樣適用。

編年對照：頻寬通縮的十年 vs AI 推論的三年。2000-2010 的頻寬故事逐年推進——2000 dot-com 崩盤後頻寬看似「過剩」（暗光纖點亮率僅個位數），2003-2007 寬頻普及讓需求逐步 backfill，2007 iPhone + 2009 雲端 + 2010 串流接力把「過剩的暗光纖」一段段點亮，總頻寬支出二十年穩步上升、單價穩步下降，贏家從「賣原始頻寬的電信商」轉到「建在便宜頻寬上的 Netflix / CDN」。AI 推論把這個十年壓縮成三年：2022 ChatGPT、2023-2024 單價崩、2025-2026 agentic 接力——時序快了一個數量級，但「單價降、總支出升、贏家在差異化層」的結構完全同構。差別是 AI 的需求彈性更高（agentic 是主動重設計去吃算力，不像頻寬被動等新應用出現），所以總支出上升的力道更猛、也更早顯現；對應的風險也更集中——純轉售層被打成 commodity 的速度，比當年電信商更快。

雲端 IaaS 商品化——routing 層的歷史對應。當基礎運算（EC2 等）商品化後，價值往「管理 / 編排層」上移（Kubernetes 生態、雲端管理 SaaS）。AI 推論的對應是 routing/orchestration 層——在一堆便宜與貴的模型之間做智能路由、快取、批次化、降級，token 越通縮、選擇越多，這層的「中介價值」越大。這是為什麼 Cloudflare 式的邊緣推論 + routing 是 token 通縮下的隱性受益者，而非受害者。

歷史的投資含義：通縮不殺需求（反而放大），但會把純轉售層打成零毛利。所以投資框架要從「推論便不便宜」換成「token 支出落在哪層、那層毛利如何」。
