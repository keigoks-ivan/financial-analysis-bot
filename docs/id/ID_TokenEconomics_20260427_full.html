§1 產業白話定義 + 歷史脈絡

白話版：每次你問 AI 一個問題，背後就消耗一批「token」，廠商按 token 收費。token 越來越便宜（像當年網路頻寬越來越便宜），但這不代表這門生意縮水——便宜讓大家用得更兇，總支出反而暴增。問題是：這筆暴增的支出，被產業鏈的哪一層賺走。

白話開場：token 是 LLM 處理文字的計費單位（約等於 0.75 個英文字）。推論（inference）是 model 回答你的問題（相對於「訓練」是事先學）。一次 agent 任務可能呼叫 model 數十次、吃掉大量 token。廠商按「每百萬 token」收費，而這個單價正在崩——但崩價的背後是 Jevons 悖論：越便宜，用越多，總支出越大。本報告問的是這門生意的經濟學：單價趨零下，錢在矽、雲、應用之間怎麼分。

邊界界定：本報告聚焦「推論 token 的價值捕獲 + 算力供給瓶頸」。納入矽（GPU/ASIC/LPU）、自有矽垂直整合者、hyperscaler、推論基礎設施（routing/cache/edge/neocloud）、端側推論的上市玩家。不展開 LLM model 本身的優劣（那是 AI 模型 ID）、也不深入訓練資本週期（見「AI 算力資本週期」）。灰色地帶是「訓練 vs 推論」算力——本報告以推論為主軸，但兩者共用先進封裝/電力瓶頸。

歷史分三段。第一段：2023 生成式 AI 爆發，token 昂貴——早期 GPT-4 每百萬 token in/out 約 $25–60，推論是成本中心、用得省。第二段：2024–2025 效率競賽——模型蒸餾、量化、推論優化 + 多模型競爭把單價往下打，「10x/年降幅」成業界常識；同時 agentic 化讓單一任務吃更多 token。第三段：2026 量補價成型——每百萬 token 跌到 ~$1.75–$5（旗艦）、~$0.3 起（budget），3 年累計崩 ~99%，但 Silicon Data 代幣支出指數自 2025 末翻倍；推論占算力 2/3–90%、capex 衝到 $765B。單價在跌、總支出在飆。

歷史類比一：頻寬崩價 → 串流巨頭（量補價）。2000 年代頻寬單位成本崩約 99%，但流量增千倍，贏家是握流量的平台（YouTube/Netflix）非賣裸頻寬的電信。量化錨：這正是「單價趨零、總支出爆發」的先例——token 降價靠 Jevons 把餅做大，賺到的是握算力/應用的人。歷史類比二：雲端降價 → 垂直整合者贏。AWS 上市以來降價 100+ 次，但營收/利潤暴增——降價刺激用量 + 規模經濟 + 自研晶片（Graviton/Inferentia）把毛利留內部。量化差異：GOOGL（TPU）、AMZN（Inferentia）、META（MTIA）正複製這條路，自有矽讓它們在 token 降價戰中既跟得了價、又留得住毛利；外購 GPU 的純 LLM 廠把一塊付給 NVDA。

本主題的「價格」是 token 單價（年年崩），用「單價 vs 用量」當時序錨。

關於 cycle 統計表的省略：token 單價單向下行、無 peak-to-trough 循環（是效率驅動的結構性下降，非供需週期），故不立傳統 ASP cycle 表；以「單價三段 + 用量爆發」的編年呈現時序。算力（GPU）本身有資本週期，見「AI 算力資本週期」ID。

這段歷史的可操作含義：token 降價不是末日、是量補價的常態（頻寬/雲端都走過），贏家是握「算力 + 自有矽 + 應用」的人。bull 的底很實（Jevons + capex $765B），bear 不是「token 沒人用」（用爆了），而是「賣裸 token 賺不到、SaaS 毛利被壓、外購矽者漏給 NVDA」。

§2 技術成熟度 + S 曲線

為什麼 token 一直降價？因為三個技術同時推進：模型效率（蒸餾/量化/MoE）、推論硬體（GPU→ASIC→LPU）、與推論優化（caching/batching/推測解碼）——三者把單位 token 成本年年砍，但也讓用量門檻降到人人能用。

token 降價的技術引擎有三。模型效率：蒸餾、量化、混合專家（MoE）、小模型路由讓「同樣任務用更少算力」。推論硬體：從通用 GPU 走向客製 ASIC（GOOGL TPU、AMZN Inferentia、AVGO 設計）與 LPU（Groq），單位 token 能耗/成本持續下降。推論優化：KV cache、prompt caching、continuous batching、推測解碼（speculative decoding）把吞吐拉高、單價壓低。三者交會，token 單價 3 年崩 99%；但這也讓 AI 從「貴到只能精用」變成「便宜到處處可用」——這是 Jevons 的技術基礎。

§3 供給側：現在與未來 + 利潤池

token 價值鏈的利潤往「握自有矽 + 收推論租金 + outcome 應用」集中——賣裸 token 的中間商被商品化，端側則把成本搬離雲端。

供給端五層：①矽層（NVDA GPU + Groq LPU、AVGO 客製 ASIC、AMD 挑戰者——收推論租金）；②自有矽垂直整合（GOOGL TPU、AMZN Inferentia/Trainium、META MTIA——成本最低、毛利不漏）；③hyperscaler 分發（MSFT/AMZN/GOOGL——賣推論服務 + 經銷 LLM）；④推論基礎設施（NET routing/caching/edge、CRWV neocloud——降 token 成本的中間層）；⑤端側（AAPL/QCOM——把推論搬到裝置）。利潤往「自有矽 + 收租算力 + outcome 應用」集中；賣裸 token 的純 LLM 中間商被多模型壓價。

利潤池遷移

整條鏈的利潤正從「賣裸 token」往「兩端」遷移：底層算力因 CoWoS/電力受限有定價權（收租）、自有矽把成本與毛利留內部、outcome 應用把 token 包成「按結果收費」；被壓的是夾在中間、外購矽、賣無差異 API 的純 token 中間商。Jevons 把總支出做大（代幣支出翻倍），但餅怎麼切由「誰握矽、誰握客戶結果」決定。（依 QC-18，缺逐段精確百分比，以定性呈現。）

Inference · 供給
token 商品化，利潤往自有矽、收租算力與 outcome 應用集中
前提：token 被多模型壓價（商品化）+ 用量爆發（Jevons）+ 自有矽留住成本/毛利 + 算力受 CoWoS/電力限（定價權）→ 利潤往矽/自有矽端 + outcome 應用端集中，純中間商被擠

當「一個 token」變成年年崩價的商品，賣裸 token 賺不到溢價；賺得到的是握自有矽的垂直整合者（GOOGL/AMZN/META——跟得了價又留得住毛利）、收推論租金的算力層（NVDA/AVGO——量爆即受益、且供給受限有定價權）、與把 token 包成 outcome 的應用層。

可證偽條件：若推論算力供給（CoWoS/電力）大幅鬆綁、定價權消失，或開源高效 model + 通用 ASIC 讓自有矽成本優勢被抹平，則「利潤往矽/整合集中」論的某一面失效。

未來供給與進入門檻：成為贏家的門檻是「自有矽（設計+量產+軟體棧）或 規模算力（CoWoS/電力配額）或 outcome 應用（握客戶結果）」。純買 GPU 跑外購 model 補不上成本優勢、賣裸 token 補不上差異化——這是 GOOGL/AMZN/META 自有矽、NVDA/AVGO 收租守得住的原因。長期變數是供給瓶頸（CoWoS/電力）會不會鬆綁、削弱算力定價權。

因果閉合（回應 §1 結構變數）：§1 提出「垂直整合者在降價戰中贏」（雲端先例）——未來 3–5 年是否仍 binding？很可能：自有矽的成本優勢隨用量規模放大，且 CoWoS/電力受限讓算力端維持定價權；唯一鬆動是供給大幅擴產 + 開源高效 model 把成本拉平——這是 §7 的反方。

12–36 個月內，token 持續商品化、利潤往「自有矽 + 收租算力 + outcome 應用」集中。對投資的含義：押握矽/自有矽者 + 推論基礎設施；避開賣裸 token 的純中間商與外購矽的純 LLM 廠。

§4 需求側：現在與未來 + 三角驗證

需求最反直覺的特徵是「越便宜、用越多」——token 單價崩 90%，總支出反而翻倍（Jevons）；最硬的瓶頸不在需求，而在供給（CoWoS、電力）跟不上。

第一引擎：Jevons 量補價（現需）。推論單價崩不會減少總支出、反而放大它——Silicon Data 代幣支出指數自 2025 末翻倍，即使單一 token 較 2023 跌約 90%；業界共識「API 成本降 98%，企業不會少花 98%，而是多蓋 50 倍應用」。agentic AI 是推論放大器：一個 agent 任務呼叫 model 數十次。推論已占 AI 資料中心算力約 2/3（Deloitte 2026）至 80–90%（部分研究）。第二引擎：供給瓶頸（未需轉現需）。需求不是問題、供給才是：2026 AI capex ~$660–765B、→2031 ~$1.6T；AWS 積壓訂單 $244B（+40% YoY）；Azure ~$80B 訂單因 GPU/電力不足無法交付；TSMC CoWoS 由 2024 ~9,000 片/月擴到 2026 年底估 ~130,000 片/月，仍「sold out through 2026」。電力與先進封裝是真瓶頸。

需求三角驗證

top-down：推論市場隨 capex（$765B 2026）與推論占比（2/3–90%）擴張。bottom-up：代幣支出指數自 2025 末翻倍（單價跌 90% 仍翻）。供給端：AWS backlog $244B、Azure $80B 待交付、CoWoS sold out through 2026。三邊一致——需求是真的、且被 Jevons 放大；瓶頸在供給。注意：不同市場規模估計口徑差異大（推論 vs 全 AI compute），不可直接相比，本報告以「代幣支出指數 + capex/backlog」為主錨。

Inference · 需求
需求被 Jevons 放大、供給受限——成長品質看「誰的毛利不被降價戰侵蝕」
前提：token 降價刺激用量（Jevons）+ 推論占比升至 2/3–90% + 供給（CoWoS/電力）受限 → 總支出爆發、算力有定價權，但賣裸 token 者毛利被壓

需求的「量」由 Jevons 保證（單價越低、用越多、總支出越大）；變數在「誰的毛利不被降價戰侵蝕」——握自有矽者成本最低、跟得了價；收租算力因供給受限有定價權；賣裸 token 的中間商被夾在「降價 vs 成本」之間。

可證偽條件：① 代幣支出指數成長停滯（量補不過價、Jevons 失效）；② 供給瓶頸快速鬆綁使算力定價權消失、推論變純價格戰。

多空為何都對：多頭測總需求（Jevons 量補價、capex $765B、推論占比升），幾乎沒爭議；空頭測毛利與分潤（SaaS 毛利壓縮、外購矽漏出、賣裸 token 賺不到）。兩者可同時為真：推論總支出爆發（多頭對），但利潤被矽/自有矽/應用層切走、純中間商吃虧（空頭對）。本報告把重心放在「單價趨零下誰捕獲價值」。

§6 產業經濟學與估值傳導

token 的錢怎麼賺：算力收租（高毛利、有定價權）+ 自有矽省成本（毛利不漏）+ outcome 應用（避開 SaaS 毛利壓縮）。賣裸 token 與 per-seat SaaS 則被夾——每次 model 呼叫都吃真實算力，把軟體毛利從 80–90% 壓到 50–60%。

看自有 vs 外購矽、SaaS vs outcome 定價的 unit economics 差異——這是價值捕獲核心。

token 經濟 unit economics（量級）
玩家類型毛利 / 經濟價值捕獲含義
收租算力 NVDA/AVGO~70–75%（受 CoWoS 限有定價權）量爆即受益賣鏟子、週期性
自有矽 GOOGL/AMZN/META推論成本最低、毛利不漏留在內部結構占優
per-seat SaaS（無優化）SaaS 80–90% → AI 壓到 50–60%被推論成本侵蝕須轉 outcome
outcome 定價應用按結果收費、毛利回升對齊價值守得住毛利

怎麼讀：收租算力與自有矽留得住毛利；per-seat SaaS 被推論成本侵蝕（80–90%→50–60%）、必須轉 hybrid/outcome 定價才回得來。NVDA/AVGO 是漏出鏈的收費方；GOOGL/AMZN/META 自有矽不漏；賣裸 token 與 per-seat 最吃虧。

估值傳導機制：算力股估值對「需求量 × 定價權（CoWoS/電力瓶頸）」敏感；自有矽整合者對「毛利優勢 × 量體」敏感；AI 軟體對「能否守住毛利」敏感。傳導鏈：Jevons 量補價 → 算力需求爆 + 供給受限 → 算力定價權 → 估值溢價；反向，供給鬆綁 → 定價權消失 → de-rate。SaaS 鏈：推論成本侵蝕毛利 → 若無 outcome 定價 → blended 毛利下行 → de-rate。

估值現在 price 了什麼：算力股（NVDA/AVGO）price 了量補價 + 定價權、沒 price「供給鬆綁」的下行；自有矽者（GOOGL/AMZN/META）的毛利優勢藏在量體裡、part-priced；AI 軟體股的毛利壓縮風險未被充分 price。會 re-rate 的是供給續緊 + 自有矽優勢擴大；會 de-rate 的是供給鬆綁 + SaaS 毛利崩。對 PM：盯代幣支出指數與 CoWoS 利用率，不是盯「token 又降價」。

§7 Non-Consensus + Priced-in 檢驗 + Kill Scenario

⚠️ 本章含 judgment：每條分歧附 [F:] cornerstone + [X:] 三情境 + priced-in 檢驗。
市場對這主題的三個共識：①「token 降價 = LLM 廠輸」；②「外購 vs 自有矽對毛利沒差」；③「AI 軟體成本下降是純利多」。三個我們都部分不同意——分歧能否賺錢取決於市場 price 了多少。
🟡分歧 1：「token 降價 = LLM 廠輸 / AI 經濟見頂」信心：高
  • 共識說 X：token 一直降價，賣 AI 的賺不到錢、AI 經濟見頂。🟢 [F: T3 — 單價崩 99%]。
  • 我們說 Y：降價是 Jevons 量補價——代幣支出指數翻倍、總支出爆發；輸的是賣裸 token 者，握算力/自有矽/應用者贏。🔵 [X: base 很可能 — 量補價續行；bear 可能 — 量補不過價]。
  • Priced-in：市場對「token 降價」的恐慌已部分 price 進賣裸 token 的純中間商，沒 price「Jevons 把餅做大 + 算力定價權」——這是最未 priced、最可操作的一條。
→ 推論邏輯：把「降價」拆成單價（跌）vs 總支出（爆）。⚠ 證偽：代幣支出指數停滯（量補不過價）。
🟡分歧 2：「外購 vs 自有矽對毛利沒差」信心：中
  • 共識說 X：用誰的晶片跑推論不影響估值/毛利。🟢 [F: T3 — 市場未細分]。
  • 我們說 Y：差很大——自有矽者（GOOGL/AMZN/META）推論成本最低、毛利不漏；外購 GPU 者把一塊付給 NVDA。🔵 [X: base 很可能 — 毛利差存在；程度待數據]。
  • Priced-in：市場幾乎沒 price 自有矽的毛利優勢——GOOGL/AMZN/META 的 TPU/Inferentia/MTIA 藏在量體裡、未被單獨定價。分歧 2 是最未 priced 的結構性洞見。
→ 推論邏輯：自有矽是隱性成本護城河。⚠ 證偽：通用 ASIC/開源讓自有矽優勢被抹平。
🟡分歧 3：「AI 成本下降是軟體股純利多」信心：中
  • 共識說 X：token 變便宜，用 AI 的軟體公司成本下降、純受惠。🟢 [F: T3 — 成本下降]。
  • 我們說 Y：每次 model 呼叫吃真實算力——AI 把 SaaS 毛利從 80–90% 壓到 50–60%，per-seat 吃到飽撐不住；要轉 outcome 定價才回得來。🔵 [X: base 很可能 — 毛利壓縮；可能 — outcome 定價回升]。
  • Priced-in：市場把 AI 當軟體股利多泛泛 price，沒 price 毛利壓縮（80–90%→50–60%）與 per-seat 模式風險——分歧 3 的下行未 priced。
→ 推論邏輯：成本下降 ≠ 毛利上升（用量同步爆）。⚠ 證偽：AI 軟體毛利證實守在 SaaS 水準。

Steel-man 反方（最有力的 3 條）

  • 反方 1（量補不過價）：若效率/開源讓單價跌得比用量快、Jevons 失效，推論總支出成長停滯、算力與相關股 de-rate。當前證據：10x/年降幅、開源高效 model。證偽窗口：代幣支出指數斜率。
  • 反方 2（供給鬆綁、定價權消失）：CoWoS/電力大幅擴產使算力供過於求、推論變純價格戰，NVDA/AVGO 定價權消失。當前證據：CoWoS 9k→130k 擴產中。證偽窗口：CoWoS 利用率、算力現貨價。
  • 反方 3（SaaS 毛利崩拖累 AI 軟體）：AI 軟體毛利持續被推論成本侵蝕、per-seat 模式崩、AI 軟體估值整體 de-rate。當前證據：SaaS 毛利 80–90%→50–60%。證偽窗口：AI 軟體毛利、outcome 定價採用率。

市場最低估的風險：不是「token 沒人用」（用爆了）、也不是「算力沒需求」（sold out），而是分歧 1/2 同源的「賣裸 token 賺不到 + 自有矽才是隱性贏家」——市場把 token 降價當末日、把外購/自有矽當沒差，最沒準備好「Jevons 量補價 + 自有矽毛利優勢」這對組合。這是推論經濟最該獨立監測的洞見。

三條分歧裡最可操作的是分歧 1/2（Jevons 量補價 + 自有矽毛利優勢）——最未被 priced 且有明確領先指標（代幣支出指數、自有矽 attach）。分歧 3（SaaS 毛利壓縮）次之、下行未 priced。

GOOGL🔴 核心受益垂直整合典範（TPU + Gemini + GCP + caching）；推論成本最低、毛利不漏
MSFT🔴 核心受益hyperscaler + LLM 經銷；Azure backlog ~$80B 受 GPU/電力限
AMZN🟡 次要受益Inferentia/Trainium 自有 ASIC + AWS backlog $244B +40%
META🟡 次要受益MTIA 自有 ASIC + 內部推論超大用戶（省成本、毛利不漏）
AMD🟡 次要受益推論 GPU 挑戰者（MI 系列）；多模型壓價下第二供給
NET🟡 次要受益推論 routing/caching/edge 純 play；token 流量收費站
CRWV🟢 邊緣受益NeoCloud GPU 租賃（推論 + 訓練 mix）
AAPL🟢 邊緣受益端側推論（Apple Intelligence）——把 token 成本移到裝置
QCOM🟢 邊緣受益端側 NPU（Snapdragon X 系列）
DDOG🟢 邊緣受益LLM observability / 推論成本治理（non-obvious 二次受益）

非顯而易見受益者：NET（Cloudflare）——它的推論 routing/caching/edge 把 token 成本壓低、是 AI 流量的天然收費站，市場曝險認知度低於 NVDA。DDOG（Datadog）同理——當企業 token 帳單暴增，「監控與治理推論成本」成剛需，是推論經濟的二次受益者，市場少注意。

最高純度 play 是收租算力 NVDA/AVGO 與自有矽整合者 GOOGL/AMZN/META——買賣由個股 DD 與 Pure MA 系統決定。被低估曝險的是 NET（推論基礎設施）與 DDOG（成本治理）；端側（AAPL/QCOM）是替代路徑；受壓的是賣裸 token 的中間商與 per-seat SaaS；OpenAI/Anthropic 私有、不可直接投資。
