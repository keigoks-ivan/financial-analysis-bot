市場把「token 單價自 GPT-3 崩約 98%」讀成 AI 基建的通縮利空——這個直覺被一個結構性悖論推翻了。Structural Jevons Paradox（Zhang & Zhang 2026-01 形式化）：當智能的單價暴跌，下游廠商不會用同樣的 workload 省錢，而是重新設計 agent 架構去吃更多算力——更深的推理迴圈、更長的 context、多 agent 工作流。結果是 token 單價崩 98%、但 token 總支出反而翻倍（Silicon Data 指數），企業 AI 預算裡推論成本占比從 2023 的 20% 衝到約 85%。

所以真正的斷層不是「便宜」，而是毛利分層的分裂：① commodity 推論（通用模型、短 context）成本逼近零、毛利被打到地板——純 GPU 雲轉售與無差異 LLM API 是輸家；② frontier 推論（最新推理模型、最長 context）不在同一條成本曲線上、仍能收高價；③ custom ASIC 推論（垂直整合的 hyperscaler）與 routing/orchestration 中介層，毛利守得住。同樣的 token 海嘯，有人賺錢、有人賠錢，差別在「在哪一層收費」。

本報告最反直覺的判斷：盯「推論變不變便宜」的人問錯問題；該問的是「不斷上升的 token 支出最後落在哪一層、那一層有沒有毛利」。Anthropic 衝到 $30B 營收、毛利含 inference 卻只 ~48%（inference 成本比預期高 23%、2026 估虧 $14B），就是「量爆發 ≠ 毛利」的活證據。押毛利守得住的層，避開被 Jevons 海嘯沖到零毛利的層。

PART I · DECISION SUMMARY

決策摘要層

三句話看完推論經濟學的現狀、走向與行動，後接 PM 級監測點。這一層是報告的全部重點——下面各節只是把這一層的每個判斷補上證據。

現在 / NOW

推論已占算力支出 ~2/3、占企業 AI 預算 ~85%（2023 僅 20%）；token 單價崩 ~98% 但總 token 支出翻倍——Jevons 悖論成立，便宜不減量反增量。

未來 / NEXT

毛利三層分裂：commodity 推論趨零毛利（GPU 雲轉售 + 無差異 LLM API 輸）vs frontier 推論 + custom ASIC + routing 層守住。Anthropic GM 含 inference ~48% 待升 70%、OpenAI 目標 2029 ≥70%。

行動 / ACTION

押「毛利守得住」的層——custom ASIC 推論（AVGO / GOOGL TPU 垂直整合）+ 自有模型 + 高階 routing；避開純 GPU 雲轉售與無差異 API。盯 token 支出指數 / agentic 滲透 / 各家 inference GM 軌跡。

The Key Call · 本報告最重要的一個判斷

「推論變便宜」被市場讀成通縮利空，這個直覺錯了。Structural Jevons Paradox 把它反轉：單價每跌一階，廠商就把省下的算力重新投進更貪婪的 agent 架構（agentic workload 每個 task 吃 5–30x 於 chatbot 的 token），所以總支出隨單價下跌而上升。真正的斷層不是價格，是毛利落在誰手上：同一波 token 海嘯，commodity 推論被沖到零毛利（GPU 雲轉售、無差異 API），而 frontier 推論、垂直整合的 custom ASIC、與 routing 中介層守住毛利。盯「便不便宜」的人問錯問題；該問「上升的支出落在哪一層、那層有沒有毛利」。

毛利三層具體是：① 輸家層——純 GPU 雲轉售（neocloud）與無差異 LLM API，commodity inference 成本趨零、被價格戰打到地板；② 中性收費層——routing/orchestration（在便宜與貴的模型之間做智能路由、抽中介費，token 通縮反而擴大其價值）；③ 贏家層——frontier 推論（最新推理模型不在零成本曲線上）+ 垂直整合 custom ASIC（GOOGL TPU、hyperscaler 自研把 inference 成本內部最佳化）。同樣賣「推論」，這三層的毛利命運完全相反。

PART II · HISTORICAL ANALOGS

兩個歷史類比：頻寬通縮與雲端商品化

「單位成本暴跌但總支出上升」不是 AI 獨有；頻寬與雲端運算都走過，教訓是「贏家在垂直整合與差異化層、輸家在純轉售層」。

頻寬通縮（2000s-2010s）——Jevons 的教科書案例。每 GB 頻寬成本在二十年內跌了好幾個數量級，但全球頻寬總支出並未崩——因為便宜的頻寬催生了串流、雲端、影音，需求量的爆發遠超單價的下跌。教訓對應本次清楚：贏家不是賣「原始頻寬」的電信商（被打成 commodity），而是建在便宜頻寬上的差異化服務（Netflix、CDN）。本次量化差異：AI token 單價跌得更快（−98% / 三年 vs 頻寬數十年），且 agentic 架構的「需求彈性」更高（單價跌 → 主動重設計去吃更多），所以總支出上升的力道更猛——但「純轉售被打成 commodity」的規律一樣適用於 GPU 雲與無差異 API。

頻寬通縮 / 雲端商品化 vs AI 推論 · 三條件對照
條件頻寬 / 雲端前例AI 推論 2026含義
單價趨勢數量級下跌−98% / 三年（更快）通縮更猛
需求彈性新應用催生需求agentic 主動重設計吃 5-30x本次彈性更高、總支出更易升
贏家層差異化服務（非純轉售）frontier + ASIC + routing規律一致：避開純轉售

怎麼讀：三條件都指向同一結論——通縮不殺死總需求（反而放大），但會把純轉售層打成零毛利。把「token 便宜」當利空、或把「總支出升」當人人都賺，都是誤讀；要分層看。

雲端 IaaS 商品化——routing 層的歷史對應。當基礎運算（EC2 等）商品化後，價值往「管理/編排層」上移（Kubernetes 生態、雲端管理 SaaS）。AI 推論的對應是 routing/orchestration 層——在一堆便宜與貴的模型之間做智能路由、快取、批次化、降級，token 越通縮、選擇越多，這層的「中介價值」越大。這是為什麼 Cloudflare 式的邊緣推論 + routing 是 token 通縮下的隱性受益者，而非受害者。

PART III · SUPPLY SIDE

供給側：同一波 token 海嘯，三層毛利命運相反

推論的「供給」是算力 + 模型 + 服務層；它們的成本結構天差地別，決定誰被 Jevons 海嘯沖到零毛利、誰守得住。

推論供給鏈分三層，毛利命運相反：① commodity 推論（輸）——純 GPU 雲轉售（neocloud）跑通用模型，成本趨零、價格戰白熱化，毛利被打到地板；無差異 LLM API 同理。② routing/orchestration（中性收費）——在模型之間智能路由、抽中介費，token 越多選擇越多，這層價值隨通縮上升。③ frontier + 垂直整合 ASIC（贏）——最新推理模型不在零成本曲線上（仍收高價），且 GOOGL（TPU）、AMZN（Trainium）、META（MTIA）用自研 ASIC 把 inference 單位成本內部最佳化，毛利守得住。同樣賣推論，三層的 ROIC 完全不同。

Inference · 供給
垂直整合的 inference 成本優勢，是毛利贏家層的護城河
前提：inference 成本 ~占 AI 預算 85% ＋ 垂直整合者用自研 ASIC（TPU/Trainium/MTIA）跑自家模型 ＋ 純轉售者付市場價買 GPU + 模型授權

當 inference 成本是最大開支，誰能把這塊成本「內部最佳化」誰就贏。GOOGL 的 TPU + 自有模型 + 自有資料中心是最完整的垂直整合，inference 單位成本結構最佳；純 GPU 雲轉售者付市場價、無模型差異，毛利最薄。這把「誰贏推論」從「模型多強」轉成「成本結構多整合」。

可證偽條件：若 commodity 推論因某種差異化（延遲、合規、地域）重獲定價權，或垂直整合者的 ASIC 成本優勢被 merchant GPU 的規模經濟追平，則「垂直整合護城河」論弱化。

推論毛利三層分裂 · 成本結構與命運
層代表玩家毛利命運為什麼
commodity 推論neocloud / 無差異 API↓ 趨零通用模型成本趨零、價格戰
frontier 推論OpenAI / Anthropic frontier→ 待升最新推理模型不在零成本曲線；惟現 GM ~48%
routing / orchestrationCloudflare (NET) 式↑token 越多、中介價值越大
垂直整合 ASIC 推論GOOGL TPU / AMZN / META↑ 守住自研 ASIC 內部最佳化 inference 成本
merchant GPU 推論NVDA（frontier）/ AMD分化frontier 推論仍強、低階商品化

怎麼讀：同樣賣「推論」，毛利往「frontier + 垂直整合 ASIC + routing」三層集中（↑/守），往「commodity 轉售 + 無差異 API」流失（↓ 趨零）。NVDA 在 frontier 推論仍強、但低階推論面臨 ASIC 與商品化雙重壓力。因果機制：inference 占 AI 預算 85% 這個事實，使「成本結構」而非「模型強度」成為毛利的主決定因子——這是垂直整合者（GOOGL）結構性領先的根。

PART IV · DEMAND SIDE

需求側：Jevons 悖論與 agentic 乘數

推論需求的引擎不是「更多人用」，而是「每個 task 吃更多 token」——agentic 架構是 Jevons 悖論的微觀放大器。

需求結構的關鍵不是用戶數，而是每個 task 的 token 消耗。Structural Jevons Paradox 的機制：單價跌 → 開發者採用更深的推理迴圈、更長 context、多 agent 工作流、chain-of-thought → 每 task token 消耗暴增（agentic workload 是 chatbot 的 5–30x）。結果：企業 GenAI 支出兩年成長 22x，而同期 per-token 價格跌 >90%——量的爆發遠超價的下跌。這是「token 單價崩 98% 但總支出翻倍」的需求側引擎。

需求三角驗證 · Jevons 悖論的量價對帳
乘數（同能力）變化方向
① per-token 單價−~98%（÷~50；GPT-3 至今）↓↓
② 每任務 token（agentic）每 task 5–30x（vs chatbot）↑↑
③ 任務量 / 採用成長企業 GenAI 2 年 ~22x（用例擴張）↑↑↑
＝ token 總支出（淨）近一年翻倍↑

怎麼讀：總支出 ＝ 單價 × 每任務 token × 任務數，三個乘數同時動。光靠 ①+② 不夠——單價 −98%（÷50）× 每任務 5–30x，淨仍是下降；真正讓總支出翻倍的是第三個乘數：任務量/採用爆發（企業 2 年 ~22x，AI 滲進更多用例）。Jevons 的完整機制是「便宜 → 用例擴張（任務數↑）＋ 每任務更貪婪（agentic↑）」兩條一起跑贏單價崩跌。⚠ 對帳結論：採信「總支出續升」（Silicon Data 指數 + 企業 22x 雙佐證），但它需要 ② 與 ③ 同時續升；任一見頂，Jevons 引擎熄火。

Inference · 需求
Jevons 引擎靠 agentic 乘數，乘數見頂則引擎熄火
前提：單價 −98% ＋ agentic workload 每 task 5-30x token ＋ 能力階梯上移（不停在被打到地板的舊能力）

token 總支出上升靠兩個乘數同時運轉：① agentic 架構吃更多 token；② 能力階梯上移（廠商不停留在零成本的舊能力，遷到 frontier 重新訂高價）。兩者一起讓總支出跑贏單價崩跌。

可證偽條件：① agentic 滲透率停滯（task token 乘數見頂）；② 能力階梯停止上移（frontier 模型不再帶來足夠新價值）；③ 企業因 ROI 壓力把 AI 從「戰略投入」改列「要當期回報的營運支出」。任一發生，Jevons 引擎熄火、總支出停滯。

變現的真風險

Jevons 讓「總支出」上升，但不保證「毛利」——Anthropic 衝到 $30B 營收、含 inference 毛利卻只 ~48%（inference 成本比預期高 23%、2026 估虧 $14B）。總支出爆發落到 frontier lab 手上，他們卻還沒把 inference 成本壓到能獲利的水準。這是「量 ≠ 利」的核心風險，也是本主題判 mixed 的根。

EDGE 🟢

routing / 中介層
NET / MRVL / AMD

NET（Cloudflare）是 token 通縮下 routing/orchestration 中介層的代表——選擇越多、中介價值越大；MRVL 是 custom 推論 ASIC #2；AMD 推論 GPU #2（ROCm 推論 parity 改善但 frontier 落後）。是「中介收費 + 第二供應」特徵。conviction：低 · satellite 觀察

估值面總結：本主題的估值分歧不在「推論會不會成長」（會，Jevons 保證），而在「毛利落在誰手上」。市場常把整個推論鏈當同一個多頭，但 commodity 轉售與無差異 API 其實是輸家——這是最大的錯誤定價。垂直整合者（GOOGL）的 inference 成本優勢未被充分 price，是風險調整後較佳的 expression；frontier lab（OpenAI/Anthropic，私募）的毛利能否升到 70% 是 AT_RISK 的關鍵假設。de-rating window 在「token 支出指數停滯」或「AI lab 毛利不升反降」那一刻。

The One Line

推論變便宜不是利空——Jevons 讓總支出隨單價崩跌而上升；真正的斷層是毛利分層：押 frontier + 垂直整合 ASIC + routing，避開被沖到零毛利的純轉售與無差異 API。
