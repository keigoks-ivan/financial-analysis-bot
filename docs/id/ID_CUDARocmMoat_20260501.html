市場把 NVDA 的 CUDA 護城河當成一道「鐵牆」——要嘛完好、要嘛崩塌的二元賭局。真相是這道牆正在分層裂解，不同層的命運完全不同。在單機推論層，牆已開始裂：ROCm 7.2（2026-03）首次達成 vLLM / llama.cpp / Ollama / SGLang 的「出箱（out-of-box）parity」，PyTorch 2.7 的 ROCm build 達 CUDA 約 95% 功能對等——對「PyTorch + vLLM、無自訂 kernel」的推論工作負載，AMD 已經夠用。

但在多卡訓練的 scale-out 層，牆紋風不動：NCCL（NVIDIA 的多卡通訊庫）+ NVLink（卡間高速互連）仍是 frontier 模型訓練的獨家路徑，TensorRT-LLM 與 FlashAttention 3 等高階核也仍 CUDA-only。換句話說——CUDA 的護城河不是「在不在」，而是「在哪一層」：推論層的水正在退，訓練 scale-out 層的牆還很高。

本報告最反直覺的判斷：這場護城河之爭的 2027-28 真贏家，不是任何一個 GPU 廠，而是控制抽象層的 framework owner——Meta（PyTorch）、Google（JAX/Pallas）、OpenAI（Triton）。當框架把硬體抽象成「可替換後端」，決定「GPU 是否可被換掉」的權力，就從晶片廠手裡轉移到框架治理者手裡。盯 CUDA vs ROCm 誰贏的人問錯了問題；該問的是「誰握有抽象層的方向盤」。

PART I · DECISION SUMMARY

決策摘要層

三句話看完護城河分層的現狀、走向與行動，後接 PM 級監測點。這一層是報告的全部重點——下面各節只是把這一層的每個判斷補上證據。

現在 / NOW

推論層 CUDA 護城河已開始裂——ROCm 7.2（2026-03）達 vLLM/llama.cpp/Ollama/SGLang 出箱 parity、PyTorch 2.7 ROCm ~95% 功能對等；但訓練多卡 scale-out（NCCL+NVLink）NVDA 仍獨家、TensorRT-LLM/FlashAttention 3 仍 CUDA-only。

未來 / NEXT

framework 抽象化把 GPU 變「可替換後端」——framework owner（META PyTorch / GOOGL JAX-Pallas / OpenAI Triton）掌握抽象治理權；推論層 AMD/ASIC 取得份額、訓練層 NVDA 護城河仍守；UALink 2.0 spec 2026 Q2 已發布、AMD/Intel/Astera 硬體 2026 H2 起初步商用，大規模 hyperscaler 採用預期 2027——採用速度是 scale-out 的關鍵變數。

行動 / ACTION

護城河交易不是「永遠 long NVDA 軟體鎖定」——是「訓練 scale-out 仍護城、推論護城河蝕」；被低估的贏家是 framework owner（META/GOOGL）+ ROCm parity 的 AMD。盯 ROCm parity 里程碑 / UALink 進度 / framework 治理權。

The Key Call · 本報告最重要的一個判斷

市場把 CUDA 護城河當「鐵牆 yes/no」，這是錯框。護城河正在分層裂解：推論層的水退了（ROCm 7.2 出箱 parity、框架日益硬體無關），訓練 scale-out 層的牆還很高（NCCL+NVLink 獨家）。而這場仗的 2027-28 真贏家不是 GPU 廠——是控制抽象層的 framework owner。因為一個機制：當 PyTorch / JAX / Triton 把硬體抽象成「可替換後端」，「能不能換掉 NVDA」的決定權就從晶片廠移到框架治理者手裡。Meta 用 PyTorch、Google 用 JAX/Pallas + 自家 TPU、OpenAI 用 Triton——他們既是最大買家、又握有抽象層方向盤，正在系統性地把 GPU 變成可商品化的後端。盯 CUDA vs ROCm 誰贏的人問錯問題；該問「誰握有抽象層」。

護城河三層具體是：① 推論層（裂）——單機 / 單節點推論，ROCm 7.2 出箱 parity、框架硬體無關化，AMD/ASIC 已能搶份額；② 訓練 scale-out 層（守）——數千卡訓練 frontier 模型需 NCCL+NVLink 的整機櫃通訊，NVDA 獨家、MSCCL++ 等替代仍 research；③ 抽象治理層（決定終局）——framework owner 決定硬體是否可替換，這層的權力 2027-28 從晶片廠移向 META/GOOGL/OpenAI。盯這三層的分別動態，比盯「CUDA 會不會被取代」更能看清誰賺到什麼。

PART II · HISTORICAL ANALOGS

兩個歷史類比：x86 生態鎖定、Android 抽象層

x86 告訴我們「生態鎖定」能撐多久、靠什麼裂；Android/JVM 告訴我們「抽象層」如何把底層硬體商品化——CUDA 同時是 x86 的生態、也面對 Android 式的抽象威脅。

x86 生態鎖定——護城河在生態不在矽，但生態會被新平台繞過。Intel x86 數十年的統治不靠單晶片最快，而靠數十年累積的軟體生態（編譯器、OS、無數應用都假設 x86）。CUDA 是 AI 時代的 x86：護城河不是 GPU 硬體，而是「過去十年每一份框架、教學、論文、部署指南都預設 CUDA」。但 x86 的教訓有兩面：① 在 PC/伺服器，生態鎖定撐了三十年；② 在手機，x86 完全沒進場——因為 ARM + Android/iOS 是新平台，不繼承 x86 生態包袱。對應本次：CUDA 在「延續既有訓練工作流」的場景護城河極深，但在「全新的推論部署 + hyperscaler 自研矽」這個新平台上，CUDA 的生態慣性較弱——這正是裂縫出現在推論層而非訓練層的歷史理由。

CUDA 護城河分層 · 對應歷史前例
層CUDA 現況歷史對應命運
推論（單節點）ROCm 7.2 出箱 parityx86 在手機被 ARM 繞過裂——新平台不繼承生態

怎麼讀：同一個「CUDA 護城河」，在不同層對應不同歷史前例、不同命運——推論層像「x86 在手機」（被新平台繞過），訓練層像「x86 在伺服器」（深鎖定），抽象層像「Android 把高通/聯發科變可替換」。把三層混為一談，就會在「CUDA 崩了沒」的假問題上反覆誤判。

Android / JVM 抽象層——把底層硬體商品化的歷史劇本。Android（與更早的 JVM「write once, run anywhere」）證明了一件事：當一個夠強的抽象層普及，底層硬體就被商品化——App 開發者寫 Android、不在乎跑在高通還是聯發科，晶片廠的議價權因此被壓縮。PyTorch / JAX 正在 AI 扮演同樣角色：當模型開發者寫 PyTorch、由框架決定後端跑 CUDA 還是 ROCm 還是 TPU，NVDA 的「軟體鎖定」就被框架層接管。本次量化差異：PyTorch 2.7 的 ROCm build 已達 ~95% 功能對等——抽象層的「可移植性」在推論場景已接近 Android 等級；但訓練 scale-out 的 NCCL+NVLink 還沒有等價的抽象層替代（這是物理互連，不只是軟體 API），所以商品化目前只發生在推論層。

PART III · SUPPLY SIDE

供給側：護城河三層，每層玩家不同

「CUDA 護城河」其實是三道厚度不同的牆疊起來；看清哪層由誰守、哪層誰能攻，才知道價值流向哪。

軟體護城河的供給端分三層：① 推論層——NVDA（CUDA + TensorRT-LLM）vs AMD（ROCm 7.2，已達 vLLM/llama.cpp 出箱 parity）vs hyperscaler 自研棧（Neuron/oneAPI）；這層牆最薄，AMD 與 ASIC 已能攻。② 訓練 scale-out 層——NVDA（NCCL + NVLink，數千卡通訊獨家）幾乎無對手，AMD 的 RCCL + Infinity Fabric 仍落後、UALink（2.0 spec 2026 Q2、AMD/Intel/Astera 硬體 2026 H2 起初步商用）大規模 hyperscaler 採用待 2027。③ 抽象治理層——framework owner（META PyTorch / GOOGL JAX-Pallas / OpenAI Triton）決定硬體可否替換；這層的權力正從晶片廠移向超大規模買家自己。利潤/議價權目前仍最集中在 NVDA（守住最厚的訓練層），但增量話語權正往抽象層滲透。

Inference · 供給
護城河在「物理互連」最硬，在「軟體 API」最易被抽象繞過
前提：推論多為單節點（軟體 API 層）＋ 訓練需數千卡（物理互連 NCCL+NVLink）＋ framework 可抽象軟體 API、但抽象不了物理互連

軟體 API 可以被框架抽象、被 ROCm 複製（推論層已發生）；但 NVLink 是卡間的物理高速互連、NCCL 是其上的通訊最佳化——數千卡訓練 frontier 模型時，這是 latency/頻寬的硬約束，不是一個 API 換掉就行。所以護城河最硬的地方不在 CUDA 這個「軟體」，而在 NVLink 這個「物理互連 + 其上的 NCCL」。

可證偽條件：UALink（開放 scale-out 互連）2.0 spec 已於 2026 Q2 發布、硬體 2026 H2 起初步商用；若 2027 hyperscaler 大規模採用、或 MSCCL++ 從 research 轉 production，則訓練 scale-out 護城河開始鬆動，NVDA 最硬的那層牆出現裂縫——故監測點 2 從 2026 H2 起即須盯（非等 2027）。

怎麼讀：牆厚由薄到厚＝推論 < 高階 kernel < 訓練 scale-out；NVDA 最該守的不是 CUDA 這個品牌，而是 NVLink+NCCL 這層物理互連。最危險的趨勢是「抽象治理層」——NVDA 根本守不住（那是框架擁有者的地盤），而 framework owner 正是 hyperscaler 自己。價值流向：推論層的價值往 AMD/ASIC + framework owner 流；訓練層的價值 NVDA 暫守；長期看，抽象層誰控制、誰就能決定 GPU 是否可商品化。

PART IV · DEMAND SIDE

需求側：誰想逃離 CUDA、為什麼逃得動

想逃離 CUDA 鎖定的，正是 NVDA 最大的客戶——hyperscaler；他們既有逃的動機（成本、議價權），又有逃的能力（自研矽 + 自有框架）。

逃離 CUDA 的需求來自 NVDA 自己的大客戶：hyperscaler（GOOGL/AMZN/MSFT/META）既付最多錢給 NVDA、又最有動機與能力擺脫鎖定。動機：CUDA 鎖定 = NVDA 的定價權 = 他們的成本與議價劣勢。能力：他們同時握有自研矽（TPU/Trainium/MTIA/Maia）與自有框架（GOOGL JAX、META PyTorch、AMZN Neuron），可以從「晶片 + 編譯棧 + 框架」三端一起繞過 CUDA。這是「逃離 CUDA」這次跟以往（小廠想挑戰 NVDA）最大的不同——這次想逃的是有資源、有框架治理權的巨頭。

怎麼讀：逃得最徹底的是 GOOGL（TPU + JAX 自有訓練棧，frontier 訓練都能繞 NVDA）；META 的關鍵武器是「擁有 PyTorch」——它不只逃，還握有讓所有人都能逃的抽象層方向盤。逃得動的層目前集中在推論；訓練 scale-out 除了 GOOGL TPU（自有互連 ICI）外，多數仍回到 NVLink。因果閉合（回應 §1 的「生態慣性」變數）：CUDA 的生態慣性在「既有訓練工作流」仍 binding（換框架成本高），但在「新建推論部署 + 自研矽」場景已鬆動——這是裂縫只在推論層、且由有自有框架的巨頭推動的根。

多空為何都對

NVDA 多頭測的是「訓練 scale-out + 既有工作流慣性」（真的還很硬），空頭測的是「推論層 parity + 抽象層商品化」（也真的在發生）——兩者測的是不同層，可同時為真。CUDA 在訓練護城河仍深（多頭對），但推論層正被 ROCm + framework 抽象繞過（空頭對）。把它讀成「CUDA 整體崩 / 整體穩」都會誤判。

PART VI · TAXONOMY

個股特徵與 conviction tier

本框架只輸出「特徵」與「分類」，買哪一檔由個股 DD 系統與 Pure MA 訊號決定。以下 tier 為產業推論的特徵歸類，提及公司僅為「該特徵的代表案例」，不附帶買賣建議。

CORE 🔴

NVDA

護城河守方——訓練 scale-out（NCCL+NVLink）仍獨家、是最硬的那層牆；但推論層護城河開始蝕、抽象治理層守不住。風險不是「CUDA 崩」，是「軟體鎖定溢價被逐步 price out」。是「軟硬整合 + 物理互連護城河」特徵。conviction：中高（訓練強、推論蝕）
CORE 🔴

AMD

護城河攻方——ROCm 7.2 達推論出箱 parity（vLLM/llama.cpp），是「推論層 CUDA 裂縫」最直接受益者；但訓練 scale-out 仍落後 NVLink、高階核仍追趕。是「開放軟體棧攻推論」特徵。conviction：中（推論入口、訓練落後）
CORE 🔴

META

抽象治理層的關鍵 owner——擁有 PyTorch（事實標準框架），決定 GPU 是否可替換後端。它既是大買家、又握抽象方向盤，是「2027-28 真贏家層」最純的代表（且自研 MTIA）。市場低估「擁有 PyTorch」的長期戰略價值。是「抽象層治理者」特徵。conviction：中高
SWING 🟡

GOOGL / AMZN / MSFT

逃離 CUDA 最完整者——GOOGL（TPU + JAX/Pallas + ICI 自有互連，訓練推論都能繞）、AMZN（Trainium + Neuron）、MSFT（Maia + 自有棧）。垂直整合程度決定繞過 CUDA 的深度；GOOGL 最深。是「自研矽 + 自有框架垂直整合」特徵。conviction：中
EDGE 🟢

AVGO / INTC

AVGO——framework 抽象化（GPU 變可替換後端）利於 custom ASIC，是抽象層之爭的間接受益者；INTC（Gaudi + oneAPI）是挑戰者但生態落後 ROCm 一個身位。是「抽象化的二次受益 / 落後挑戰者」特徵。conviction：低 · satellite 觀察

估值面總結：本主題不是 TAM 題，而是「NVDA 的軟體鎖定溢價值多少」題。市場給 NVDA 的估值隱含「CUDA 鎖定難以撼動」——但護城河分層意味這個溢價該分層定價：訓練 scale-out 的溢價該保留，推論層與抽象層的溢價該逐步折讓。最大的錯誤定價是把「CUDA 整體不可取代」當前提；一旦市場開始 price「推論層已商品化 + 抽象權移向 framework owner」，NVDA 的軟體溢價會被重定價（不是崩，是 de-rate 到「最強但可競爭」）。de-rating window 在「ROCm 達訓練 parity」或「UALink 大規模商用」那一刻。framework owner（META/GOOGL）的抽象治理價值則尚未被 price。

The One Line

CUDA 護城河不是一道牆、是分層的——推論層水退、訓練 scale-out 牆still 高；2027-28 真贏家不是 GPU 廠，是控制抽象層的 framework owner。

  1. 不告訴你哪一檔該買——NVDA / AMD / META 等是「特徵的代表案例」，買賣由你的個股 DD 與 Pure MA 系統決定，本報告非買入推薦。
  2. 不預測「CUDA 哪年崩」——只給「護城河分層、各層命運不同」的結構框架與觸發訊號，不押二元崩塌點。
  3. 不假裝知道知識邊界以外的事——UALink 硬體 2026 H2 已初步商用，但「hyperscaler 大規模採用速度」、framework 多後端能否推進到訓練，目前無定論，已誠實標為情境變數。
