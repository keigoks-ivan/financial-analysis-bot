💡 §1 Insight
1. 4 層軟體棧的真正分水嶺在 L2（math libraries）+ L3（framework backend）— 不在 L1 driver
事實：ROCm L1 (HIP) 已達基本 parity；L4 vLLM 也達 parity；但 L2 cuDNN / NCCL 與 L3 framework backend 仍是 NVDA 領先 1-2 年。
推論：投資觀察 NVDA moat 進度應看 L2/L3，不是 L1；MIOpen 與 cuDNN 的 feature parity 是真正的 NVDA 護城河指標。
投資意義：每季 AMD 法說會應追蹤 MIOpen feature roadmap（vs cuDNN 9.x），這是真正的 moat-erosion 進度報告。
💡 §1 Insight 2
2. 邊界爭議揭示 hyperscaler 三家在做的不是「換 GPU」，是「重定義 abstraction 層」
事實：META 主導 PyTorch；GOOGL 主導 JAX/XLA/Pallas；AMZN 開源 NKI (Apache 2.0)。三家都在 L3-L4 層建立 hardware-agnostic abstraction。
推論：hyperscaler 並不期望 CUDA 完全消失，而是期望 framework 層獨立於任何 GPU vendor，使 GPU 變成 swappable commodity。
投資意義：META / GOOGL / AMZN 的軟體 moat 被市場低估（estimated TAM $30B by 2030 cross all 3）；NVDA 軟體 moat 估值需要打折。

§10.6 Inference TAM 三段式拆解（v1.1 critic 修補 — 防止 zero-sum 框架誤讀）

讀者常把「AMD 進入 inference 20% share」與「ASIC 整體取代 NVDA 40-50%」當成同一件事來算（zero-sum），這會嚴重誤判競爭結構。實際上 inference 市場可結構性拆成 3 個獨立 bucket，三家終局並行而非互斥：

💡 §10.6 三段式拆解的投資意涵
三家 endpoints 並行不互斥：NVDA 25-30% (commodity) + AMD 25-30% (external + on-prem) + Hyperscaler ASIC 35-45% (captive cloud)

投資意義：long basket 應分 bucket 獨立計算，避免重複扣減。NVDA 在 inference 即使 share 跌至 25-30%，仍是巨大絕對營收（2030 inference TAM ~$200B × 25-30% = $50-60B）。AMD upside 主要看 Bucket B 滲透率（即「外部 model 公司 + on-prem enterprise」），不是搶 hyperscaler 自家的 inference。
