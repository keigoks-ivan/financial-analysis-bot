§1 產業白話定義 + 歷史脈絡

這份報告講的不是某一顆晶片，而是「全球科技巨頭為了跑 AI，正在花多少錢蓋運算工廠、這些錢由誰最後買單、會不會蓋過頭」。買方是 Microsoft、Google、Amazon、Meta 這些雲端巨頭（hyperscaler，指自建超大型資料中心的雲服務商）；他們把錢付給晶片設計商、代工廠與記憶體廠；最終要靠 AI 應用的營收把這筆投資賺回來——而那一步，正是爭議所在。

白話開場：所謂「AI 算力資本週期」（capex supercycle，capex = 資本支出，指買地、蓋廠、買伺服器這類長期投資），就是一條從「花錢蓋運算產能」到「靠 AI 服務變現」的資金鏈——這條鏈在 2026 年規模已達史上首個兆元年（~$1.04T）。鏈的最上游是出錢的 hyperscaler 與新興的「主權 AI」（sovereign AI，指國家為資料在地化自建的 AI 基建）；中間是把錢變成算力的晶片設計（NVIDIA）、代工（TSMC）、記憶體（HBM，高頻寬記憶體，AI 晶片的專用高速記憶體）；最末端是 OpenAI、Anthropic 與雲端 AI 業務這些真正向終端用戶收錢的人。整個週期的命門就一句話：上游花的錢，最末端賺得回來嗎？

邊界界定：本報告把焦點放在「資本週期的永續性與過剩風險」這個母題（涵蓋 2022 年至今的這一輪 build-out），因此 in-scope 的是 capex 規模、供給瓶頸（CoWoS / HBM / 電力）、利潤池分布、終端變現缺口與資本週期訊號（折舊、發債）；out-of-scope 的是單一晶片世代的技術細節（那是子題 ID_AIAcceleratorDemand 的事）與單一資料中心的工程設計。灰色地帶在「neocloud（算力轉售商，如 CoreWeave）」——它既是需求方（向 NVDA 買卡）又是供給方（向 OpenAI 租算力），本報告把它當「資本週期的高槓桿放大器」看，而非獨立環節，因為它的脆弱性正來自跨越供需兩端的舉債。

第一次轉折：2022-11 ChatGPT 發布，兩個月內 MAU 衝破 1 億；自此四大 hyperscaler 的 AI 相關 capex 從 2022 約 $162B 一路翻到 2025 約 $448B（近 3 倍），Epoch AI 直接以「自 GPT-4 以來 hyperscaler capex 翻 4 倍」為標題。轉折的機制是三力共振：① frontier model 訓練成本隨 compute 規模指數上升；② 已部署產品的 inference（推論，指模型實際回答問題的運算）需求超出現有容量；③ 各家自研 silicon（如 Google TPU、Amazon Trainium）以建護城河、降低對 NVDA 的依賴。

第二次轉折：2025-01-21 Stargate Project 宣布——4 年投 $500B 建 AI 基建（供 OpenAI 用），$100B 立即部署，由 SoftBank / OpenAI / Oracle / MGX 領投。這標誌資本週期從「hyperscaler 內生現金流投資」升級到「跨主體、帶 SPV 槓桿的兆元級承諾」。到 2026-Q1 法說，四大 2026 capex guidance 合計已上看 $725B（YoY +77%）：Amazon ~$200B、Microsoft $190B（其中 $25B 來自記憶體/零組件漲價）、Alphabet up to $190B、Meta $125-145B；含 Oracle 的五大則 widely forecast >$600B（+36%）。

歷史類比一：2000 電信光纖泡沫。當年（2000）peak telecom capex 約 $120B，boom 期（1995-2000）累計投入估達 $2 trillion，鋪了 80-90M 哩光纖。殺手數據是：2001 年僅 5% 光纖容量被點亮（95% dark fiber），到 mid-2000s 仍約 85% 閒置，bandwidth 價格早 2000s 跌約 90%，這個 glut 花了近 10 年才 backfill。當年多頭（WorldCom / Global Crossing）錯在押「網路流量每 100 天翻倍」的假設過度建設；空頭則低估了長期流量真的會長上來（只是慢了 10 年）。本次的量化差異：① 光纖資產壽命長但利用率近零、無對應營收，而 GPU 折舊僅 3-5 年卻能即刻產出可計價 token；② 電信當年靠高槓桿舉債，本次多為自有現金流（惟 2025-09 起已現外部發債裂縫，見 §5）。

歷史類比二（最貼近）：2018-2019 雲端第一波 digestion。Amazon capex 2018 $27.6B → 2019 ~$32B（僅 +15%，增速大降）；同期 NVDA Data Center 營收的時序最說明問題——Q3 FY2019（peak）record $792M（+58% YoY）→ Q4 FY2019 $679M（+12% YoY、環比 -14%）→ Q1 FY2020 $634M（-10% YoY）。也就是說，capex YoY 從高雙位數降速後，約 2-3 個 quarter 就把 NVDA DC 營收從 +58% 翻成 -10%。教訓清楚：本次若 2026 capex +77% 在 2027 急降速，傳導到上游營收的時滯也約 2-3 季——這正是 §5 三視野表 12M 與 3Y 分歧的根源。

§2 技術成熟度 + S 曲線

為什麼算力需求是「現在」爆發，不是三年前、也不是三年後？因為兩件事剛剛同時發生：模型從「會聊天」進化到「會自己做事」（agentic），讓單一任務的 token 消耗暴增 5-30 倍；而推論成本同時年降 5-10 倍，讓「用 AI 做事」第一次在經濟上划算。需求曲線與成本曲線的剪刀差，就是這輪資本週期的引擎。

技術成熟的因果鏈有兩段。第一段是能力門檻被跨過：2025 起 pre-training（預訓練，用海量資料把模型「教大」）的 scaling law 出現 diminishing returns（收益遞減，因資料瓶頸），前沿轉向 test-time / inference compute（推論時運算，o1/o3、Gemini Flash 這類「邊想邊算」的模型）。這個解耦很關鍵——它把運算需求從「一次性的超大訓練」轉成「每次回答都吃算力的持續性負載」，需求從脈衝變成洪流。第二段是 NVIDIA 在 GTC 2026（San Jose）給的框架：Jensen 稱過去 2 年 computing demand 增加「100 萬倍」（單一 work 的 compute 需求增 ~10,000x × usage 增 ~100x），並宣告「inference inflection has arrived」——AI 已能做 productive work，需求結構從訓練轉向推理與即時行動。agentic 是乘數：reasoning 模型（會反思、規劃）→ Claude Code 這類「第一個 agentic model」（自己讀檔、編碼、編譯、測試、迭代）→ 多步 workflow 自主執行，每一步都吃 token。佐證在量上：NVDA 揭露 AI inference 的 token 生成「一年內增 10 倍」。

因果閉合（回應 §1 提出的結構變數）：§1 點出三個結構變數——CoWoS/HBM 雙鎖、自研 silicon 護城河、token 成長機制。其中 CoWoS/HBM 雙鎖在未來 3-5 年是否仍 binding？答案是「2026-27 仍 binding、2028 後鬆動」：CoWoS 2027 底擴到 170K wpm、HBM4 三家增產，到 2028 供給彈性才追上，故瓶頸的議價權紅利還有約兩年。而自研 silicon 護城河正在從「降低對 NVDA 依賴」變成「實際侵蝕 NVDA 份額」（ASIC 44.6% CAGR），這個結構變數在未來 3-5 年只會更 binding，不會消退——這也是為何 §7 把「利潤池會否被自研侵蝕」列為核心分歧。

供給側給出一個反直覺但可操作的結論：瓶頸（CoWoS/HBM/電力）是這輪資本週期的朋友，不是敵人。因為產能上線有 2-4 年的結構性時滯（封裝滿到 mid-2027、變壓器 2-4 年、interconnection 4-10 年），即使資金狂湧，短期內也很難真的「蓋過頭」——這把過剩風險從 2026 往後推到 2027-28。對 PM 的含義：掌握瓶頸的 TSM（CoWoS 守門人）與 HBM 三寡占享有最持久的議價權紅利（約兩年），是比 merchant GPU 更安全的押注；而 ODM 的 ↓ 利潤池與 neocloud 的未證利潤，則是這條鏈裡該避開的「量大錢薄 / 高槓桿」陷阱。

§4 需求側：現在與未來 + 三角驗證

誰在買算力、為何買、哪塊需求最不可替代？現在的買家高度集中在四大 hyperscaler（2026 占 ~$725B / $1.04T 總 capex），但需求正在外溢到主權 AI 與企業端；最不可替代的是 frontier lab 的訓練需求（連兩代舊卡都租不到），最脆弱的是企業推論需求（MIT 稱 95% pilot 零 ROI）。

現需結構：2026 是史上首個算力 capex 兆元年（~$1.04T），拆分為 Big-4 hyperscaler $725B + Oracle $50B + Apple $13B + Neoclouds $60B + 中國 $80B + 主權 $60B + 其他 $48B。需求廣化的最強訊號來自 NVDA：Q1 FY2027 DC 營收 $75.2B 首次出現 Hyperscale（~$38B）/ ACIE（cloud builder + enterprise + sovereign，~$37B）近乎對半——代表 AI 需求已外溢出 Big-4。pricing power 的分層很關鍵：frontier lab 訓練需求最 price-inelastic（Jensen 指連兩代舊卡 spot 租金仍漲、租不到，capacity-constrained）；企業推論需求較 elastic（對 token 單價 / ROI 敏感），是過剩風險的主要承壓點。

用三情境把 2030 TAM 框起來，每個數字都能回溯到「假設改了什麼」——但先看口徑，三家機構的數字根本不是同一指標，不可直接並列。

因果閉合（回應 §1 的需求驅動）：§1 提的需求驅動是「生成式 AI + agentic 拐點」。未來 5-10 年走向？需求量能側強多頭：Google token 量 3.2 quadrillion/月（YoY +7x）、agentic 每 task 耗 token 為 chatbot 的 5-30x、Anthropic 5 個月 5x。但拐點的持續性取決於 Gartner 點出的關鍵——「迄今 AI 支出主要由科技公司 / hyperscaler 驅動，企業端尚未真正發力，2026 預期為 inflection year」。也就是說，需求驅動的下一棒（企業端買單）尚未驗證，這是把 thesis 從「結構性」打成「mixed」的原因：量能（token）真實，但變現（企業 ROI）未證。

需求側留給 PM 兩個可操作判斷。其一，TAM 的口徑亂象（$1.7T vs $7T）提醒我們別用機構大數字當 thesis 錨——真正該盯的是 7-9x 的變現缺口能否收斂，而不是 TAM 多大。其二，需求的「不可替代性」是分層的：押 frontier 訓練曝險（NVDA 高階 / TSM N2 / HBM4）比押企業推論曝險（吃 token 單價的環節）更安全，因為前者 price-inelastic、後者是 MIT「95% 零 ROI」的承壓點。這直接影響 §9 的 ticker 深度排序。

§5 供需裁決 + 短中長期推估 + 投資時鐘

⚠️ 本章含 judgment：🟡 bullet 代表推論，附事實鏈與信心度 + ⚠ 證偽條件
綜合 §3（供給）與 §4（需求），結論方向是：未來 12 個月供給短缺（lead time 36-52 週、CoWoS 滿到 mid-2027、租價反漲）；未來 3 年轉向平衡偏緊，因為產能 2027-28 才追上而變現缺口同時擴大。這不是過剩週期的頂，而是「build-out 末段、早期裂縫已現」的 Phase II。

資本週期證據（裁決的量化依據，至少引 2 項）：用「高報酬引資本→過剩→均值回歸」框架檢查三個指標：

  • ① 折舊年限變更（最硬訊號）：hyperscaler 集體把 server 折舊由 3-4y 延至 5-6y（Alphabet 2023 server 4y→6y、Amazon 5y→6y、Meta 漸延至 5.5y），合計每年省 ~$18B 折舊、機械性 flatter earnings。但 Amazon Q1 2025 反向把部分 server 6y→5y，理由「AI/ML 技術迭代加速」——這是過剩風險的早期反向訊號（T1 動作）：當折舊開始縮短，代表資產實際壽命被重估、回收期被質疑。但須限定 scope：此次縮短僅適用部分 server 資產（AI/ML 伺服器子集），非 Amazon 全艦隊折舊政策翻轉，財務影響約 $0.7B/年（季度 D&A +~$217M）——故這是「一家、一子集」的早期訊號，本 thesis 監測的是「是否有第二家跟進」（§7 NC#2 / §8 證偽表），而非把它讀成 fleet-wide 政策反轉。矛盾點在於 NVDA 旗艦 GPU 實際 ~3y 換代（每代效能 2-3x），hyperscaler 卻仍多記 5-6y 折舊。
  • ② capital intensity / ROIC（方向性）：capital intensity 達營收 45-57%（歷史「不可想像」水準）[A: 以 Introl 2026 估算為錨，各家口徑略異]；capital base 增速 >> 營收/EPS 增速 → ROIC 受壓（mixed，META 因增收 + 減本 ROIC 反升）。缺各家精確逐年 ROIC 數列，故此項為方向性判斷。
  • ③ FCF 拐點 + 外部發債（最清晰拐點）：2025-09 起 hyperscaler 集體轉外部融資，capex 開始超過 operating cash flow——META 2025-10 發債 $30B、ORCL 2025-09 發債 $18B、2025 全年 AI-related 新債 ~$121B，MS/JPM 預估 tech 未來數年需再發 ~$1.5T。這是資本週期進入「外部融資驅動」late-cycle 特徵的定義性訊號。
§3 + §4 → 供需裁決

依本章 §3 與 §4 的推導，未來 12 個月供給短缺、未來 3 年轉向平衡偏緊，因為：（短期）供給端三個硬證據——GPU lead time 36-52 週、Blackwell sold out 至 mid-2026（backlog 3.6M units）、CoWoS full allocation 至 mid-2027——加上 H100 一年租約價反漲 +40%，沒有任何全面 glut 跡象；（中期）2027-28 產能（CoWoS 170K wpm、HBM4 三家增產）才追上，而同期變現缺口 7-9x 若未收斂、折舊縮短若擴散，需求端會出現「急煞」風險，使供需從短缺翻向平衡偏緊。不騎牆的關鍵：現在的風險不是產能過剩，是「變現追不上 + 槓桿上升」——這是 financial 過剩而非 physical 過剩。

§6 產業經濟學與估值傳導

這條鏈的錢怎麼賺、估值跟著什麼變數動？簡單說：上游靠「瓶頸 + 高毛利 + 量價齊揚」賺結構性利潤，估值則跟著「成長假設兌現度」走——而非倍數本身。當倍數已不貴（NVDA Fwd P/E 低於歷史均值），re-rating 的鑰匙就交到 E（盈餘）能否追上 consensus 手裡。

估值傳導：產業變數如何 pass-through 到公司 Fwd PE/PEG？關鍵在於本輪的傳導不是透過倍數擴張，而是透過 E 的高速 catch-up。以 NVDA 為例，Fwd P/E ~22.5-24x 反而低於半導體 industry median 34（-33.8%）與自身 3/5/10 年均值——這代表市場並未給泡沫式倍數，估值壓力全部轉移到「FY27 consensus ~$391B 營收這個 E 能否兌現」。敏感度錨點：若 FY27 營收兌現度從 100% 降到 75%（bear），E 下修約 25%，在倍數不變下股價約 -25%；若變現缺口收斂帶動 consensus 上修 +1pp 成長預期，因倍數已低、re-rating 空間反而大（low base effect）。換句話說，這是一個「倍數安全、E 危險」的估值結構——與 2000 電信「倍數瘋狂」恰好相反。

估值傳導給 PM 一個關鍵校準：別用「PE 不貴」當安全感。NVDA Fwd P/E 低於歷史均值看似便宜，但那是因為 E 已被 plug 進高速成長——真正的風險不在倍數會不會壓縮，而在那個高 E（FY27 ~$391B）的成長假設會不會落空。這把監測重心從「估值分位」轉到「§8 證偽表的營收 / capex 增速 metric」。對 entry 的含義：倍數低 = 不必等大幅 de-rating 才進場，但要在 E 假設出現裂縫（capex 增速反轉）時果斷降曝險。

  • 反方 2：circular financing 崩塌——路徑：NVDA→OpenAI $100B、Stargate $300B、CoreWeave↔Oracle↔Microsoft 資金循環一旦一環斷裂（如 Stargate 傳「stall」），連環營收同時縮水。當前證據：Bloomberg 統計 >$800B 連環交易；2026-06 selloff 點名此疑慮。證偽窗口：任一大型 SPV / neocloud 出現債務違約或合約取消。
  • 反方 3：企業端 ROI 證偽（MIT 路徑）——路徑：MIT 稱 95% 企業 GenAI pilot 零可量測 P&L 影響，若 2026「企業 inflection year」落空（Gartner 預期），需求的下一棒接不上，token 量能成長無法轉成終端營收。當前證據：MIT NANDA 報告（$30-40B 企業支出基礎）。證偽窗口：2026H2 企業 AI 採用率 / ROI 數據。

市場最低估的風險：不是「需求消失」（多頭空頭都在吵這個），而是折舊政策的反身性——一旦第二、三家 hyperscaler 跟進 Amazon 把折舊縮短，會同時做兩件事：壓低 reported earnings（讓本來就高的 E 假設更難達標）+ 公開承認資產壽命下修（坐實 GPU 回收期疑慮）。這個會計訊號比任何分析師的 bear note 都更有殺傷力，因為它是 hyperscaler 對自己資產的誠實重估，市場目前幾乎沒在 price 這條路徑。

三條分歧裡，分歧 1（供給短缺）最可操作——它對（租價硬證據）且未 priced（倍數偏低），是清晰的 long 理由。分歧 3（利潤池集中）次之，對且部分已 priced，風險報酬轉佳，支持逢 selloff 建立 AVGO。分歧 2（變現缺口）雖對但樂觀已 price 進 E，故它不是 long 理由而是風險控制線——當折舊縮短擴散或變現比惡化，這條分歧由 INTACT 轉 BROKEN，就是降曝險的訊號。
