§1 產業定義與邊界

要討論 Agentic AI 平台的投資機會，最容易犯的錯是把「AI 」「LLM」「Agent」「Copilot」攪在一起。這個產業其實有四個截然不同的層次，每一層的贏家邏輯、護城河來源、以及估值倍數都完全不同。本 ID 把這四層拆開來看，讓你看到資金在哪一層流動、誰在哪一層建立真正的護城河。

四層分工（In-scope）

L1 — Horizontal 平台（跨 vertical 通用）是指 PLTR AIP+Foundry、CRM Agentforce 360、MSFT Copilot Studio+Agent 365、GOOGL Agentspace+Vertex+A2A、以及 SAP Joule 這一批。它們的核心是 ontology / data cloud / agent builder / orchestrator，賣的是「在我的平台上可以建任何垂直的 agent」。2026 年這一層已從「demo 期」進入「商業化驗證期」——CRM Agentforce 已公告 $800M ARR（+169% YoY）、PLTR 美國商用 +137% YoY、NOW Now Assist ACV 突破 $600M，這三個數字是本 ID 最硬的量化 anchor。

L2 — Vertical SaaS 內嵌 Agent是現有 vertical SaaS 把 AI agent 塞進自家產品裡，依賴既有的 regulated data 與 workflow schema 做差異化。NOW Now Assist、WDAY Illuminate、HUBS Breeze、INTU GenOS、ADBE Firefly+Orchestrator、TEAM Rovo、ZM AI Companion 3.0、DOCU IAM、TWLO CustomerAI 都屬此層。關鍵分化點是 data moat 深度：INTU（IRS e-file + 財稅資料）、WDAY（payroll ledger）、NOW（IT CMDB）、DOCU（合約簽署資料）因為守著 regulated schema，短期被 horizontal 平台吞食的風險低；相反，HUBS、ZM、TWLO 這種以 communication data 為主的玩家，已經開始被 Agentforce / MSFT Copilot 擠壓（HUBS 2026-04-14 被迫把 Customer Agent 從 $1 降至 $0.50/resolved）。

L3 — Framework 與 Agent Infra是開發者端的基礎設施，分兩個子層。Framework 層有 LangChain/LangGraph、CrewAI，以及 LLM 原廠自家的 Anthropic Agents SDK、OpenAI Agents SDK、MSFT Agent Framework 1.0、Google ADK。Infra 層有 Cloudflare Workers+Code Mode、AWS Bedrock AgentCore、以及 Anthropic 捐給 Linux Foundation 的 MCP 協議（已有 10,000+ servers）。這層的利潤分化極端：Framework 正在商品化（LangChain 估值 $1.25B 但 ARR 只 $12-16M，是 75-100x multiple），而 Infra 則是結構性贏家（Cloudflare Q4 2025 單一合約 $42.5M ARR，Workers AI req YoY +4,000%）。

L4 — Pure-play AI Agent 新創是專攻單一垂直的 agent 公司：Sierra（客服，$4.5B 估值）、Decagon（客服，$4.5B，14 個月三倍跳）、Harvey（法律，$11B / $190M ARR）、Cognition/Devin（coding，$10.2B）、Cohere（企業 LLM，$7B / $240M ARR）、11x（SDR）、Clay（GTM enrichment，$3.1B）、Poolside（coding LLM，$3B）。這一層融資非常熱，2024-2026 累計超過 $5B。Cohere 是 2026 年最可能的 IPO 候選，Harvey 58x ARR multiple 已經逼近 Wiz 併購水位（32x），是下一個 Wiz-like exit 最可能的標的。

Out-of-scope（本 ID 不主攻）

幾個相關但獨立成篇的主題不在本 ID 範圍內：LLM 原廠模型能力評比由 M11（ID_LLMVendorSecurityEconomics）處理；AI × 資安四層由 M2（ID_AICybersecurityDoubleEdge）處理；身份 / NHI / agent identity 由 M5（ID_IdentityNewPerimeter）處理；半導體 / AI 推論晶片由 ID_AIAcceleratorDemand 處理；AI 攻擊面 / AI-SPM 由 M2 處理。這些議題在本 ID 出現時會以「見 XX ID」方式引用，不重複鋪陳。

灰色地帶

這 4 層分工並非完全乾淨，有幾個玩家卡在分類邊界上，而邊界爭議本身常常是 alpha 來源。

MSFT 究竟是 Horizontal 還是 LLM 原廠？三個角色它都有：Copilot Studio（horizontal agent builder）、Azure AI Foundry（LLM infra）、MAI-1 / Phi-4（自建模型）。本 ID 把它歸為 L1 Horizontal，因為 Copilot Studio 已鋪在 230K+ 組織的 Fortune 500 80% 中。但這個分類遺漏了它的真正價值——2025-10 MSFT 重簽 OpenAI 協議後獲得獨立 AGI 權利、Phi-4 / MAI-1 走 on-device + routing 路線、2026-04-02 首次發表自家 MAI-Transcribe-1 / MAI-Voice-1 / MAI-Image-2 對標 Whisper / DALL-E。換句話說，MSFT 的「自主可控度」是所有 agentic 玩家中最高的，這是市場還沒充分定價的點。

PLTR 算 Horizontal 還是 Framework？Foundry + AIP + Apollo 三位一體既是 agent orchestration 平台（類 L3）也是 vertical deployment 工具（類 L1+L2 混合）。本 ID 歸 L1 的理由在於它賣的是「一個平台跑所有垂直」，但真正的護城河不在 orchestration，而在可以在 FedRAMP High / IL5-6 這種分級環境部署 agent——這是 CRM、MSFT、GOOGL 短期內都做不到的，也是 Maven $1.3B 合約擴到 2029 的根本原因。

Cohere 算 LLM 原廠還是 Pure-play？它是 enterprise-only LLM（跟 OpenAI / Anthropic 的 consumer + enterprise 雙軌不同），ARR $240M（2025）、估值 $7B、Nvidia + AMD + Salesforce 戰略持股。本 ID 暫歸 L4 pure-play，但如果 2026 IPO 估值超過 $10B，就該重新分類為小型 LLM 原廠。觀察點：Cohere 是不是繼 Wiz 之後下一個資安以外的 IPO 大戲。

💡 Insight §1：三個 take-away
第一，要把 Agentic AI 拆成四層來看，而不是籠統地問「AI agent 賺錢嗎」——每一層的贏家邏輯、估值倍數、退出路徑都不同，L1 靠 distribution + data moat、L2 靠 regulated schema、L3 infra 和 framework 兩極分化、L4 靠 vertical specialization。第二，灰色地帶就是 alpha：MSFT 的自主可控度、PLTR 的 FedRAMP 護城河、Cohere 的 IPO 時機，這三個點都是市場分類框架沒抓到的機會。第三，Agent identity / NHI / MCP 協議在 M11（ID_LLMVendorSecurityEconomics）已經處理過，本 ID 不重複，重點擺在 application 和 vertical 的利潤如何分配。

§2 技術演進 + S 曲線 + 歷史類比

Agentic AI 平台演進：五個階段的贏家與輸家

要理解現在這個時點的投資機會，先把過去 10 年拆成五個階段看。每個階段都有一批贏家和一批被錯過的輸家，而現在我們正處於第三階段——也就是商業化驗證期——的中段。

第一階段（2015-2022）是 RPA + UI automation 時代。這是沒有認知能力的 rule-based bot 時代，UiPath 靠這個在 2021 IPO 衝上 $35B 市值。但當時的架構假設是錯的——bot 不懂上下文，只會執行寫死的腳本，這讓 UiPath 被後來的 LLM 時代打了一個 80% 的折扣（股價從 $85 peak 跌到 $10）。Blue Prism 更慘，2022 被 SS&C 以 $1.6B 私有化，幾乎沒討論空間。

第二階段（2022-2024）是 LLM chatbot 嵌入 SaaS 的時代。ChatGPT 爆紅後，MSFT 365 Copilot 是最早的跟隨者，SAP Joule、Salesforce Einstein 也同期推出。這些第一代產品的問題是它們只是「單一 LLM call 輔助使用者」——沒有真正的 agentic workflow。這解釋了為什麼 Einstein 1.0 在 2022-2024 幾乎沒貢獻 ARR，直到 2024 底被重新包裝成 Agentforce 才開始有實質數字。MSFT Copilot 是個例外，因為它搭在 M365 的 distribution 上，靠 seat 經濟先取得規模。

第三階段（2024-2028，現在）是 Agentic 商業化驗證期——這是本 ID 的核心。這階段的產品已經是 multi-step agent + tool use，並且開始有 outcome-based pricing（Sierra、Decagon、HUBS 的 $0.5/resolved 都屬此類）。贏家名單現在已大致清晰：PLTR、CRM、MSFT、NOW、WDAY 這五家 Horizontal + Regulated Vertical 拿走主要的 agentic ARR；Harvey、Cohere、Sierra、Decagon 則是 pure-play 中的 Wiz-like 候選。輸家也很明確——Framework 獨立廠正在商品化（LangChain 估值跟 ARR 脫鉤），communication SaaS（HUBS、ZM、TWLO）被 horizontal 擠壓。

第四階段（2028-2031）會是 Agent-to-Agent + Autonomous 時代。跨廠 agent 互通（A2A、MCP）成為標準，多 agent 可以自主跑 L3-L4 workflow。Gartner 預測那時 agent 數量會是使用者的 10 倍。贏家條件會變成「有 ecosystem + identity 中立層」——OKTA XAA（已在 M5 覆蓋）、Cloudflare agent infra、Anthropic MCP 這三個中立層地位會是關鍵。封閉 stack（如果 MSFT Agent 365 走向封閉）反而會是輸家。

第五階段（2031 後）是 AGI-assisted enterprise 時代，frontier model 達到通用 AGI 等級。這階段還太遠，贏家未定，但可以肯定的是本 ID 的投資判斷都要重估。

這段演進告訴我們什麼？當前最關鍵的認知是「我們還在第三階段的早到中段」——Gartner 預測 2026 年才會有 40% 的企業應用含 agent（對比 2025 年的 < 5%），這意味第三階段 2-3 年內的成長動能還非常強。但同時，贏家名單已經開始分化，純觀望會錯過 catalyst；而跟風 framework 廠（LangChain、CrewAI）則有下行風險。

★ 歷史類比：三個前例，以及為什麼都不完全一樣

Agentic AI 這波的最大風險是「歷史類比用錯劇本」。投資人常把這波比作三種歷史情境，但每一種都有關鍵量化差異，直接套劇本會得出錯誤結論。

類比一：1995-2005 ERP 雲端轉型（Salesforce.com 顛覆 Siebel）。這波最常被拿來類比的前例——當時 Salesforce 靠 SaaS 模式把 Siebel 完全顛覆，Siebel 2005 被 Oracle 以 $5.85B 收掉。當時空頭低估 SaaS 的取代速度，多頭則低估 Siebel 客戶的黏性，結果空頭先輸、多頭後輸。本次的相似處在於 Horizontal 平台（Agentforce）正在顛覆 vertical legacy（例如早期 CRM on-prem），軌跡也很像——CRM 2010 市佔從 0 起步、2020 達 20%，Agentforce 現在走類似曲線。但關鍵差異是 Vertical SaaS 如 WDAY、NOW 已經是 SaaS-native 架構（不是 on-prem），它們的護城河來自 regulated schema 而非「雲端 vs on-prem」的差異；這讓「Siebel 劇本」不能直接套在 WDAY/NOW 身上，這些公司不會像 Siebel 一樣被完全取代。

類比二：2015-2022 第一代 AI copilot（Salesforce Einstein / SAP Leonardo / Oracle AI）為什麼失敗？這個前例常被拿來警告「Agentforce 會不會是第二個 Einstein 1.0」。當時 SAP Leonardo 在 2022 直接被放棄、Einstein 1.0 必須重新包裝為 Einstein Copilot。空頭因此懷疑 Agentforce 也會失敗。但關鍵差異在於基礎技術：GPT-4 / Claude 3.5 的能力遠勝 2018 年的 Einstein，而且 Agentforce 2025-2026 已實際收 $800M +169% YoY 的真實 ARR（不是 Einstein 1.0 的空氣行銷）。相似的警訊是 70% human-in-the-loop 的採用天花板仍適用，但這個限制已反映在 Agentforce 的 outcome-based pricing（客戶只有在真的有結果時才付錢）上，結構上已不同於 Einstein 1.0 時代的 seat-only 銷售。

類比三：2008-2018 雲端 infra（AWS 崛起）vs SaaS 應用。這個前例關心的是「Anthropic / OpenAI 這種 LLM 原廠會不會吃下 SaaS 應用層」？當時空頭誇大 AWS 對 SaaS 的吞食，但多頭也忽略了 Snowflake、MongoDB、Databricks 這種靠 data gravity moat 守住獨立性的玩家。Anthropic 現在正走 AWS 劇本——push primitives（MCP 協議、Agent SDK）讓上層應用繁榮而不是自己直接吃。Claude Code（$2.5B ARR）是唯一的 vertical 例外，且它的客戶結構（Amazon $25B 投資、Google 14% 持股）在結構上限制了 Anthropic 往其他 vertical 擴張的動機。歷史數字也支撐這個判斷——AWS 在 2015-2020 並沒有把 SaaS 吃完，平均 vertical SaaS 還是 30%+ CAGR 成長。

總結歷史類比的啟示：Agentforce 最接近「ERP 雲端轉型」而不是「Einstein 1.0 崩盤」；Anthropic 最接近「AWS push primitives」而不是「MSFT 壟斷 90 年代桌面」。錯套劇本會導致兩個錯誤——把 WDAY/NOW 當 Siebel short、或把 Anthropic 當未來的 CRM vertical SaaS 吃食者。

⚠ 反向類比警示：最常見的錯誤類比是「所有 vertical SaaS 都會像 Siebel 一樣被 Agentforce 殺掉」。這個類比的四個量化前提都不成立——NOW / WDAY / INTU 擁有 regulated data + workflow schema（Siebel 當年沒有）；CRM 本身就是 vertical SaaS 而非通用 database 玩家；今天是 MSFT M365 distribution 與 CRM Data Cloud 雙龍頭格局，不像 2000 年 Siebel 是單一玩家；Outcome-based pricing 讓 vertical SaaS 有抵擋 horizontal 的機制。四條全破，所以不可直接套用 Siebel 崩盤劇本。但 communication / 輕 CRM vertical（HUBS / ZM / TWLO）確實在被擠壓，這是一條真實存在的細分劇本。
💡 Insight §2：三個 take-away
第一，Einstein 1.0 之所以失敗不是 Salesforce 策略錯，而是 2018 年 LLM 基礎能力不足。要到 2022 ChatGPT 之後 Agentforce 才有可能真正商業化，所以當前類比應該用「ERP 雲端轉型」而不是「Einstein 崩盤」。第二，AWS-SaaS 這段歷史最接近 Anthropic 的現在策略——push primitives（MCP 協議、Agent SDK）讓上層應用繁榮，自己靠下游 token 經濟賺錢，不做 vertical 競爭。第三，Claude Code 的 $2.5B ARR 是 Anthropic 自己打臉這個策略的 exception——它是個直接做應用層的案例。這個例外會不會擴展到 security、legal、medical 是 2027 年最重要的觀察點；如果會，那 Anthropic 就是 LLM 原廠中唯一向 vertical SaaS 伸手的玩家，投資結論會反轉。

§3 核心技術棧拆解

Agentic AI 平台由 8 個關鍵子技術堆疊而成。不是每個都同等重要，有 3 個是「kingmaker」——控制這 3 個的玩家就能長期佔位。以下先看 8 個子技術的競爭格局，再解釋為何 3 個特別關鍵。

Ontology / Data Fabric是最底層，負責跨 silo 資料連結、語意層、以及 permission / action binding。這是 PLTR Foundry 的護城河所在——它手握 3,500+ 相關專利，且 Foundry 把 ontology 定位成 agent 行動的「canonical schema」，客戶一旦把 business logic 寫進去就難以遷出。CRM Data Cloud、MSFT Fabric 正在追趕，但 PLTR 的獨家性最深。Agent Reasoning Engine則是最上層的認知能力——Anthropic Claude Mythos 以 OSWorld 79.6% 奪下 SOTA，超過 OpenAI ChatGPT Agent（CUA 38.1%）、GOOGL Gemini 3 Flash、以及 CRM 自家的 Atlas Reasoning Engine。Mythos 目前限制在 Project Glasswing < 40 家夥伴 exclusive（CRWD / PANW / MSFT 等資安龍頭），這是市場沒完全看到的訊號——頭部資安 SaaS 已取得最強 frontier model 的代差優勢。

Agent Builder / Low-code是 MSFT 的勝場——Copilot Studio 鋪在 230K+ 組織、Fortune 500 的 80%，這是目前 agent builder 市佔最寬的基礎，CRM Agent Builder 緊追在後。Agent Orchestrator是多 agent 協調與衝突解決層，CRM Agentforce 360 Command Center、PLTR Apollo、NOW Now Assist Orchestrator、MSFT Agent 365（2025-11 Public Preview）、UiPath Maestro 都屬此類。Agent Identity / NHI（OAuth 2.1 credential + MCP PKCE）由 OKTA XAA、MSFT Entra Agent ID、CYBR-Venafi 主導，詳細已於 M5 ID_IdentityNewPerimeter 覆蓋。

Consumption Metering是 2026 的轉折點——從 seat-based 轉為 outcome-based。CRM Flex Credits（$2/conv 或 $0.10/action）、MSFT $30/user Copilot、HUBS $0.50/resolved、Harvey hours-replaced 是四種不同的變種。outcome-based 在 2026 年正式變成主流範式。Agent Observability（trace / eval / prompt debugging）是 LangSmith、Datadog LLM Observability、Arize、Helicone 的戰場，但這層商業化薄——LangSmith 整家 LangChain 的 ARR 只 $12-16M，說明觀察層很難獨立變現。MCP / A2A 協議是最新的戰場——Anthropic MCP 2025-12 捐給 Linux Foundation 後有 10,000+ servers、97M/月 SDK 下載，已成事實標準；Google A2A 則 150+ production orgs 並行，互補不衝突。

★ 三個 Kingmaker 子技術

8 個子技術中有 3 個特別關鍵，控制這 3 個的玩家能長期佔位。

第一個是Ontology 加 Regulated Data。PLTR 的 ontology、WDAY 的 payroll + business process framework、NOW 的 CMDB 都屬此類——這三類 schema 是 LLM 原廠無法自建的，因為它們代表的是客戶企業內部的 business logic 與資料存取權限。LLM 原廠能做出 reasoning engine，但做不出來「這個員工可以存取哪張表、可以執行什麼動作」的 schema。這解釋為何 Anthropic 與 OpenAI 都沒有（也不會）做 HR SaaS。

第二個是Agent Builder 的 distribution 規模。MSFT Copilot Studio 已有 230K+ orgs 使用、Fortune 500 的 80% 部署，這是 agent builder 市佔最寬的基礎。CRM 緊追，但 MSFT 靠 M365 既有通路佔得先機。這在 2027-2028 會放大——當更多企業開始實際部署 agent，已有 builder 的玩家拿到回饋資料、更容易優化，形成正向循環。

第三個是Outcome-based Pricing 的 measurement infrastructure。HUBS 從 $1/conversation 改為 $0.50/resolved conversation、Harvey 用 hours-replaced 計費——這是 agentic SaaS 的新定價範式。誰先建立「如何量化 agent 成果」的 infra，誰就能在 2026-2028 價格戰中拿到定價權。這個 measurement 能力比 agent 能力本身更有護城河——因為定價結構一旦客戶接受，切換成本極高。

💡 Insight §3
PLTR 的 ontology 是 Horizontal 平台中最深的護城河，但也是採購 friction 最高的——客戶必須先把 business logic 寫進去才能鎖定，所以雖然護城河深，卻 growth 曲線較陡、採購週期較長。MSFT 則走相反路線：Copilot Studio 230K 組織是最寬的 agent builder distribution 基礎，勝利方式是「先佔位再優化」，它的 230K orgs 會在未來變成 agent 的 flywheel。而 Outcome-based pricing 則是 Vertical SaaS 反守為攻的關鍵——HUBS 的 $0.5/resolved 和 Harvey 的 hours-replaced 是先行者，如果這個定價模式 2026-2027 擴散到整個 SaaS 類別，會重新定義「AI ARR 品質」這件事的衡量方式。

§4 市場規模 + CAGR

Agentic AI 的 TAM 估算是本 ID 最令人困惑的部分——不同研究機構給出的數字差距可達 10 倍以上。理解這個差距比記住某個數字更重要，因為不同的 TAM 數字反映了分析師對「什麼算 agentic」的假設差異。我們先釐清口徑，再看實際數據。

最窄的嚴格口徑是 MarketsandMarkets 的數字：2025 $7.84B、2030 $52.6B，CAGR 46.3%——這是純 Agentic AI Platform（不含 LLM infra、不含普通 AI 功能）。Bain / McKinsey 的廣口徑（含 platform + applications）則估 2025 $12-15B、2030 $80-100B。最廣的 IDC AI spending 涵蓋所有 AI 項目（不僅 agent），2029 到 $1.3T。Gartner 不估 dollar amount，改用滲透率——2025 < 5% 的企業應用含 task-specific agent，預測 2026 跳到 40%、2035 達到 30% 的應用營收占比（隱含 $450B+）。最具體的 vertical 示例是 Agentic SCM（供應鏈管理），Gartner 2025 < $2B、2030 估 $53B。

這組數字告訴我們什麼？第一，Agentic 嚴格市場才 $7.84B，但 CRM Agentforce 一家就有 $800M ARR、PLTR US 商用 $507M、MSFT AI $13B 加起來超過 $15B——已公告的 agentic ARR 超過嚴格 TAM 的 2 倍。這代表研究機構的 TAM 估算明顯滯後於現實商業化速度。第二，IDC 的 $1.3T AI 市場是「天花板」，Agentic 純市場只吃其中 5-10%，但成長速度遠高於整體。第三，滲透率從 5% 跳到 40%（Gartner 預測的 1 年變化）是非常激進的假設——這個預測可信度中等，但反映了市場對 2026 年的整體樂觀情緒。

★ Pure-play 新創融資 landscape（2025-2026）

Pure-play AI agent 新創在 2024-2026 累計融資超過 $5B，其中幾家已進入「下一個 Wiz」的候選名單。

§5 Value Chain

議價權近 5 年轉移

Value chain 上各層的議價權在 2022-2028 這個 6 年窗口劇烈重組。理解這個重組是理解「誰賺錢」的關鍵——因為議價權決定誰能把成本往下游 push、誰被迫吃 margin 壓力。

LLM 原廠（Anthropic、OpenAI）的議價權從 2022 的「極強」下降到 2026 的「中強」、再到 2028E 的「中」。原因是 MCP 2025-12 捐給 Linux Foundation 之後 LLM 變得可以互相替代，開源模型（Llama、DeepSeek、Qwen）也持續追趕。更關鍵是 LLM 原廠在企業 vertical 的收入佔比持續低（估 < 5%），因為企業 agentic SaaS 的錢主要不流向 LLM 廠——這是 M11 的核心結論。

Horizontal 平台（CRM、PLTR、MSFT）是議價權上升幅度最大的環節：從 2022 的「中」暴漲到 2026 的「極強」、維持到 2028E 的「強」。原因是 distribution + data moat 兩條護城河同時加深——MSFT 有 230K 組織、CRM 有 Data Cloud、PLTR 有 ontology + FedRAMP。前 5 家到 2028 會吃掉 70% 的 agentic ARR。

Vertical SaaS 出現明顯分化。Regulated data 類（NOW、WDAY、INTU）從 2022 的「強」維持到 2028 的「強」——因為 payroll、IT CMDB、稅務資料是 LLM 或 Horizontal 都複製不了的。Communication 類（HUBS、ZM、TWLO）則從「強」下滑到 2026 的「中弱」、2028 的「弱」——通信資料沒有 regulated 特性，被 Agentforce / Copilot 直接蠶食，HUBS 已被迫轉 outcome-based、ZM 推 $10 standalone。2028 前至少兩家會 PE 化或被併購。

Framework 和 Agent Infra 是兩個相反方向的新玩家。Framework（LangChain、CrewAI）從 2022 不存在到 2026 的「弱」、再到 2028 的「極弱」——LangChain $1.25B 估值對 $12M ARR 是 100x multiple，市場在定價「未來」而非「現在」，但下一輪融資若 down round 整類會 de-rate。Agent Infra（Cloudflare Code Mode、AWS AgentCore）則從不存在跳到「強」再到「極強」——Cloudflare 已有 $42.5M ARR 單合約、Workers AI req +4,000% YoY 驗證 per-invocation 收費模式可行。

Pure-play（Harvey、Sierra、Cohere）從 2022 不存在到 2026 的「強」（融資熱），2028 會分化：Harvey、Sierra、Cohere 這種 regulated 或 outcome-based 玩家能 IPO 成功；11x、Crew 類沒有 moat 的會被併購或消失。

⚠ QC-I19：議價權採定性「極強 / 強 / 中 / 弱 / 極弱」分級，非絕對百分比。這是因為「議價權」是相對概念，無法用單一數字表達，但趨勢方向（上升 / 持平 / 下降）本身就是投資訊號。
💡 Insight §5
議價權轉移圖指向三個投資結論。第一，Horizontal 平台的升幅最大（中 → 極強），是本 ID 投資清單的首選——CRM、PLTR、MSFT 都在此列。第二，Framework 議價權崩跌（弱 → 極弱）是市場尚未完全反映的風險，LangChain 的 $1.25B 估值是在定價未來、不是現在，如果下一輪融資是 down round 整個 framework 類別會被 de-rate。第三，Vertical SaaS 的分化會在 2026-2028 劇烈化——regulated 類（NOW / WDAY / INTU）是安全選擇，communication 類（HUBS / ZM / TWLO）是結構被擠壓的 short 候選。這個分歧在 P/S 倍數上會越拉越開，是配對交易的機會。

Agent Pricing 模型演進

Agent pricing 在 2024-2026 這三年走了三個階段，這個演進本身是理解「誰在競爭、誰在守」的關鍵切口。2024 年是 seat 時代——MSFT Copilot $30/user/月是代表，客戶買 seat 後就能用，distribution 強但定價沒有跟 agent 的實際產出對齊。2025 年進入 consumption 時代——CRM Flex Credits（$0.10/action）和 HUBS credit 是代表，開始按實際使用量計費，彈性高、規模可量化。2026 年主流化 outcome-based——HUBS Customer Agent $0.5/resolved、Sierra 和 Decagon 的 per-resolved、Harvey 的 hours-replaced 都屬此類，pricing 直接對齊客戶 ROI。

CRM 內部其實同時跑三種：$2/conversation（保留）、$0.10/action（Flex Credits 主流）、以及 Agentforce 360 的 custom enterprise pricing。PATH Maestro 和 Cognition Devin 則走 $20-500/task 的 consumption 模式（技術導向 vertical）。最極端的是 Harvey，企業合約 $100K-$1M/年按 hours-replaced 計算，這是 legal vertical 專屬的定價——客戶可以直接對比「AI agent 節省了幾個律師工時」。

Pricing 演進給投資人兩個啟示。第一，outcome-based pricing 考驗 measurement infrastructure——誰先建好「如何量化 agent 成果」誰就能在 2026-2028 定價戰中贏。第二，communication vertical（HUBS、Sierra、Decagon）之所以被迫走 outcome-based，是因為傳統 seat pricing 抵擋不住 Agentforce 的 Flex Credits 擠壓；這個「被迫轉型」本身就是被擠壓的訊號。

💡 Insight §7
這一節三個發現對投資決策最關鍵。第一，PLTR Rule of 40 = 127 是全 SaaS 第一，但估值容錯也最低——下一季 US commercial YoY 若 < 100%，65-80x EV/Sales 會立刻壓縮。第二，CRM 的「10% 整體 / 169% Agentforce」分歧不是問題而是機會——它代表投資人應該把 CRM 估值錨定在 Agentforce 的軌跡（2027-2028 能否做到 $3-5B ARR），而非 traditional CRM 的 10% 成長。第三，MSFT 的 $13B AI run rate 透明度低是 MSFT 估值的最大限制——如果 2026 Q2-Q3 獨立揭露 Copilot Studio ARR，可以 re-rate；如果持續混報，市場會給 discount。
