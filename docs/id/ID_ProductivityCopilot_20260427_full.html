核心 Thesis
AI Copilot 不只是 per-seat 升級金礦，而是商業模式相變——per-seat 轉向 consumption/outcome 計費、agent 取代席次造成 seat compression（已抹約 $2T 軟體市值）、且 88% 用僅 39% 有 EBIT 影響；贏家是握分發+專屬資料且成功轉型計費的 incumbent（MSFT/GOOGL），輸家是純 per-seat、被 agent 取代的單點 SaaS。 🔴 [I: 採用爆發（20M 席次）+ 計費相變（per-seat→consumption）+ seat compression + ROI gap → 結構看多分發者、但 per-seat 模式被自家 agent 侵蝕、價值高度集中]。
KEY CALL — 本報告最重要的一個判斷

市場把 Copilot 當 per-seat 升級金礦，真正的非共識是商業模式相變 + 三重威脅——① 計費從 per-seat 轉向 consumption/metered（Cowork 計量、GitHub Copilot 帳單 $29→近 $750 引 backlash）；② seat compression：agent 取代人 → 少買席次，恐慌已抹軟體類股約 $2 兆市值；③ ROI gap：88% 用、僅 39% 認得到 EBIT 影響（49pp）。贏家 = 握分發 + 專屬資料、且成功轉成 consumption/outcome 計費的 incumbent（MSFT 365 捆綁、GOOGL Workspace 免費捆綁、CRM per-action 先驅）；輸家 = 純 per-seat、無資料護城河、被 agent 直接取代的單點 SaaS。

§1 產業白話定義 + 歷史脈絡

白話版：「Copilot」是坐在你旁邊、幫你寫信改簡報的 AI 助手；微軟、Google 把它直接內建進你每天用的 Office/Gmail。問題是——這門生意過去靠「每人每月收一筆」，但當 AI 變成會自己做完工作的 agent，「人數」這個收費基礎本身正在鬆動。

白話開場：先分清三種收費方式，因為它們是這門生意的命脈。per-seat（按席次）：每個使用者每月固定一筆（如 Copilot $30/人/月），是傳統 SaaS 的模式。consumption/metered（按用量）：按實際呼叫/任務的算力收費，像水電表。outcome（按結果）：按交付的成果（如處理幾份合約）收費。AI Copilot 正從第一種往後兩種遷移——因為每次 AI 回應都吃真實的推論成本（見「Token 經濟學」），固定收一筆的 per-seat 撐不住重度用戶。本報告問的是這場相變下的經濟學：誰捕獲價值、誰被顛覆。

邊界界定：本報告聚焦「生產力 AI（office suite + 協作 + 文件/合約 + 垂直 SaaS）的價值捕獲 + 計費模式相變」。納入做生產力 Copilot 的上市玩家；底層 LLM/agent 平台見「Agentic AI 平台」、推論成本見「Token 經濟學」。灰色地帶是「Copilot vs agent」——多數產品正從被動 copilot 往主動 agent 過渡，本報告以此過渡為主軸。

歷史分三段。第一段：2023 生成式 AI 爆發，產品是「聊天助手」，各家急著貼上 AI。第二段：2024–2025 Copilot per-seat 商業化——微軟、Google、Salesforce 把 AI 嵌進 suite，按席次加價賣（Copilot $30/人/月）。第三段：2026 大規模商業化 + 模式相變——MSFT 365 Copilot 付費席次破 2,000 萬、run-rate $7.2B；但同時計費開始轉 consumption（Cowork 計量、GitHub Copilot token 計費），且 agent 取代人引發 seat compression 恐慌、ROI gap 浮現。

歷史類比一：on-prem → SaaS 訂閱（成功轉型者贏）。上一次軟體模式相變是賣永久 license 轉訂閱——贏家是主動轉型的 incumbent：Adobe 2013 把 Creative Suite 轉 Creative Cloud，短期陣痛、長期營收與股價數年數倍。量化錨：今天 per-seat → consumption/outcome 是下一次相變，像當年 Adobe 主動轉的（CRM Agentforce per-action、MSFT Cowork 計量）會贏。歷史類比二：行動時代免費捆綁（單點被吞）。智慧手機普及後，OS 內建免費地圖/相機/筆記 app 把無數單點 app 逼到牆角。今天 Zoom AI Companion 全免費、MSFT/Google 把 AI 內建進 suite——單點協作工具難對 AI 單獨收費。量化錨：seat compression 與捆綁雙重擠壓，已抹軟體類股約 $2 兆市值。

本主題的「價格」是計費模式（per-seat→consumption），用「採用 vs 計費模式」當時序錨。

關於 cycle 統計表的省略：生產力 AI 為單向結構成長、無 ASP peak-to-trough，故不立 cycle 統計表；以「產品形態三段 + 計費相變」的編年呈現時序。

這段歷史的可操作含義：Copilot 同時是「訂閱相變」（per-seat→consumption，轉型者贏）與「行動捆綁」（MSFT/GOOGL 免費/捆綁殺單點）。bull 的底很實（20M 席次），bear 不是「Copilot 沒人用」（用爆了），而是「per-seat 模式被自家 agent 侵蝕、純單點 SaaS 被捆綁吞、估值 de-rate」。

§2 技術成熟度 + S 曲線

為什麼相變是現在？因為 agent 的自主性夠強到能「做完」而非「協助」一件事——這同時讓 Copilot 更有用（採用爆發）、也讓 per-seat 模式失效（一個 agent 抵多個人、且成本隨用量爆）。

相變的技術拐點有三。agent 自主性：model 能規劃、呼叫工具、多步執行（見「Agentic AI 平台」），讓 Copilot 從「幫你寫一段」升級到「替你做完一件事」——這是 seat compression 的技術根源。推論成本變動：每次 AI 回應吃真實 token 成本（見「Token 經濟學」），重度用戶的成本遠超 per-seat 固定費，逼著計費轉 consumption。專屬資料介接：Copilot 要有用，必須接到企業的郵件/文件/CRM/HR 資料——這是分發 incumbent 與垂直 SaaS 的護城河來源。三者交會，Copilot 從「貼 AI 標籤」變成「會做事、但計費要重設計」的產品。

§3 供給側：現在與未來 + 利潤池

生產力 AI 的利潤往「握分發通路 + 專屬資料 + 成功轉型計費」的 incumbent 集中——純單點工具被免費捆綁與 agent 取代。

供給端是「分發 incumbent + 垂直專屬資料 + 受脅單點」三層。分發 incumbent：Microsoft（365 Copilot，345M 席次基底捆綁）、Google（Workspace + Gemini 免費捆綁）——AI 內建進既有 suite，分發成本近零。垂直專屬資料：Salesforce（Slack + Agentforce per-action）、Adobe（Acrobat AI）、Atlassian（Rovo）、Workday（Illuminate）、DocuSign（Iris）、Box（多 LLM）——靠專屬資料與工作流守住。受脅單點：Zoom（AI Companion 免費送）、HubSpot（mid-market 被擠）——難對 AI 單獨收費。利潤往「分發 + 專屬資料 + 成功轉計費」集中。

利潤池遷移

整條鏈的利潤正從「純 per-seat 單點工具」往「分發 + 專屬資料 + 轉型計費」遷移：分發 incumbent（MSFT/GOOGL）靠既有席次近零成本捆綁、握專屬資料者（CRM/ADBE/WDAY）有高轉換成本且能轉 consumption；被搶的是無資料護城河、靠單一功能收費的單點工具（會議、筆記、mid-market CRM）。MSFT 365 Copilot 20M 席次（產品有機成長、非併購灌入）證明分發捆綁的威力。（依 QC-18，缺逐段精確百分比，以定性呈現。）

Inference · 供給
分發 incumbent 與專屬資料者贏，純單點工具被吞
前提：分發 incumbent 近零成本捆綁（MSFT/GOOGL）+ 專屬資料高轉換成本（CRM/ADBE/WDAY）+ 單點工具無護城河、被免費捆綁/agent 取代 + 計費轉 consumption → 利潤往分發端 + 專屬資料端集中

當 AI 變成 suite 內建功能，分發 incumbent 靠既有席次近零成本獲客；握專屬資料與工作流者有高轉換成本、能轉 consumption/outcome；純單點工具對 AI 很難單獨收費，被免費捆綁與 agent 取代。MSFT 365 Copilot 20M 席次證明分發捆綁的威力。

可證偽條件：若某純 AI-native 生產力新創用「跨 suite、AI-first」獨立長成大廠（不被捆綁吞），或開源/通用 agent 讓專屬資料優勢被抹平，則「分發+資料集中」論的某一面失效。

未來供給與進入門檻：成為贏家的門檻是「分發通路（既有席次基底）或 專屬資料/workflow（高轉換成本）+ 成功轉 consumption 計費」。純單點工具補不上分發與資料——這是 MSFT/GOOGL 捆綁、CRM/ADBE/WDAY 專屬資料守得住的原因。長期變數是分發 incumbent 會不會用「免費捆綁 + agent」把垂直 SaaS 的價值也收走。

因果閉合（回應 §1 結構變數）：§1 提出「握資料+分發是 SaaS 換代不變的贏家邏輯」——未來 3–5 年是否仍 binding？很可能：agent 越自主越需要可信的專屬資料與分發入口，門檻越高；唯一鬆動是分發 incumbent 用捆綁把垂直價值收走、或開源 agent 抹平資料優勢——這是 §7 的反方。

12–36 個月內，純單點工具被免費捆綁/agent 擠、利潤往「分發 + 專屬資料 + 轉型計費」集中。對投資的含義：押分發 incumbent（MSFT/GOOGL）+ 垂直專屬資料（CRM/ADBE/TEAM/WDAY）；避開純 per-seat、無資料的單點 SaaS。

§4 需求側：現在與未來 + 三角驗證

需求最剛性的是「員工已經在用 AI」——88% 員工常用、20M 付費席次；但最大的變數是「用了不等於有 ROI」——僅 39% 認得到可量化 EBIT 影響，落地門檻高。

第一引擎：採用爆發（現需）。需求已從試點變普及：MSFT 365 Copilot 付費席次破 2,000 萬、單季 +500 萬、逾 60% Fortune 500 有 ≥1 萬席、每用戶查詢數季增近 20%、每週黏著度達 Outlook 等級。員工要的是「嵌在既有工作流、自動做掉雜事」。第二引擎：ROI gap（反需求/篩選器）。同時微軟自己的 Work Trend Index：88% 員工常用 AI、但僅 39% 認得到可量化 EBIT 影響（49pp 落差）；AI 工具的「licensed 席次 vs active 使用」落差是所有軟體裡最大的。這缺口既篩掉「買了沒用」的虛胖席次，也創造「證明 ROI、變更管理、agent 治理」的新需求。

需求三角驗證

採用（bottom-up）：M365 Copilot 20M 付費席次、60% F500 ≥10k 席。營收：年化 run-rate ~$7.2B，但扣企業折扣 40–60% 後實際 ~$1.5–2.5B（名目與實際差很大、勿混用）。落地篩選器：88% 用 AI、僅 39% 有 EBIT 影響（49pp）。三邊一致——採用真實、但商業化的「品質」（折後營收 + ROI 兌現）仍待證明。注意：名目 run-rate（$7.2B）與折後實際（$1.5–2.5B）口徑不同，不可混用。

Inference · 需求

§6 產業經濟學與估值傳導

生產力 AI 的錢怎麼賺：分發近零成本獲客 + 專屬資料溢價，但要扣推論 token 成本。per-seat 固定費撐不住重度用戶（成本爆），逼轉 consumption；seat compression 又侵蝕席次「量」——估值對「席次 × ARPU × ROI」三者敏感。

看分發 vs 垂直 vs 單點的 unit economics 與估值，理解誰守得住。

生產力 AI unit economics（量級）
玩家類型毛利 / 經濟估值定位含義
分發 incumbent MSFT/GOOGL近零成本獲客 + 捆綁大盤型（AI 樂觀已 part-price）分發護城河、量體稀釋
垂直專屬資料 CRM/ADBE/WDAY高轉換成本 + 轉 consumption溫和–中高守得住毛利

🟡分歧 1：「Copilot 是 per-seat 升級金礦」信心：高
  • 共識說 X：每人每月加 $30，AI Copilot 是 SaaS 的 per-seat 升級金礦。🟢 [F: T1 — Copilot $30/席、20M 席次]。

🟡分歧 2：「採用爆發 = 營收必然」信心：中
  • 共識說 X：20M 席次、$7.2B run-rate，營收已兌現。🟢 [F: T1 — 20M 席次、run-rate $7.2B]。

→ 推論邏輯：名目 ≠ 實際、採用 ≠ ROI。⚠ 證偽：EBIT 兌現比例快速上升。
🟡分歧 3：「分發 incumbent 通吃、垂直 SaaS 必死」信心：中
  • 共識說 X：MSFT/GOOGL 捆綁通吃，所有 SaaS 都會被內建殺死。🟢 [F: T1 — Copilot/Gemini 捆綁]。

Steel-man 反方（最有力的 3 條）

  • 反方 1（seat compression + AI 退燒）：agent 取代席次擴大、per-seat 收入被自家產品侵蝕、ROI 落空，軟體估值整體 de-rate。當前證據：seat compression 抹 ~$2T 市值、ROI gap 49pp。證偽窗口：軟體 seat 成長、ROI 兌現率。
  • 反方 2（計費 backlash）：consumption 計費讓帳單暴增（GitHub Copilot $29→$750），企業反彈、放慢採用或砍預算。當前證據：GitHub Copilot backlash。證偽窗口：consumption 採用率、客戶流失。
  • 反方 3（分發 incumbent 通吃垂直）：MSFT/GOOGL 用免費捆綁把垂直 SaaS 的價值也收走，垂直 SaaS TAM 被壓縮。當前證據：Zoom AI Companion 免費、Copilot 內建擴張。證偽窗口：垂直 SaaS NRR、被捆綁侵蝕程度。

市場最低估的風險：不是「Copilot 不重要」（重要）、也不是「MSFT 不強」（強），而是分歧 1/2 同源的「per-seat 模式被自家 agent 侵蝕 + 名目營收 ≠ 實際 ROI」——市場把採用爆發當營收必然、把 per-seat 當穩固模式，最沒準備好「seat compression + ROI gap」這對組合。這是整個 per-seat SaaS 板塊最該獨立監測的尾部。

三條分歧裡最可操作的是分歧 1/2（per-seat 被侵蝕 + 名目≠實際）——最未被 priced 且有明確領先指標（軟體 seat 成長、ROI 兌現率、consumption ARPU）。分歧 3（分發 vs 垂直）次之、part-priced。

最高純度 play 是分發 incumbent MSFT/GOOGL 與專屬資料垂直 CRM/ADBE/TEAM——買賣由個股 DD 與 Pure MA 系統決定。被低估韌性的是垂直 regulated 資料者（DOCU/BOX/WDAY）；受害方是單點工具（ZM/HUBS）；Notion 等私有、不可直接投資。
