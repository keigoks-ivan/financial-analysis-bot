§1 產業白話定義 + 歷史脈絡

Hybrid bonding（混合鍵合）是把兩片晶片不靠焊錫凸塊、而靠「銅對銅 + 介電層對介電層」直接黏合的技術——像把兩塊磨到原子級平整的金屬玻璃板貼合後加熱融成一體。它的價值在於把晶片間的連線密度推到微凸塊做不到的等級，是 AI 晶片把「多顆 die 疊成一顆」（3D 堆疊）的物理前提。誰付錢？做 AI 加速器的 TSMC SoIC 客戶（AMD / Apple / NVIDIA）與 HBM 記憶體三寡占。

白話開場：傳統封裝用「微凸塊」（micro-bump，pitch > 30-40μm 的小焊錫球）把上下兩片晶片接起來，就像兩塊樂高用一排凸點對接。但凸點越密、焊錫越容易短路與孔洞，到了 sub-10μm pitch 就做不下去。hybrid bonding 乾脆拿掉焊錫，把銅 pad 直接對銅 pad、介電層直接對介電層融合——pitch 可以縮到 6μm（量產）甚至研究級的 sub-1μm。這就是它能把 die-to-die 連線密度做到傳統 2.5D 的 56 倍（TSMC SoIC vs CoWoS 數據）的物理根。

邊界界定：本報告聚焦 sub-10μm pitch 的 Cu-Cu 直接鍵合（bumpless）這個 3D 堆疊核心技術——包含設備（BESI / ASMPT / AMAT / EV Group）、IP（Adeia DBI）、製程整合（TSMC SoIC / Intel Foveros Direct / Samsung X-Cube），以及下游採用者（AI 加速器、HBM）。CoWoS-S / CoWoS-L（仍有微凸塊）、EMIB（橫向 2.5D 互連）、玻璃基板（substrate 層、非 die 堆疊層）屬母題先進封裝或兄弟 ID，本篇只 cross-link。灰色地帶是 fluxless TCB（先進熱壓鍵合）——它做 >10μm pitch、是 hybrid bonding 在過渡期的競爭/共存者，本篇追蹤但非主軸。

第一次轉折：2015 Tessera（後改名 Xperi、再分拆為 Adeia）收購 Ziptronix，把 DBI（Direct Bond Interconnect）/ ZiBond 專利族納入授權版圖——奠定整個 hybrid bonding IP 收租的基石。次年 2016 Sony 用 DBI 把堆疊式 CMOS 影像感測器（CIS）裝進三星 Galaxy S7，這是 W2W hybrid bonding 第一次高量產商用。第二次轉折：2020 TSMC 正式對外發表 SoIC（System on Integrated Chips），確認 AMD 與 Google 為開發夥伴，初始 bond pitch 9μm（N7 世代）、roadmap 指向 6μm（N5）；同年 Intel shipped Lakefield（Foveros，雖仍用 Cu pillar 非純 Cu-Cu）。2022 AMD 推 Ryzen 7 7800X3D，用 TSMC SoIC 的 3D V-Cache 把 64MB L3 SRAM 直接疊在 Zen die 上——第一批消費級 hybrid bonding 大量出貨。

第三次轉折：2025-04 Applied Materials 買 BESI 約 9% 股權成最大股東、co-develop「業界首套全整合 die-based hybrid bonding 設備」——hybrid bonder 正式進入「設備寡占期」。到 2026-02，TSMC 確認 SoIC 6μm pitch 達 HVM、四大客戶（Apple / NVIDIA / AMD / Broadcom）就位；同月 BESI Q1-26 hybrid bonder 訂單翻倍破 2024 峰值——產業進入「SoIC 邏輯堆疊量產、設備寡占化、HBM 尚未接棒」的階段。

歷史類比（兩個有用、一個誤導）。① 2010-2018 EUV 雙寡占（有用，框上行形狀 + 提醒「現在是 2016 不是 2014」）：ASML 獨家 EUV 機台、Zeiss（ASML 持股 ~25%）獨家光學，股價在 EUV 從研發走向量產後 10 年約 19x（2016→2026）。BESI + AMAT 同構，但 BESI 約在「2016 ASML」階段——首批客戶機台正出、HVM 未放量。② 1995-2010 Dolby 授權（有用，框收租倍數）：Dolby AC-3 被 DVD Forum 訂強制標準，營收 $328M（2005）→ $923M（2010）五年 2.8x，收租毛利 ~87-90%。Adeia DBI 同構，約在「1998-2002 Dolby」階段。③ 2000 電信光纖（誤導，反向類比警示）：有人把 hybrid bonding 設備擴張比作當年「鋪過頭的暗光纖」，但 hybrid bonding 設備是按 wafer bonding 通量收費、即刻對應 SoIC HVM 真實營收，不是「鋪了沒人用」——直接套用會錯判時點。

把敘事歷史升級成可用於 timing 的統計——hybrid bonding 屬新興結構成長，無完整可統計 cycle，故列「採用滲透」時序而非 peak-to-trough 振幅。

這段歷史對今天的意義：hybrid bonding 的市場分界不是「設備 vs 製程」，而是「誰擁有 IP」與「誰握鎖喉點」。設備商（BESI / ASMPT）賣硬體、Adeia 收 IP 授權、AMAT 握 CMP、TSMC 有自有 IP 不付 Adeia——四家各賺不同的錢、業績不完全同步。配置邏輯是「兩端（設備 + IP）一起看」而非挑一家，但兩個 pure play 的估值都已先 re-rate，方向對不等於現價是進場點。

§2 技術成熟度 + S 曲線

為什麼是現在、不是三年前、也不是三年後？因為兩個 bottleneck 剛被跨過：TSMC SoIC 6μm pitch 在 2026-02 達 HVM（邏輯堆疊量產可行），而 HBM 的層數還沒撞到微凸塊的物理牆（要等 HBM5 20-Hi）。技術成熟度卡在「邏輯堆疊已熟、HBM 量產未到」這個中間態。

為什麼是現在：hybrid bonding 的核心物理門檻有三道，前兩道剛被量產化跨過、第三道（HBM 高層堆疊良率）還沒。第一道是 CMP 超平整化——介電層（SiO₂ / SiCN）表面粗糙度要做到 RMS 0.1-0.5nm、銅 pad 凹陷 ≤5nm，過拋會 dishing、欠拋接觸不上；這只有 AMAT 有量產資歷。第二道是 對位精度——W2W 達 <50nm overlay、D2W 要 tens-of-nm 級放置精度，BESI 2024 訂單已含 100nm alignment 能力的機台、研發級展示到 50nm。第三道是 潔淨度——單一 1μm 高的微粒會造成直徑 10mm 的 void，所以 hybrid bonding 強制 ISO Class 1-2 cleanroom（建置成本是標準 fab 的 3-4 倍）。前兩道跨過讓 SoIC 邏輯堆疊量產，第三道在 HBM 16+ 層的累積良率上仍是卡點。

§3 供給側：兩端收租，中段被擠

誰在供給、瓶頸在哪、利潤被誰賺走？供給端三類玩家賺不同的錢——設備（BESI ~60-67% 市占 + ASMPT + AMAT 整合線與 CMP）、IP（Adeia DBI 收租，毛利 ~100%）、製程整合（TSMC SoIC 自有 IP / Intel / Samsung）。利潤往設備與 IP 兩端集中，中段 OSAT 被擠成「量大錢薄」的承包方。

現供結構：設備端 BESI 是 hybrid bonder 龍頭（Datacon 8800 CHAMEO 系列，~60-67% 市占，Q1-26 訂單 €269.7M +104.5% YoY、hybrid bonder 單季訂單翻倍破 2024 峰值、累積 >150 units 跨 18-20 客戶），ASMPT 第二（但近期 HBM 主力是 TCB +146% YoY 而非 hybrid bonding），EV Group 守 W2W oxide bonding 利基。AMAT 不直接賣 bonder，但握兩個結構槓桿：① 買 BESI ~9% 股權 + co-develop 全整合線（Kinex 平台），② hybrid bonding 必需的 CMP 製程近乎獨佔——「不管誰做 bonder 都得先過 AMAT 的 CMP」是隱形鎖喉點。IP 端 Adeia 的 DBI / DBI Ultra 是任何 Cu-Cu 鍵合繞不開的專利族（除 TSMC 自研外）。

§4 需求側：SoIC 先量產、HBM 後接棒

誰在買、為何買、哪塊需求最不可替代？最不可替代的是 TSMC SoIC 邏輯堆疊需求（已在 6μm HVM、AMD / Apple / NVDA 鎖定），最被市場誤讀的是 HBM——HBM4 不用 hybrid bonding、HBM5 才物理強制。這個「一代之差」是 thesis 從「2026 爆發」打成「兩段式滲透」的根源。

現需：兩條時間線。近端（已量產）是邏輯堆疊——TSMC SoIC 6μm pitch HVM，AMD MI400（432GB HBM4、12 顆 compute/IO die 用 SoIC-X 疊、占 TSMC 2026 先進封裝產能 ~11%）、Apple M5 Pro/Max/Ultra（SoIC-mH，Apple 首個 chiplet 設計，月產 ~10,000 wafers）、所有 3D V-Cache，再加 NVDA Rubin Ultra（2027）/ Feynman（2028）排隊。遠端（2028+ 接棒）是 HBM——當 HBM 堆到 20-Hi（HBM5），微凸塊在 JEDEC 高度限制下物理上塞不下，hybrid bonding 才從「可選」變「強制」。

HBM bonding 製程的真相——與舊敘事最大的事實落差。HBM4 12-Hi（2026 量產）走的是 MR-MUF（SK Hynix）與 TC-NCF（Samsung 2026-02 量產、Micron Q2-26 ramp），完全沒用 hybrid bonding。甚至 HBM4 / HBM4E 16-Hi，SK Hynix（NVDA Vera Rubin HBM4 約 60-70% 份額）也明說維持 MR-MUF——因為 JEDEC 2025-04（JESD270-4）把高度上限從 HBM3E 的 720μm 放寬到 775μm，配 30μm 晶圓磨薄就能塞下 16 層。Samsung 是唯一在 HBM4E 16-Hi 押 hybrid bonding（HCB）的，但供 NVDA 的樣品報 ~10% 良率。SK Hynix 雖在 2026-04 完成 12-Hi hybrid bonding 驗證、2026-03 下首張 mass-production-grade 設備訂單（AMAT+BESI Kinex，~20B KRW），但明說是「development line 預備步驟、量產要次次世代」。

估值現在 price 了什麼、哪個變數動會 re-rate / de-rate：BESI 現價已 price 了「設備寡占 + SoIC HVM」；Adeia 已 price 了「半導體授權 ramp 開始」。會再 re-rate 的變數是「HBM5 確認 + Adeia per-stack 落地」（上行 catalyst）；會 de-rate 的變數是「SoIC 訂單降溫 + Samsung 繞過 Adeia + 高倍數回吐」（下行 catalyst）。風報不對稱——這是為何 conviction 是 mid 不是 high。

§7 Non-Consensus + Priced-in 檢驗 + Kill Scenario

市場主流怎麼看、我們哪裡不同、為什麼這分歧還能賺錢？三條分歧分別針對「需求時間線（共識看 HBM4）」「設備估值（共識已追）」「IP 收租（共識在重估）」——但每條都要對照「市場已 price 多少」，分歧對但已 priced 就不可操作。
本章含 judgment，已標信心度與證偽條件。
NC#1：真量產引擎是 SoIC 邏輯堆疊，不是 HBM4 信心：高
  • 共識：多數產業報導把 hybrid bonding 的 2026 爆發歸因於「HBM4 量產帶動」[T3 一般財經媒體]。
  • 我們說：HBM4 / HBM4E 16-Hi 仍用 MR-MUF / TC-NCF（JEDEC 775μm 放寬續命）；當前已量產的 hybrid bonding 引擎是 TSMC SoIC 邏輯堆疊（6μm HVM、AMD MI400 / Apple M5），HBM 要等 HBM5（2028-2029）才物理強制。[F: T2 TrendForce 2026-01 SK Hynix 維持 MR-MUF]
  • 市場 price 多少：BESI 估值已 price「設備短缺」但市場敘事仍掛在「HBM4 帶動」——分歧可操作，因為錯誤歸因會讓投資人在 2026 HBM4 量產卻沒看到 hybrid bonding 放量時錯誤失望、或在 SoIC 訂單才是真驅動時看錯 leading indicator。
  • → 投資意義：盯 BESI 季訂單（SoIC 驅動）與 HBM5 規格凍結（2027-Q4），而非 HBM4 量產時點。
  • ⚠ 證偽：若 2026-2027 HBM4 16-Hi 大規模轉 hybrid bonding（Samsung 良率突破帶動 SK Hynix 跟進），則「HBM 要等 HBM5」的判斷提前失效。
NC#2：利潤往設備 + IP 兩端集中，中段 OSAT 被擠 信心：高
  • 共識：把 BESI 當「荷蘭中型設備商」、AMAT 9% 股權當「策略合作」、Adeia 當「Tivo 遺產 IP 公司」。
  • 我們說：這是「硬體寡占版 EUV（BESI+AMAT+CMP 鎖喉）+ IP 版 Dolby（Adeia DBI）」的雙端收租生態；利潤往兩端集中、中段 foundry / OSAT（除 TSMC 自有 IP）被擠成承包方。[I: BESI GM 63.5% + Adeia 毛利 100% + AMAT CMP 獨佔 → 兩端議價權]
  • 市場 price 多少：已大幅 priced——AMAT 2025-04 買 BESI 9% 後 BESI 已 re-rate 到 Fwd P/E ~56x / EV/EBITDA ~57x（10 年中位 ~27x 之上）。隱含成長推算：56x Fwd P/E 在 ~10x 終值倍數假設下，反推市場已 price ~30%+ 多年期 EPS CAGR（推導：56/10 ≈ 5.6x 需用 ~6 年 30% 複利消化）——這要求 SoIC + HBM5 兩段引擎都如期兌現，容錯極小。Adeia 市值已從 ~$1.5B 翻到 ~$3.5B。分歧方向對但部分不可操作——結構對、但便宜的 entry 已過。
  • → 投資意義：兩端收租 thesis 結構成立，但要等 correction 才有風報；現價追高的人是在高倍數買確定性。
  • ⚠ 證偽：ASMPT / Korean 新進者搶 BESI >10pp 份額；或 Samsung 自研繞過 Adeia——任一打破「兩端議價權」。
NC#3：Adeia per-stack 收租被低估但 binary 信心：中
  • 共識：Adeia 半導體授權 +808% YoY 已被市場看到、股價已反映（市值翻倍）。
  • 我們說：已反映的是「AMD 簽約 + 既有授權」；未反映的是 Samsung / SK Hynix 的 per-stack 經濟（現有授權多無 per-unit HBM royalty，2027+ 續約才是真 step-up）。若兩大 memory 廠簽 per-stack，Adeia 往 Dolby 區間（15-25x EV/EBIT）re-rate。[X: base 可能 — 2027+ 續約逐步簽；bull 可能 — 提前簽 per-stack；bear 不太可能但機率不低 — Samsung 自研繞過]
  • 市場 price 多少：EV/EBIT ~18x 在 Dolby 區間（15-25x）低端。隱含成長推算：18x EV/EBIT 對一家全公司營收將降（FY26 guide $395-435M vs FY25 $443M）的公司，市場其實只 price 了「半導體授權成長抵銷 media 衰退、整體持平」——未 price Samsung/SK Hynix per-stack 的 step-up，故若 per-stack 落地有往 Dolby 區間上緣 re-rate 的空間；但 binary（Samsung 繞過則 thesis 半廢）。分歧對但高度 binary、不完全可操作。
  • → 投資意義：Adeia 是最 asymmetric 也最 binary 的標的，建倉應拆「核心 + 觀望 Samsung 動作」。
  • ⚠ 證偽：Samsung 宣告自研避開 Adeia 專利成功；或 Adeia 主要授權訴訟敗訴。

Steel-man 反方（3 條最強空頭論證）

反方 1：fluxless TCB 把 hybrid bonding 的 HBM inflection 一直往後推。具體路徑：ASMPT AOR TCB、Kulicke & Soffa APTURA FTC 做到 sub-1μm 放置精度、<10μm bump pitch，成本只有 hybrid bonder 的一半（~$1-2M vs ~$3M），良率更穩——讓 HBM5 甚至 HBM6 都還能用先進 TCB，hybrid bonding 在 HBM 的強制 inflection 一直不來。當前證據：ASMPT TCB +146% YoY、首套 fluxless AOR 系統 2025-12 已出貨給大 memory 客戶。證偽窗口：2027-2028 HBM5 規格凍結。

反方 2：Samsung / SK Hynix 自研繞過 Adeia，打破收租模式。具體路徑：Samsung 歷史上多次選自研繞過授權金（如部分 memory IP），在 HBM5 世代完整自研 hybrid bonding 流程避開 Adeia 所有專利，SK Hynix 跟進，Adeia 半導體授權 2028-2030 停滯。當前證據：Samsung / SK Hynix 現有授權多無 per-stack HBM royalty；Samsung HBM4E 押自家 HCB 技術。證偽窗口：2027 HBM5 規格 + per-stack 續約期。

反方 3：AI capex digestion 砍 SoIC 邏輯堆疊訂單。具體路徑：AI 訓練 / 推論 capex 2027-2028 放緩，hyperscaler 減少 AI 加速器訂單，連帶 SoIC 邏輯堆疊量（AMD MI400 / NVDA Rubin）下修，BESI 設備訂單 run-rate 回落——近端引擎熄火、遠端 HBM5 還沒接上，青黃不接。當前證據：AI capex 變現缺口（見 ID_AIComputeCapexCycle）；BESI 訂單對 SoIC 需求高度敏感。證偽窗口：2027 AI capex guidance 方向。

這三條分歧裡哪條最可操作（對 + 未 priced）：NC#1（真引擎是 SoIC 非 HBM4）最可操作——方向對、且市場敘事仍錯誤掛在 HBM4，盯對 leading indicator（BESI 訂單 + HBM5 規格）的人有資訊優勢。NC#2（兩端集中）對但已大幅 priced。NC#3（Adeia per-stack）對但高度 binary。最該盯的反方是「Samsung 繞過 Adeia」（對 NC#3 致命）與「AI capex digestion 砍 SoIC」（對近端引擎致命）。
