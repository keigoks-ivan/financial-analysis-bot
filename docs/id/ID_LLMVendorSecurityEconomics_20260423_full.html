§1 產業白話定義 + 歷史脈絡

白話版：當資安公司用 AI（agentic SOC）幫你防駭，它得跟 OpenAI/Anthropic 租模型、跟 NVDA 租算力。所以「AI 安全」這門生意賺的錢，會在「資安公司、模型公司、算力公司」之間分。本報告問的是：這塊餅怎麼切。

白話開場：把「AI×資安」拆成兩件事。第一是「用 AI 做資安」——資安公司（CRWD/PANW/S）把大型語言模型嵌進產品，做自動化的 agentic SOC（見子題「AI 資安雙刃」）。第二是「保護 AI 本身」——當企業自己導入 AI agent，得防止 prompt injection（惡意提示注入）、model jailbreak（越獄）、MCP（AI agent 連工具的協議）被濫用，這需要新的「AI runtime security」工具。本報告聚焦這兩件事的經濟學：錢怎麼在棧裡分。

邊界界定：本報告聚焦「AI×資安棧的價值捕獲 + AI 安全新類別的定義權」。納入做 agentic SOC、AI runtime security、AI gateway 的上市與重要私有/被併玩家，以及漏出鏈上的 LLM 廠與算力（NVDA）。不展開個別 LLM 廠的估值（那是 AI 模型 ID）、也不深入攻防戰術（見「AI 資安雙刃」）。

歷史分三段。第一段：2023 生成式 AI 爆發，資安公司開始把 LLM 當「助手」嵌進產品。第二段：2024–2025 agentic 化——AI 從助手變自主執行，資安公司大量採用 frontier model（付推論成本給 OpenAI/Anthropic），同時「保護 AI 本身」的新創（Lakera、Protect AI）崛起。2025-07 Palo Alto 併 Protect AI、2025-09 Check Point 以 $300M 併 Lakera（Q4 2025 完成；Zurich 成 CHKP 全球 AI 安全 R&D 中心）——純工具被平台收編。第三段：2026 類別定義戰——Palo Alto 推 Prisma AIRS 3.0（AI runtime firewall / model security / posture），最積極搶定義「AI runtime security」這個 greenfield 類別；CRWD、Microsoft、雲端原生新創都在賽跑。

歷史類比一：雲端「賣鏟子 vs 賣服務」。AWS 賣算力賺豐厚毛利，但建在上面的 SaaS（Salesforce 等）利潤池更大——握客戶分發與資料者贏。對本次的含義：LLM 廠/NVDA 是「賣鏟子的」（分推論經濟），資安平台是「賣服務的」（最大利潤池），這是 bull 的底。量化差異：AI 推論成本占資安產品成本的比重還在快速變動（模型降價 vs 用量上升），漏出幅度未定。歷史類比二：防毒引擎授權。早年很多防毒貼牌同一底層引擎，貼牌商賺通路、引擎商賺授權費——利潤分層。對本次的 bear 含義：若 frontier model 成 agentic SOC 不可或缺的「引擎」、而引擎只有少數幾家，外購模型的資安廠就像當年貼牌商，要付一塊給引擎商；自有模型者（MSFT/GOOGL）則無此漏出。

本主題無價格 cycle，用「AI 安全新類別採用」當時序錨。

關於 cycle 統計表的省略：AI 安全為 0 基期 greenfield、無價格 cycle，故不立 cycle 統計表；以「AI×資安三段」（2023 助手 → 2024–25 agentic+新創 → 2026 類別定義戰）的編年呈現時序。

這段歷史的可操作含義：每一波技術換代，「握分發 + 資料」者賺最大利潤池（資安平台仍贏），但「賣鏟子的」（模型/算力）分一塊；本次特殊在「引擎只有少數幾家 + 自有模型者不付費」，所以價值捕獲的關鍵變數是「自有 vs 外購模型」的毛利差。

§2 技術成熟度 + S 曲線

為什麼 AI 安全是現在？因為 agentic AI 進生產製造了新攻擊面（prompt injection、MCP 濫用、GPU 盲區），傳統工具（WAF / EDR）看不見也擋不住，必須新建 AI runtime security——這是 0 基期的 greenfield。

技術拐點是「攻擊面換代」。傳統 web 防火牆與 API gateway 不是為 LLM 設計的：prompt injection（把惡意指令藏在輸入裡）、系統提示外洩、模型越獄、tool-calling agent 的「過度自主」是 OWASP 2025 LLM 應用 Top 10 的新風險。傳統 EDR 又看不見 GPU / AI 推論工作負載（「GPU Blind Spot」）。所以需要新的一層：AI runtime firewall（執行前檢查 prompt、執行後檢查輸出）、model security（保護模型 artifact 與訓練管線）、MCP gateway（管 AI agent 連工具的協議）、posture management（持續合規）。Palo Alto Prisma AIRS 3.0 把這四件事打包成一個平台。

§3 供給側：現在與未來 + 利潤池

AI×資安棧的利潤往「兩端」集中——握客戶分發的資安平台（一端）+ 收推論租金的 NVDA/模型商（另一端），中間的純 AI 安全工具最容易被併。

供給端五層：①底層算力（NVDA，收推論租金）、②LLM 模型（Anthropic/OpenAI，分推論經濟、但被多模型壓價）、③資安平台（CRWD/PANW/S/NET，握分發+資料飛輪、最大利潤池）、④自有模型平台（MSFT/GOOGL，垂直整合不付外部 LLM 廠）、⑤AI 安全新創（Lakera/Protect AI 等，做 guardrail——正被併進平台）。利潤往「兩端」集中。

Inference · 供給
純 AI 安全工具被併，利潤往分發與算力兩端集中
前提：AI 安全工具單點功能、易被平台內建 + frontier model 只少數幾家（議價強）+ 資安平台握客戶分發 → 利潤往「平台分發端 + 算力/模型端」集中，中間被擠

純做 prompt injection 防護或 LLM firewall 的新創，功能容易被平台內建或併購（Check Point/Lakera $300M、PANW/Protect AI 已示範），很難獨立成大廠；frontier model 只有少數幾家、議價力強，分走一塊推論經濟。最大利潤池仍在「握分發 + 資料飛輪」的資安平台，但其增量毛利被模型/算力端分走。

可證偽條件：若某 AI 安全新創用「AI-native 平台」獨立長成大廠（不被併、自建分發），或開源安全模型讓資安廠不需付外部 LLM 廠（漏出消失），則「兩端集中」的某一面失效。

§4 需求側：現在與未來 + 三角驗證

需求最剛性的是「企業導入 AI agent 卻叫不停」的治理缺口——100% 把 agentic AI 列 roadmap、但 63% 承認無法在 agent 出錯時終止（60% 無法強制限制其行為範圍）；這個缺口就是必須新建的 AI 安全 TAM。

第一引擎：治理缺口（現需）。Kiteworks 對 225 位安全/IT/風險主管的調查：100% 把 agentic AI 列入 roadmap，但 63% 承認「agent 出錯時無法終止」、60% 無法強制限制 agent 行為範圍——2026 的治理-containment 缺口。另有 86% 企業稱需升級 tech stack、90% 視安全與可觀測為必要（此兩項出自 MintMCP/TrueFoundry 等較低層調查，供方向參考）。企業不敢在沒有「prompt injection 防護 + MCP gateway + AI runtime firewall + 預算/速率上限」下放 agent 進生產。第二引擎：GPU 盲區（未需）。傳統 EDR 看不見 GPU / AI 推論工作負載，開出一個必須新建的「AI 工作負載可見性」TAM——0 基期 greenfield。

需求三角驗證

top-down：AI 安全新類別 TAM 早期、~$10–30B 量級、~25%+ CAGR（第三方估計分歧大，反映早期類別、定義未統一）。bottom-up：治理缺口（100% agentic roadmap、63% 叫不停、60% 無法限制範圍，Kiteworks 225 主管 T1 錨）+ 採用門檻（86% 需升級 stack、90% 視安全必要，較低層調查供參考）。兩邊方向極一致——需求是真 greenfield；缺口在「規模估計分歧大」（早期類別正常）。採信「治理缺口」作需求真實性的最硬證據。

§6 產業經濟學與估值傳導

AI 安全的錢怎麼賺：賣產品的毛利減掉「付給模型/算力的推論成本」。自有模型者（MSFT/GOOGL）毛利不漏、結構占優；外購模型者（CRWD/PANW/S）的 AI 模組毛利低於傳統 SaaS。

看自有 vs 外購模型的 unit economics 差異——這是價值捕獲的核心。

AI×資安棧 unit economics（量級）
玩家類型AI 模組毛利價值漏出含義
自有模型（MSFT/GOOGL）~傳統 SaaS（~75%+）不漏（內部）結構占優
外購模型（CRWD/PANW/S）低於 SaaS（扣推論成本）漏給 LLM 廠 + NVDA增量毛利被分走
LLM 廠（Anthropic/OpenAI）—收推論費、但被多模型壓價分一塊、競爭激烈
算力（NVDA）~75%（GPU 毛利）收推論租金漏出最終受益方

怎麼讀：自有模型者 AI 模組毛利接近傳統 SaaS（不漏）、外購模型者低於 SaaS（漏給 LLM 廠/NVDA）——這就是「價值捕獲」在毛利上的具現。NVDA 是漏出鏈的最終受益方（每次推論都過 GPU）。市場目前把「AI = 資安利多」泛泛 price，沒 price 這個毛利差。

估值傳導機制：外購模型的資安廠估值對「AI 模組毛利能否守住」敏感。傳導鏈：AI 安全 greenfield 推營收 → 但若 AI 模組毛利明顯低於 SaaS → 市場重新定價「AI 營收的品質」→ de-rate。敏感度錨點：若資安廠 AI 模組毛利比 SaaS 低 ~10–15pp，且 AI 營收占比上升，blended 毛利下行、估值承壓。自有模型者（MSFT/GOOGL）無此傳導。

估值現在 price 了什麼：市場把 AI 安全當「資安股齊漲利多」泛泛 price，沒 price「自有 vs 外購模型的毛利差」與「增量毛利漏出」——這是最未 priced 的一面。會 re-rate 的是自有模型者份額加速 + AI 安全成大類別；會 de-rate 的是外購者 AI 模組毛利明顯低於 SaaS。對 PM：盯 AI 模組毛利差，不是盯「AI 安全營收成長」。

  • 共識說 X：PANW/CRWD 等平台廠是 AI runtime security 的當然贏家。🟢 [F: T1 — PANW Prisma AIRS 3.0]。

Steel-man 反方（最有力的 3 條）

  • 反方 1（毛利大漏）：frontier model 推論成本居高，外購模型的資安廠 AI 模組毛利明顯低於 SaaS，AI 營收占比上升反而拖累 blended 毛利。當前證據：agentic SOC 大量呼叫 LLM。證偽窗口：資安廠 AI 模組毛利揭露。
  • 反方 2（Microsoft 內建壟斷）：Microsoft 用 E5 把 AI 安全內建免費送，純資安廠與新創吃不到 greenfield、類別 TAM 被結構性壓縮。當前證據：Microsoft 在各資安層的捆綁。證偽窗口：Microsoft AI 安全份額 vs 純玩家。
  • 反方 3（greenfield 是空頭支票）：AI 安全需求雖在 roadmap，但落地慢、長期停在試點，新類別營收遲遲不放量。當前證據：86% 需升級 stack（採用門檻高）。證偽窗口：AI 安全新類別營收能否從零放量。

三條分歧裡最可操作的是分歧 1/2（價值捕獲 + 自有模型毛利優勢）——最未被 priced，且有明確領先指標（AI 模組毛利 vs SaaS、自有 vs 外購份額）。分歧 3（類別定義權）次之、part-priced。

最高價值捕獲是自有模型者 MSFT/GOOGL（不漏）與定義類別者 PANW——買賣由個股 DD 與 Pure MA 系統決定。被低估曝險的是 NVDA（漏出受益）與 NET（AI 流量入口）；S 是「外購模型、毛利漏出最明顯」的純玩家（雙面）；純 AI 安全工具已被併、不可獨立投資。
