§1 產業定義與邊界

本 ID 以「Hyperscaler 雲端基建」為核心，定義為「全球部署 ≥ 5 個區域、≥ 30 個 availability zone、年化 capex ≥ $50B、自有 AI accelerator silicon roadmap 的超大規模 cloud infrastructure 提供商」。本定義刻意排除「pure GPU rental cloud」（CRWV / Lambda / Together）與「enterprise-only / private cloud」（VMware / Oracle on-prem 等），因為這兩類缺乏多 region 規模 × ASIC 自研雙條件，屬於不同 thesis 框架。

三巨頭核心是 AWS（Amazon Web Services）/ Azure（Microsoft Azure）/ GCP（Google Cloud Platform）。本 ID 額外納入 ORCL OCI（第四 hyperscaler，capex 規模 $25-30B 接近 entry threshold + OpenAI Stargate 合作）+ 中國雲三巨頭（Alibaba Cloud / 騰訊雲 / 華為雲）作為 cross-geography 對照（QC-I34 強制要求）。Meta 雖然 capex 體量等同 hyperscaler（FY26 $125-145B）但**不對外提供 cloud service**，屬「captive AI infra」而非 hyperscaler，本 ID 列為「鄰近玩家」cross-link 但不在主框架。

關鍵邊界爭議是 ORCL 是否算 hyperscaler。本 ID 視為「準 hyperscaler」— OCI capex F4Q26 ~$25-30B（vs Big Three 各 $75-105B 級距）+ RPO $130B+ + OpenAI Stargate $300B+ 合作確立 4th place，但 multi-region footprint 仍不及 Big Three 1/3、自研 silicon 缺、operating margin 結構性低（OCI infra ~20% vs AWS 35%）。**置於框架但定位「fringe player + differentiated 飛輪」**。Meta 反向 — capex 巨大但無 external cloud service，純為自家 AI 訓練 / Reels / FB / IG / Reality Labs 服務，列為 cross-link 不列為 hyperscaler。

第二個邊界爭議是「中國雲三巨頭」如何納入。本 ID 採取「對照不對標」原則 — Alibaba Cloud / 騰訊雲 / 華為雲在中國市場規模驚人（FY26 阿里雲 capex RMB 380B+ 加碼 ≈ $52B / 年）但**全球 footprint <5%、AI accelerator 受 BIS 出口管制 + 自研 silicon 較弱**（華為 Ascend 受限 / 阿里 PPU + 含光自研在 7nm 節點）。納入 §6 玩家矩陣作為地緣對照 + 中國 detw thesis 的 hedge，但不列入核心 cross-vertical basket。

💡 §1 Insight
1. 「Big Three」這個共識框架本身就是錯的 — 三家從 2025 開始已經是「3 個結構不同的 thesis」（GCP 是 ASIC + AI accelerated growth / Azure 是 OpenAI inflection + 利潤領先 / AWS 是規模 + Anthropic 集中度風險）；任何「Big Three basket equal-weight」配置會失去 alpha。
2. ORCL 是 binary call、不是 incremental position — OpenAI Stargate $300B+ 兌現與否是 ORCL 估值倍數 +30-50% 與 -20% 的分水嶺；不應該配 ORCL 作為「diversification」（與 Big Three 相關性高 + binary risk 倍增）。

§2 技術演進史 + S 曲線 + 歷史類比

Hyperscaler 三巨頭從 2006-2026 經歷四個結構性轉折：(1) 2006-2012 IaaS 大眾化（AWS EC2 / S3 確立 + Azure 2010 launch + GCP 2012 launch）；(2) 2013-2018 Enterprise migration（Azure 對 AWS 反擊 + Office 365 + GCP 落後雙頭）；(3) 2019-2023 AI 訓練爆發（NVDA A100 / H100 主導 + 三巨頭 capex 分化 + GCP TPU v4/v5 自研領先）；(4) 2024-2026 Capex 軍備競賽 + 自研 ASIC 加速 + 三家結構性分化加速（GCP TPU v6/v7 + Anthropic-AWS Trainium 4B + Apple-TPU 採用 + Maia 100/200）。

本次 2024-2026 循環最特別的是 「capex 規模 + 自研 silicon + 客戶議價」三軸同步質變：(a) Big Three FY24 合計 capex ~$155B → FY26 ~$260-290B = 1.7x 兩年；(b) 自研 ASIC 占 AI accelerator 從 2023 ~10% → 2026E 25-30% → 2028E 35-45%（vs NVDA + AMD GPU）；(c) Anthropic 三 cloud 同時 commit（AWS Trainium 400K + GCP TPU 1M chip + Azure deployment）+ Apple TPU adoption + OpenAI 失去 Azure exclusive = 客戶議價權從 LLM 廠 → 三巨頭 vendor 之間轉移。這個結構分化的速度比 2010-2015 雲端革命快 1.5-2x，意味著「等等看 capex digestion」的投資思維會 miss 2026-2027 estimate revision 機會。

★ 歷史類比（2 個正類比 + 1 個反類比）— QC-I28

正類比 1：1990s Telecom Capex Bubble（1996-2001）。當年 AT&T / WorldCom / Global Crossing / Sprint 全球光纖鋪設競賽 = 5 年 capex 從 $30B → $120B+ / 2000-2001 over-build / 2002 WorldCom 倒閉 + 大規模 capex digestion。量化對照：1996-2000 telecom capex 5 年 +300% / 但 capacity utilization < 5% (overbuild) / 隨後 -85% peak-to-trough；本次 hyperscaler 2024-2026 capex +175% / 但 utilization 40-60% (NVDA Jensen 多次提及 capacity constrained) / digestion 預計 2027-2028 mild not 1:1 reset。差異：(a) 客戶 visibility 不同（telecom 客戶分散且長期、hyperscaler 客戶 Anthropic / OpenAI / Apple 三家 = $50B+ 多年 commit）；(b) 商業模式不同（telecom 是 long-term contract underutilized / cloud 是 on-demand 已 utilized）。反向警示：若 2027-2028 大型 LLM 廠（Anthropic / OpenAI）任一 ARR growth 大放緩 → capex digestion 風險升 — 此為本 ID §13 falsification metric 之一。

正類比 2：2010-2015 公共雲第一波（AWS dominance + Azure 反擊）。當年 AWS 從 2010 ~$1B → 2015 ~$10B revenue / 期間 Azure 從 ~$0 → ~$5B 反擊成功 / GCP 仍落後 / 期間 enterprise IT spending 結構性 shift 至 cloud。量化對照：2010-2015 cloud revenue +10x 5 年 / on-prem IT spending CAGR -5% / Oracle on-prem 倍數從 18x → 12x（受擠壓）；本次 2024-2030 AI cloud revenue 預估 +5-8x 6 年 / on-prem AI infra 邊際化 / NVDA on-prem GPU rental（CRWV）受擠壓。差異：本次「客戶端 LLM 廠 vs hyperscaler vendor」議價權更平衡（Anthropic 三 cloud commit / OpenAI 失去 Azure exclusive），不是當年 AWS 一家獨大格局。

反類比：2000-2002 Fiber Overbuild（避免「這次不一樣」陷阱）。當年 1999-2001 全球海底光纖 + 北美 long-haul 光纖鋪設 capex ~$500B 累積（5 年）/ 2002 utilization < 5% / 倒閉潮（Global Crossing / 360networks / Williams Communications）/ 隨後 5-7 年才 digest。反類比警示：AI hyperscaler capex 2024-2026 累積 ~$700B+ / 若 LLM ARR 增速放緩 + Agentic AI 需求 < 預期 + Apple AI on-device 替代效應 → 2027-2028 過剩風險。多條件對照：fiber overbuild 當年 (a) 客戶 visibility 弱（dot-com 倒閉 = 客戶消失）、(b) capacity 過剩物理可見（大量 dark fiber）、(c) 商業模式 unproven。AI 目前 (a) 客戶 visibility 強（Anthropic + OpenAI ARR $30B + $25B）、(b) capacity 高 utilization（hyperscaler commentary 明確）、(c) 商業模式（per-token + per-seat + per-action）已 viable。多條件不成立但仍是 cautionary tale — 本 ID §9.5 Kill Scenario 將具體量化此情境的衝擊（連 capex 7-10% 削減 = AVGO -25% / NVDA DC rev -15% / hyperscaler stocks -15-20%）。

💡 §2 Insight
1. 本 ID thesis 的 time bound 是 2027 capex peak / 2028 ASIC 35-45% inflection / 2030 結構分化定型：2026-2027 是加速期 buy GCP + AVGO（自研最大受益者）；2027-2028 inflection 後應 reposition 至 NVDA share trough 反彈（55-60% 是 floor 不是 0%）；2028-2030 分化定型期 cross-vertical pair trade。
2. 1990s telecom 與 2000s fiber 反類比警示：capex digestion 風險 真實但不對等 — telecom 倒閉是「客戶消失」/ AI cloud 客戶（Anthropic / OpenAI / Apple）visibility 強，digestion 是「mild slowdown」而非 1:1 reset；本 ID base case 預估 2027 capex YoY +5-10%（vs 2026 +30-40%）非絕對值下滑，是 estimate revision 而非崩盤。

§3 核心技術棧拆解 — 6 個子技術 × 自研進度

Hyperscaler AI infra 技術棧由 6 個子技術構成：(1) AI accelerator silicon（NVDA / AMD / 自研 ASIC）；(2) Networking（NVLink / UEC / Ethernet / AVGO Tomahawk + Jericho）；(3) Storage（NVMe / Z-NAND / SCM）；(4) Power & Cooling（800V DC / 液冷 / 高效 PSU）；(5) CPU（自研 ARM 如 Graviton / Cobalt / Axion）；(6) Software stack（CUDA / ROCm / Pallas / Pathways）。本章節聚焦 silicon 自研進度（kingmaker 子技術），其餘交給專屬 ID 處理。

三家自研 silicon 進度差距大，**GCP TPU 領先 5-7 年（v1 2015 → v7 Ironwood 2026 H2 即將 ramp）/ AWS Trainium 約落後 4-5 年（Trainium 2018 試水 → Trainium2 2024-2025 大規模 / Trainium3 2026 H2）/ Azure Maia 落後 6-8 年（Maia 100 announce 2023 → 2024 部署 → Maia 200 2026 H2 ramp）**。這個 5-7 年的時間差解釋了為何 GCP 在 ASIC 經濟 (cost per token) 領先 — TPU v7 Ironwood 30-41% lower TCO/hr vs GB200/300（[T1-GOOGL Cloud Next 2025 keynote]）/ TPU v6e 4x perf/$ vs H100 + Trainium2 30-40% perf/$ premium（[T1: AMZN re:Invent 2025]）。
