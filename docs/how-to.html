Skill 之間的整合關係

11 套 skill 如何協作 — 兩條主動線

個股研究流：DD → DCA → comparator

stock-analyst 寫 DD（讀 docs/id/INDEX.md 命中 ID 則引用，不重寫產業通論）
deep-conviction-analyst 跑 DCA — Phase A4 自動讀 dd-meta JSON 取 trap / signal / moat（fallback HTML extract）
multi-stock-comparator-v1 讀本地 dca/ 或 dd/ 對 2-5 檔做四時距對比（不重做個股）
portfolio-manager 複盤時掃 docs/dd/INDEX.md 與 docs/dca/INDEX.md → 計算組合的 ID 曝險矩陣 (§7.5)

產業研究流：ID v2（已併入 DS）→ Flagship

industry-analyst 產 ID v2 敘事版（敘事為骨 + 表格為窗，併入原 DS 供需敘事弧）+ §11 列關聯個股 + §11.5 cross-ID 依賴圖
2026-06 起 DS 線已併入 ID v2 — 不再另產 DS 報告；docs/ds/ 8 份歷史 DS 凍結為 legacy 存檔（照常可讀）
id-review 對 ID 跑 mandatory critic（Step 8.7 強制 spawn）；legacy DS 仍可用 --mode ds
geopolitics-dd 產情境 → industry-analyst 可建對應風險 ID；portfolio-manager 拿情境 × 產業矩陣做組合壓測
多份 ID 累積 5+ → 建 flagship（跨 ID synthesis）

維運流：發布 + 更新

earnings-synthesis 每月跑 30 天統整
push-earnings 發新財報日報 + 更新 index
refresh-eps-screener 每月用新 Excel 更新 /dd-screener/

常見情境 / Troubleshooting

情境怎麼做
我不確定該跑哪個 skill describe 想要什麼結果即可（「我想知道 NVDA 現在該不該加倉」 → Claude 自動判斷該跑 DCA + PM）
Skill 執行到一半卡住 說「繼續」或「接下去」即可續跑；也可以說「換用 B 方案」讓 Claude 改路徑
數據跑出來感覺過時 說「這個數字有更新嗎」→ Claude 重跑 WebSearch；或直接呼叫 B refresh
