使用說明 — 如何呼喚 skill / SOP

四種觸發節奏：A 事件補丁 / B 季度 refresh / C 擴充 / D 重寫

B 季度 refresh 每 90 天主幹全面校正（對齊 Q 報季節） 季度 refresh flagship 每 90 天（主幹）

Skill 之間的整合關係

四套 skill 如何協作
industry-analyst 產 ID → §11 列關聯個股、§11.5 cross-ID 依賴圖
stock-analyst 建 DD 時自動掃 docs/id/INDEX.md → 命中 ID 則 §9/§11 引用（不重寫產業通論）
portfolio-manager 複盤時掃 docs/id/INDEX.md → 計算組合的 ID 曝險矩陣（§7.5）
geopolitics-dd 產生情境 → industry-analyst 可建對應產業風險 ID；portfolio-manager 拿情境 × 產業矩陣做組合壓測
多份 ID 累積 5+ → 建 flagship（跨 ID synthesis）
flagship 的 thesis 被 event 打臉 → A 補丁；90 天 → B refresh

四套 skill 共用 docs/dd/INDEX.md、docs/id/INDEX.md 和 references/ 作為單一事實源。任何一處更新會透過索引檔自動傳導。

心智模型 · Munger Latticework P1

20 個核心模型 · 五大領域 · 文章閱讀式
docs/mental-models/

心智模型頁依 Charlie Munger latticework 理念，整理 20 個 P1 核心模型，分屬五大領域：心理學（認知偏誤 / 激勵機制）、微觀經濟（護城河 / 激勵結構）、數學統計（複利 / 貝氏）、工程物理（臨界點 / 冗餘設計）、生物學（演化 / 生態競合）。

頁面採文章閱讀式設計：左側 sticky TOC 可快速跳轉各領域；每個模型為獨立可展開 section，含定義、Munger 原文引述、投資與商業實例、關聯模型 cross-reference；支援列印（展開全部 sections）。

⚡ 適用場景：投資決策前自檢思考偏誤（如可得性偏誤、確認偏誤）；商業案例分析時快速定位適用框架（如護城河五力、激勵機制）。P1 目前收錄 20 個模型，未來將擴充至 P2 / P3。
前往： /mental-models/ 或點選頂部導覽「🧠 心智模型」

常見情境 / Troubleshooting

情境怎麼做
我不確定該跑哪個 skill describe 想要什麼結果即可（「我想知道 NVDA 現在該不該加倉」 → Claude 自動判斷該跑 PM / DD / ID）
Skill 執行到一半卡住 說「繼續」或「接下去」即可續跑；也可以說「換用 B 方案」讓 Claude 改路徑
數據跑出來感覺過時 說「這個數字有更新嗎」→ Claude 重跑 WebSearch；或直接呼叫 B refresh
想同時跑多個產業研究 說「批次跑 ID_XX、ID_YY、ID_ZZ」但注意品質可能略降（參考：一個一個跑最穩）
發布前要確認沒錯 skill v1.4 會自動跑 7 Gates；若想手動 peer review 說「讓我看草稿，還不要 push」
💡 最重要一句：你不需要記所有指令。描述想要的結果即可，Claude 會判斷該跑哪個 skill。這頁只是列出「最精準的指令」方便你想要精準控制時用。

自動更新頁面 — Markets / Sectors / Screener

GitHub Actions 定時排程，無需人工介入；亦可手動觸發
scripts/ + .github/workflows/
頁面網址更新內容排程手動觸發
Markets /markets.html 28 個全球指數 level、YTD、RS + Trend（benchmark VT） 每週六 08:00 TW（00:00 UTC） GitHub → Actions → Weekly Market Update → Run workflow
Sectors /sectors.html 美股 23 檔板塊 ETF price、YTD、PE、AUM 每週六 08:00 TW（同上）
Screener 美股 /screener.html QGM 選股清單（latest.json + fundamentals.json） 每日 06:30 TW（22:30 UTC 前日） GitHub → Actions → Daily Screener US → Run workflow
Screener 台股 /screener-tw.html QGM 台股選股清單（tw_latest.json + tw_fundamentals.json） 每日 07:00 TW（23:00 UTC 前日） GitHub → Actions → Daily Screener TW → Run workflow
⚡ 手動觸發步驟：GitHub repo 頁面 → 上方 Actions tab → 左側選對應 workflow → 右側 Run workflow 按鈕 → 選 main branch → Run workflow。約 1-3 分鐘後 commit 自動推上 main，頁面即更新。
⚡ 腳本位置： scripts/update_markets.py（Markets + Sectors）、 scripts/screener.py（美股）、 scripts/screener_tw.py（台股）。 資料寫入 docs/markets.html、docs/sectors.html、docs/screener/latest.json 等，GitHub Actions 自動 commit + push。
⚡ 不更新的欄位：PE、PB、PEG、EPS 成長率、展望、P/NAV 等基本面敘述欄位需人工維護，腳本只更新量化市場數據。
你：（想立刻刷新 Markets 數據）
做法： GitHub → Actions → Weekly Market Update → Run workflow；或在本地執行 python scripts/update_markets.py（需已安裝 yfinance）
