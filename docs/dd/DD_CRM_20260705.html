§2｜序章：第一性原理 × 逆向思考

存在意義（第一性原理）

企業要成長，必須知道「誰是客戶、他們買什麼、下一步該賣什麼」。這件事在紙本時代靠銷售員的腦袋與 Rolodex，在雲端時代必須有一個單一、共用、隨時更新的客戶真相來源（system of record）。Salesforce 的第一性存在意義就是這個 system of record：把一家公司所有與客戶互動的資料（銷售管線、客服工單、行銷觸點、電商訂單）鎖在一個平台裡，讓每個部門看同一份客戶真相。這是它 25 年、$41.5B 營收、150,000+ 客戶的地基。

agentic AI 對這個地基的意義是雙面的。威脅面：若 AI agent 替代人類完成工作，公司需要的「人類 seat」下降，而 CRM 的傳統計價正是按 seat 收費 → 收入基礎被侵蝕。機會面：AI agent 要能自主行動，必須讀取「乾淨、結構化、有權限治理的客戶資料」— 而這正是 Salesforce 25 年鎖住的東西。誰擁有 system of record，誰就擁有 agent 的燃料。Agentforce（agent 執行層）＋ Data 360（資料層）＋ Flex Credits（$0.10/action 的消費計價）就是把「按 seat 收費」轉成「按 agent 工作量收費」的商業模式手術。這場手術是否成功，是本報告的核心命題。

為什麼是 Salesforce（競爭起源）

1999 年 Marc Benioff 用「No Software」的口號賭一件事：企業願意把最敏感的客戶資料放到別人的雲上，只要換來「不用維護、隨處可用、持續更新」。這在當時是異端（甲骨文、SAP 賣的是 on-prem 授權）。Salesforce 贏的不是技術，而是商業模式的時間差 — 它比所有 on-prem 巨頭早 5-10 年 all-in 訂閱雲，等對手回神，它已累積了足以形成 switching cost 的裝機基礎與生態（AppExchange、Trailhead 開發者）。這條護城河的本質是「先發 + 資料累積 + 生態鎖定」的複利，不是任何單一產品的優越。

而這正是 agentic AI 對它的雙面性根源：先發優勢在「system of record」層仍成立（25 年資料難複製），但 agentic AI 是一次新的商業模式時間差 — 這次的異端是「按工作量（agent action）計費」取代「按人頭（seat）計費」。歷史會不會重演、只是這次 Salesforce 站在被顛覆的一方（就像當年它顛覆 Oracle）？還是它能用資料層優勢把自己從 seat 模式平滑轉到 consumption 模式、成為 agentic 時代的 system of record？這是本檔 3-5 年的核心賭注。

Munger Inversion：三維失效路徑

  • 技術維：agentic AI 讓「客戶自己在 Bedrock / Azure OpenAI / Vertex 上組裝 agent」比買 Agentforce 便宜（build-vs-buy）→ system-of-record 的資料優勢被「客戶用自己的 LLM 直連自己的資料」繞過。歷史對標：2000 年代 on-prem CRM 被 SaaS 顛覆——顛覆者往往是「架構更輕」的一方。
  • 管理維：連續高階主管流失（3 個月 4 位 senior exec 離職，含 AI/Agentforce GM Adam Evans）+ ~1,000 人裁員（含 Agentforce 團隊）→ AI 執行力在最關鍵時刻動盪。歷史對標：Slack 併入後兩年減損約 50%，整合執行是 Salesforce 的歷史弱點。
  • 結構維：per-seat 計價被 consumption 計價取代的過程中，若「消費金額成長 < seat 流失金額」，總營收淨縮。加上 Salesforce 自己把客服團隊 9,000→5,000（用 Agentforce），這既證明產品有效、也證明整個企業軟體的 seat-TAM 可能系統性縮小。

D｜邊際貢獻判斷框架

F｜Single Thing（binary discrete event）

Single Thing：Q2 FY27 法說（~2026-08-27）的 cRPO YoY 與有機成長，是否確認「Agentforce consumption 已淨蓋過 seat 蠶食」。

元素定義
① Binary outcome成功：Q2 cRPO YoY ≥ 12% 且有機營收 ≥ 8% 且 Agentforce ARR 續增至 ~$2B → H1「淨」效應轉正證據。失敗：cRPO YoY < 10% 或有機 < 6% → 「seat 蠶食 > consumption 新池」確認，R1/R3 升溫。
② Trigger event2026-08-27 前後 Q2 FY27 earnings：cRPO YoY、有機成長（剔 Informatica）、Agentforce ARR QoQ 增量、FY27 guidance 是否上修

  • 轉換成本：換掉 Salesforce = 重建銷售/客服流程、遷移多年客戶互動資料、重接數十個整合（AppExchange、ERP、行銷）→ 大型企業需 12-36 個月 + 數百萬美元 + 業務中斷風險。這是 CRM 25 年最硬的護城河。
  • 資料壁壘：Data 360 統一 150k 客戶的結構化客戶資料 = agent 的燃料。agentic AI 時代「誰有乾淨、有權限治理的客戶資料，誰的 agent 才可信」。
  • 網路效應：AppExchange 第三方應用生態（數千 app）+ 開發者社群（Trailhead）= 平台黏性。

競爭鴻溝（為何資本充足對手仍跨不過）

① 時間壁壘：25 年累積的 system-of-record 資料與流程綁定，非資本可短期複製；② 認證/整合壁壘：企業 IT 生態圈的深度整合需多年建立。但關鍵警示：agentic AI 正在降低這道鴻溝的相關性 — 若客戶用自己的 LLM 直連自己的資料（build），Salesforce 的「資料+流程」優勢被繞過。這是護城河的結構性侵蝕向量（非 point-to-point）。

估值隱含成長反推（市場在定價什麼？）

用簡化的股權自由現金流永續模型反推：現價 $166、FY27 FCF/股 ~$17.7（FCF $14.4B / 819M sh，且 FCF > non-GAAP NI），P/FCF ~9.4x。以股權成本 ~9%（beta 1.18）反推，P/FCF 9.4x 隱含的永續成長率僅 ~GDP 水準（~3-4%）。換言之，市場已把 Salesforce 定價成「有機成長沉澱到 GDP+、ex-growth 成熟軟體」。
