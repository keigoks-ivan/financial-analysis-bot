§4｜序章：第一性原理 × 逆向思考

存在意義

Cloudflare 解決的是「全球互聯網在一個不可信任、高延遲、遭受攻擊的世界中如何繼續運作」的問題。若它明天消失：

  • 全球 ~20% 的網站即時降級（Cloudflare 處理 8,000 萬+ HTTP 請求/秒）
  • DDoS 攻擊從「被過濾的背景雜音」回到「直接打倒服務」
  • Zero Trust 取代 VPN 的遷移進度倒退 2-3 年
  • Workers / R2 取代 Lambda / S3 的邊緣運算敘事消失，hyperscaler 壟斷加深

逆向推演（Munger Inversion）

若要讓這筆投資在 3 年內虧損 50% 以上，最可能路徑：

  • 地緣政治：中國資料當地化政策擴散（2026 中國 JV 拆分已發生），若 EU Digital Sovereignty Act 禁止美國 CDN 處理 EU 公民資料 → 歐洲收入 ~25% 面臨重構
  • 技術替代：AWS 於 2027 推「Bedrock at CloudFront edge」且零 egress → NET 的 Workers AI + R2 核心論點被複製。機率：中（已觀察到 AWS 對 egress 策略鬆動）
  • 管理層失職：Prince 在 agentic internet 敘事上過度 allocate capex 至 GPU PoP，若 Workers AI 營收 < $200M 年化（FY27），毛利率跌破 72% = 市場對「edge 輕資本」敘事徹底 re-rate
  • 價值陷阱路徑：上方 4 項都 Out → 情境仍可能走「估值過熱型」：公司基本面無問題，但 Fwd PE 180x → 60x 的 mean reversion 本身就是 -65% 回撤（類似 2022 SaaS 泡沫破裂）

最可能的陷阱模式：非護城河侵蝕、非盈餘品質，而是「估值多重壓縮」——市場願意付的倍數從 180x 降至 80x，即便 EPS 成長完全 on track。

§9｜護城河分析

評分：7 / 10（S 級門檻為 8.5+，Cloudflare 尚未達到 S 級，屬 A 級護城河）

護城河來源

  • 網路效應：✅ 強。330+ 城市 PoP + 8,000 萬請求/秒形成的威脅情報池，每個新客戶貢獻 attack pattern = 所有客戶受益。
  • 無形資產：✅ 中。品牌在開發者社群強（GitHub stars、技術聲譽），但無專利/監管壁壘。
  • 轉換成本：⚠️ 中等。Free tier → Paid 漏斗容易黏住，但 enterprise 遷移至 AWS/Akamai 技術門檻不高（API 相容）。Zero Trust 整合後黏性提升。
  • 規模優勢：✅ 強。Anycast 網路的邊際擴張成本遠低於單點建置，獨佔 "every server is every server" 的架構。

競爭鴻溝

AWS、Akamai、Google 都有技術能力複製 CDN/edge compute，但：

  • AWS 的 incentive 結構（egress 收租模式）使其不願主動打破 zero-egress 敘事
  • Akamai 架構不支援 Workers-style 0ms cold start，需要重建平台
  • Google Cloud CDN 是 hyperscaler 捆綁產品，非獨立 platform

這不是 unbreakable 的鴻溝，而是「AWS 選擇性不全力反擊」的結構。一旦 AWS 決定打破 egress 收費（如 2027 可能事件），護城河深度顯著下降。
