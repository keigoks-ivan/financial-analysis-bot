§4 序章：第一性原理 × 逆向思考

存在意義

AppLovin 解決「廣告預算的邊際 ROAS 無法即時歸因」這個自數位廣告誕生以來的基本痛點。傳統 DSP（The Trade Desk 類）把決策權交給買方——買方須自選 audience、自配預算、自測 creative；AXON 2.0 把決策權交給 AI：每秒 200 萬筆 auction、10 億裝置的 cross-app behavioural graph、即時 ROAS 反饋。對中小型遊戲廠商，AXON 的「黑箱但可預測」比 Meta Advantage+（封閉生態）或自建 DSP（缺資料）更有效。若 APP 明天消失：中小 dev 被迫選擇（a）把錢丟進 Meta/Google 封閉生態、失去跨平台歸因；（b）自建 DSP、承受 60% 的資料劣勢。它取代的不是一家公司，而是「中小廣告主獲得可預測 ROAS 的能力」。

逆向推演（Munger Inversion）

  • 地緣政治曝險：研發在美國/EMEA，資料處理雲端 AWS/GCP 美國為主；手遊發行業務已於 2025/5 以 $400M + 20% 股份賣給 Tripledot，地緣風險大幅降低。主要曝險是「AI/隱私監管」而非生產地。
  • 技術替代：① Apple iOS ATT+ / Privacy Sandbox 若封鎖 cross-app device ID，AXON 資料飛輪被截（時間 12-18 個月）；② Meta 若開放 Advantage+ API 給第三方 dev（可能性中）；③ 新進 AI-agent 廣告 orchestrator（OpenAI Operator 類）整合 GPT-5 + API（2026-2027）。
  • 管理層失職：CEO Foroughi 近 5 年 25 筆內部人交易全為賣出（2026/3/12 再賣 $19M，2025 套現 $80M）；資本配置激進回購（2025 買 $2.58B = 65% FCF）。潛在失職：併購分心（Wurl CTV 整合失敗）、或在短報告壓力下過度回購耗損投資彈性。
  • 價值陷阱路徑（四選一）：護城河侵蝕型。若 SEC 認定 AXON 跨 app 資料提取違規且強制修改歸因架構，AUC 回退 → ROAS 下滑 → 中小 dev 流失 → 規模飛輪反轉。此為主要下行劇本。

F｜Single Thing（v12.3 新增 — 單一 Binary Trigger）

🎯 Single Thing：「SEC 發出正式 Wells Notice 或達成強制修改歸因架構的和解協議」

為什麼是 Single Thing，不是 R1 過程性風險？

R1（SEC 裁罰）是持有期間的過程性監測（⚡ 短期尺度），配合連續監測指標（Wells Notice / 和解 / MTD 被拒）。但「Wells Notice 正式發出」這一刻是 discrete binary event——它不是「成長率下滑 X%」這種連續漂移，而是在某一天突然從「調查中」變為「正式指控」。

觸發後的判斷改變路徑（可觀測、不模糊）：

  • SEC 發正式 Wells Notice → AXON 歸因架構「可能被迫重建」已從理論變為法律程序 → H1「AXON ROAS 優勢結構性」的前提被質疑 → 立刻重評護城河至 5/10（C 級）→ 訊號從 B 降至 X
  • SEC 和解但不要求修改歸因 → H1 反而得到正面確認 → 訊號可從 B 升至 A

與 §5.C R1 職責切分：§5.C R1 是持續監測 + 漸進警戒；§5.F 是「一旦發生就立刻改判斷方向的 single moment」。二者不重疊。

Cross-check vs DD §5.F：本 DCA（若有）§5 Single Thing 應一致引用此 trigger；若 DCA 的 single thing 不同（如 e-com ARR），需在 DCA 中說明分歧理由。

對上游供應商議價權

AppLovin 的上游供應商主要為雲端基礎設施商（AWS / Google Cloud Platform），集中度高（2-3 家主導），但替代性尚可——APP 具備跨雲部署能力，且軟體廣告平台的「算力」是彈性可擴展資源，非定制硬體（不同於 TSMC 代工依賴）。AWS 協議議價能力隨 APP 使用量增長而提升（enterprise pricing tiers）。2026/5 時點無公開的供應商集中度或議價糾紛披露。關鍵觀察：Capex/Rev 僅 0.5%（FY2025），說明 APP 幾乎不依賴資本密集的自建基礎設施，對供應商的依賴屬「必要成本」而非「被動接受」——若 AWS/GCP 大幅漲價，APP 有技術能力遷移（但短期切換成本存在）。上游議價權整體：中等偏高（多雲能力 + 低 Capex 需求）。
