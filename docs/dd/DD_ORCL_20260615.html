§4　序章：第一性原理 × 逆向思考

存在意義

Oracle 解決兩個長期且高度不可逆的痛點：①企業「記錄真相」的資料庫 — 全球銀行、電信、航空、政府的交易系統跑在 Oracle Database 上，一筆轉帳、一張機票、一筆稅，背後是 ACID 一致性與數十年累積的維運慣性；②企業「營運骨架」的 ERP/HCM（Fusion、NetSuite）— 總帳、薪資、供應鏈的 system of record。這兩者的共同點是：遷移成本不是金錢問題，是「停機 = 業務癱瘓」的存亡問題。若 Oracle 明天消失，全球後台會出現的不是「效率下降」，而是金融清算、電信計費、政府發餉的系統性中斷。
2023 年起 Oracle 疊上第三條腿：OCI（雲基建） — 把同一批 mission-critical 客戶的資料庫搬上自家雲，並切入 AI 訓練/推論的 GPU 算力出租。前兩條腿是 80%+ 毛利的護城河資產；第三條腿（OCI）目前是 ~14% 毛利、資本密集的「規模賭注」，是本份 DD 全部張力的來源。

護城河如何累積而成，見 §9.D 時間軸。

逆向推演（Munger Inversion）— 3 年內虧 50%+ 的路徑

  • 技術替代維度：AI infra 客戶（尤其 OpenAI）轉向自建資料中心或改用更便宜的 neocloud（CoreWeave 類）/ 超大規模自研晶片，Oracle 的 GPU 出租被當作「過渡產能」用完即棄 → $638B RPO 中 backlog 的可實現性折價。時間軸：12–24 個月（合約可重議/延後提貨）。
  • 管理層失職維度：在 14% 毛利仍以債 + 股權（$40B/年）擴 capex（FY27 淨 $70B），若終端需求或毛利不如預期，等於用昂貴資本買低報酬資產 → 自我毀滅式資本配置。自我毀滅決策路徑詳見 §12 資本配置計分卡。
  • 結構/產業過時維度：AI 算力供給若由短缺轉為過剩（如 Micron 對記憶體週期的警告外溢），GPU 租金 ASP 崩 → Oracle 已投入的折舊資產面臨閒置減值。架構性替代分級見 §9 QC-23 ⛔。
  • 價值陷阱路徑（四選一）：隱性資本密集型 — 帳面 EPS 看似成長，但維持 OCI 競爭力所需的資本支出被市場系統性低估，真實 owner earnings 遠低於非 GAAP NI。歷史對標：2000 年代電信/光纖「蓋了就有人來」的產能泡沫。

② M&A 飛輪判定

Oracle 歷史是「FCF → 併購 → 整合 → 更多 FCF」的飛輪（PeopleSoft→Sun→Cerner）。但 FY26 起飛輪已切換：不再是「FCF 驅動併購」，而是「外部融資（債+股權）驅動 capex」。新模式的增量 ROIC 取決於 OCI 毛利回升，目前未證實 > WACC。風險：FCF 不僅沒變股東現金，反而被「越來越大的下一輪 capex」吃掉，且須靠稀釋股權與舉債填補。

③ 一句結論

這檔的成長最終被下一輪 capex 吃掉（FCF −$23.7B、capex > OCF、須外部融資），增量 ROIC ≈ WACC（OCI 段未證實 >），飛輪由「併購複利機器」切換為「需警戒出價/投資紀律的重資本賭注」。對齊 §8.0 第 4 問（最弱一題）裁決。
