§4｜序章：第一性原理 × 逆向思考

存在意義

ALAB 解決的核心痛點是「AI 計算系統內部高速數據傳輸的訊號完整性與規模化」。當 GPU / CPU / Memory / Storage 之間的訊號需要跨越 PCB、cable、rack、跨 rack 傳輸時，訊號會衰減、扭曲、丟失同步。ALAB 透過 Aries（PCIe retimer）/ Scorpio（AI Fabric Switch）/ Taurus（Ethernet AEC）/ Leo（CXL Memory Controller）這四個產品線，分別解決四種互連場景的訊號完整性問題。

若 ALAB 明天消失，AI cluster 的建構成本與時程會立即受影響：每一條 PCIe Gen 6 鏈路需要 Aries 重新標定訊號，否則會限制 GPU 與 CPU 跨 socket 距離；每一個 8-GPU+ cluster 需要 Scorpio 提供 fabric switch；每條 50m+ rack-to-rack ethernet 需要 Taurus AEC。短期沒有完全替代品（Broadcom / Marvell 在 PCIe Gen 6 still 落後 12-18 月）；長期 silicon photonics 可能取代部分 use case，但時程在 5-7 年外。

逆向推演（Munger Inversion）三維展開

① 技術維度：什麼技術替代會讓 ALAB 3 年內失去 50% 價值？

  • Silicon photonics 取代電子 retimer：CPO（Co-Packaged Optics）若在 PCIe Gen 7（2027-2028）成熟，將部分取代 retimer use case。歷史對標：Intel optical interconnect 嘗試 2015-2020，量產延後 5+ 年，目前 Intel / NVIDIA / TSMC 仍以 packaging 內 photonics 為主，rack-level 仍需 retimer。
  • NVIDIA / Hyperscaler 自研 retimer ASIC：CRDO 2025 教訓 — AWS Trainium 3 將 SerDes 從 Credo 換到 Synopsys IP。若 ALAB top 2-3 hyperscaler（NVIDIA、Google、AWS）任一自研 PCIe retimer ASIC，可能蠶食 30-40% 市佔。歷史對標：AWS Annapurna 收購（2015）後 Nitro chip 取代部分 Mellanox 功能。
  • Broadcom / Marvell PCIe Gen 6 量產：兩者預期 2026 H2-2027 H1 量產，會直接壓 ALAB ASP 10-20%。歷史對標：Marvell vs Inphi（被併）SerDes 競爭，Inphi 護城河仍保持高 GM，但成長放緩。
  • UALink 2.0 標準化後降低差異化：開放標準若使 retimer/switch 成為 commodity，ALAB 高 GM 將難維持。但 UALink 2.0 spec 仍需高品質 implementation，ALAB 的 IC 設計優勢仍保留。

② 管理維度：什麼決策路徑會自我毀滅？

  • 過度併購整合失敗：FY26 H2 可能 tuck-in M&A（如 silicon photonics IP），若大型併購（> $1B）整合失敗會稀釋 EPS + 分散管理層注意力。歷史對標：Marvell-Inphi 併購（2021，$10B）整合 3 年後仍未完全發揮綜效。ALAB 創辦人 CEO Jitendra Mohan + CTO Casey Morrison 雖技術強但無大型 M&A 經驗，是潛在風險點。
  • Capex / R&D 慣性：FY25 R&D +60%（vs Rev +115%）合理，但若 FY27-28 hyperscaler 需求放緩仍維持 R&D +50% 將侵蝕 OPM。歷史對標：Intel 14nm/10nm 延後期間維持 R&D 高位導致 ROIC 連 5 年下滑。
  • 創辦人退場 / 主管離職：CEO + CTO 為共同創辦人，IPO 後 lock-up 已解。若 2026-2027 任一離開將觸發市場信心問題。

③ 結構維度：什麼產業變化會讓商業模式過時？

  • AI capex 進入 plateau（FY28-29）：若 hyperscaler AI capex 在 FY28 開始 plateau（從 +30% YoY 降至 +5%），ALAB 高 PE 將難維持。歷史對標：cloud 1.0 capex 2016-2018 plateau 後 Equinix / Coresite 估值倍數 5 年下移 40%。
  • 監管型分拆 / Hyperscaler 反壟斷：若 Google / Amazon 被分拆，會打亂 ALAB 大客戶 ASIC roadmap。短期可能性低（2026-2027 大選後可能性更低）。
  • 計算範式轉換（量子 / 神經型計算）：5+ 年外，量子計算或神經型晶片若取代部分 AI 訓練，會改變 互連需求。歷史對標：2010s GPU 取代 CPU 訓練 → NVIDIA 取代 Intel 領導地位。對 ALAB 影響：5+ 年外，目前不影響近期。

價值陷阱路徑判定：四類陷阱中，最可能的是「盈餘品質型」+「產業結構衰退型」雙重風險（hyperscaler capex plateau + 自研替代），但這些風險的時程在 FY28+，已部分被 Forward PE 75x 提前定價。短期最大風險仍為「估值 mean reversion」而非基本面陷阱。

競爭鴻溝

為什麼 Broadcom（市值 $1T+）、Marvell（市值 $80B+）、Microchip 等資本充足對手仍無法跨越？

  • 多代產品 IP 連續性：retimer / switch IP 不是單代設計，需要從 Gen 4 → Gen 5 → Gen 6 連續迭代（每代都需要重新驗證 SI/PI、driver、firmware）。Broadcom / Marvell 在 Gen 5 沒大規模量產，Gen 6 從 0 起步進度慢。
  • Hyperscaler 信任壁壘：hyperscaler 對 retimer / switch 的 firmware bug 容忍度極低（一個 bug = $10M+ data center 重啟成本），需要 2-3 個成功代別才建立信任。ALAB 在 Aries Gen 4-5 累積信任後 Gen 6 自然延續。
  • Fabless 系統公司模式：ALAB 是「IC 設計 + system-level cabling + driver / firmware + 服務」整合，後進者只賣 IC 不夠，需要建構整套 ecosystem。
