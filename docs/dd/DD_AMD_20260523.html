§4 序章：第一性原理 × 逆向思考（v12.2 擴充）

存在意義

AMD 的核心解決方案是「在 NVDA 一家獨大的 AI 計算市場提供 hyperscaler-grade 可信第二供應商」+「在 INTC 數十年寡占的 x86 伺服器市場提供高性能替代」。AI 時代 hyperscaler / 雲廠商 / 大型企業 / 主權國家對單一供應商依賴的厭惡程度，遠高於 2010 年代 Intel 寡占時期 — 因為 AI compute 的 stake 是國家級戰略資產，不是邊際 IT 採購。若 AMD 明天消失，世界會出現：(1) NVDA 對 hyperscaler 的議價權再翻倍，hyperscaler 對 AI infra 的 capex 效率惡化 30%+；(2) Intel x86 重新獲得 monopoly pricing power（無 EPYC 對手），雲廠商伺服器 CPU 成本上升 15-20%；(3) 全球 AI 供給端形成單點故障風險（geopolitical / supply chain）。

逆向推演（Munger Inversion）三維展開

① 技術維度：什麼技術替代會讓 AMD 3 年內失去 50% 價值？

  • NVDA Rubin / Rubin Ultra 全面壓制：若 Rubin 2026 H2 + Rubin Ultra 2027 Q2 性能領先 MI400/MI450 ≥ 40%（FP4 compute + memory bandwidth + NVLink 6），hyperscaler workload 全面回流 NVDA，AMD AI GPU 收入停在 $25B 不再加速。歷史對標：Intel Itanium 2001 vs AMD64 — AMD 突破後 Intel 從未追回。反向風險。
  • Hyperscaler 自研 ASIC 全面取代通用 GPU：AVGO 70% 客製 ASIC 市佔 + Marvell Trainium3 + Google TPU + Microsoft Maia 持續搶 hyperscaler 增量。AMD 在 inference 通用 GPU 增量需求和自研 ASIC 之間夾縫。歷史對標：Apple silicon 從 Intel switch，3 年從 0 至 100% Mac CPU。
  • 量子計算 / 光計算商用化：若 2028-2030 量子或光計算在 specific workload（藥物發現、密碼、優化）成本 / 性能優於 GPU ≥ 10x，AI infra capex 結構性轉向。歷史對標：FPGA 對 ASIC 競爭 — GPU 對 ASIC 競爭仍非 zero-sum。低機率。
  • 軟體層級 abstract away hardware vendor：若 OpenAI / Anthropic / Google 等模型廠商通過 compiler layer（OpenAI Triton 等）完全 abstract 硬體差異，CUDA lock-in 消失但同時 AMD 的差異化也消失，GPU 變成 commodity，pricing power 全壓縮。歷史對標：x86 server 商品化 → margin 從 70% → 30%。

② 管理維度：什麼決策路徑會自我毀滅？

  • Capex 慣性過度擴張：若 AMD 為配合 OpenAI 6GW + Oracle 50K 而大量 prebook TSMC 2nm capacity，但 hyperscaler 後續 capex 環境轉差（2027 經濟下行），AMD 卡在 capacity commitment 而 demand 不足，inventory 暴增 + GM 壓縮 1500bps。歷史對標：Intel 14nm 過度投資 + Cisco 2001 庫存暴雷。
  • Lisa Su 離任繼任問題：Lisa Su 是 AMD 護城河的核心驅動，若 2028-2030 退休而繼任者執行力不及，technology roadmap delay + culture 削弱。歷史對標：Apple Tim Cook 接 Jobs 順利，但 Intel 持續換 CEO 後執行力惡化。機率中。
  • 整合失敗 — Xilinx synergies 未實現：$49B 收購 Xilinx 至今 4 年，data center networking + FPGA / AI integration 仍在路徑上，若 2028 仍未產生 visible 業務貢獻，視為 capital misallocation。歷史對標：HP 收購 Autonomy / 各大 telco 收購案多失敗。

③ 結構維度：什麼產業變化會讓商業模式過時？

  • AI 訓練 vs 推理結構性轉變：若 2027-2028 AI 演算法（agent + retrieval）導致大模型訓練需求飽和，市場主流轉向小模型 + 推理 + edge，AMD 在 推理性能優勢被 ASIC + edge SoC 稀釋。歷史對標：手機 SoC 取代 PC CPU 增量需求 — Intel 錯過 mobile。機率中高。
  • 監管型分拆：若 OpenAI 因 antitrust 被分拆 + AMD 因 OpenAI 6GW 與 OpenAI 過度綁定 → 監管要求 AMD 開放架構 / 限制獨家合約，pricing power 壓縮。歷史對標：Microsoft IE 反壟斷案。低機率（10 年內）。
  • 計算範式轉換 — neuromorphic chip 商用化：若 IBM / Intel 等的 neuromorphic chip 在 specific workload 上 10-100x 性能領先傳統 GPU，AI infra 結構轉變。歷史對標：CPU → GPU 轉換用了 15 年。低機率（2030 前）。

價值陷阱路徑判斷：若這是價值陷阱，最可能的惡化劇本是「成長股估值陷阱」反向變體 — 不是基本面崩壞，而是估值倍數壓縮（從 FY+2 36x → 25x，即 -30%）+ 成長率 reset（CAGR 從 56% → 30%）雙殺。當前 $467 entry 沒有 margin of safety，這是核心風險。

ROCm 軟體生態仍在 building phase — 當前 PyTorch native 支援程度 ~70%（vs CUDA 100%），TensorFlow ~60%。規模閾值估計：需 PyTorch native ROCm contributor base 達 CUDA 的 50%（當前 ~20%）才觸發 network flywheel。當前不算成熟網路效應。

無形資產

① x86 cross-license（與 INTC 1976 簽署，是 institutional moat）— 新進者無法跨越；② Xilinx FPGA IP（5000+ 專利 + 工業 / 通信 / 國防客戶 deep relationships）；③ EPYC architecture 累積 5 年 iteration 的軟體優化；④ ROCm 軟體棧（雖落後 CUDA 但已是業界第二大 GPU compute 生態，~2000+ contributors）。

轉換成本

EPYC ↔ Xeon：x86 ISA 相容，業務 logic 可移植但需重新 BIOS / driver / 性能優化 — 中等成本（hyperscaler 部署需 3-6 個月）；MI300X ↔ NVDA H100：CUDA lock-in 是巨大障礙，hyperscaler 需重寫 GPU kernel + 重新優化 PyTorch model — 高成本（6-12 個月 + 性能風險）。

規模優勢

AMD 年營收 ~$35B vs NVDA ~$200B — 規模仍小 ~6x。在 TSMC capacity allocation、HBM4 採購等 supply chain 環節，NVDA 有 absolute priority。AMD 在 hyperscaler 整體 spend 中佔比 ~7-8% vs NVDA 65-70%。要追上需 5-7 年。

議價權三段獨立（v12.3 強化）

① 對上游供應商

AMD 供應商集中度極高：TSMC（fab capacity，~90% 集中度）+ SK Hynix/Samsung/Micron（HBM4，三家寡占）+ ASML（光刻設備間接相關）+ Synopsys/Cadence（EDA 工具雙寡占）。替代難度：High。近 3 年議價案例：(1) TSMC 2nm 價格從 N3 +15-20%，AMD 接受並 secure allocation（沒有壓低能力）；(2) HBM4 緊缺，AMD 與 SK Hynix 簽署 multi-year LTAs，價格高於 spot；(3) ASML EUV 設備支出共擔機制（與 TSMC、Samsung 等 share fab capex）。整體：對上游議價權極弱，TSMC 是不可替代的單一 fab partner，HBM4 是 AI GPU 性能核心瓶頸。

② 對下游客戶

客戶集中度（引用 §8.H）：Top 5 客戶 ~50%（hyperscaler 集中：MSFT, Meta, Google, AWS, Oracle）。合約結構：(1) 一般 EPYC：訂單型 + multi-quarter LTA；(2) MI300X / MI400：MOU 型 multi-year 框架（OpenAI 6GW、Oracle 50K MI450 multi-year）；(3) consumer / embedded：訂單型 spot。近 3 年客戶 case：(1) Microsoft MAIA 自研啟動但 MSFT 仍持續加碼 AMD MI300X；(2) Meta 採購 MI300X 高於原計劃；(3) Oracle 50K MI450 是首次大型 hyperscaler exclusive design win。定價能力：(1) MI300X / MI450 ASP 維持高位（hyperscaler 願 pay for diversification 而不僅是 cost-down）；(2) EPYC 對 INTC 仍 maintain pricing parity（不需大幅折讓）。

③ 地緣曝險

核心生產地：TSMC Taiwan（~85%，N3P / 2nm）+ Samsung Korea（部分 HBM4）+ Globalfoundries USA（少量 client / embedded）。供應鏈最高集中點：TSMC Taiwan（單一 fab partner，地緣風險最高）+ HBM4 Korea（地緣風險中）+ EDA Synopsys/Cadence USA（受美國 export control 直接影響）。政策風險具體路徑：(1) 美國 export control to China（AMD 已停止 MI300X 對 China 銷售，影響 ~5% 營收）；(2) 中國反制（若 GPU 限購反制 + 國內 GPU 自研加速）；(3) Taiwan 地緣事件 → 全面 supply chain 危機。已有分散化措施：(1) AMD 與 GlobalFoundries 簽 wafer LTA 至 2028（client / embedded 多元化）；(2) 探討 TSMC Arizona Fab 6 量產可能性（2027+）；(3) 與 Intel Foundry 探討合作（早期）。整體曝險中高。
