## Batch 4 Systemic Findings

### 問題 1：§13.5 砍除不徹底（影響 3017 — FAIL 等級）

3017 版本記錄聲稱已砍 §13.5，但節體完整殘留（含舊五角驗證表 + 舊 v11.0 §2 D 交叉驗證邏輯）。這是 v12.3 upgrade patch 的假完成模式：**changelog 更新了，body 沒砍**。

此問題在 3017 為 FAIL 等級（整個 §13.5 h3 節仍在），在 2454 為 WARN 等級（§13.5 節已砍但 §2 內有兩處引用殘留），在 2383TW 為 WARN 等級（§13.3 被引用於 §3）。

**Systemic pattern**：v12.3 升級 patch 在更新版本記錄時，未做「砍除章節後全文搜尋替換所有引用」的完整清理步驟。

### 問題 2：§11 議價權三段格式 vs 內容（影響 2330TW — WARN 等級）

TSMC 的 §11 議價權三段以「段落標題 + 表格/bullet」呈現，v12.3 spec 要求「各 ≥ 60 字獨立 `<p>`」。2330TW 的內容品質無疑問（遠超 60 字），但格式不符規範。對於 TSMC 這類議價關係複雜的標的，表格可能確實比 `<p>` 更清晰，但這是 spec 偏差。

**無 systemic pattern**：其他 4 個 ticker 均有正確的獨立 `<p>` 段落。2330TW 是個案。

### 問題 3：§8.H 台灣年報匿名慣例處理正確（全 5 ticket）
