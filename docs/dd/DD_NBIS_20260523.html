§4 序章：第一性原理 × 逆向思考

存在意義

Nebius 解決一個技術經濟學的根本問題：當生成式 AI 的核心算力（NVIDIA GPU）是稀缺、昂貴、且需要極高度專業化資料中心配置時，誰能比 hyperscaler 更快、更便宜、更靈活地把這些算力交付給 AI builder？NBIS 的存在前提是：(1) NVIDIA 願意把早期 chip access（如 Vera Rubin）優先給「中立 / 戰略合作夥伴」而非完全綁定大客戶；(2) AI builders（Microsoft 自己、Meta、AI labs、enterprise）並不希望 100% 算力依賴 AWS/Azure/GCP，需要 multi-cloud 抗議價；(3) 全棧 AI cloud 平台（GPU + storage + network + ML platform tools）的整合複雜度高到不是「白標 GPU 出租」可以解決。如果 NBIS 明天消失，AI 算力市場短期會出現 ~$10-20B 的供給缺口（FY26-27 backlog），長期 hyperscaler 自建會填補但代價是 AI lab 議價能力進一步喪失。

逆向推演（Munger Inversion）— 三維展開

① 技術維度：什麼技術替代會讓 NBIS 3 年內失去 50% 價值？

  • NVIDIA / GPU 範式被效率更高的 ASIC 取代（如 Microsoft Maia v3、Meta MTIA v4、Google TPU v7 + 開源化）→ 歷史對標：Intel x86 vs ARM/RISC-V，10 年範式轉換完整路徑（雖然 NVIDIA 短期仍是王者，但 5-7 年後 ASIC 在 inference 端可能取代 50%+ workload）
  • NBIS 自家 AI cloud 平台被 NVIDIA 自有 DGX Cloud + AWS / Azure 強化整合替代（如 NVIDIA 不再 prioritize NBIS chip access）→ 歷史對標：Sun Microsystems 失去 Solaris 戰略護城河
  • 分散式訓練架構成熟（model parallel + edge inference）使集中式 hyperscale GPU cluster 需求下降→ 歷史對標：Mainframe → distributed computing 50 年轉換
  • Quantum computing 在 AI training 某些子問題（optimization、sampling）取代 GPU→ 歷史對標：2030+ 才會有可能，但風險不可忽略

② 管理維度：什麼決策路徑會自我毀滅？

  • Capex 慣性過度：若 management 在 FY27-28 即使 demand 放緩仍維持 $20-25B/年 capex，FCF 黑洞無底 → 歷史對標：Cisco 2000 年代 capex 對應收入失調，5 年後股價跌 80%
  • 創辦人 Volozh 退場 / 健康問題：Volozh 是 NBIS 戰略中樞，若退場 NVIDIA 戰略合作可能弱化 → 歷史對標：Steve Jobs 離開 Apple 1985-1996 期間公司戰略漂移
  • M&A 整合失敗：若 NBIS 大規模收購其他 neocloud（如 Lambda、Crusoe）後整合不力，文化衝突 + 客戶流失 → 歷史對標：HP-Compaq 整合失敗

③ 結構維度：什麼產業變化會讓商業模式過時？

  • 監管型分拆：若美 / 歐反壟斷將 hyperscaler 強制 spin-off cloud business 或開放 GPU 平等競價 → 商業模式優勢被消除 → 歷史對標：AT&T 1984 年強制分拆 Baby Bells
  • 計算範式轉換：Transformer 架構被 next-gen model（如 state-space model、neuromorphic computing）取代，新範式可能對應全新硬體需求 → 歷史對標：Intel/x86 1990s 在桌面端壟斷但被 ARM 在移動端 完全顛覆
  • 能源價格大幅上漲 / 電力短缺：若 AI 用電佔美國總用電 10%+ 觸發政治反彈，capex bloat 變成負債 → 歷史對標：1970 年代石油危機對能源密集產業的衝擊

價值陷阱路徑：NBIS 最有可能變成「隱性資本密集型」陷阱 — Capex $20-25B 折舊期 4-5 年但 GPU 經濟壽命可能僅 3 年，殘值大幅折損 + 客戶在合約期內換代要求降價，導致 FCF 永遠無法兌現 — 這是當前估值定價未充分反映的最大風險。

議價權三段獨立（v12.3 強化）

① 對上游供應商（NVIDIA、Bloom Energy、Vertiv、AMD 等）

NVIDIA 是單點極高集中度供應商（GPU 90%+ 來自 NVIDIA，AMD MI 系列佔 5-10%）。NVIDIA 同時是 NBIS 0.5%-1% 持股股東 → 雙重關係：戰略合作 + 商業競爭（DGX Cloud）。近 3 年議價：NBIS 成功在 Vera Rubin 早期 access 拿到「次於 hyperscaler 的優先級」（vs CRWV 同級或更高優先），這是 pricing 微弱 + 量供給拉到天花板的結果。Bloom Energy $2.6B 燃料電池長約 = 長期 power 議價，鎖定 5+ 年成本。Vertiv（資料中心冷卻）為 NBIS 部分採用，無單一獨家 lock-in。

② 對下游客戶

客戶集中度極高（top 2 占 contracted ~75-80%）。Microsoft / Meta 因為 in-house ASIC 路線清晰，議價地位嚴重失衡 — NBIS 是 bridge provider，2031 後續約風險高。合約結構：Microsoft 5 年（含 $6.96B 前期款，2031/10 到期）+ Meta 5 年（含 $12B base + 最多 $42B option，2031-2032 到期）。前期款雖然提供短期 cash flow 強化，但削弱長期 pricing power。近 3 年客戶流失：尚未發生大規模流失，主要因產品快速擴張期 + 容量不足以服務所有 client。長尾客戶 sticky 但規模小。

③ 地緣曝險

核心生產地 = 北美（Pennsylvania 1.2GW AI factory）+ 歐洲（芬蘭 / 法國資料中心 + 以色列 R&D）。供應鏈最高集中點：NVIDIA chip allocation = TSMC 台灣製造（雙重台灣風險：地緣 + 供應）。政策風險：(1) 若 Trump 政府進一步加緊 GPU export control → 對中國市場影響但 NBIS 主市場美 + 歐，影響小；(2) 若以色列地緣風險升級 → R&D 中斷風險；(3) 歐盟 AI Act / EU cloud sovereignty → 對 NBIS 中性偏正（NBIS 歐洲背景可符合 EU sovereignty 要求）。已有分散化措施：Pennsylvania 自建 + 多區域資料中心部署。
