### QC-9｜品質分強制計算

§2 開頭必須先計算品質分,公式:

  底分 = (護城河 + 成長持久性) / 2
  品質分 = 底分 + 體質加減分

  - 護城河:§10 評分(1-10)
  - 成長持久性:綜合 §9 成長跑道 + 成長品質 + AI 影響,給 1-10 分
    (AI 受益加分、AI 威脅扣分,不再獨立計算)

  兩個輸入分數必須在 §10、§9 各章節明確列出,不得只給綜合分。

  體質檢核 5 項各 ±0.5 分(全過 +2.5,全不過 -2.5),結果加到底分上。

  品質分決定品質等級和底倉比例(查表):
    ≥ 7.5    → A 級,底倉 25%(目標倉位的 25%)
    6.0-7.4  → B 級,底倉 12%
    < 6.0    → 迴避,底倉 0%

### QC-10｜Bollinger 位置 Python 計算

§2 必須執行 Bollinger 位置計算,使用 yfinance 自動取得週線資料,
Python 計算 BBand 2σ,判定現價位於上軌之上 / 軌道之間 / 下軌之下。

若 yfinance 安裝失敗或網路不通,標註「數據限制:Bollinger 無法計算」並使用預設「合理」處理(不停下)。

### QC-11｜底倉框架定義

v10.0 所有「底倉 X%」均為「目標倉位的 X%」,**不是組合的 X%**。

HTML 必須在 §1 投資結論加註:

## 【價值陷阱警戒協議】

**本框架內建四層防禦機制，在以下節點強制觸發反向驗證：**

| 觸發位置 | 觸發條件 | 反向驗證任務 |
|:---|:---|:---|
| §6 門檻檢核 | Forward P/E 或 P/FCF 位於各自 5 年歷史分位 ≤ 25% | 觸發「低估值四問」篩選 |
| §9 長期成長性 | 衰退信號偵測表有任一項亮燈 | 強制輸出「價值陷阱風險評級」 |
| §4 估值診斷 | Forward P/E 或 P/FCF 歷史分位 ≤ 25%，或 Reverse DCF 顯示保守 | 強制執行「便宜的理由檢驗」 |
| §1 最終結論 | 選擇進場且估值偏低 | 必須回答「這是價值陷阱嗎？」 |

**四種最常見的價值陷阱模式**（分析全程保持警覺）：
- **護城河侵蝕型**：定價能力下滑、市佔流失，低估值反映未來盈利能力惡化
- **盈餘品質型**：EPS 靠回購或會計調整撐起，FCF 與淨利背離，真實獲利能力被高估
- **產業結構衰退型**：低 P/E 是因為市場已定價產業長期萎縮，成長天花板已到
- **隱性資本密集型**：FCF 看似良好，但維持競爭力所需的資本支出被系統性低估

## §5｜序章：第一性原理 × 逆向思考

兩部分完整輸出

### 存在意義

剝離財務表象，這家公司解決了哪個長期且不可逆的人類痛點？若它明天消失，世界會出現什麼不可替代的混亂？論述須具體，不接受「提升效率」這類空泛答案。

### 逆向推演（Munger Inversion）

若要讓這筆投資在 3 年內虧損 50% 以上，最可能的路徑是什麼？逐一分析：

- **地緣政治曝險**：生產地 / 供應鏈集中度與相關政治風險
- **技術替代風險**：誰或什麼技術可以讓它變得多餘，時間軸為何
- **管理層失職**：資本配置錯誤 / 激勵錯位 / 執行力崩壞的具體情境
- **價值陷阱路徑**：若這是一個價值陷阱，最可能的惡化劇本是哪一種？（護城河侵蝕 / 盈餘品質 / 產業衰退 / 隱性資本密集，四選一，說明理由）

## §6｜核心門檻檢核

### 【EPS CAGR 強制搜尋協議】

執行本節前，必須完成以下步驟取得 EPS 共識預估，禁止使用訓練資料估算：

**步驟零（v9.0 新增）：yfinance 程式化抓取（最高優先）**

在執行任何 web_search 之前，先用 Python yfinance 抓取即時共識預估。
此資料源與 TradingView 同源（Refinitiv/LSEG），為即時共識中位數，
可靠度遠高於 web_search 散落片段。

```python
import yfinance as yf
t = yf.Ticker("TICKER")  # 替換為實際 ticker（台股用 "2383.TW" 格式）
ee = t.earnings_estimate  # 欄位：avg, low, high, numberOfAnalysts, growth
                          # 列：0q(本季), +1q(下季), 0y(今年), +1y(明年)
```

記錄以下數字（若 yfinance 成功取得）：
- FY+1E（0y 列）：avg = __, n = __ 位分析師，YoY growth = __%
- FY+2E（+1y 列）：avg = __, n = __ 位分析師，YoY growth = __%
- 季度趨勢（0q, +1q）：用於交叉驗證年度數字合理性
- EPS Trend（t.eps_trend）：觀察 7d/30d/60d/90d 修正方向（上修 or 下修）

**yfinance 限制**：僅提供 FY+1 和 FY+2 兩年年度預估，無 FY+3。

**FY+3 推導方式（v9.0 邏輯分析法，禁止機械外推）**：
FY+3 EPS 必須基於 §9 長期成長性分析推導，不得用「FY+2 growth × 0.7」等機械公式。
具體步驟：
1. 從 §9 A（成長跑道 Runway）判斷 3 年後所處的成長階段
2. 從 §9 B（成長驅動結構）判斷定價 vs 量的貢獻是否遞減
3. 從 §9 D（ROIC × 再投資率）計算內生成長率作為 FY+3 成長率的上限參考
4. 綜合以上三點，給出 FY+3 YoY growth 的具體數值與一句話邏輯依據
5. FY+3E = FY+2E × (1 + FY+3 growth)，標注「§9 邏輯推導」

禁止出現「遞減外推」「線性外推」等無邏輯的描述。
每個 FY+3 推導必須有具體的業務邏輯支撐（例如：「M9 材料全年貢獻 + 越南廠滿載」
或「Azure AI 營收佔比從 15% 升至 25%」）。

**若 yfinance 失敗**（網路不通或 ticker 不支援），退回步驟一的 web_search 流程。

**步驟一（備援）：web_search 搜尋關鍵字**
僅在 yfinance 失敗時執行。
「[標的代碼] EPS analyst consensus estimate [FY+1 年份] [FY+2 年份] [FY+3 年份]」
同時搜尋：「[標的代碼] earnings per share forecast [FY+1] [FY+2] [FY+3] Zacks」

**步驟二（備援）：來源優先順序**
① Zacks Research（區分 GAAP / Non-GAAP，記錄分析師覆蓋人數 n）
② StockAnalysis.com → Forecast 頁籤
③ Yahoo Finance → Analysis → EPS Trend
④ 若以上均無 FY+3 數據，以 FY+2 替代並標注「FY+2 外推」

**步驟三：記錄數字**
- 基期 EPS：最新完整財年實際值（已公布，標注 GAAP）
- FY+1E：共識中位數（標注來源 yfinance / Zacks / 其他，n=__ 位分析師）
- FY+2E：共識中位數（標注來源，n=__ 位分析師）
- FY+3E：共識中位數或遞減外推（標注來源或外推方法，n=__ 位分析師）

**步驟四：計算**
- GAAP CAGR    = (FY+3E GAAP    ÷ 基期 GAAP EPS)    ^ (1/3) − 1
- Non-GAAP CAGR = (FY+3E Non-GAAP ÷ 基期 Non-GAAP EPS) ^ (1/3) − 1

**步驟五：口徑差距說明**
說明 GAAP 與 Non-GAAP 的差距來源（SBC / 重組費用 / 攤銷 / 其他調整項）
台灣上市公司不區分 GAAP/Non-GAAP，此步驟標注「不適用（台股法定口徑）」即可。

## §4｜估值診斷

### 設計原則

**本章的核心任務是「讀懂當前股價隱含了什麼預期」，而非「預測股票應該值多少」。**

五個工具依序回答五個問題：
1. **估值體系診斷（7.0）**：市場現在用什麼邏輯定價這家公司？這把尺會不會換？
2. **歷史分位（7.1）**：市場現在給的估值，在歷史上屬於哪個情緒區間？

5. **同業比較（7.4）**：相對同行，現在的溢 / 折價有無基本面支撐？

**本章不輸出目標股價。** 估值結論以「當前定價是否合理」表達，具體進場條件在 §1 以條件句給出。

#### Bollinger 位置(週線 2σ)

執行以下 Python 腳本判定現價位於 Bollinger Band 的位置:

import numpy as np

ticker = "TICKER_PLACEHOLDER"  # 替換為實際標的

closes = weekly['Close'].iloc[-20:].values
sma = np.mean(closes)
std = np.std(closes, ddof=0)
upper_2 = sma + 2 * std
lower_2 = sma - 2 * std
current_price = float(weekly['Close'].iloc[-1])

if current_price > upper_2:
    position = "偏貴(上軌之上)"
    adjustment = 0.5
elif current_price < lower_2:
    position = "偏宜(下軌之下)"
    adjustment = 1.5
else:
    position = "合理(軌道之間)"
    adjustment = 1.0

print(f"現價: {current_price:.2f}")
print(f"週線 BB 上軌(2σ): {upper_2:.2f}")
print(f"週線 BB 下軌(2σ): {lower_2:.2f}")
print(f"位置判定: {position}")
print(f"部位調整係數: {adjustment}")

### 功能規格

- 頁首固定：標的代碼 ｜ 資料抓取時間 ｜ 最新股價 ｜ DD Schema v10.0

- 同時在 HTML <head> 區加一個 meta 標籤供程式化解析：`<meta name="dd-schema-version" content="v10.0">`，meta content 同樣取自 frontmatter version 欄位

### 輸出規格(Claude Code 本地環境)

- 檔名格式：`DD_[標的代碼]_[YYYYMMDD].html`
- 使用 Write 工具輸出至 `docs/dd/`(相對於專案根目錄 `financial-analysis-bot/`)
- **輸出完成後,必須執行以下三步驟,不得省略**:
  1. **更新 INDEX.md**:使用 Edit 工具 append 一行到 `docs/dd/INDEX.md`,格式為(7 欄):
     `| YYYY-MM-DD | TICKER | vX.X | 進場/觀望/迴避 | 🟢非陷阱/🟡觀察期/🔴高風險陷阱 | ✅/❌/⛔ / ✅/❌/⛔ / ✅/❌/⛔ | DD_TICKER_YYYYMMDD.html |`
     - 第 1 欄:報告日期(YYYY-MM-DD)
     - 第 2 欄:Ticker
     - 第 3 欄:Schema 版本(取自本檔案 frontmatter 的 version 欄位,例如 v7.0)
     - 第 4 欄:裁決(取自 §1「進場 / 觀望 / 迴避」最終結論)
     - 第 5 欄:陷阱定性(取自 §1「這是價值陷阱嗎?」最終定性)
     - 第 6 欄:v10.0 雙 R:R 判定,格式為 `品質等級 / 短期R:R / 中期R:R`(取自 §2 E)。品質等級:A/B(A=7.5+,B=6.0-7.4),短期R:R 和中期R:R 各到小數點 1 位(如 2.3x)
     - 第 7 欄:檔名(不含路徑)
  2. **觸發網站同步**:執行 `python scripts/update_dd_index.py` 讓首頁自動列出新 DD。
     若腳本執行失敗(例如 Python 環境問題),改為提示使用者手動執行該命令,不得跳過這一步。
     理由:docs/index.html 的深度研究報告表格是快照,不會自動更新;此腳本會掃描
     docs/dd/DD_*.html 並重寫該表格段落。
  3. **terminal 摘要**:在 terminal 輸出一段精簡摘要供使用者快速確認,格式為:
     ```
     ✅ DD 完成:[TICKER]
     📄 檔案:docs/dd/DD_TICKER_YYYYMMDD.html
     💰 最新股價:$__
     🎯 最終裁決:[進場/觀望/迴避]
     ⚠️ 陷阱定性:[🟢/🟡/🔴]
     📊 v10.0 評估:品質 [A/B] | 短期 R:R [Xx] | 中期 R:R [Xx] | 底倉 [N%(目標倉位)]
     📈 長期持有信心:[高/中/低]
     🔗 首頁同步:[✅ 成功 / ❌ 失敗,需手動執行 python scripts/update_dd_index.py]
     ```
     🔗 首頁同步欄位的狀態取自上方第 2 步的執行結果。
     這是 terminal 允許輸出的**唯一**文字,章節內容仍然嚴禁在對話框顯示。
