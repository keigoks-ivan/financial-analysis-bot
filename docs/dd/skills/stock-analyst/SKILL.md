## 【v12.0 重要變更：分析師 / 基金經理人分工】

**本 skill 是「分析師」角色。輸出為個股綜合訊號燈，不建議倉位 %。**

- DD 輸出：quality_tier（S/A/B/C/拒絕）× 估值燈（🟢🟡🟠🔴）× MA 狀態 × 大盤豁免 × trap → 綜合成 `final_signal: A+ / A / B / C / X`
- 倉位決策：由 **portfolio-manager skill**（基金經理人）基於組合狀態決定
- 原 v12.0 §2 H「最終倉位 X%」→ v12.0 改為「綜合訊號 A+/A/B/C/X」
- §1 Dashboard 不再顯示 "建議倉位 X%"，改為 "訊號：A+"

**綜合訊號對應表**（供下游 PM skill 引用）：

| 訊號 | 條件 | PM 層面語意 |
|:---|:---|:---|
| **A+** | quality S/A + gate 🟢 + MA 🟢/✅ + trap 🟢 + 大盤正常 | 核心候選，滿格訊號 |
| **A** | quality S/A + gate 🟡 + MA ✅ + trap 🟢 | 核心候選，進場訊號 |
| **B** | quality B + gate 🟢/🟡 + trap 🟢/🟡，或 S/A 級有一項訊號減半 | 衛星候選 |
| **C** | 多項訊號減半 / 觀察池狀態 | 觀察名單 |
| **X** | trap 🔴 或 gate 🔴 或 MA ❌ 或 quality 拒絕 | 迴避 / 清倉訊號 |

### QC-9｜品質分強制計算

§2 開頭必須先計算品質分,公式:

  底分 = (護城河 + 成長持久性) / 2
  品質分 = min(底分 + 體質加減分, 10)　　← 滿分 10 分

  - 護城河:§9 評分(1-10)
  - 成長持久性:綜合 §8 成長跑道 + 成長品質 + AI 影響,給 1-10 分
    (AI 受益加分、AI 威脅扣分,不再獨立計算)

  兩個輸入分數必須在 §9、§8 各章節明確列出,不得只給綜合分。

  體質檢核 5 項各 ±0.5 分(全過 +2.5,全不過 -2.5),結果加到底分上。

  品質分決定品質等級和底倉比例(查表):
    ≥ 7.5    → A 級,底倉 25%(目標倉位的 25%)
    6.0-7.4  → B 級,底倉 12%
    < 6.0    → 迴避,底倉 0%

### QC-24｜Intraday 訊號檢查（v12.0 新增，ONTO 教訓）

yfinance 取資料時必須同時抓**近 5 交易日**的 Open / High / Low / Close:

```python
daily5 = yf.download(ticker, period="5d", interval="1d", progress=False)
for idx, row in daily5.iterrows():
    gap_up = (row['Open'] - prev_close) / prev_close if prev_close else 0
    intraday_range = (row['High'] - row['Low']) / row['Open']
    close_vs_high = (row['High'] - row['Close']) / row['High']
    # 警示條件
    if close_vs_high > 0.05:
        flag "盤中衝高回落 shooting star"
    if gap_up > 0.05 and row['Close'] < row['Open']:
        flag "gap up 收紅 K"
```

若任一日觸發，§2 F 建倉建議必須標註「近 5 日 intraday 警示」並在 §1 提醒。

範例：ONTO 4/16 intraday $295 vs 收 $266 = 10% 衝高回落 = 🔴 動能末段訊號。

| < 30% | 使用 yfinance 值 |
| ≥ 30% | 兩者都顯示，WACC 計算用**較高值**（保守） |
| ≥ 50% | §13.3 DCF 必須同時呈現高/低 Beta 兩個情境的隱含 CAGR |

範例：CRDO yfinance 2.72 vs TradingView 3.35（差 23%）；ONTO 1.47 vs 2.35（差 60%，必須雙情境）。

### QC-26｜Margin 結構性測試（v12.0 新增，ONTO 教訓）

毛利率 YoY 下滑 > 1.5pp 時，必須執行 3 項對照:

1. **同業同期毛利率趨勢**（上升 / 持平 / 下滑）
2. **產品組合 mix 變化**（新品低毛利？還是舊品流失？）
3. **同業用相似技術 margin**（若對手用較低階技術但 margin 反而高 → 結構性問題）

| 結果 | 判斷 |
|:---:|:---|
| 同業擴張，本標的下滑 | **結構性質變**，護城河 −0.5 分，§1 陷阱定性降為 🟡 觀察期 |
| 全產業同步下滑 | 週期性，短期噪音，不扣分 |
| 產品組合一次性稀釋 | 記錄為「管理層承諾 recover」，列入監測指標 |

範例：ONTO 49.7% vs Camtek 51.6%（同業擴張），用更低階技術反而 margin 更低 = 結構性質變。

### QC-27｜Revenue vs OI 增長率 Divergence（v12.0 新增，AMZN 教訓）

§4 即時財報情報章節必須額外計算:

```
核心業務（如 AWS、Azure）:
  Revenue YoY growth - Operating Income YoY growth = divergence
```

| Divergence | 判斷 |
|:---:|:---|
| < 0% | OI 成長快於 Revenue（margin 擴張） |
| 0 ~ 3% | 接近平衡 |
| **> 3%** | **Margin 壓縮警示**，列入 §8 E 壓力測試 |
| > 7% | 嚴重 margin 壓縮，§1 Dashboard 加紅色警示 |

範例：AMZN Q4 2025 AWS Rev +24% vs OI +17% = 7pp divergence → 已達「嚴重壓縮」閾值，但 v11.0 未警示。

### QC-28｜絕對成長 vs 相對成長對照（v12.0 新增，AMZN 教訓）

對於有明確可比競爭對手的業務（雲端、HBM 設備、AEC 晶片等），§9 市佔競爭表必須呈現:

| 項目 | 標的 | 主要對手 |
|:---:|:---:|:---:|
| 營收 YoY % | 24% | 34% |
| **絕對美元新增**（季）| $10.3B | $10.0B |
| 市場份額變化 | -1.5pp | +1pp |

當**對手絕對美元新增 ≥ 本標的 90%** → 觸發「規模優勢質變」警示，護城河評分重審。

範例：AMZN AWS vs Google Cloud 2025Q4，GCP 絕對新增 $10.0B ≈ AWS $10.3B → AWS 規模優勢在消失，報告未反映。

### QC-29｜R:R 壓力測試（v12.0 新增，AMZN 教訓）

§2 E 必須執行 4 情境壓力測試（Base + 3 壓力）:

| 情境 | 假設 | 目的 |
|:---:|:---|:---|
| Base | 報告原始假設 | 基準 |
| 壓力 A | 3Y EPS CAGR −10% | 成長放緩 |
| 壓力 B | 合理 PE −20%（倍數壓縮） | 估值重設 |
| 壓力 C | 適用時：同業溢價收斂 50% | 競爭收斂 |

**壓力測試通過率** = 中期 R:R ≥ 2.0 的情境 / 4

若通過率 < 75%（即 < 3 個情境過關），§1 必須標註「R:R 對壓力情境敏感」，綜合訊號**降一級**（A+ → A、A → B、B → C）。

範例：AMZN 2.15x 在 Base 下勉強過，但 3 個壓力情境下跌至 0.1-0.2x = 通過率 25%，應降倉。

### QC-30｜同業溢價收斂壓測（v12.0 新增，LLY 教訓）

若標的 Fwd PE > 同業中位 50% 以上:

1. §13.4 同業比較必須加「收斂情境」分析:
   - 情境 A：標的 PE 不變，對手 PE 上修至同業均 → 上行空間壓縮
   - 情境 B：對手 PE 不變，標的 PE 收斂 50% 至 (自身 + 同業) / 2 → 股價下行空間
2. 情境 B 的股價必須作為**替代 Bear Case** 與 §8 E Bear 取保守者
3. §1 必須標註「相對同業溢價 __%，收斂風險列為 R4」

範例：LLY Fwd PE 26x vs NVO 11x = 138% 溢價 → 必須測試 NVO 追趕情境。

### QC-19｜標的自身重大事件強制搜尋（v11.2 新增，BSX 教訓）

QC-12 只搜「產業/競爭者」新聞。本規則補強:必須針對**標的公司自身**近 12 個月的重大事件執行至少 4 組搜尋:

```
[TICKER] [公司名] acquisition merger 2025 2026
[TICKER] [公司名] class action lawsuit securities fraud
[TICKER] [公司名] clinical trial FDA approval 2025 2026        ← 醫療/生技類必搜
[TICKER] [公司名] product launch recall warning letter          ← 有產品風險類必搜
[TICKER] [公司名] SEC investigation restatement                 ← 治理風險類必搜
```

**針對不同產業額外強制搜尋**:

- **醫療/生技**: 臨床試驗結果（Phase II/III readout）、FDA 審查進展、MDL 訴訟
- **半導體**: 客戶流失公告、產能意外、export control 政策更新
- **金融**: 監管處罰、壞帳率異常、存款流動
- **消費**: 食安/產品召回、集體訴訟
- **所有行業**: **10-K / 10-Q / 8-K 近 90 天重大公告**

**必須檢查的重大事件類型**(任一發現必納入 DD):

1. **併購 / 分拆**(M&A): 金額 > 當前市值 5% 或 > 過去 5 年單筆最大併購 2 倍 = 🔴 重大事件，必須列入 §5 R 風險、§10/§12 治理、§13 估值稀釋計算
2. **集體訴訟 / 證券詐欺訴訟**: Lead Plaintiff 階段已啟動 = 必須列入 §12 治理（對管理層誠信的挑戰，護城河扣分）、§1 陷阱定性需重新評估
3. **臨床試驗 / FDA 讀數**: 過去 90 天已發表結果 = 必須引用具體讀數，不得寫「即將發表」
4. **CEO / CFO 離職、SEC 調查、財報重編**: 自動觸發 🔴 高風險陷阱初篩
5. **主要客戶流失公告、長期合約終止**: 必須重算 §8 成長假設

若 web_search 確認「無上述重大事件」,在 §12 治理章節末尾標註:
「🟢 過去 12 個月無併購金額 > 市值 5%、無集體訴訟、無管理層異動或重大監管事件」,作為正面確認而非跳過。

### QC-20｜催化劑實現檢查（v11.2 新增）

DD 中若出現「即將到來」「即將發表」「預期在 X 月」「倒數中」「trial readout 接近」等表述,**必須搜尋該催化劑是否已經發生**:

```
[catalyst 名稱] results announced [當月] [當年]
[catalyst 名稱] readout data [當月] [當年]
```

**處理規則**:

| 催化劑狀態 | 處理方式 |
|:---|:---|
| 尚未發生（搜尋無結果 或 發表日明確在未來） | 維持「催化劑即將到來」表述，併列出具體日期與驗證指標 |
| **已發生，市場已消化**（搜尋有結果 + 發表日 < 當前日） | **絕對禁止**寫成「即將到來」；必須引用實際結果與市場反應（股價、分析師評等調整） |
| 已發生但結果模糊 / 部分達標 | 必須獨立分析「技術細節是否與表面結論一致」（例:CHAMPION-AF 達標但 ischemic stroke rate 較高）|

**強制機制**: §6 Earnings Call 萃取、§5 H 假設、§13.5 結論等處凡提及具體時程催化劑(如 clinical trial、FDA 決議、重大併購關閉、新品發表),**必須有對應的 web_search 查核紀錄**。若未查核即寫入,視為違反 QC-20。

### QC-21｜R:R 數學假象防禦（v11.2 新增，BSX 教訓）

v11.0 §2 E R:R 計算在以下情境會產生數學假象:

```
當 Bear 股價 ≈ 現價（差距 < 5%）時:
下行距離 → 0 → R:R 分母趨近 0 → R:R 爆到 100x 以上但無實質意義
```

**這不代表「margin of safety 巨大」,而代表「Bear Case 已被市場完全定價」——安全邊際實際為零**。

**強制判斷邏輯**:

計算 R:R 後,檢查「下行距離」的絕對百分比:

| 下行距離 | 判斷 | R:R 處理 |
|:---|:---|:---|
| > 15% | 正常 | R:R 有效,直接使用 |
| 5-15% | 警示 | R:R 有效但於 §2 E 備註「Bear Case 已接近定價,下行空間已壓縮」 |
| < 5% | **R:R 失效** | R:R 欄位標註「⚠️ 數學假象 - Bear 與現價幾乎重合」,**禁止**直接引用 R:R 數字做進場判定 |
| Bear > 現價 | Bear 高於現價 | 標註「即使 Bear Case 仍高於現價，表示市場過度悲觀或 Bear 假設過樂觀，需重新檢視 §8 E 壓力情境」 |

**當 R:R 失效時的替代判定流程**:

1. 計算「極端 Bear」: Bear PE × 0.8 + Bear EPS × 0.85（假設市場繼續壓縮 20% PE 且 EPS 再下修 15%）
2. 以「極端 Bear 股價」作為替代下檔錨點重算 R:R
3. 若替代 R:R 仍 ≥ 2.0 → 可進場但底倉減半（原 25% → 12%）
4. 若替代 R:R < 2.0 → 判定為「觀望偏進場」而非「進場」,並在 §1 加註「現價接近 Bear 錨點,下行空間已壓縮,建議分批建倉而非一次進場」

**§2 E 判定表新增一列**:

| 條件 | 結果 |
|:---|:---|
| 短期 R:R ≥ 2.0 且中期 R:R ≥ 2.0 且**雙方下行距離 ≥ 5%** | 進場 |
| ...（其他原規則不變） | ... |
| 短期/中期任一「R:R 失效」（下行 < 5%） | 自動降級,不得直接判進場 |

### QC-31｜v12 綜合訊號燈定義表（強制對映）

dd-meta JSON 的 `signal` 欄位（A+/A/B/C/X）必須按以下定義表對映，**不得自由發揮**。
這張表是 thesis-level 分類，與 §12 E（v12 新編號 §2 E）的 R:R 進場時機判定**正交獨立**：

| 訊號 | 含義 | 觸發條件（thesis-level） | 進場行動建議 |
|:---|:---|:---|:---|
| **A+** | 進場 | 品質 ≥ 7.5（A 級） + 估值🟢 + 短期 R:R ≥ 2.0 + 中期 R:R ≥ 2.0 + Pure MA 多頭 | 立即建倉至底倉 25%（或目標倉位 100%） |
| **A** | 觀望偏進場 | 品質 ≥ 7.5（A 級） + 中期 R:R ≥ 1.0 + 長期持有信心 = 高 | 建倉至底倉 15-20% |
| **B** | 觀望 | 品質 ≥ 6.0（A/B 級）+ thesis 完整，但時機不到（R:R 不足 / 估值🔴 / MA 過熱 / Bollinger 上軌之上） | 不新進，等估值或時機修復；已持倉者視情況部分獲利了結 |
| **C** | 迴避（thesis 失敗） | 任一觸發：(1) 品質分 < 6.0；(2) 護城河 X 級或侵蝕信號 ≥ 3 個；(3) 價值陷阱定性 = 🔴；(4) §6F AI 取代風險 = 🔴 | 不持有，已持有者清倉 |
| **X** | 黑名單 | 任一觸發：(1) 重大治理問題 / 舞弊；(2) 結構性產業衰退（TAM 萎縮 + 估值倍數系統下移）；(3) 獲利品質崩壞（FCF/NI < 0.5 連續 2 年） | 不分析、不討論、不持有 |

**核心規則**：

1. **R:R 不足 ≠ C**：R:R 是「進場時機」參考，不單獨觸發 thesis-level 迴避。品質 A/B 的標的若 R:R 不過進場門檻，落 **B（觀望）**，等估值或時機修復。
2. **估值🔴 ≠ C**：估值極端高估只代表現在不該進，標的本身（品質 + 護城河 + 長期信心）若仍為 A 級，落 **B（觀望）**。
3. **C/X 必須有 thesis-level 的失敗證據**：不是「現在 R:R 不夠」，而是「這標的長期不該持有」。撰寫 §1 時若選 C/X，必須在「結論 Dashboard」明確列出觸發條件對應的具體章節數據（§9 護城河、§8 衰退信號、§7 陷阱四問、QC-9 品質分等）。
4. **§2 E（v12 新編號）的「都不過 → 迴避」是時機語意，不是 signal 對映**：該行的「迴避」描述「現在不進場」的決策，**不是 signal grade 的指派**。signal grade 必須回到本表查對。範例：PANW 短期 R:R −0.17、中期 0.23，§2 E「都不過」 → 不新進；但 thesis 完整（品質 10、護城河 A、估值🟡）→ signal = **B 觀望**，不是 C。
5. **批次 DD 一致性**：同一批 DD 中，所有「估值🔴 + 4 週動能爆衝 + 品質 A/B」的標的（典型如 RMBS/FORM/CIEN/MOD/FIX）一律落 B（觀望，等修復），不得隨機歸 C。

**四種最常見的價值陷阱模式**（分析全程保持警覺）：
- **護城河侵蝕型**：定價能力下滑、市佔流失，低估值反映未來盈利能力惡化
- **盈餘品質型**：EPS 靠回購或會計調整撐起，FCF 與淨利背離，真實獲利能力被高估
- **產業結構衰退型**：低 P/E 是因為市場已定價產業長期萎縮，成長天花板已到
- **隱性資本密集型**：FCF 看似良好，但維持競爭力所需的資本支出被系統性低估

## §7｜核心門檻檢核

### 【EPS CAGR 強制搜尋協議】

執行本節前，必須完成以下步驟取得 EPS 共識預估，禁止使用訓練資料估算：

**步驟零（v9.0 新增）：yfinance 程式化抓取（最高優先）**

在執行任何 web_search 之前，先用 Python yfinance 抓取即時共識預估。
此資料源與 TradingView 同源（Refinitiv/LSEG），為即時共識中位數，
可靠度遠高於 web_search 散落片段。

```python
import yfinance as yf
t = yf.Ticker("TICKER")  # 替換為實際 ticker（台股用 "2383.TW" 格式）
ee = t.earnings_estimate  # 欄位：avg, low, high, numberOfAnalysts, growth
                          # 列：0q(本季), +1q(下季), 0y(今年), +1y(明年)
```

記錄以下數字（若 yfinance 成功取得）：
- FY+1E（0y 列）：avg = __, n = __ 位分析師，YoY growth = __%
- FY+2E（+1y 列）：avg = __, n = __ 位分析師，YoY growth = __%
- 季度趨勢（0q, +1q）：用於交叉驗證年度數字合理性
- EPS Trend（t.eps_trend）：觀察 7d/30d/60d/90d 修正方向（上修 or 下修）

**yfinance 限制**：僅提供 FY+1 和 FY+2 兩年年度預估，無 FY+3。

**FY+3 推導方式（v9.0 邏輯分析法，禁止機械外推）**：
FY+3 EPS 必須基於 §8 長期成長性分析推導，不得用「FY+2 growth × 0.7」等機械公式。
具體步驟：
1. 從 §8 A（成長跑道 Runway）判斷 3 年後所處的成長階段
2. 從 §8 B（成長驅動結構）判斷定價 vs 量的貢獻是否遞減
3. 從 §8 D（ROIC × 再投資率）計算內生成長率作為 FY+3 成長率的上限參考
4. 綜合以上三點，給出 FY+3 YoY growth 的具體數值與一句話邏輯依據
5. FY+3E = FY+2E × (1 + FY+3 growth)，標注「§8 邏輯推導」

禁止出現「遞減外推」「線性外推」等無邏輯的描述。
每個 FY+3 推導必須有具體的業務邏輯支撐（例如：「M9 材料全年貢獻 + 越南廠滿載」
或「Azure AI 營收佔比從 15% 升至 25%」）。

**若 yfinance 失敗**（網路不通或 ticker 不支援），退回步驟一的 web_search 流程。

**步驟一（備援）：web_search 搜尋關鍵字**
僅在 yfinance 失敗時執行。
「[標的代碼] EPS analyst consensus estimate [FY+1 年份] [FY+2 年份] [FY+3 年份]」
同時搜尋：「[標的代碼] earnings per share forecast [FY+1] [FY+2] [FY+3] Zacks」

**步驟二（備援）：來源優先順序**
① Zacks Research（區分 GAAP / Non-GAAP，記錄分析師覆蓋人數 n）
② StockAnalysis.com → Forecast 頁籤
③ Yahoo Finance → Analysis → EPS Trend
④ 若以上均無 FY+3 數據，以 FY+2 替代並標注「FY+2 外推」

**步驟三：記錄數字**
- 基期 EPS：最新完整財年實際值（已公布，標注 GAAP）
- FY+1E：共識中位數（標注來源 yfinance / Zacks / 其他，n=__ 位分析師）
- FY+2E：共識中位數（標注來源，n=__ 位分析師）
- FY+3E：共識中位數或遞減外推（標注來源或外推方法，n=__ 位分析師）

**步驟四：計算**
- GAAP CAGR    = (FY+3E GAAP    ÷ 基期 GAAP EPS)    ^ (1/3) − 1
- Non-GAAP CAGR = (FY+3E Non-GAAP ÷ 基期 Non-GAAP EPS) ^ (1/3) − 1

**步驟五：口徑差距說明**
說明 GAAP 與 Non-GAAP 的差距來源（SBC / 重組費用 / 攤銷 / 其他調整項）
台灣上市公司不區分 GAAP/Non-GAAP，此步驟標注「不適用（台股法定口徑）」即可。

### 【Munger 護城河維度強制搜尋協議】

執行門檻檢核表前，須補充搜尋以下數據（與 EPS CAGR 搜尋並行執行）：

**FCF Margin 歷史**（需 5 年年度數據）
搜尋：「[標的代碼] free cash flow margin history Macrotrends」或「[標的代碼] FCF margin annual」
記錄每年 FCF Margin，計算 5 年均值與最低谷年份

**ROIC 穩定性**（需 10 年年度數據）
搜尋：「[標的代碼] ROIC history 10 year annual Macrotrends」或「[標的代碼] return on invested capital annual」
來源優先：Macrotrends → TIKR → Wisesheets；記錄每年數值，計算達標年數

**毛利率趨勢**（需 10 年年度數據）
搜尋：「[標的代碼] gross margin history 10 year Macrotrends」
記錄每年毛利率，計算「改善年份數 / 總年數」

**Capex / Revenue**（需近 3 年）
來源：最新 10-K 或 §6 財報數據即可，通常已有

## §13｜估值診斷

### 設計原則

**本章的核心任務是「讀懂當前股價隱含了什麼預期」，而非「預測股票應該值多少」。**

五個工具依序回答五個問題：
1. **估值體系診斷（13.0）**：市場現在用什麼邏輯定價這家公司？這把尺會不會換？
2. **歷史分位（13.1）**：市場現在給的估值，在歷史上屬於哪個情緒區間？
3. **PEG 診斷（13.2）**：以分析師共識成長率衡量，現在的價格算貴還是便宜？
4. **Reverse DCF（13.3）**：要辯護當前股價，市場隱含的成長預期是否合理？
5. **同業比較（13.4）**：相對同行，現在的溢 / 折價有無基本面支撐？

**本章不輸出目標股價。** 估值結論以「當前定價是否合理」表達，具體進場條件在 §1 以條件句給出。

### 【即時數據搜尋協議】

執行本章前，必須完成以下搜尋，禁止使用訓練資料：

- 當前 P/E（Trailing & Forward）、P/S、P/FCF、EV/EBITDA

### D｜估值燈（v12.0 新，取代 R:R 裁決）

**v12.0 變更**：廢除 v11.x 的「合理 P/E → 目標價 → R:R → 裁決」鏈式邏輯。改為單一「估值燈」直接對最終倉位下 modifier。

**資料來源**：
- Fwd PE 5Y 分位 = (當前 Fwd PE − 5Y 低) / (5Y 高 − 5Y 低) × 100%
- PEG = 當前 Fwd PE ÷ 3Y EPS CAGR（§7）

| 燈號 | 條件 | 最終倉位 modifier | 邏輯 |
|:---:|:---|:---:|:---|
| 🟢 **便宜** | Fwd PE 5Y 分位 < 30% **或** PEG < 1.0 | **等級 +1**（B→A、A→S、S→S+）| 歷史少見便宜，給予加碼許可 |
| 🟡 **合理** | 30-70% 分位 **且** 1.0 ≤ PEG ≤ 2.0 | 不變 | 正常估值 |
| 🟠 **偏貴** | 70-85% 分位 **或** 2.0 < PEG ≤ 2.5 | **× 0.5** | 偏高位置，半倉入場 |
| 🔴 **過熱** | > 85% 分位 **或** PEG > 2.5 | **直接拒絕**（0%）| 透支未來，不進場 |

**優先序**：分位與 PEG 取**較嚴**的一個（例：分位 40%🟡 但 PEG 2.8 → 🔴 過熱）。

### F｜Pure MA 狀態機（v12.0 新增，Stage 4 覆蓋層）

**六狀態機**（週線 W52 / W104 / W250，SMA）：

| 狀態 | 條件 | 部位乘數 |
|:---:|:---|:---:|
| 🟢 **最佳進場** | 價 < W52 **且** 價 > W104 **且** W52/W104/W250 三條斜率全正 | **× 1.2** |
| ✅ **強勢進場** | 價 > W52 > W104 > W250 **且** W250 斜率正 | × 1.0 |
| 🟡 **減半進場** | 四條件過但 W250 斜率持平（\|13 週變化\| < 3%）| × 0.5 |
| 🟡 **觀察池** | 剛站回 W104（站上 < 4 週） | 0（等 4 週確認） |
| 🟠 **暫不進場** | 價 < W104 但 > W250，W250 斜率仍正 | 0（加追蹤池） |
| ❌ **系統失效** | 價 < W250 **或** W250 斜率轉負 | 0（拒絕） |

**W250 斜率定義**：當週 W250 值 vs 13 週前 W250 值
- 正（> +3%）：強正
- 持平（−3% 到 +3%）：持平
- 負（< −3%）：轉負

### 🛡️ 盲點 2 防禦：V 型反轉早期不誤殺（v12.0 正式化）

**問題情境**：深度下跌的結構性好標的（2023 年 Meta $88、2024 年中 Dollar Tree 深跌後、2022Q4 半導體谷底等）會觸發 ❌ 系統失效（跌破 W250 或 W250 斜率轉負）。v12.0 會自動歸類「拒絕」。但若產業風向燈 🟢 + 護城河未壞，這是 V 型反轉的早期買點——MA 最晚才轉正，會讓人完全錯失底部。

**救援規則**：當 MA 狀態為 ❌ 系統失效，但以下**三條件全滿足**，降級為「🟠 暫不進場（追蹤池）」而非「拒絕」：

| 條件 | 門檻 |
|:---|:---|
| ① §2 I 長期持有信心 | **高**（成長跑道充裕 + 護城河趨勢加深或穩定）|
| ② §8 長期成長性綜合評級 | 🟢 高確信 |
| ③ 護城河（§9） | ≥ 8 |

**額外條件**（保障下行停損）：
- 若同時觸發大盤系統性風險（大盤破 W104）→ 救援**失效**，維持「拒絕」
- 若 QC-19 發現結構性重大利空（集體訴訟、管理層誠信問題）→ 救援失效
- 若 W250 斜率 < −10%（深度失速）→ 救援失效（跌得太猛不救）

**救援結果**：
- MA 狀態從 ❌ 改標記為「❌→🟠（盲點 2 救援）」
- 加入追蹤池，設定「站回 W104 後 4 週」作為再評估閾值
- 必須在 §2 F 明確標註：「⚠️ 盲點 2 救援觸發：§2 I 信心高 + §8 🟢 + 護城河 __，❌ 降為追蹤池」

**Python 計算**（強制執行）：

```python
import yfinance as yf
import numpy as np

ticker = "TICKER_PLACEHOLDER"
weekly = yf.download(ticker, period="6y", interval="1wk", progress=False, auto_adjust=True)
closes = weekly['Close'].values.flatten()
current = float(closes[-1])

# 三條週線 MA
w52 = float(np.mean(closes[-52:])) if len(closes) >= 52 else None
w104 = float(np.mean(closes[-104:])) if len(closes) >= 104 else None
w250 = float(np.mean(closes[-250:])) if len(closes) >= 250 else None

# W250 斜率（當前 vs 13 週前）
if len(closes) >= 263:
    w250_13w_ago = float(np.mean(closes[-263:-13]))
    slope_pct = (w250 / w250_13w_ago - 1) * 100
else:
    slope_pct = None

# W52 / W104 斜率（輔助）
w52_slope = (w52 / float(np.mean(closes[-65:-13])) - 1) * 100 if len(closes) >= 65 else None
w104_slope = (w104 / float(np.mean(closes[-117:-13])) - 1) * 100 if len(closes) >= 117 else None

# 狀態判定
if w250 is None:
    state = "樣本不足（< 250 週）"
elif current < w250 or slope_pct < -3:
    state = "❌ 系統失效"
elif current < w104:
    state = "🟠 暫不進場"
elif current < w52 and w52_slope > 0 and w104_slope > 0 and slope_pct > 0:
    state = "🟢 最佳進場"
elif current > w52 > w104 > w250 and slope_pct > 3:
    state = "✅ 強勢進場"
elif current > w52 > w104 > w250 and abs(slope_pct) <= 3:
    state = "🟡 減半進場"
else:
    state = "🟡 觀察池"

print(f"現價: {current:.2f}")
print(f"W52: {w52:.2f}" if w52 else "W52: N/A")
print(f"W104: {w104:.2f}" if w104 else "W104: N/A")
print(f"W250: {w250:.2f}" if w250 else "W250: N/A")
print(f"W250 斜率（13週）: {slope_pct:.2f}%")
print(f"狀態: {state}")
```
