### 問題 A：§3 空頭表格殘留 `§13.3` cross-ref（MU 行 355 / NET 行 266）

**受影響**：MU / NET（2 份）

**根因**：v12.3 升級時 §13.3 Reverse DCF 章節 body 被正確砍除，但 §3 三方辯論空頭表格中引用 `（§13.3）` 的 inline cross-ref 未清除。典型的「body 砍了但 cross-ref 未追蹤」pattern。

**修法**：搜尋全文的 `§13.3` 並改為 `§13.1`（歷史估值分位）或直接移除括號引用。無需更動估值判斷本身。

**影響評級**：WARN（不影響裁決，不影響數字，但會讓讀者以為存在 §13.3）。

**根因**：§1 前有舊版 note `「完整推理過程見 §13.5 投資結論」`，但 §13.5（五角驗證 + 便宜理由五問）在 v12.3 已移除，v12.3 §13 只剩 13.1/13.2/13.4 + 結論段。此 note 可能是 v12.2 時代殘留的「todo」句，升級 §13 後未同步更新 §1 note。

### 問題 D：MOD §8.H Top 10 匯總行缺失

**受影響**：MOD（1 份）

**根因**：MOD 的客戶結構本質上難以三層量化（10-K 不揭露，SEC 門檻 10%+），§8.H 以業務段（DC hyperscaler / CV OEM / HVAC）取代排名切割，並以 Data Gap Note 誠實說明。相比 MA（支付網路，客戶分散度定義清晰）或 MU（半導體，有 10-K 匿名揭露），MOD 的資料可達性確實較低。

**修法**：可選擇性補充一行「Top 10 客戶合計（估）：~60-70%（DC hyperscaler + auto OEM + HVAC 合計）」，並標 Data Gap。非強制修復。

**影響評級**：WARN（資料限制所致，不構成 FAIL）。
