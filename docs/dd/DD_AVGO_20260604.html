§4 序章：第一性原理 × 逆向思考

存在意義

剝離財務，Broadcom 解決兩個不可逆痛點：① 讓 hyperscaler 擺脫單一 GPU 供應商的壟斷定價。當 Google / Meta / OpenAI / Anthropic 的 AI 工作負載達到 GW 級，買 NVDA GPU 的毛利率全進了 NVDA 口袋；唯一的解是設計自家 XPU，但 hyperscaler 沒有 SerDes（224G）、先進封裝（CoWoS 協同）、高速網路（Tomahawk/Jericho）與 TSMC 2nm 設計協同的完整能力 — Broadcom 是少數能把這些拼起來交付量產 ASIC 的公司。② VMware 是全球企業資料中心虛擬化的事實標準基座，數十萬企業的私有雲跑在上面，遷移成本以年計。若 Broadcom 明天消失，AI 基礎設施的「去 NVDA 化」會倒退數年、企業私有雲虛擬化層會出現真空。這不是「提升效率」，是 AI 算力產業鏈的結構性瓶頸供應商。

逆向推演（Munger Inversion，三維展開）

① 技術維度（什麼技術替代讓 AVGO 3 年內失去 50% 價值？）

  • NVDA 全棧反攻：NVLink Fusion + 開放 NVLink 給客製晶片，使 hyperscaler 願意留在 GPU 生態而非自研 XPU → 歷史對標：Intel 用 x86 + 軟體生態長期壓制 RISC 替代。
  • 客戶內製成熟：Google TPU 設計團隊、Meta 矽團隊壯大到不再需要 Broadcom turnkey → 歷史對標：Apple 自研 A 系列/M 系列後踢開供應商、再內製 modem 踢 Qualcomm。
  • MediaTek/Marvell 在 custom ASIC 設計服務追平 → 點對點 socket 流失（§9 已列 🟡）。
  • chiplet/UCIe 標準化降低「全棧協同」門檻，讓客戶混搭多供應商 → 稀釋 Broadcom 的整合溢價。

② 管理維度（什麼決策路徑會自我毀滅？）

  • 併購慣性：Hock Tan 的成長模式高度依賴大型 M&A；若為維持成長硬做下一筆超大併購（市值 5%+）且整合失敗 → 歷史對標：AT&T 連串媒體併購減損。
  • 創辦人/CEO 退場：Hock Tan 是資本配置與客戶關係核心，接班斷層風險（§12）→ 歷史對標：奇異 Welch 後繼任失速。

③ 結構維度（什麼產業變化讓商業模式過時？）

  • AI capex 超級週期反轉：少數 hyperscaler 同步砍 capex（ROI 不如預期/監管/宏觀）→ $73B「committed」backlog 被重新談判 → 歷史對標：Cisco/Sun 2001 電信 capex 崩盤。
  • 監管型分拆：反壟斷或國安要求拆分半導體/軟體 → 估值雙引擎協同消失。
  • 計算範式轉移：若推論轉向極致 edge/光子/新架構，現有 XPU 設計資產貶值（慢變數，§5 R3）。

價值陷阱路徑四選一：最可能是產業結構衰退（AI capex 週期反轉）而非護城河侵蝕 — AVGO 護城河與盈餘品質皆健康，唯一系統性下行情境是其客戶（少數 hyperscaler）的需求集中度。

② 對下游客戶
