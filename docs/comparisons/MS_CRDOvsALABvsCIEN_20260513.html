業務地圖三個關鍵洞察(這節的 alpha):

① 業務 overlap 最深的是 CRDO ↔ ALAB(在 PCIe Gen6 retimer + SerDes IP) — 但兩家定位完全不同:CRDO 是 AEC 主場(rack 內短距 copper 互連)、PCIe retimer 只佔 <10% 為次要產品;ALAB 是 PCIe Retimer + AI Fabric Switch + 軟體三位一體的「rack 平台」。Broadcom PCIe 6.0 2026 H2 量產同時威脅兩家,但 ALAB 因 COSMOS 軟體層 36 月 API lock-in,被擠壓速度遠慢於 CRDO。

② CIEN 業務最 differentiated — 三檔中唯一做長距 backbone coherent optical(WaveLogic 6 1.6T),不在 CRDO / ALAB 產品線裡。CIEN 直接競爭對手是 Cisco / Nokia(不在這個比較圈),所以 CIEN vs CRDO/ALAB 的勝負並不取決於 socket 對打,而是「1.6T coherent temporal monopoly 12-18 月窗口能否轉成 EPS 兌現 + 倍數防守」這個獨立問題。

③ 護城河 mechanism 結構性 vs 執行性 vs temporal 三分類(這是這個比較最關鍵的判斷):
  • 結構性硬 lock-in(軟體層 / 平台 / 36+ 月):ALAB COSMOS 是唯一 — 三檔中最 durable
  • 執行 + 物理優勢混合(主場 + 平台 design-in 18-24 月):CRDO AEC — 主場物理優勢真實,但 SerDes IP 已被 Synopsys 部分追上,雙引擎方向相反
  • Temporal monopoly(規格首發 12-18 月唯一 window,非永久):CIEN WaveLogic 6 — 護城河本質上是「時間窗口」,Cisco/Nokia 2027 對位後即進入三足鼎立,PE compression 不可避

這個三分法直接解釋下方 §B.3 排序為何 ALAB > CRDO > CIEN(IRR 排序)— 護城河 durability 從 36 月平台 lock-in → 18 月硬體領先 → 12 月 temporal 規格首發,本質上就是「時間 lock-in 長度」的排序。

§B.3 中長期視角 3-5 年 — 護城河 mechanism 解剖 + IRR + TAM

§B.3.a 護城河來源解剖 — 每檔 mechanism 級判斷

不是評分,是 anatomy。對每檔回答三個問題:(1) mechanism 具體是什麼? (2) 為什麼能撐 3-5 年? (3) 3-5 年內最可能讓 mechanism 失效的單一事件?

CRDO — AEC 主場物理優勢(已穩) + SerDes IP(已部分失,已驗證)

Mechanism:AEC 護城河 = 主場物理優勢(銅線短距功耗低 30-50% + reliability 1000x vs 光模組)+ Aries 平台 Intel/AMD reference design 深度整合 18-24 月 design-in 切換成本;SerDes IP 護城河 = 224G PAM4 領先,但縮短至 6-12 月(AWS Trainium3 已轉 Synopsys 證實侵蝕)。

為什麼撐 3-5 年:(a) AEC 短距主場是物理層問題,光模組功耗與成本短期(3-5 年)無法在 rack 內取代銅線;(b) AWS / Azure / Google / Meta 的 AI rack scale-out 客製化粘性(每代 ASIC + cable 共設計);(c) 73% AEC 市佔 + 連 5 季 GM 擴大至 68.6% 證明定價力仍在。

最可能讓 mechanism 失效的單一事件:第二個 hyperscaler(Azure / Meta / Google 任一)公告下一代 ASIC 改用非 CRDO SerDes(複製 AWS Trainium3 案例)— 即證明 SerDes 護城河完全失效,等級從 B 跌至 C,Top 5 客戶集中度 60-70% 結構性風險具象化。Marvell Golden Cable 2026/2 FII 完成首設計是次要催化(吃 AEC 份額)。
ALAB — COSMOS 軟體層 36 月 API lock-in(三檔唯一結構性硬 moat)

Mechanism:硬體層 Aries Gen 6 PCIe Retimer + Scorpio AI Fabric Switch 領先 Broadcom 12-18 月 + 軟體層 COSMOS API 在 hyperscaler AI rack 端到端最佳化的設計-驗證-部署形成 36 月平均 design-lock-in cycle。硬體可追上但軟體層 hyperscaler 不會輕易切換。

為什麼撐 3-5 年:(a) 2027-2028 PCIe/UALink silicon 多代併行(Gen6/7 + UALink 1.0/2.0),ALAB COSMOS 縱深強使替代成本 extremely high;(b) Broadcom/Marvell 硬體追趕需 18 月但無對標軟體;(c) Hyperscaler 自研 retimer 機率低(<15% by 2029)因軟體層價值 >60% 不易內製;(d) NVDA NVLink Fusion 主動公告 ALAB 配套是平台站隊的反映,進一步深化 lock-in。

最可能讓 mechanism 失效的單一事件:Broadcom 2027 上市 PCIe Gen7 並 同步推出競爭軟體層(模仿 COSMOS)— 機率 20-25%。一旦發生,hyperscaler 採購策略從 ALAB single-source 轉向 dual-source(Broadcom 40% + ALAB 60%),IRR composition 中 +18.5%/yr EPS 引擎會被擠壓至 +10-12%/yr,Base IRR 從 +12.4% 降至 +6-8%/yr。
CIEN — WaveLogic 6 1.6T temporal monopoly(window 12-18 月,非 durable)

Mechanism:WaveLogic 6 1.6T coherent 業界首發(2024 GA / 3nm / 50% 功耗節省)+ 12-18 個月「唯一供應者」窗口 + 3nm 製程 DSP IP 獨家優化 + 90+ hyperscaler 簽約 WaveLogic 6e roadmap + Backlog $7B (1.4x annual revenue)提供 12-18 月訂單能見度。

為什麼撐 3-5 年(僅短中期):(a) 3nm DSP IP 獨家優化(華為 / 三星短期內無法快速複製);(b) 90+ 簽約客戶提供 18 月訂單鎖定;(c) Backlog $7B 現金流確定性。但本質上是「規格首發 temporal monopoly」而非永久 mechanism — 12-18 月後 Cisco / Nokia 對位即進入三足鼎立。

最可能讓 mechanism 失效的單一事件:Cisco 或 Nokia 在 2026 EOY 前公告 1.6T commercial GA 並拿下 ≥ 1 個 hyperscaler(GOOGL / META / MSFT 任一)大單。機率 35-40%(2027 H1 前)。一旦發生,即刻破裂「唯一」敘事,PE 從當前 ~77x compression 至 22-25x EMS / Network OEM 中樞水平(-70%),即使 EPS +19%/yr 也被 re-rate −32%/yr 完全吃掉(IRR composition 結構性問題)。
三檔護城河 mechanism 級比較總結:

  • ALAB:軟體層 36 月 API lock-in + 硬體 12-18 月領先 + 4 條獨立產品線分散風險 — 三檔唯一「durable」護城河,失效需要硬體 + 軟體雙重事件
  • CRDO:AEC 主場物理優勢真實但 SerDes IP 已部分失分,雙引擎方向相反 — 失效路徑單一事件(第二 hyperscaler SerDes 改用非 CRDO)即可觸發
  • CIEN:Temporal monopoly 本質上是「時間窗口」,3-5 年中段(2027 後)必然進入三足鼎立 — 機率 60%+ 的 PE compression 是 base case 而非 risk case

這個 mechanism 級排序(ALAB > CRDO > CIEN)與下方 IRR 加權排序完全一致 — Durability 等級直接驅動 IRR composition 質感(ALAB EPS 引擎可保 / CRDO 主場可保但 SerDes 易失 / CIEN EPS 對了但 PE 必壓縮)。
