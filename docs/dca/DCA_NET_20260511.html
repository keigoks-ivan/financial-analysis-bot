§3｜Key Drivers + 🔄 Consensus 分歧

#Key Driver為什麼是命脈對應可觀測指標
1AI agentic internet 路由層位置（供給側 moat）Cloudflare 的物理 anycast 網絡 + AI Gateway 是 inference / agent 流量的天然中介層。LLM token 經濟學決定 AI 公司必須優化 latency（每跳 50ms 就影響 UX）+ caching（同類請求 60-80% cost saving）。CF 是少數能同時提供「edge inference + 全球路由 + token caching」的單一平台。**這是供給端結構性護城河，非單純需求側風口。**(1) Workers AI 季度年化收入（首次揭露 = thesis 確認）；(2) AI Gateway 處理的 token 量 / 月；(3) 大模型供應商（Anthropic/OpenAI/Google）公開合作宣告

⚖ Driver 抽象層 guardrail 校驗

Driver 1（供給側 moat：物理網絡 + AI 路由）vs Driver 2（需求側持續性：大客戶 expansion）— **能否在不假設 driver 1 成立的情況下，driver 2 單獨被推翻？** 可以。例：若 Workers AI 完全失敗（driver 1 推翻），CF 仍可靠 CDN + Zero Trust + Application Security 三段式 land-expand 維持 DBNR ≥ 115%（driver 2 獨立站立）。**通過正交性測試。**

為什麼是這 1-2 個，而不是其他？

Cloudflare 是「平台戰」的典型 case，但平台戰的命脈不是「產品數量」（CF 已有 30+ 產品 SKU），而是兩個獨立 vector：(a) 平台是否處於 AI 時代的關鍵供給位置（driver 1）、(b) 平台是否被企業真實預算採用（driver 2）。前者決定 TAM 樓上限，後者決定 TAM 變現節奏。其他常被市場討論的（如 DDoS protection 市佔、IPv6 推廣、CDN 速度 benchmark）都是「結果指標」而非「命脈」，列為命脈會誤導 thesis。
