## 1. Mission

DCA Phase A4「既有報告數據擷取」是 DCA 與 DD 唯一的 structural interface。當前 v1.2 對這個介面的描述已落後 DD 6+ 個版本：

| DCA v1.2 寫的 | 對應 DD 實際版本 | 過時程度 |
|:---|:---|:---|
| `R:R ratio (MA60 / MA200 / MA104w 三版本)` | v8.x 格式（pre-v10.0） | OBSOLETE 自 v10.0（13 個月前） |
| `三引擎目標價` | v9.x 格式（pre-v10.0） | OBSOLETE 自 v10.0 |
| `§0.5 核心假設 (H1/H2/H3)` | v9.0/v9.1 章節編號 | OBSOLETE 自 v9.2 章節重編號 |
| `§8 最終結論` | v9.0/v9.1 章節編號 | OBSOLETE 自 v9.2（現為 §1） |

更嚴重：**DCA 完全沒讀 `dd-meta` JSON**（v12.1 起 DD 都有的 structured data block），全靠 LLM 鬆散解讀 HTML。

本 v1.3 patch 的目標：
1. **修 stale references**（4 處長期積欠）
2. **Pivot Phase A4 到 dd-meta JSON 為 primary source**（HTML 為 fallback）
3. **加 §5.F single thing cross-check rule**（DCA §5 vs DD §5.F 必有對應）
4. **可選 adopt execution + pricing power 二維 moat 框架**（與 DD §9 對齊）

**不動 DCA HTML 輸出 schema（§2-§8 結構不變）**。

### Edit 2：§5 Single Thing 加 cross-check rule（L424 後）

**Current** (v1.2): §5 Single Thing 完全獨立寫，與 DD §5.F 各說各話。

**v1.3 addition**（在「目前離發生有多遠」row 之後加入新區塊）:

```
**Cross-check with DD §5.F single thing（v12.3+ 強制）**：

DD v12.3 起 §5.F 加入了 single thing 概念（從 DCA borrowed back）。DCA §5 與 DD §5.F 必須有對應關係，**禁止各自寫不相關 trigger**。

| Alignment 狀態 | 規則 |
|:---|:---|
| **✅ Aligned** | DD §5.F 與 DCA §5 指向同一事件 → 註明「DD §5.F 一致，DCA confirm」+ 加強監測 detail（資料來源、頻率、最近 status） |
| **⚠ Partial** | DD §5.F 與 DCA §5 部分重疊（如同一 mechanism 但不同 trigger point） → 註明分歧並寫「兩者都監測」 |
| **❌ Diverge** | DD §5.F 與 DCA §5 完全不同 → 必須說明 **why DCA picks different trigger**（如：DD 從執行風險角度、DCA 從 PM 部位風險角度），並標 "DCA 認為 DD §5.F = secondary, DCA §5 = primary" 或反之 |

**禁止**：寫 DCA §5 而完全不 reference DD §5.F（DD 已存在時）。若 DD 無 §5.F（v12.2 之前的 DD），DCA §5 為唯一 single thing，標 "no DD §5.F counterpart"。
```

當 DD 已有 `moat_execution` + `moat_pricing_power` sub-scores（dd-meta JSON）→ DCA Phase A1 應 inherit + 加自己獨立評估：

| 維度 | DD §9 評分 | DCA 獨立評估 | Alignment |
|:---|:---:|:---:|:---|
| Execution moat | DD __/10 | DCA __/10 | aligned / partial / diverge |
| Pricing power moat | DD __/10 | DCA __/10 | aligned / partial / diverge |
| **合併** | DD __/10 | DCA __/10 | — |

**判斷規則**：
- 若 DD 為 single-axis（SaaS / 銀行 / 保險 / 寡占公用事業）→ DCA 維持 single-axis + narrative 說明為什麼這業務不適合二維

`moat_trend ↑/→/↓` 仍為單一指標（DCA Phase A1 必填欄），不需拆二維。
