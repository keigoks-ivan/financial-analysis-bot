§A1｜護城河深掘（execution + pricing power 二維）

Execution Moat — 7/10

① 17 年 NoSQL document DB 技術 lead + developer mindshare

MongoDB 從 2007 年（10gen 成立）起累積 17 年 document model 技術；對手 Cosmos DB（2017 GA）/ DocumentDB（2019 GA）雖 API-compatible 但 ecosystem mindshare 弱。MongoDB University 100 萬+ developer certifications + GitHub mongodb org 100+ repos + Stack Overflow MongoDB tag 500K+ questions 形成自我維持的 developer funnel。但這個 mindshare 優勢在 2024-26 受 PostgreSQL 復興擠壓 — PostgreSQL 自身 ecosystem（pgAdmin / Supabase / Neon / Crunchy）也在 cloud-native 浪潮中復興，年輕開發者新項目選 PostgreSQL 比例上升。

⚠️ 關鍵矛盾 #1：MDB 仍是 NoSQL document DB 領導者，但 NoSQL document 戰場本身在縮減

核心問題：「NoSQL document DB」這個 category 是否仍是企業偏好？

正方：

  1. schema-less 在 web / mobile / 微服務時代仍是 startup default
  2. MDB 65,200+ 客戶 + Atlas $2B+ ARR 證明 mass adoption
  3. JSON-first API 是 cloud-native 開發者偏好

反方：

  1. PostgreSQL JSONB 11 年成熟（2014 GA），現代 PostgreSQL + JSONB 體驗已接近 document model
  2. Neon / Supabase 提供 cloud-native PostgreSQL 體驗（schema migrations + auto-scaling），消除過去 PostgreSQL ops 痛點
  3. 2024-26 年 startup 新項目選 PostgreSQL 比例上升至 ~55-60%（vs 2018 年 ~35-40%），MDB 從 ~40% 降至 ~25%
  4. Databricks Lakebase + Snowflake Crunchy Data 將 PostgreSQL 整合進數據平台，企業可 one-stop 處理 OLTP + Analytics

結論：NoSQL document DB 仍是 viable category，但已從「最佳選擇」變「合理選擇之一」。MDB 護城河仍在（17 年技術 + ecosystem + 切換成本），但長期 share 預計從 NoSQL 子集 25-30% 緩降至 18-22%。這與 FY27 guide +16-18% 一致 — 不是 thesis 破裂，是 mature SaaS 自然 trajectory。
