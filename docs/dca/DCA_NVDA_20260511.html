div+回 +0.4%
re-rate −10.4%

護城河細節深度（補充分析）

(1) CUDA 軟體生態系：18 年累積的 PyTorch / TensorFlow / JAX 底層整合 + cuDNN / cuBLAS / TensorRT / NCCL 庫。任一個 ML researcher 跑訓練腳本 `pip install torch` 後預設 backend 是 CUDA — 這是「預設選項偏誤」型 moat，不是性能差距 moat。即使 ROCm 性能追到 95%，「重寫 + 認證」一整套 production pipeline 仍需 6-18 個月人力。Stack Overflow CUDA 問題數 / GitHub repo 數估計是 ROCm 的 50-100x。

(2) 系統整合 (NVLink / NVSwitch / NVL72 / NVL576)：Rubin Ultra NVL576 集成 576 GPUs / 144 modules / 共享 1TB HBM。這不是「單顆 GPU 規格」競爭，而是「整個 rack 系統 throughput」競爭 — AMD 即使單卡追上，缺乏對應的 fabric & memory architecture 整體 stack，hyperscaler 仍會選 NVDA 因為「scale-out 比較容易」。Feynman 2028 將上 3D stacking + 自研 HBM，這層 moat 短期看不到對手。

(3) TSMC 製程優先權：NVDA 是 TSMC 第二大客戶（僅次 Apple），CoWoS 配額分配優先；Rubin 用 TSMC 2nm，Feynman 預期 A16。對 AMD / Intel 而言，即使能拿到同代製程，CoWoS 排隊得等。這層 moat 在「先進製程 + 高 ASP 場景」最強，但若 ASIC（Google TPU 用 Samsung、Trainium 用 TSMC 但二線）滲透加速，這層 moat 對總體 wallet share 保護力會降低。

(4) Moat trend 動態：軟體層 → widening 訓練側 / → narrowing 推論側（OpenAI Triton + vLLM + TensorRT-LLM 的競爭 stack 在推論側慢慢 commodify）；系統層 → widening（每代 NVL rack 規模翻倍）；製程層 → holding（TSMC 2nm 全市場可用但 NVDA 仍享優先 CoWoS）。整體 moat_trend = → holding 為合理判斷。
