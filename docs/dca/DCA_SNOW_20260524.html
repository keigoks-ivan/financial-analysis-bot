§A1｜護城河深掘（execution + pricing power 二維）

Execution Moat（製造工藝 / 研發深度 / 規模效率）— 9/10

① Separated Compute/Storage 架構：14 年技術 lead

SNOW 三位創辦人（Marcin Żukowski 前 VectorWise、Benoit Dageville 前 Oracle、Thierry Cruanes 前 Oracle）在 2012 年做出 separated compute / storage 架構決策：把資料常駐物件儲存（S3 / GCS / Blob），運算層（virtual warehouses）按需求啟動 + 自動擴縮。這在 2012 年是 contrarian 決定 — 當時 Redshift / BigQuery 仍是 compute + storage 緊耦合，無法做 multi-tenancy concurrent scaling。SNOW 領先 5-7 年；對手 Redshift 至 2019 才推出 RA3（separated storage），但仍未做到 SNOW 的 multi-tenant cloud-native 程度。

② Multi-cloud 中立性：2018-19 戰略決定的 10 年累積

SNOW 2018 上 Azure、2019 上 GCP — 在當時是 contrarian（多數 SaaS 仍 AWS-only）。今天 SNOW 在 AWS 占 ~55% 工作負載、Azure 占 ~30%、GCP 占 ~15%。客戶要的不是「在哪個雲」，而是「不被 lock-in 任何一個雲」— SNOW 提供唯一中立 substrate。Databricks 雖也 multi-cloud，但歷史上深綁 AWS（Spark on EMR 起家），中立 narrative 較弱。Hyperscaler 自有方案（Redshift / BigQuery / Synapse）天然 single-cloud，無法提供中立性。

③ Sridhar Ramaswamy 接班後 product velocity 大幅加速

Sridhar Ramaswamy（前 Google 搜尋廣告 SVP，2023 Neeva 創辦人）2024 Feb 接任 CEO 後，產品 velocity 從「年度 GA 3-5 個新功能」加速至「FY26 累計 430+ 新功能」（公司 Investor Day 數字）。具體 milestone：

  • 2024 H2：Cortex AI Functions（內建 LLM 推理）GA
  • 2025 H1：Snowflake Intelligence agentic platform GA
  • 2025 H2：Cortex Agents（自定義 agent 建構）GA
  • 2026 Q1（4/21）：Cortex Code（AI coding agent）GA + Snowflake OpenFlow（資料整合）GA + Snowflake Postgres 預覽

對比 Databricks 同期：Mosaic AI（MML 結合 LLM）GA 2024 H2、Unity Catalog 開源 2024 H2、AI/BI Genie 2025 H1。兩者產品 velocity 接近 par，但 SNOW 在 governance + native app + Marketplace 仍 broader。

④ 規模效應：cloud cost discount + R&D leverage

SNOW FY26 cloud infra cost 估 $300-400M（佔 Rev 7-9%），來源 AWS / Azure / GCP enterprise contract。新進對手如 Firebolt / ClickHouse Cloud 採購規模 1/100，unit cost 高 2-3x — 在 commodity compute 戰場結構性處於劣勢。R&D $1.6B/年 vs MDB $450M / CFLT $300M — 同 tier 內 SNOW 規模優勢 2-5x，可同時 fund 5+ 條產品線。

Pricing Power Moat（轉換成本 / 網路效應 / 客戶依賴度）— 9/10

① Consumption-based 計價 + 自然 expansion：NRR 125%

SNOW 按 credit consumption 計費（非 seat-based），客戶資料量增加 / 查詢頻率增加 / Cortex AI workload 上線 = 自然 expansion。Q4 FY26 NRR 125%（FY24 158% → FY25 130% → FY26 125% 已 stabilized）；同業 DDOG 115% / MDB 119% / NET 110% — SNOW 仍最高。

NRR 從 FY24 158% 滑落至 125% 的原因（公司 Q4 FY24 法說明確）：

  • 2023-24 客戶執行 cost optimization（query 優化 + 移除冗餘資料 + 自動 suspend warehouse），單客戶 credit consumption 增速放緩
  • 大客戶從 "land & expand" 進入 "expansion mature" 階段，growth 自然降速
  • 2025 開始 Cortex AI 帶來新一波 incremental consumption — Q4 FY26 NRR 125% 連 2 季穩定，顯示 stabilization

③ Marketplace 雙邊網路效應

3,000+ data + AI app providers（LiveRamp 第三方身份資料、Foursquare 位置資料、S&P Global 金融資料、AccuWeather 氣象、Equifax 信用、Nielsen 收視率等）；11,000+ data consumers。每新增 provider → 對 consumer 吸引力 +1。Snowflake Native Apps（50+ GA）讓 ISV 在「資料不離開客戶 Snowflake account」前提下提供功能，這在 multi-tenant SaaS 圈是獨家模式 — Databricks Unity Catalog 雖也支援 Delta Sharing，但 native app 生態落後 2-3 年。

④ RPO $9.77B 多年合約鎖定

Q4 FY26 RPO 達 $9.77B（+42% YoY），約等於 FY27 product revenue guide ($5.66B) 的 1.73x，意指企業已預付 ~2 年 product revenue。RPO 成長 (+42%) 遠快於 Revenue 成長 (+30%) 12pp 剪刀差 — 領先指標健康，未來 4-6 季 revenue 有 multi-year contract 支撐。

Pricing Power 扣 1 分原因：Databricks lakehouse + Iceberg 開放範式壓 upside

Apache Iceberg（開源 table format）2024 後快速普及，企業可在 S3 / GCS / ADLS 上以 Iceberg 表存資料，再用任意 query engine（Trino / Polaris / Databricks SQL / SNOW External Tables / Athena）查詢 — 理論上不再需要 SNOW 專有 storage。SNOW 2024 主動支援 Iceberg（External Tables + Iceberg Catalog），不抗拒此趨勢，但等於承認「storage 層 lock-in 已弱化」。長期可能從「資料 + 計算 + 應用」三層綁定退至「計算 + 應用」二層綁定，pricing power upside 受限。短期（3-5 年）影響有限因 Iceberg 生態不夠成熟；長期需監控。

⚠️ 關鍵矛盾 #1：Databricks growth 是 SNOW 2.2x，但 SNOW 更賺錢；誰是贏家？

表面看：Databricks 完勝（成長 2.2x + AI run rate 14x + NDR 高 15pp）。

實質分析：

  1. Databricks 65% growth 含「相對小基數效應」— $5.4B ARR 從 $3.3B 跳上來，base 1.5 年前還小；SNOW 從 $3.6B 漲到 $4.68B 是 29%，base 大絕對增量也大（+$1B vs +$2.1B，比例差但絕對額在同 ballpark）
  2. Databricks AI run rate $1.4B 包含 Mosaic AI（模型訓練平台）+ Lakebase + ML pipelines — 是 ML 工作負載集中地；SNOW $100M 集中在「企業既有資料 + Cortex 推理 agent」— 兩者服務不同的 AI workload type
  3. Databricks NDR 140%+ 是「分母不同」— 計算 base 是 $1M+ customers（650 家）；SNOW NRR 125% 是「全客戶 base」（11,000+）。可比 metric 應該是 SNOW $1M+ 客戶的 NRR，那大概 ~135-140%（與 Databricks par）
  4. Databricks 是私募，沒有公開財報嚴格度；其 FCF 數字（估 ~10% margin）需 IPO 後才能驗證 — 歷史上私募高成長 SaaS 在 IPO 後常 FCF 數字打折

結論：Databricks 在 AI / ML 訓練端領先，SNOW 在企業資料治理 + Marketplace + 政府 / 受監管行業領先 — 是不同細分而非「Databricks 全面碾壓」。但 SNOW 必須維持 ≥ 25% growth + Cortex 變現加速，才能在「AI 推理工作負載」維持中介層地位。

⚠️ 關鍵矛盾 #2：hyperscaler 既是 SNOW 客戶（雲服務商）又是競爭者（自有 data warehouse）；長期共存還是淘汰？

結構性思考：

  • 過去 10 年（2014-2024）：hyperscaler 從 SNOW 賺基礎設施錢，雙方共存 — 因為大企業客戶要 multi-cloud 而 SNOW 是唯一中立 substrate
  • 2024-2026 變化：hyperscaler 開始 push「AI 應用層」（Bedrock / Azure AI / Vertex AI），這層 SNOW 介入有限。但企業資料治理層 SNOW 仍主導
  • 未來 5-10 年路徑：(A) Cloud-native 客戶（startup / mid-market）可能直接用 hyperscaler one-stop solution；(B) Enterprise / 受監管行業仍需中立 substrate → SNOW 護城河
  • SNOW 的「策略防禦」：(1) Cortex 整合 Anthropic / OpenAI / Meta 模型 → 不依賴單一 hyperscaler；(2) Marketplace 雙邊網路 → 跨雲生態；(3) Snowflake Postgres（Crunchy Data 整合）→ 補齊 OLTP 缺口減少客戶 spillover

結論：hyperscaler 自有方案在中小型企業 + 純 single-cloud 工作負載會吃 SNOW share；但 SNOW 在 Fortune 1000 + 受監管行業 + multi-cloud 客戶 (Top 1000) 仍 lock-in 強。預計 2030 SNOW market share 從 18% 降至 12-15%，但 TAM 擴張 2.2x 使絕對營收仍 4-5x。
