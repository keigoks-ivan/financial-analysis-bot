公開資料端點文件

本站選股與跨資產監測底層由一組公開 JSON 端點驅動，供程式化讀取。以下逐一說明每個端點的用途、更新頻率（來源 GitHub Actions workflow）、as_of 時效欄位語意與 schema。所有端點皆為靜態檔，直接 GET 即可，無需驗證、無速率限制。

使用條款：資料僅供研究，無擔保（as-is, no warranty），來源請註明「InvestMQuest Research」。端點為靜態快照，時效以各檔 as_of / generated_at 為準。

DD 選股宇宙

/dd-screener/latest.json

全站個股 DD 宇宙的機器可讀快照——約 240 檔已寫過 DD 報告的個股，帶基本面訊號（FCF／ROIC／EPS 成長／PEG／負債）、護城河評分、統一裁決（dca_verdict）、五年情境價與 EPS 估計。是 /dd-screener/、/cockpit/、/research/ 三頁的共同資料源。

更新頻率每日（Tue–Sat 09:00 台北）· workflow daily-non-fundamental-refresh 跑 build_dd_screener.py；每份新 DD 發布時亦即時重建
as_of 語意as_of = 快照交易日；eps_estimates_source.snapshot_date = EPS 估計 Excel 快照日
欄位型別意義
schema_versionstring此檔 schema 版本（現 "1.3"）
as_ofstring快照交易日 YYYY-MM-DD
run_timestampstring產生時戳（ISO 8601，含時區）
universe_sizeint宇宙成員數（現 241）
criteriaarray五道基本面閘定義（key／label／threshold／invert／unit）
funnel_configobject漏斗排序權重、品質映射、可原諒失敗數、moat 趨勢乘數、修正門檻
summaryobject通過閘數分布（pass_5／pass_4／pass_3／pass_lt3／no_data）
eps_estimates_sourceobjectEPS 估計來源（snapshot_date／source_file／coverage_count／fallback 名單）
stocksarray個股列（見下方逐檔欄位）
stocks[] 主要欄位型別意義
ticker / name / sectorstring代碼／名稱／產業
signalstring基本面評級 A+/A/B/C/X（metadata，非裁決）
dca_verdictstring統一裁決：進場／觀望／迴避
dca_rolestring倉位角色（核心持倉／衛星…）
moat_grade / moat_score / moat_trendstring/num護城河評等（S/A/…）、分數、趨勢（↑→↓）
fcf / roic / eps2y / peg / denumberFCF%、ROIC%、2 年 EPS 成長%、PEG、負債比
fpe_fy2 / pct_5ynumberFY+2 forward P/E、5 年 P/E 分位

bull_5y_price / bear_5y_price / p_bull_pct / p_bear_pctnumber5 年多空情境價與機率

eps_fy_curr / eps_fy_next / eps_fy3 + *_revision_pctnumber各財年 EPS 估計與修正幅度%
eps_display_currency / eps_fx_ratestring/num非美股顯示幣別與換算匯率
dd_path / dd_datestring對應 DD 報告路徑與日期
live_fpe_est / pe_drift_pct / dd_age_daysnumber即時 forward P/E 估計、相對 DD 當日漂移%、DD 齡

穩定性：schema 隨 DD skill 版本演進為 additive（下游 dual-read）。signal 為基本面 metadata，投資裁決以 dca_verdict 為準。任何破壞性變更會列於本頁。

產業輪動（RRG 主題）

/rotation/data/latest.json

64 個站內主題於週線宇宙的相對輪動位置（RRG：相對強度比值 × 動能），含四象限分類、產業時鐘相位、修正寬度與廣度。描述器非擇時——相對排名於本站宇宙內，非市場絕對輪動。

更新頻率每週（週日 12:00 UTC）· workflow crossasset-weekly 跑 build_rotation.py
as_of 語意as_of / data_asof_weekly = 週線資料截止日；generated_at = 產生時戳
欄位型別意義
as_ofstring資料截止日 YYYY-MM-DD
benchmarkstring相對基準（SPY）
quadrant_countsobject四象限成員數（leading／weakening／lagging／improving）
methodobject方法定義（breadth／gate／quadrants／disclaimer 等）
insufficientarray因成員／覆蓋不足而未評分的主題
themesarray主題列（見下）
themes[] 欄位型別意義
themestring主題名
rs_ratio / rs_momnumberRRG 相對強度比值、動能（100 為基準）
quadrantstring象限：leading／weakening／lagging／improving
clock_phase / sd_verdict / convictionstring產業時鐘相位、供需裁決、信心
ret_12m / rev_med / rev_up_fracnumber12 月報酬、EPS 修正中位、上修成員占比
breadth_ma40 / n_membersnumber/int成員站上 40 週均線占比、成員數
picks / trailarray代表成員、RRG 尾跡座標序列

穩定性：主題集合與方法門檻可隨校準調整；method 物件自帶當期定義，請以其為準。

輪動雷達（多框架 RRG 尾跡）

/rotation/data/radar.json

跨資產／產業／因子三個宇宙的多時間框架（120／60／20 交易日）RRG 尾跡動畫資料，日更。每日凍結副本存於 /rotation/data/snapshots/{YYYY-MM-DD}.json（同 schema）。

更新頻率每日（Tue–Sat 22:40 UTC）· workflow rotation-radar-daily
as_of 語意as_of = 最新交易日；snapshots 檔名即該日
欄位型別意義
as_ofstring最新交易日
framesarray時間框架（交易日數）：["120","60","20"]
methodobjectRRG 計算方法定義
universesarray三個宇宙，各含 key／label／benchmark／members
universes[].members[]object成員，含 frames.{window}.trail[]（d＝日期、r＝ratio、m＝momentum）

穩定性：框架與宇宙集合穩定；trail 座標每日追加最新點、滾動視窗。

擁擠交易監測

/crowding/data/latest.json

三層代理三角測量的資料層：15 個期貨市場的 COT 部位極端、15 檔跨資產 ETF 的動能／偏離、64 個主題的四維擁擠分數。描述器非擇時——衡量部位集中與下方緩衝，不預測轉折時點。

更新頻率每週（週日 12:00 UTC）· workflow crossasset-weekly 跑 build_crowding.py
as_of 語意cot_as_of = CFTC COT 報告日（常滯後數日）；generated_at = 產生時戳；ETF 時效見 meta
欄位型別意義
cot_as_ofstringCOT 報告 as-of 日
cotarray15 市場：market／net_pct_oi／pctile_3y／pctile_5y／delta_4w／delta_13w／direction／extreme
etfarray15 檔：ticker／label／ret_12m／momentum_pctile／deviation_pctile／volume_z
themesarray64 主題：name／rank／score／score_4dim／dims／members_covered／members_total／low_coverage
gapsarray未取得或低信心的資料點（誠實列出，不捏造）
metaobject各層時效與方法註記

穩定性：COT 市場數與主題數可調；gaps 為刻意保留的缺口清單。

精選清單（候選 + 正式榜）

/picks/candidates.json

由 build_picks.py 規則自動判定的兩軌清單：長熬（趨勢對）與爆發（循環形狀＋站上年線＋非過熱）。含正式榜（official）與候選，兩者互斥；持有人保留 veto。

更新頻率事件驅動——每份 DD 發布／同步時經 update_dd_index.py 連鎖 build_picks.py；每日 daily-non-fundamental-refresh 亦刷新
as_of 語意as_of = 判定基準交易日；generated_at = 產生時戳
欄位型別意義
countsobject各清單成員數
official_changhao / official_baofaarray正式榜：長熬／爆發
changhao / baofaarray候選：長熬／爆發
notestring判定規則說明
清單項目欄位型別意義
ticker / name / sectorstring代碼／名稱／產業
why / triggerstring入選理由、觸發條件
multiple / horizon / position / exitstring目標倍數、期程、倉位、退出條件
moat_grade / moat_trend / runway / rolestring護城河等級／趨勢、跑道、角色
ev5y_pct / above_w52 / trend_statusnum/bool/str5 年期望值%、是否站上年線、趨勢狀態
id_theme / dd_path / sourcesstring/array關聯產業主題、DD 路徑、來源
not_promoted_reasonstring候選未升正式榜的原因（僅候選項目）

穩定性：規則門檻於校準輪 PREREG 凍結；欄位為 additive。

全資產市場監測

/monitor/data/latest.json

近百條跨資產序列（指數／類股／因子／利率／波動／信用／匯率／商品／加密／流動性）的日更明細＋統計異常偵測，含當日 alerts。schema=monitor-v1。描述器非擇時。

更新頻率每日（Tue–Sat 22:52 UTC）· workflow monitor-daily 跑 build_monitor.py
as_of 語意as_of = 資料交易日；部分序列自帶 stale 旗標
欄位型別意義
schemastring"monitor-v1"
as_ofstring資料交易日
categoriesarray10 類（key／label／items[]）：indices／sectors／factors／rates／vol／credit／fx／commodities／crypto／liquidity
alerts_todayarray當日異常：cat／key／msg／rule／sev（嚴重度）
statusobject綜合狀態（殖利率曲線 curve_10y2y、流動性 liquidity、vix_term…）
fear_greedobject恐懼貪婪：score／prev／rating
rulesobject異常偵測規則定義
gapsarray未取得資料點

穩定性：schema 版本化為 monitor-v1；類別新增為 additive。另有 /monitor/data/alerts.json 留痕與 editorial.json（人工解讀層）。

大類資產 Regime

/regime/data/latest.json

六軸記分卡（利率／成長／通脹／信用／美元／波動）的機械＋editorial 兩層讀數，收斂成單一 regime 綜合定位（0–1）。非可交易指數——環境判讀而非擇時訊號。

更新頻率每週（週日 12:00 UTC）· workflow crossasset-weekly 跑 build_regime.py
as_of 語意各軸 reading 內含來源日期；meta 帶 cot_as_of／etf_as_of；generated_at = 產生時戳
欄位型別意義
compositeobject綜合定位：label_zh／label_en／pos_0to1／method
axesarray六軸：key／name／pos（0–1）／pill（狀態標籤）／reading（敘述含來源）／src
cot_percentile_5yobject各市場 COT 部位 5 年百分位
growth_defense_ratiosobject成長／防禦相對比值
triggersarray/obj證偽／狀態轉換觸發條件
meta / gapsobject/array時效註記、未取得資料點

穩定性：pos_0to1 為質化加權讀數，非回測指數；軸集合穩定。

全站搜尋索引

/assets/search-index.json

全站報告的輕量檢索索引（短鍵，緊湊），驅動 /search.html。涵蓋個股 DD、產業 ID、總經、對比、期望綜合、財報、擁擠交易、產業輪動、供應鏈與 legacy DS/DCA。

更新頻率隨報告發布重建（build_search_index.py）
as_of 語意每筆 d = 報告日；供應鏈頁無日期（d 空字串）
欄位型別意義
vint索引 schema 版本（現 1）
entriesarray報告條目（見下）
entries[].ustring相對 URL
entries[].tstring標題
entries[].kstring類型（個股DD／產業ID／總經／…）
entries[].sarray可搜尋關鍵字（tickers／slug）
entries[].dstring報告日 YYYY-MM-DD（可空）

穩定性：短鍵 {u,t,k,s,d} 凍結；新類型加入 k 值集合為 additive。同一批發布內容不變則零改動（zero-churn）。同源 RSS：/rss.xml。
