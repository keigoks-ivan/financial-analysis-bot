## 條目

**1. Opportunity-cost 鏈拼接**（MU/SNDK/GLW）｜把每檔來源報告 opportunity-cost 段點名的「更優同類」串成有向鏈——鏈的終點在比較集內嗎？集內是否有檔被**集外**標的 dominate（「memory 兩兄弟都 dominated、GLW 與鏈脫鉤」）？多份報告的指向互相印證還是矛盾？

**2. 跨報告引用的前提時效驗證**（MELI/SE/GRAB）｜引用 A 報告對 B 檔的相對判斷時，該判斷依賴的數字前提（「前提是 SE PEG 1.8」——已被更新報告打破至 0.65）以最新資料重驗；不成立則明文推翻該引用。與財報 staleness gate 不同——這是報告間交叉引用的前提校驗。

**3. 品質首選 vs 此刻可動手首選分離**（SNDK/GLW/APH）｜兩者是否同一檔？不同 → 寫出讓兩答案**合一的具體價位/事件**（「APH 回測 ≤$145 兩答案合一」），並給「可動手」裁決自己的失效價位（「GLW 漲回 ≥$190 作廢」）。防止品質裁決被誤讀成即刻買進訊號。

**4. 替代框架 steelman-then-veto**（MU/SNDK）｜是否存在某個持有期/mandate 使目前輸家變第一（「純動能視角 SNDK 第一」）？有 → 明文點名該框架與贏家，再寫出否決它的具體紀律理由。禁止假裝替代框架不存在，也禁止被它帶走。

**5. Archetype 可比性＋全勝反轉解讀**（SNDK 逐出長抱軸）｜墊底檔是「同軸較差」還是「屬於另一個 archetype/持有期」？後者 → 逐出比較軸並在它成立的軸上給獨立動作（「SNDK＝<2 年 NAND 投機，satellite 軸另議」），不是排最後。某檔四層全勝時必答：贏家強還是**對手不適格**（「GLW 全勝＝另兩檔結構性不適合 buy-and-hold」）。

**6. 共同 beta 切割與管轄權移交**（SNDK/GLW/APH）｜這幾檔的共同風險因子是什麼（「三檔都是 AI capex 曝險，選哪檔都躲不掉」）？該因子的裁決屬於哪份 ID＋industry-thesis-critic、不屬於本報告——明文移交。並驗證相對排序對該因子漲跌方向是否穩健。

**7. 排序穩定度 meta 層**（NXPI/RCL 案）｜四層排序完成後必答三題：首選在哪幾層換人、為何？第二名在哪些層交換、對應什麼用戶意圖？排序對時間軸的（不）敏感性本身給出什麼結論（「NXPI 對時間軸最不敏感＝不需問持有期就能選」＝報告核心 alpha）。

**8. 便宜法醫學三分類**（MU peak/STM trough/APH 舊錨）｜比較集中估值最低的那檔，便宜的成因是——① peak EPS 分母假象（陷阱）② trough 分母反彈虛胖＋倍數 mean-revert 雙殺 ③ 市場沿用過期估值錨（真錯價、機會）？結論必須寫明屬於哪類；「選最便宜那檔」在未分類前禁止。

**9. Alpha 方向標注**（SNDK/GLW/APH）｜每檔寫「市場定價的假設 vs 本報告的不同假設」＋方向符號：**偏多／偏空／僅 entry 紀律**。偏空 alpha（「SNDK 的 edge 在賭空頭」）是研究價值不是買進理由——禁止出現在推薦理由中。

**10. Pattern match 血統分級＋衰減折扣**（MSFT）｜各檔 §11.7 樣本是**自身已驗證歷史**還是**跨公司類比**？可信度分級後才可跨檔比；沿用對標報酬前列出當前 setup 與樣本的差異因子並明示折扣（「capex 強度 2.4 倍＋競爭激烈 → 折至 +57%/5Y」）。

**11. 原版勝複製版**（GRAB＝MELI lite／ON vs NXPI）｜某檔的 bull case 若建立在「像集內另一檔的早期」上，必答「為什麼不直接買原版」——答不出即判複製版 dominated。跨檔比較特有，單檔 DD 看不到。

**12. Bear 性質分類 → 抱期焦慮成本**（STM class action vs NXPI）｜每檔 Bear case 分類：季度可監測執行風險／進行式蠶食（程度問題）／binary 範式（36+ 月才 settle）。分類直接回答「多數人會不會在 −40% 區段被洗出」——行為可持有性是正式排序維度。

**13. Max DD 起算點正規化**（SE 已 −52%）｜各檔 Max DD 的起算價位是什麼？已從高點回落幅度不一時，**換算成從本報告擬進場價起算**再排序——照抄來源報告的高點跌幅是不同基準的假比較。

**14. 不可復原性 trump 深度**（SNDK $1,562 上限 +15%/下限 −72%）｜每檔 Max DD 觸發後的復原路徑：幾季／幾年／**不可復原**（下輪 peak EPS 也撐不回原價的 PE×EPS 乘積）？不可復原＝永久資本損失，風險排序中優先於回撤深度。

**15. 落選檔升級觸發**（MELI/GRAB）｜每個落選檔給至少一條「升級回候選」的正向觸發（指標＋門檻＋預計時點）入監測表——比較結論保持活體，非一次性淘汰。

**16. 來源報告內部一致性當證據權重**（SE pre-mortem↔§5 重疊）｜各檔來源 DD 的 pre-mortem 敗局觸發與 Single Thing 是否重疊？重疊＝風險已被單一可監測指標覆蓋（加分）；脫節＝風險辨識存疑（降權）。

**17. Spread 單一決定因素歸因**（MSFT 案：進場估值起點）｜排序差距最大的維度，其跨檔 spread 的最大單一決定因素是什麼？壓縮成一句可遷移的因果句（「Bear 跌幅的最大決定因素＝進場估值起點——這就是 MSFT 勝出的根本」）。

**18. Holdings-conditional 第二檔切換**（RCL+RMS vs RCL+VIK）｜用戶已持有集內任一檔（或同籃標的）時，「加碼第二檔」的答案是否改變？用兩兩相關度/曝險重疊說明（cruise 同籃 0.6+ vs 跨籃 0.3-0.4）——單檔排名第三的檔在組合視角可以勝過第二名。
