## 【v12.4 變更：EPS estimates source 接 Excel buy-side consensus】

**核心改動**：寫 §13 估值前先跑 `python3 scripts/get_eps_for_ticker.py {TICKER}`，Excel 覆蓋（目前 141/146 含 US/ADR/TW 自動換 USD）時直接用 Excel 的 FY+1/+2/+3 + 2Y CAGR 當 anchor，**毋需再做 yfinance fetch 或 web_search 找 EPS consensus**。

**為什麼**：
1. Excel/Koyfin 是 buy-side 標準的 normalized consensus mean（非 GAAP 對 SaaS 自動處理）
2. 月度凍結，比 yfinance 每天變、web_search 散亂片段穩
3. ADR / TW listings 自動 USD 統一單位（解決 yfinance financialCurrency 是 TWD/EUR/CHF 的 FX-ratio 陷阱 — 之前 TSM 從這條路算出 -39.8% 鬼數）
4. FY+3 也有 consensus（yfinance 沒），不需要原 v9.0 的「§8 邏輯推導」機械外推

**判讀規則**（在「§13 估值 → 步驟零'」詳述）：
- `exit 0` Excel 覆蓋 → 用 Excel，§13 標註「EPS 來源：Excel snapshot YYYY-MM-DD」
- `exit 1` 未覆蓋 → fall back yfinance（步驟零）→ web_search（步驟一）
- `exit 2` Excel 缺檔 → 通知用戶 `cp ~/Downloads/DD_universe_EPS_estimates_*.xlsx data/eps-estimates/`

**§8 sanity check 仍必做**：Excel consensus 是 sell-side 共識，若與你 §8 的成長跑道 / 定價權 / 內生成長率分析明顯矛盾，要在 §13 標明分歧並提出 PM 自家修正值（不是無腦照搬）。

**配套**：`deep-conviction-analyst` skill v1.4 同步加 Phase A3 預備步驟（同樣呼叫 CLI）— DD 與 DCA 引用同一份 Excel snapshot 確保 §13（DD）與 Phase A3（DCA）的 EPS anchor 一致。

## 【品質管控強制規則(v9.0)】

**執行原則**:本 skill 必須一口氣跑完整份 DD,不得中斷詢問使用者
任何問題,除非資料經 2 次 web_search 仍無法取得(此時直接停止並回報「資料不足」,
不要問「你能提供嗎」)。v9.0 在 v8.1.2 八條品質規則基礎上,新增三條(QC-9 到 QC-11)
針對綜合品質分、波動率燈號 Python 計算、目標倉位框架。所有 11 條規則在每份 DD 執行時無條件強制遵守。

### QC-1｜業務權重必須引用 §11

§8 B 成長品質判斷中涉及的各業務段營收權重**必須**直接引用 §11 產業格局
的營收組成數字。禁止自行估算權重。所有超過 5% 營收占比的業務段必須
完整列出,權重加總必須 = 100%。若 §11 尚未完成,先完成 §11 再回來算。

### QC-2｜MA104w 必須查實際數據

MA104w（104 週移動均線）**必須**從 web_search 查詢實際數值。
搜尋查詢範例:
```
[TICKER] 104-week moving average
[TICKER] MA104 weekly TradingView
```
**禁止**使用「雙年均值外推」「起點終點平均」或任何估算法。
若 2 次搜尋無法取得 MA104w,可用 MA100w 或 MA520d 替代,
但必須在表格備注「替代指標:MA100w（非 MA104w）」。
同理,MA60 若無法取得而使用 MA50,必須備注「替代指標:MA50（非 MA60）」。

### QC-3｜下檔錨點必須標示數據來源

§2 的下檔錨點必須明確標示:
- Bear PE 的來源（引用 §8E 成長熄火壓力測試的「成長降至 10%」情境）
- Bear EPS = FY+1 EPS × 0.9 的計算過程
- Bear 股價 = Bear PE × Bear EPS
- W52 Low 和 W104 Low 仍須查詢並列為參考（yfinance / web_search），但不用於 R:R 計算
- 若 Bear 股價 > 現價，標註「上行空間為負，即使 Bear Case 仍高於現價」

### QC-4｜Forward P/E 分位必須列公式

§13.1 的 Forward P/E 歷史分位必須用以下公式計算並顯示:
  分位 = (當前值 − 5Y 低) / (5Y 高 − 5Y 低) × 100%
不得使用「~70%」等概數,必須算到整數位（如 68%）。
§3 或 §1 引用 5Y 均值時,數字必須與 §13.1 完全一致,
禁止出現同一份 DD 內兩個不同的 5Y 均。

### QC-5｜§10 同業比較組必須與 §13.4 一致

§10 財務品質的同業比較表必須使用與 §13.4 橫向比較相同的對手組
（至少 3 家直接競爭對手）。禁止只放 1 家對手做比較。

### QC-6｜§12 治理必須涵蓋四項

§12 公司治理章節必須包含以下四項:
1. 股東/股權結構（含 dual-class 投票權說明）
2. 資本配置方向（Capex / 回購 / 分紅的優先順序與金額）
3. 管理層薪酬結構（固定薪 vs 績效獎金 vs 股權激勵的比例）
4. 過去 12 個月重大內部人交易（買入/賣出/金額/對象）
若 web_search 無法取得第 3 或第 4 項,在該小節標註「數據限制:
無法取得管理層薪酬/內部人交易資訊」,不得跳過整個小節。

### QC-9｜品質分強制計算

§2 開頭必須先計算品質分,公式:

  底分 = (護城河 + 成長持久性) / 2
  品質分 = min(底分 + 體質加減分, 10)　　← 滿分 10 分

  - 護城河:§9 評分(1-10)
  - 成長持久性:綜合 §8 成長跑道 + 成長品質 + AI 影響,給 1-10 分
    (AI 受益加分、AI 威脅扣分,不再獨立計算)

### QC-29｜R:R 壓力測試（v12.0 新增，AMZN 教訓）

§2 E 必須執行 4 情境壓力測試（Base + 3 壓力）:

| 情境 | 假設 | 目的 |
|:---:|:---|:---|
| Base | 報告原始假設 | 基準 |

```
當 Bear 股價 ≈ 現價（差距 < 5%）時:
下行距離 → 0 → R:R 分母趨近 0 → R:R 爆到 100x 以上但無實質意義
```

**這不代表「margin of safety 巨大」,而代表「Bear Case 已被市場完全定價」——安全邊際實際為零**。

**強制判斷邏輯**:

計算 R:R 後,檢查「下行距離」的絕對百分比:

| 下行距離 | 判斷 | R:R 處理 |
|:---|:---|:---|
| > 15% | 正常 | R:R 有效,直接使用 |
| 5-15% | 警示 | R:R 有效但於 §2 E 備註「Bear Case 已接近定價,下行空間已壓縮」 |
| < 5% | **R:R 失效** | R:R 欄位標註「⚠️ 數學假象 - Bear 與現價幾乎重合」,**禁止**直接引用 R:R 數字做進場判定 |
| Bear > 現價 | Bear 高於現價 | 標註「即使 Bear Case 仍高於現價，表示市場過度悲觀或 Bear 假設過樂觀，需重新檢視 §8 E 壓力情境」 |

**當 R:R 失效時的替代判定流程**:

1. 計算「極端 Bear」: Bear PE × 0.8 + Bear EPS × 0.85（假設市場繼續壓縮 20% PE 且 EPS 再下修 15%）
2. 以「極端 Bear 股價」作為替代下檔錨點重算 R:R
3. 若替代 R:R 仍 ≥ 2.0 → 可進場但底倉減半（原 25% → 12%）
4. 若替代 R:R < 2.0 → 判定為「觀望偏進場」而非「進場」,並在 §1 加註「現價接近 Bear 錨點,下行空間已壓縮,建議分批建倉而非一次進場」

**§2 E 判定表新增一列**:

| 條件 | 結果 |
|:---|:---|
| 短期 R:R ≥ 2.0 且中期 R:R ≥ 2.0 且**雙方下行距離 ≥ 5%** | 進場 |
| ...（其他原規則不變） | ... |
| 短期/中期任一「R:R 失效」（下行 < 5%） | 自動降級,不得直接判進場 |

### QC-31｜v12 綜合訊號燈定義表（強制對映）

dd-meta JSON 的 `signal` 欄位（A+/A/B/C/X）必須按以下定義表對映，**不得自由發揮**。
這張表是 thesis-level 分類，與 §12 E（v12 新編號 §2 E）的 R:R 進場時機判定**正交獨立**：

| 訊號 | 含義 | 觸發條件（thesis-level） | 進場行動建議 |
|:---|:---|:---|:---|
| **A+** | 進場 | 品質 ≥ 7.5（A 級） + 估值🟢 + 短期 R:R ≥ 2.0 + 中期 R:R ≥ 2.0 + Pure MA 多頭 | 立即建倉至底倉 25%（或目標倉位 100%） |
| **A** | 觀望偏進場 | 品質 ≥ 7.5（A 級） + 中期 R:R ≥ 1.0 + 長期持有信心 = 高 | 建倉至底倉 15-20% |
| **B** | 觀望 | 品質 ≥ 6.0（A/B 級）+ thesis 完整，但時機不到（R:R 不足 / 估值🔴 / MA 過熱 / Bollinger 上軌之上） | 不新進，等估值或時機修復；已持倉者視情況部分獲利了結 |
| **C** | 迴避（thesis 失敗） | 任一觸發：(1) 品質分 < 6.0；(2) 護城河 X 級或侵蝕信號 ≥ 3 個；(3) 價值陷阱定性 = 🔴；(4) §6F AI 取代風險 = 🔴 | 不持有，已持有者清倉 |
| **X** | 黑名單 | 任一觸發：(1) 重大治理問題 / 舞弊；(2) 結構性產業衰退（TAM 萎縮 + 估值倍數系統下移）；(3) 獲利品質崩壞（FCF/NI < 0.5 連續 2 年） | 不分析、不討論、不持有 |

**核心規則**：

1. **R:R 不足 ≠ C**：R:R 是「進場時機」參考，不單獨觸發 thesis-level 迴避。品質 A/B 的標的若 R:R 不過進場門檻，落 **B（觀望）**，等估值或時機修復。
2. **估值🔴 ≠ C**：估值極端高估只代表現在不該進，標的本身（品質 + 護城河 + 長期信心）若仍為 A 級，落 **B（觀望）**。
3. **C/X 必須有 thesis-level 的失敗證據**：不是「現在 R:R 不夠」，而是「這標的長期不該持有」。撰寫 §1 時若選 C/X，必須在「結論 Dashboard」明確列出觸發條件對應的具體章節數據（§9 護城河、§8 衰退信號、§7 陷阱四問、QC-9 品質分等）。
4. **§2 E（v12 新編號）的「都不過 → 迴避」是時機語意，不是 signal 對映**：該行的「迴避」描述「現在不進場」的決策，**不是 signal grade 的指派**。signal grade 必須回到本表查對。範例：PANW 短期 R:R −0.17、中期 0.23，§2 E「都不過」 → 不新進；但 thesis 完整（品質 10、護城河 A、估值🟡）→ signal = **B 觀望**，不是 C。
5. **批次 DD 一致性**：同一批 DD 中，所有「估值🔴 + 4 週動能爆衝 + 品質 A/B」的標的（典型如 RMBS/FORM/CIEN/MOD/FIX）一律落 B（觀望，等修復），不得隨機歸 C。

### QC-36｜5Y 目標價一致性（v12.2 新增，2308.TW Inception DD 教訓）

§2 E R:R 表的「長期（FY+5）」格 + §1 dashboard「參考 R:R 長期」+ dd-meta `upside_5y_pct` **三處數字必須完全一致**（v12.5：原 §13.5 目標價表已隨 §13.5 砍除，5Y 目標價單一居所改為 §2 E）。

**強制計算公式**：
```
5Y 目標價 = §8 E Base 情境 5Y EPS × §2 E 長期合理 PE（由 §9 護城河分數 + §8 成長評級決定）
upside_5y_pct = (5Y 目標價 − 當前股價) ÷ 當前股價 × 100%
```

**禁止錯誤**（2308.TW v12.2 第一份 DD 實際發生過）：
- ✗ 用 §8 E 的 10Y EPS 當 5Y 計算（會把目標價虛高一倍）
- ✗ §2 E R:R 與 §8 E 用不同的 EPS 來源 / PE 倍數
- ✗ dd-meta upside_5y_pct 與 HTML 內顯示數字不一致
- ✗ 用「FY28E 推算 +X%」這類間接推導取代直接引用 §8 E 的 5Y EPS

**自驗步驟**（HTML 寫完後在 LLM 內部跑一次）：
1. 從 §8 E 表格抓出 Base 情境 5Y EPS（例：2308.TW = 130）
2. 從 §2 E 合理 P/E 基準抓出長期合理 PE（例：32x）
3. 計算 5Y 目標價 = 130 × 32 = 4,160
4. 確認 §2 E 長期格、§1 dashboard、dd-meta 全部寫的是 4,160（不是 8,000、不是 3,904 推算）
5. upside_5y_pct = (4,160 − 2,215) ÷ 2,215 × 100 = 88（rounded）

**Bear Case 5Y 處理**：5Y 後股價可能高於現價（因為 Bear A 仍給 +13% 報酬）。此時 R:R 計算的「下行」應取**短中期 Bear**（持有期間最大跌幅）作為下行錨點，而非 5Y end state Bear。否則會出現「Bear 仍 +X%」的反直覺數字。

## 【價值陷阱警戒協議】

**本框架內建四層防禦機制，在以下節點強制觸發反向驗證：**

**四種最常見的價值陷阱模式**（分析全程保持警覺）：
- **護城河侵蝕型**：定價能力下滑、市佔流失，低估值反映未來盈利能力惡化
- **盈餘品質型**：EPS 靠回購或會計調整撐起，FCF 與淨利背離，真實獲利能力被高估
- **產業結構衰退型**：低 P/E 是因為市場已定價產業長期萎縮，成長天花板已到
- **隱性資本密集型**：FCF 看似良好，但維持競爭力所需的資本支出被系統性低估

**FY+3 推導方式（v9.0 邏輯分析法，禁止機械外推）—— 僅在步驟零' Excel 未覆蓋時適用**：

2. 從 §8 B（成長驅動結構）判斷定價 vs 量的貢獻是否遞減
3. 從 §8 D（ROIC × 再投資率）計算內生成長率作為 FY+3 成長率的上限參考
4. 綜合以上三點，給出 FY+3 YoY growth 的具體數值與一句話邏輯依據
5. FY+3E = FY+2E × (1 + FY+3 growth)，標注「§8 邏輯推導」

或「Azure AI 營收佔比從 15% 升至 25%」）。

（v12.4 補：Excel 步驟零' 直接給 FY+3 consensus 時，仍要寫 §8 邏輯描述

**步驟二（備援）：來源優先順序**

**ROIIC 與內生成長天花板（v12.5 新增，強制計算）**

複利引擎的真正來源不是 ROIC 水準，而是**增量**投入資本的回報 × 再投資跑道。§7 FY+3 推導步驟 3 已引用「從 §8.D（ROIC × 再投資率）計算內生成長率」，此塊補上這個過去從未強制的計算。四列必算，附完整推導數字：

| 列 | 計算 | 規則 |
|:---|:---|:---|
| ROIIC（3Y） | (NOPAT_t − NOPAT_t−3) ÷ (投入資本_t − 投入資本_t−3) | 附兩端數字；ΔIC ≤ 0 或受大型 M&A 扭曲 → 標「剔除 M&A 重算」或「不適用 + 原因」，禁止硬算 |
| 再投資率 | (Capex − D&A + ΔWC + 收購淨額) ÷ NOPAT，3Y 均 | 與上方表 Reinvestment Rate 列一致 |
| 內生成長天花板 | ROIIC × 再投資率 | |
| 共識對賬 | §7 共識 EPS CAGR − 內生天花板 = 缺口 __pp | 缺口強制歸因：margin 擴張（引 §8.B③ 增量利潤率）/ 淨回購（引 §10）/ 無法歸因 |

**判定規則**：
- 缺口可閉合（歸因到 margin 或回購）→ 標「共識不依賴 re-rate」。
- 缺口無法歸因 → §5.B 對應假設強制加註「依賴 re-rate / 無基本面引擎支撐」警示，且 §2 I 長期持有信心上限「中」。
- 內生天花板數字寫入 dd-meta optional 欄位 `endo_growth_ceiling`（供 DCA §4 IRR sanity check 引用）。

**範例（示意數字）**：
> ROIIC(3Y)：NOPAT $14.2B→$22.6B（Δ$8.4B）÷ IC $48B→$76B（Δ$28B）= 30.0%。再投資率 ≈ 55%。內生天花板 = 16.5%/yr。對賬：共識 EPS CAGR 21.0%，缺口 4.5pp = 增量 margin +2.0pp（§8.B③）+ 淨回購 +2.3pp（§10）→ 可閉合 → 複利引擎可自我支撐，支持 5Y+ 持有。

elif current < w104:
    state = "🟠 暫不進場"

    state = "🟢 最佳進場"
elif current > w52 > w104 > w250 and slope_pct > 3:
    state = "✅ 強勢進場"
elif current > w52 > w104 > w250 and abs(slope_pct) <= 3:
    state = "🟡 減半進場"
else:

print(f"W250: {w250:.2f}" if w250 else "W250: N/A")

print(f"狀態: {state}")

### 功能規格

- 頁首固定：標的代碼 ｜ 資料抓取時間 ｜ 最新股價 ｜ DD Schema v12.3

- 同時在 HTML <head> 區加一個 meta 標籤供程式化解析：`<meta name="dd-schema-version" content="v12.3">`，meta content 同樣取自 frontmatter version 欄位
