## Systemic Issues Identified (Across Batch)

### Issue 1: Stale §13.3 cross-references in §3 body (ADI line 285, APP line 316)
Pattern: when §13 is pruned but §3 三方辯論 body text contains "§13.3" references, they are not updated. In ANET this is worse because §13.3 was not actually removed. In ADI/APP, §13.3 heading is correctly absent but the body text reference remains.

**Root cause**: The upgrade process correctly removes §13 section headings but misses inline cross-references in §3 body.

**Recommended fix**: When doing v12.3 upgrades, run a grep for "§13\.3\|§13\.0\|§13\.5" across the full body text and clean all inline references, not just the section headings.

### Issue 2: ANET dd-meta JSON not bumped / §13 not pruned (fake completion)
This is the batch's only FAIL. The ANET upgrade log claimed §13 pruning and dd-meta schema bump were done, but neither actually happened. The upgrade session (win-20260516-1641-4703) appears to have written the §5/§8/§9 additions correctly but stopped before completing §13 pruning and dd-meta JSON update.

**Root cause**: Likely session interruption or premature completion detection. The presence of the upgrade note text creates a false sense of completion.

**Recommended fix**: In future upgrade sessions, add an explicit final verification step: grep for `<h3>13.0\|<h3>13.3\|<h3>13.5` to confirm absence before marking as complete.

### Issue 3: ANET §11 three-segment bargaining power missing from §11 proper
The ANET DD has downstream customer analysis in §8.H but lacks explicit upstream and geopolitical bargaining power `<p>` paragraphs in §11. For a network infrastructure company, upstream (TSMC for custom ASIC chips, Marvell/Broadcom for components) and geopolitical (US-China Ethernet switch export controls) are material.
