## Systemic Issues 觀察（跨 batch 累積）

本 batch 的系統性問題有以下型態：

### 1. Stale §13.5/§13.3 cross-reference（GEV / GFS / GRAB）
- 最常見型態：`→ §13.5 implication：` 前綴殘留在 §2.D / §2.E / §13.1 / §13.2 的 `<div class="reasoning">` 內

- 根本原因：v12.3 upgrade 刪除 §13.3/§13.5 section headers 但未清理 body 內的 forward references

### 2. §13.0 section body 未刪（GRAB）

- 型態：upgrade 時只刪 §13.3 / §13.5，遺漏 §13.0
- **建議 upgrade prompt 補充**：「v12.3 刪除清單 = 13.0 + 13.3 + 13.5，三個全刪，不是兩個」

### 3. Schema meta tag 未更新（GRAB）
- `content="v12.2"` 未更新是 DELL 案例同型的 silent bypass 問題
- 高風險：CI validator 不會 catch，只有 cold review 才能發現
- **建議 upgrade prompt 補充**：「最後一步強制 check meta tag: content="v12.3" 是否已更新」

### 4. Stale version label in section headers（GFS）

- `§3 body "v12.1 訊號燈"` stale
