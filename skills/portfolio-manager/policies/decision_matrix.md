## 組合級檢核（套用於個股裁決之後）

按順序檢核，先到先判：

### 1. 類別檔數檢核
- 核心 ≥ 5 → 新核心候選凍結（除非先 trim 現有核心）
- 衛星 ≥ 3 → 新衛星候選凍結

### 2. 單檔上限檢核
- core > 20% → **強制 trim 建議**（降至 18-20%）
- satellite > 7% → 強制 trim 建議

### 3. 單檔下限檢核
- 任一持倉 < 2% → **建議清倉**（位置太小，佔 slot 無意義）

### 4. Sector / Theme cap 檢核
- 合計超過 cap → 該 sector/theme 新進凍結
- 已超過 cap 的持倉：若同時是加碼候選 → 觸發 L1 override，PM 可保持現有倉位但不加碼

### 5. 現金下限檢核
- cash < 5% → **強制 trim 最低 conviction 倉位**（優先順序：B 衛星 → C 觀察 → 核心末位）

### 6. Top 3 集中度檢核（資訊性，不強制）
- top 3 > 60% → HTML §7 警示，但允許 PM 判斷是否 trim
- top 3 > 75% → 強烈建議分散（但仍不強制）

### 7. benchmark 狀態（資訊性，不直接影響現金）
- 破 W104 → HTML §7 標註「大盤豁免層已透過 DD 個股 gate degrade 吸收」
- 破 W250 → 標註「系統性風險警戒；核心加碼需更嚴篩選（A+ only）」
- **禁止**加硬性現金下限

## Rebalance 執行順序（建議）

1. 清倉（trap 🔴 / final_signal X）→ 釋出現金
2. 減碼（超漲 / 超上限 / C 訊號）→ 調整至目標
3. 新進（核心 → 衛星）→ 用釋出的現金
4. 加碼（觸發價內的 A+ 持倉）→ 補滿目標
5. 檢核現金下限（若 < 5%，再 trim 最低 conviction）
